Remove Nodes From Linked List

Linked ListStackRecursionMonotonic Stack
01

Problem Overview

You are given the head of a linked list.

Remove every node which has a node with a strictly greater value anywhere to its right side.

Return the head of the modified linked list.

Constraints:

  • The number of nodes in the list is in the range [1, 10^5].
  • 1 <= Node.val <= 10^5
02

Examples

Example 1​

Input:  head = [5,2,13,3,8]
Output: [13,8]
Explanation:
  Nodes 5 and 2 are removed because 13 (to their right) is greater.
  Node 3 is removed because 8 (to its right) is greater.
  Nodes 13 and 8 have no greater value to their right — they survive.

Pattern Recognition

→ Pattern Decision Tree
?
Must we remove nodes dominated by a later greater value?
✓ Yes — Any node smaller than something to its right is invalid. Walking right-to-left or using a monotonic stack lets us enforce this.
?
Is a monotonic decreasing stack the right tool?
✓ Yes — Push nodes left-to-right. Before pushing, pop everything from the top that is smaller than the current node — those are dominated. The stack contains only survivors in order.
Tap to reveal pattern →
04

Approach

Use a monotonic decreasing stack of ListNode?. Walk left-to-right. Before pushing the current node, pop all stack entries whose value is strictly less than the current node's value — these nodes have a greater value to their right and must be removed.

After processing all nodes, the stack holds the surviving nodes in order (largest-first from bottom to top... actually, since we never push a node smaller than the top, the stack is non-increasing from bottom to top). Rebuild the linked list by chaining nodes in stack order.

Rebuilding with a dummy head

Iterate through the stack in order (bottom to top = left to right). Use a dummy ListNode(0) as the new head; chain each node and advance curr. Null the last node's next. Return dummy.next.

06

Step-by-Step Solution

Build monotonic decreasing stack —

Initialise var stack = [ListNode?](). Walk curr = head while curr != nil. Before pushing, while stack.last??.val is less than curr!.val, pop from stack. Push curr. Advance curr = curr?.next.

Rebuild linked list —

Create var list = ListNode(0) (dummy). Let curr = list. For each node in stack: curr?.next = node; curr = curr?.next. Null the last node's next: curr?.next = nil.

Return —

Return list.next.

07

Code Implementation

class Solution {
  func removeNodes(_ head: ListNode?) -> ListNode? {
    var stack = [ListNode?]()

    var curr = head
    while curr != nil {
      while let top = stack.last??.val, curr!.val > top {
        stack.removeLast()
      }
      stack.append(curr)
      curr = curr?.next
    }

    var list = ListNode(0)
    curr = list
    for node in stack {
      curr?.next = node
      curr = curr?.next
    }
    curr?.next = nil

    return list.next
  }
}

09

Common Pitfalls

✕01
Using strictly less-than instead of less-than-or-equal-to when popping
The problem says remove nodes with a strictly greater value to their right. Equal values should NOT be removed. The pop condition must be strictly greater: curr.val > top.
Fix: Pop condition: while curr!.val > stack.last??.val. Equal values stay.
✕02
Not nulling the last node's next pointer
Stack nodes still carry their original next pointers. The last node in the rebuilt list may point to a removed node, creating a corrupt list.
Fix: After rebuilding, set curr?.next = nil to terminate the list cleanly.
✕03
Iterating stack in reverse
The stack stores nodes in left-to-right insertion order (index 0 = leftmost survivor). Iterating forward gives the correct order.
Fix: Iterate stack with a standard for-in loop (not reversed) when rebuilding.
