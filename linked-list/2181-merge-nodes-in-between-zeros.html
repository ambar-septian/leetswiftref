Merge Nodes in Between Zeros

Linked ListSimulation
01

Problem Overview

You are given the head of a linked list, which contains a series of integers separated by 0s. The beginning and end of the linked list will have Node.val == 0.

For every two consecutive 0s, merge all the nodes lying in between them into a single node whose value is the sum of all the merged nodes. The modified list should not contain any 0s.

Return the head of the modified linked list.

Constraints:

  • The number of nodes in the list is in the range [3, 2 * 10^5].
  • 0 <= Node.val <= 1000
  • There are no two consecutive nodes with Node.val == 0.
  • The beginning and end of the linked list have Node.val == 0.
02

Examples

Example 1​

Input:  head = [0,3,1,0,4,5,2,0]
Output: [4,11]
Explanation: Segment [3,1] → sum=4. Segment [4,5,2] → sum=11.

Example 2​

Input:  head = [0,1,0,3,0,2,2,0]
Output: [1,3,4]

04

Approach

Use a dummy node pointer that starts at the result head. Walk current from head.next (skipping the first 0). For non-zero nodes, add to dummy.val. When a zero node is found and current.next != nil (not the trailing zero), move dummy to a new zero node and continue.

Reuse zero nodes

Instead of allocating new ListNodes for each segment, reuse the zero nodes already in the list. Set dummy = dummy.next (the zero node), then accumulate into it. No extra allocation needed.

06

Step-by-Step Solution

Initialise —

dummy = ListNode(0) (the result head). result = dummy. current = head.next (skip leading 0).

Walk until current.next is nil —

For each node: if current.val == 0, move to a new node (dummy.next = ListNode(0), dummy = dummy.next). Otherwise, dummy.val += current.val.

Return result —

Return result — the head of the accumulated list.

07

Code Implementation

class Solution {
  func mergeNodes(_ head: ListNode?) -> ListNode? {
    var current = head?.next
    var dummy: ListNode? = ListNode(0)
    let result = dummy

    while current?.next != nil {
      if current?.val == 0 {
        dummy?.next = ListNode(0)
        dummy = dummy?.next
      } else {
        dummy?.val += current!.val
      }
      current = current?.next
    }

    return result
  }
}

09

Common Pitfalls

✕01
Not skipping the leading zero
The head is always 0. Starting current at head would accumulate 0 into the first result node or trigger a spurious new-node creation.
Fix: Start current = head?.next to skip the leading sentinel zero.
✕02
Processing the trailing zero
The last node is also 0. If you loop while current != nil, you create an empty extra node at the end.
Fix: Loop while current?.next != nil — this stops before processing the trailing zero.
✕03
Forgetting to initialise dummy with val=0
The first result node accumulates into dummy.val. Starting with a non-zero value would corrupt the first segment sum.
Fix: Initialise dummy = ListNode(0) so the first segment starts clean.
