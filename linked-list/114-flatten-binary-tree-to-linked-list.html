01

Problem Overview

Given the root of a binary tree, flatten the tree in-place into a linked list that follows preorder traversal order. The left child of every node must be set to null, and the right child should point to the next node in the preorder sequence.

The "linked list" uses the same TreeNode class where the right pointer plays the role of the next pointer.

Constraints:

  • The number of nodes in the tree is in the range [0, 2000].
  • -100 <= Node.val <= 100
  • You must flatten the tree in-place (O(1) extra space).

04

Approach

Recursively flatten both subtrees using a dfs function that returns the tail node of the flattened result. After flattening:

  1. Store root.right (original right subtree).
  2. Move root.left to root.right.
  3. Set root.left = nil.
  4. Walk to the tail of the new right chain (the leftTail) and attach the original right subtree there.
Why return the tail?

After flattening the left subtree the last node needs to be connected to the right subtree. Returning the tail avoids an extra traversal to find it.

Return priority

rightTail ?? leftTail ?? root — if right subtree exists, its tail is the overall tail; else the left tail; else the node itself (leaf case).

06

Step-by-Step Solution

Base case —

If root is nil, return nil.

Recurse post-order —

let leftTail = dfs(root.left) then let rightTail = dfs(root.right). Both subtrees are fully flattened before we touch the current node.

Stitch left tail to right subtree —

If leftTail exists: leftTail.right = root.right, then root.right = root.left, then root.left = nil.

Return the tail —

Return rightTail ?? leftTail ?? root. This gives the caller the endpoint of this subtree's flattened list.

07

Code Implementation

class Solution {
  func flatten(_ root: TreeNode?) {

    @discardableResult
    func dfs(_ root: TreeNode?) -> TreeNode? {
      guard let root else { return nil }

      let leftTail  = dfs(root.left)
      let rightTail = dfs(root.right)

      if let leftTail {
        leftTail.right = root.right
        root.right = root.left
        root.left  = nil
      }

      return rightTail ?? leftTail ?? root
    }

    dfs(root)
  }
}

09

Common Pitfalls

✕01
Losing the original right subtree before stitching
If you set root.right = root.left before saving leftTail.right = root.right, the original right subtree is lost.
Fix: Always attach leftTail.right = root.right (old right) BEFORE overwriting root.right with the left subtree.
✕02
Not setting root.left = nil
After moving the left subtree to root.right, the left pointer still references the moved subtree, violating the flat list invariant.
Fix: Always set root.left = nil after the stitch.
✕03
Wrong tail return order
Returning leftTail ?? rightTail ?? root is incorrect: if a right subtree exists, its tail is the last node in the flattened list, not the left tail.
Fix: Return rightTail ?? leftTail ?? root — right tail has higher priority.
