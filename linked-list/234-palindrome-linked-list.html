01

Problem Overview

Given the head of a singly linked list, return true if it is a palindrome or false otherwise.

Constraints:

  • The number of nodes in the list is in the range [1, 10^5].
  • 0 <= Node.val <= 9

04

Approach

Three phases:

  1. Find the midpoint — move slow one step and fast two steps until fast.next == nil or fast.next.next == nil. slow ends at the last node of the first half.
  2. Reverse the second half — iterative three-pointer reversal starting from slow, producing a reversed chain (prev).
  3. Compare both halves — walk p1 from head and p2 from prev simultaneously; if any values differ, return false.
Odd vs even length

For odd-length lists the middle node is skipped — only the nodes after it form the second half. The fast/slow condition fast?.next != nil and fast?.next?.next != nil correctly positions slow at the end of the first half in both cases.

06

Step-by-Step Solution

Find the midpoint with fast/slow pointers —

Move slow one step and fast two steps each iteration until fast?.next == nil or fast?.next?.next == nil.

Reverse the second half —

Starting from slow (inclusive), apply the iterative reversal: save next, redirect curr.next = prev, advance both.

Compare first and second halves —

Walk p1 from head and p2 from prev. Return false on any value mismatch.

Return true —

If p2 is exhausted without a mismatch, the list is a palindrome.

07

Code Implementation

class Solution {
  func isPalindrome(_ head: ListNode?) -> Bool {
    var slow = head
    var fast = head

    while fast?.next != nil && fast?.next?.next != nil {
      slow = slow?.next
      fast = fast?.next?.next
    }

    // Reverse second half starting from slow
    var prev: ListNode? = nil
    var curr = slow

    while curr != nil {
      let next = curr?.next
      curr?.next = prev
      prev = curr
      curr = next
    }

    // Compare both halves
    var p1 = head
    var p2 = prev

    while p1 != nil, p2 != nil {
      if p1?.val != p2?.val { return false }
      p1 = p1?.next
      p2 = p2?.next
    }

    return true
  }
}

09

Common Pitfalls

✕01
Wrong fast/slow stopping condition
Using fast != nil and fast?.next != nil moves slow one step too far for even-length lists, leaving the second half with one extra node that skews the comparison.
Fix: Use fast?.next != nil and fast?.next?.next != nil to stop slow at the last node of the first half.
✕02
Starting the reversal from slow.next instead of slow
For even-length lists the midpoint has two equal middle nodes. Starting from slow.next skips one and causes a false negative.
Fix: Start the reversal from slow (inclusive). The comparison will naturally stop when p2 is exhausted.
✕03
Using p1 == nil as the termination condition
The first half may be longer than the reversed second half (for odd-length lists). p2 is exhausted first — that is the correct termination.
Fix: Loop while p1 != nil and p2 != nil. When p2 runs out the check passes.
