Design Circular Queue

ArrayLinked ListDesignQueue
01

Problem Overview

Design your implementation of the circular queue. The circular queue is a linear data structure in which the operations are performed based on the FIFO (First In First Out) principle, and the last position is connected back to the first position to make a circle (also called a "Ring Buffer").

Implement the MyCircularQueue class:

  • MyCircularQueue(k) — Initializes the object with the size of the queue to be k.
  • boolean enQueue(int value) — Inserts an element into the circular queue. Returns true if the operation is successful.
  • boolean deQueue() — Deletes an element from the circular queue. Returns true if the operation is successful.
  • int Front() — Gets the front item from the queue. If the queue is empty, returns -1.
  • int Rear() — Gets the last item from the queue. If the queue is empty, returns -1.
  • boolean isEmpty() — Checks whether the circular queue is empty or not.
  • boolean isFull() — Checks whether the circular queue is full or not.

You must solve the problem without using the built-in queue data structure in your programming language.

Constraints:

  • 1 <= k <= 1000
  • 0 <= value <= 1000
  • At most 3000 calls will be made to enQueue, deQueue, Front, Rear, isEmpty, and isFull.
02

Examples

Example 1​

MyCircularQueue cq = new MyCircularQueue(3);
cq.enQueue(1);  // true
cq.enQueue(2);  // true
cq.enQueue(3);  // true
cq.enQueue(4);  // false (full)
cq.Rear();      // 3
cq.isFull();    // true
cq.deQueue();   // true
cq.enQueue(4);  // true
cq.Rear();      // 4

Can we use a fixed-size array with modulo arithmetic to simulate a ring?
✓ Yes — Incrementing head and tail with modulo maxCount wraps the pointers around, giving circular behaviour on a linear array.
?
Do we need a separate count variable?
✓ Yes — head == tail when both empty AND when full. A separate count variable disambiguates these two states.
?
What is the tail-to-rear offset?
✓ Yes — tail points to the next EMPTY slot. The rear (last inserted value) is at (tail - 1 + maxCount) % maxCount.

04

Approach

Allocate items: [Int?] of size maxCount. Maintain head (index of front element), tail (index of next write slot), and count. For enQueue: place at items[tail], advance tail = (tail + 1) % maxCount, increment count. For deQueue: clear items[head], advance head = (head + 1) % maxCount, decrement count. Rear() reads from (tail - 1 + maxCount) % maxCount.

Circular array of capacity 5. head=0, tail=0 (both point to slot 0).
Why tail needs +maxCount before modulo

(tail - 1) can be -1 when tail == 0. In Swift (and most languages), -1 % 5 = -1, not 4. Adding maxCount before the modulo guarantees a non-negative result: (0 - 1 + 5) % 5 = 4.

06

Step-by-Step Solution

Initialise fields —

head = 0, tail = 0, count = 0, items = [Int?](repeating: nil, count: maxCount).

enQueue — guard full, write, advance tail —

If count == maxCount, return false. Else: items[tail] = value, tail = (tail + 1) % maxCount, count += 1, return true.

deQueue — guard empty, clear, advance head —

If count == 0, return false. Else: items[head] = nil, head = (head + 1) % maxCount, count -= 1, return true.

Front — guard empty, read head —

guard let value = items[head] else { return -1 }; return value.

Rear — guard empty, read tail-1 with wrap —

let realTail = (tail - 1 + maxCount) % maxCount; guard let value = items[realTail] else { return -1 }; return value.

07

Code Implementation

class MyCircularQueue {
  var head = 0
  var tail = 0
  var count = 0
  var items: [Int?] = []
  let maxCount: Int

  init(_ k: Int) {
    self.maxCount = k
    self.items = Array(repeating: nil, count: maxCount)
  }

  func enQueue(_ value: Int) -> Bool {
    guard count < maxCount else { return false }
    items[tail] = value
    tail = (tail + 1) % maxCount
    count += 1
    return true
  }

  func deQueue() -> Bool {
    guard count > 0 else { return false }
    items[head] = nil
    head = (head + 1) % maxCount
    count -= 1
    return true
  }

  func Front() -> Int {
    guard let value = items[head] else { return -1 }
    return value
  }

  func Rear() -> Int {
    let realTail = (tail - 1 + maxCount) % maxCount
    guard let value = items[realTail] else { return -1 }
    return value
  }

  func isEmpty() -> Bool { count == 0 }
  func isFull() -> Bool  { count == maxCount }
}

09

Common Pitfalls

✕01
Using head == tail to detect both empty and full
After filling the queue completely, head and tail point to the same slot (tail wrapped back to head). Without count, isEmpty and isFull are indistinguishable.
Fix: Track count separately. isEmpty returns count == 0; isFull returns count == maxCount.
✕02
Negative modulo for Rear() when tail == 0
(tail - 1) % maxCount = -1 % maxCount which in Swift evaluates to -1, not maxCount - 1.
Fix: Use (tail - 1 + maxCount) % maxCount to guarantee a non-negative result.
✕03
Reading Front() or Rear() without checking isEmpty first
When the queue is empty, items[head] is nil. Force-unwrapping will crash.
Fix: Use guard let value = items[...] else { return -1 } to safely handle the empty case.
