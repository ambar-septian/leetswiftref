Remove Nth Node From End of List

Linked ListTwo Pointers
01

Problem Overview

Given the head of a linked list, remove the nth node from the end of the list and return its head.

Constraints

  • The number of nodes in the list is sz.
  • 1 <= sz <= 30
  • 0 <= Node.val <= 100
  • 1 <= n <= sz
Core Insight

Use a fast pointer that moves 2 steps at a time and a slow pointer that moves 1 step. When fast stops, count gives the total node count. Convert that to a 0-based front index, walk a dummy-headed cursor to the node just before the target, then skip the target with curr.next = curr.next.next.

04

Approach

Start slow at head and fast at head.next, count = 1. Advance both while fast != nil && fast.next != nil (fast moves 2 steps, slow moves 1 step, count increments). After the loop, count encodes half the list length. Compute the true length: count * 2, then subtract 1 if fast is nil (odd-length list). The front index of the target is length - n. Walk a dummy-headed cursor that many steps, then unlink via curr.next = curr.next.next.

Why the count formula works

Fast moves 2 steps per iteration. If the list has L nodes: odd L → fast ends at nil after (L-1)/2 iterations, so count = (L+1)/2 and length = 2count - 1. Even L → fast ends at the last node after L/2-1 iterations... actually fast ends when fast.next is nil, so for even L fast stops at the last node (not nil), count = L/2, length = 2count. The fast==nil check distinguishes the two cases.

Step-by-Step Solution

Initialise fast/slow and count —

slow = head, fast = head?.next, count = 1.

Advance to count nodes —

While fast != nil && fast?.next != nil: move slow one step, fast two steps, count += 1.

Compute true length —

count *= 2. If fast == nil (odd length), count -= 1. target = max(0, count - n).

Attach dummy head —

Create dummy = ListNode(0), curr = dummy, curr.next = head.

Walk to the node before target —

Advance curr exactly target times via curr = curr.next.

Unlink the target —

curr.next = curr.next?.next. Return dummy.next.

07

Code Implementation

class Solution {
  func removeNthFromEnd(_ head: ListNode?, _ n: Int) -> ListNode? {
    var slow = head
    var fast = head?.next
    var count = 1

    while fast != nil, fast?.next != nil {
      slow = slow?.next
      fast = fast?.next?.next
      count += 1
    }

    count = count * 2
    if fast == nil { count -= 1 }
    let target = max(0, count - n)

    var dummy: ListNode? = ListNode(0)
    var curr = dummy
    curr?.next = head

    for _ in 0..<target {
      curr = curr?.next
    }
    curr?.next = curr?.next?.next
    return dummy?.next
  }
}

09

Common Pitfalls

✕01
Removing the wrong node without a dummy head
If the head itself is the target (n == list length), there is no previous node to update. Without a dummy, this requires a separate branch.
Fix: Prepend a dummy node. The cursor starts at dummy, so even the head can be unlinked uniformly via dummy.next = dummy.next.next.
✕02
Walking to the target node instead of the node before it
To unlink node at index i, we need curr to point to index i-1 so we can update curr.next. Walking to the target itself leaves no way to update the previous pointer.
Fix: Walk exactly target steps from dummy (not target+1), stopping one node before the target.
✕03
Off-by-one in the odd/even length formula
The fast==nil check distinguishes odd-length lists (fast ran off the end) from even-length ones (fast stopped at the last node). Missing this check gives length = 2*count for all lists, which is off by 1 for odd lengths.
Fix: After count *= 2, subtract 1 if fast == nil.
