Design HashSet

01

Problem Overview

Design a HashSet without using any built-in hash table libraries.

Implement MyHashSet:

  • MyHashSet() — Initializes the object with an empty set.
  • void add(int key) — Inserts the value key into the HashSet.
  • bool contains(int key) — Returns whether the value key exists in the HashSet or not.
  • void remove(int key) — Removes the value key in the HashSet. If key does not exist in the HashSet, do nothing.

Constraints:

  • 0 <= key <= 10^6
  • At most 10^4 calls will be made to add, remove, and contains.

Can we use a fixed-size array for O(1) bucket access?
✓ Yes — Allocate 10,000 buckets and use hash(key) = key % 10,000 to map any key to a bucket index.

Do we need to handle hash collisions?
✓ Yes — Multiple keys can map to the same bucket. A linked list at each bucket resolves collisions via chaining.

✓ Yes — A dummy head node simplifies insertion and deletion — removal never needs to special-case the head of the chain.

Approach

Allocate 10,000 buckets. hash(key) = key % 10_000. For add: locate the bucket; if empty, create dummy → node(key); otherwise scan for duplicates before appending. For remove: scan for the predecessor of key, splice out. For contains: scan the chain for the key.

Step-by-Step Solution

Initialise buckets —

array = (0..10_000).map { _ in nil } — 10,000 optional ListNode slots.

add(key) —

hashKey = hash(key). If array[hashKey] == nil, create dummy and append node(key). Else scan chain; if key already exists return; else append at tail.

remove(key) —

hashKey = hash(key). If bucket is nil, return. Walk chain; when current.next.val == key, set current.next = current.next.next.

contains(key) —

hashKey = hash(key). If bucket is nil, return false. Walk chain; if current.val == key, return true. Return false at end.

Hash function —

private func hash(_ key: Int) -> Int { key % 10_000 }.

07

Code Implementation

class ListNode {
  var val: Int?
  var next: ListNode?
  init(_ val: Int?) { self.val = val }
}

class MyHashSet {
  var array: [ListNode?]

  init() {
    array = (0..<10_000).map { _ in nil }
  }

  func add(_ key: Int) {
    let hashKey = hash(key)
    if let listNode = array[hashKey] {
      var current: ListNode? = listNode
      while current?.next != nil {
        if current?.next?.val == key { return }
        current = current?.next
      }
      current?.next = ListNode(key)
    } else {
      let listNode = ListNode(nil)  // dummy
      listNode.next = ListNode(key)
      array[hashKey] = listNode
    }
  }

  func remove(_ key: Int) {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return }
    var current: ListNode? = listNode
    while current != nil {
      if current?.next?.val == key {
        current?.next = current?.next?.next
        break
      }
      current = current?.next
    }
    array[hashKey] = listNode
  }

  func contains(_ key: Int) -> Bool {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return false }
    var current: ListNode? = listNode
    while current != nil {
      if current?.val == key { return true }
      current = current?.next
    }
    return false
  }

  private func hash(_ key: Int) -> Int { key % 10_000 }
}

09

Common Pitfalls

✕01
Duplicate insertion without checking
HashSet semantics require uniqueness. Appending without scanning for the key first creates duplicate nodes in the chain.
Fix: In add(), scan the chain fully before appending. If the key is found, return early.
✕02
Skipping the dummy head node in contains
The dummy head stores val=nil. Starting the scan from the dummy and checking val==key will never match nil-val to an integer key, but it wastes one iteration.
Fix: In contains(), scanning from the dummy is safe (nil != any Int), but starting from dummy.next is slightly cleaner and avoids confusion.
✕03
Not reassigning array[hashKey] after remove
In Swift, ListNode is a class (reference type), so mutations propagate automatically. But reassigning array[hashKey] = listNode after the removal loop is a good habit in case of future refactors.
Fix: Reassign array[hashKey] = listNode after the remove loop to make the mutation explicit and safe.
