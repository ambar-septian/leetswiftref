01

Problem Overview

The DNA sequence is composed of a series of nucleotides abbreviated as 'A', 'C', 'G', and 'T'.

Given a string s that represents a DNA sequence, return all the 10-letter-long sequences (substrings) that occur more than once in a DNA molecule. You may return the answer in any order.

Constraints:

  • 1 <= s.length <= 10^5
  • s[i] is either 'A', 'C', 'G', or 'T'.
02

Examples

Example 1​

Input: s = "AAAAACCCCCAAAAACCCCCCAAAAAGGGTTT"
Output: ["AAAAACCCCC","CCCCCAAAAA"]

Example 2​

Input: s = "AAAAAAAAAAAAA"
Output: ["AAAAAAAAAA"]

Pattern Recognition

✓ Yes — Every substring is exactly 10 characters — a fixed-size sliding window.
?
Do we need to detect duplicates?
✓ Yes — Use a 'seen' set to track substrings encountered once; a 'result' set for those seen more than once.
?

✗ No — Use a set for results to avoid duplicates in the output.

04

Approach

Use a fixed window of size 10 defined by a left and right String.Index. At each position, extract the 10-character Substring s[left...right]. If it is already in the seen set, add it to result. Otherwise add it to seen. Advance both indices by one. Convert the result set to [String] at the end.

Set for results

Using a Set for results automatically deduplicates: if "AAAAAAAAAA" appears three times, it still appears once in the output.

Substring vs String hashing

Storing Substring views in sets avoids copying the 10-character window on every step. The final result.map(String.init) materialises the strings only once.

Step-by-Step Solution

Guard short input —

If s.count less than 10, return [] — no 10-letter windows possible.

Initialise indices and sets —

var left = s.startIndex, var right = s.index(startIndex, offsetBy: 9). var seen = Set(Substring)(), var result = Set(Substring)().

Slide the window —

While right less than s.endIndex: extract s[left...right], check and update seen/result, advance both indices.

Return result —

result.map(String.init).

07

Code Implementation

class Solution {
  func findRepeatedDnaSequences(_ s: String) -> [String] {
    guard s.count >= 10 else { return [] }

    var seen   = Set<Substring>()
    var result = Set<Substring>()

    let startIndex = s.startIndex
    var left  = startIndex
    var right = s.index(startIndex, offsetBy: 9)

    while right < s.endIndex {
      let sub = s[left...right]

      if seen.contains(sub) {
        result.insert(sub)
      } else {
        seen.insert(sub)
      }

      left  = s.index(after: left)
      right = s.index(after: right)
    }

    return result.map(String.init)
  }
}

09

Common Pitfalls

✕01
Using a single set and checking for duplicates on insert
A single set cannot distinguish 'seen once' from 'seen twice or more' — it only tracks membership.
Fix: Use two sets: seen (first occurrence) and result (repeats). Only add to result when the substring is already in seen.
✕02
Using an inclusive right index but advancing incorrectly
The window uses left...right (inclusive). After processing, both left and right must advance by one. Forgetting to advance right by one causes windows of length 11 on the second step.
Fix: Always advance both left and right by s.index(after:) at each iteration.
✕03
Initialising right to index offset 10 instead of 9
The window is 10 characters, indices 0 to 9 (inclusive). Initialising right at offset 10 creates an 11-character window.
Fix: Use s.index(startIndex, offsetBy: 9) — offset is 9 for a 10-element inclusive range.
