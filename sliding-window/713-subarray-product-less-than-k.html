Subarray Product Less Than K

#713MediumVariable Sliding WindowLeetCode ↗

Subarray Product Less Than K

ArrayBinary SearchSliding WindowPrefix Sum
01

Problem Overview

Given an array of integers nums and an integer k, return the number of contiguous subarrays where the product of all the elements in the subarray is strictly less than k.

Constraints:

  • 1 <= nums.length <= 3 * 10^4
  • 1 <= nums[i] <= 1000
  • 0 <= k <= 10^6
02

Examples

Example 1​

Input:  nums = [10,5,2,6], k = 100
Output: 8
Explanation: The 8 subarrays that have product less than 100 are:
  [10], [5], [2], [6], [10,5], [5,2], [2,6], [5,2,6]
  Note that [10,5,2] and [10,5,2,6] are NOT included as their products are >= 100.

Example 2​

Input:  nums = [1,2,3], k = 0
Output: 0
03

Pattern Recognition

→ Pattern Decision Tree
?
Does shrinking the left boundary always reduce the product?
✓ Yes — All nums[i] >= 1, so dividing by the outgoing element never increases the product. The window is monotone in product as we shrink.
?
How many new subarrays does adding nums[right] create?
✓ Yes — Every subarray ending at right with left endpoint from left..right is new. That is right - left + 1 new valid subarrays.
?
Do we need k less than or equal to 1 as a special case?
✓ Yes — Since all elements are at least 1, any single element has product >= 1 >= k when k is 0 or 1 — no valid subarrays exist.
Tap to reveal pattern →
04

Approach

Guard k <= 1 and return 0. Otherwise use a variable-size window: maintain total (the product of elements in [left, right]). For each right, multiply total by nums[right]. While total >= k, divide out nums[left] and advance left. If total < k, there are right - left + 1 valid subarrays ending at right — add that to result.

Counting subarrays ending at right

When the window [left, right] has product less than k, all subarrays ending at right with a start in [left, right] also have product less than k (since they are sub-windows). There are right - left + 1 such subarrays.

06

Step-by-Step Solution

Guard edge case —

if k less than or equal to 1 { return 0 }. When k is 0 or 1, no subarray (all elements >= 1) can have a product strictly less than k.

Initialise window —

var left = 0, var total = 1, var result = 0.

Expand right —

For each right in 0 ..&lt; nums.count: total *= nums[right].

Shrink left while product too large —

While total >= k and left less than nums.count: total /= nums[left]; left += 1.

Count valid subarrays —

if total < k { result += right - left + 1 }.

Return result —

Return result.

07

Code Implementation

class Solution {
  func numSubarrayProductLessThanK(_ nums: [Int], _ k: Int) -> Int {
    if k <= 1 { return 0 }

    var left = 0
    var total = 1
    var result = 0

    for right in 0..<nums.count {
      total *= nums[right]

      while total >= k, left < nums.count {
        total /= nums[left]
        left += 1
      }

      if total < k {
        result += (right - left + 1)
      }
    }

    return result
  }
}

09

Common Pitfalls

✕01
Not handling k less than or equal to 1
When k is 0 or 1, total starts at 1 and every multiplication keeps it at least 1. The shrinking loop would drive left past right and the count formula would produce a negative value.
Fix: Return 0 immediately when k less than or equal to 1 — no subarray of positive integers can have a product strictly less than 1.
✕02
Counting subarrays with the wrong formula
After shrinking, there are right - left + 1 new valid subarrays ending at right — not just 1.
Fix: Add right - left + 1 (not 1) to result at each step to count all subarrays with right as the rightmost element.
✕03
Using multiplication overflow
With up to 30000 elements each up to 1000, the running product can grow enormous before being shrunk. In Swift, Int is 64-bit and won't overflow for these constraints, but watch for this in other languages.
Fix: For Swift with these constraints, Int (64-bit) is sufficient. In languages with 32-bit int, use long or check bounds carefully.
