Maximum Average Subarray I

#643EasyFixed-Size Sliding WindowLeetCode ↗

Maximum Average Subarray I

ArraySliding Window
01

Problem Overview

You are given an integer array nums consisting of n elements, and an integer k.

Find a contiguous subarray whose length is equal to k that has the maximum average value and return this value. Any answer with a calculation error less than 10^-5 will be accepted.

Constraints:

  • n == nums.length
  • 1 <= k <= n <= 10^5
  • -10^4 <= nums[i] <= 10^4
02

Examples

Example 1​

Input:  nums = [1,12,-5,-6,50,3], k = 4
Output: 12.75000
Explanation: Maximum average is (12 - 5 - 6 + 50) / 4 = 51 / 4 = 12.75

Example 2​

Input:  nums = [5], k = 1
Output: 5.00000
03

Pattern Recognition

→ Pattern Decision Tree
?
Is the window size fixed at k?
✓ Yes — Every candidate subarray must be exactly length k — classic fixed-size sliding window.
?
Can we update the sum in O(1) per slide?
✓ Yes — Add the incoming right element and subtract the outgoing left element — no need to recompute the full sum each step.
?
Should we track average or sum?
✓ Yes — Track the running sum. Convert to average (divide by k) only when updating the result — avoids repeated floating-point conversion.
Tap to reveal pattern →
04

Approach

Build the initial sum over the first k elements. Initialise result as sum / k. Slide the window: for each new position i from k to nums.count - 1, add nums[i] and subtract nums[i - k]. Update result = max(result, Double(sum) / Double(k)). Return result.

Avoid recomputing the sum from scratch

Recomputing the sum by iterating all k elements on each slide costs O(n*k) total. The sliding window maintains the sum in O(1) per step for O(n) overall — just one addition and one subtraction.

06

Step-by-Step Solution

Build initial window sum —

Sum the first k elements. result = Double(sum) / Double(k).

Slide the window —

For i in k ..&lt; nums.count: sum = sum - nums[i - k] + nums[i].

Update result —

result = max(result, Double(sum) / Double(k)).

Return result —

Return result after the loop.

07

Code Implementation

class Solution {
  func findMaxAverage(_ nums: [Int], _ k: Int) -> Double {
    var sum = 0
    for i in 0..<k {
      sum += nums[i]
    }

    var result = Double(sum) / Double(k)

    for i in k..<nums.count {
      sum = sum - nums[i - k] + nums[i]
      result = max(result, Double(sum) / Double(k))
    }

    return result
  }
}

09

Common Pitfalls

✕01
Recomputing the sum from scratch on each slide
Iterating k elements at each of the n-k positions costs O(n*k) total, which can be up to O(n²) when k is large.
Fix: Maintain a running sum: sum = sum - nums[i - k] + nums[i]. One subtract and one add per step — O(1).
✕02
Initialising result to 0
If all elements are negative, the maximum average is negative and initialising result to 0 returns the wrong answer.
Fix: Initialise result from the first window: result = Double(sum) / Double(k) before the sliding loop.
✕03
Integer division when computing the average
sum / k in Swift performs integer division, truncating the fractional part.
Fix: Use Double(sum) / Double(k) to get the floating-point average.
