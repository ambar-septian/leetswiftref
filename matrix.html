Matrix

A matrix is a 2D array accessed by (row, column) index. Matrix problems involve row/column aggregation, 4-directional (or 8-directional) traversal for BFS/DFS, and in-place transformation. The key is clean index arithmetic and boundary checks before accessing neighbouring cells.

What are Matrix Patterns?

Matrix problems generalise array problems to two dimensions. Every 1D array pattern (linear scan, prefix sum, sliding window) has a 2D counterpart, and matrix problems additionally involve traversal across rows and columns.

PatternDescription
Row/column aggregateCompute row sums, column maxes, or row-by-column products
4-directional BFS/DFSExplore cells reachable via up/down/left/right moves
Diagonal traversalr + c == constant for anti-diagonals
In-place rotation/transposeModify the matrix without extra space
Boundary guardr >= 0 && r < rows && c >= 0 && c < cols before accessing neighbours
Key Constraint

Always check bounds before accessing a neighbour cell. In Swift, accessing an array out of bounds causes a runtime crash — unlike some other languages that return nil or zero. The idiomatic pattern is guard r >= 0, r &lt; rows, c >= 0, c &lt; cols else { continue } inside a BFS loop, or an early return in a DFS function. The four directions are encoded as [(-1,0),(0,1),(1,0),(0,-1)] — up, right, down, left.

Rule of Thumb

If the problem asks to compute something for each row or column — scan row by row or column by column. If it asks to find connected regions or shortest paths — BFS/DFS with a 4-directional direction array. Always write the bounds check first before worrying about the logic.
