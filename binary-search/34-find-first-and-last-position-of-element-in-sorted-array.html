Find First and Last Position of Element in Sorted Array

#34MediumBinary SearchLeetCode ↗

Find First and Last Position of Element in Sorted Array

ArrayBinary Search
01

Problem Overview

Given an array of integers nums sorted in non-decreasing order, find the starting and ending position of a given target value.

If target is not found in the array, return [-1, -1].

You must write an algorithm with O(log n) runtime complexity.

Constraints

  • 0 <= nums.length <= 10^5
  • -10^9 <= nums[i] <= 10^9
  • nums is sorted in non-decreasing order.
  • -10^9 <= target <= 10^9
Core Insight

Run binary search twice — once biased left (when target is found, keep searching left by setting r = m-1) and once biased right (keep searching right by setting l = m+1). Each pass records the last index where target was found.

02

Output: [-1, -1]
Explanation: 6 is not in the array.

Example 3​

Input:  nums = [], target = 0
Output: [-1, -1]

04

Approach

A single binarySearch helper takes an isFindingMostLeft flag. Standard binary search runs with l=0, r=nums.count-1. When nums[m] == target: record result = m, then bias the search — if finding the left boundary set r = m - 1; if finding the right boundary set l = m + 1. Call this helper twice and return both results.

Why record result and continue

A normal binary search returns immediately on finding target. Here we record the index and keep shrinking the window in the biased direction. This guarantees we land on the extreme occurrence, not an arbitrary middle one.

Step-by-Step Solution

Define binarySearch helper —

Parameters: nums, target, isFindingMostLeft: Bool. Returns the boundary index or -1.

Standard binary search loop —

l=0, r=nums.count-1, result=-1. While l <= r, compute m = (l+r)/2.

Handle the three cases —

nums[m] < target → l = m+1. nums[m] > target → r = m-1. Equal → result = m, then bias: left search sets r = m-1, right search sets l = m+1.

Call twice and return —

[binarySearch(nums, target, true), binarySearch(nums, target, false)].

07

Code Implementation

class Solution {
  func searchRange(_ nums: [Int], _ target: Int) -> [Int] {
    let firstItem = binarySearch(nums, target, isFindingMostLeft: true)
    let lastItem  = binarySearch(nums, target, isFindingMostLeft: false)
    return [firstItem, lastItem]

    func binarySearch(_ nums: [Int], _ target: Int, isFindingMostLeft: Bool) -> Int {
      var l = 0
      var r = nums.count - 1
      var result = -1

      while l <= r {
        let m = (l + r) / 2
        if nums[m] < target {
          l = m + 1

        } else if nums[m] > target {
          r = m - 1
          
        } else {
          result = m
          if isFindingMostLeft { r = m - 1 }
          else                 { l = m + 1 }
        }
      }

      return result
    }
  }
}

09

Common Pitfalls

✕01
Returning immediately when target is found
A standard binary search returns on the first match, which may be any occurrence. We need the leftmost or rightmost.
Fix: Record result = m when nums[m] == target, then continue narrowing in the biased direction instead of returning.
✕02
Biasing in the wrong direction
For the left boundary, we must set r = m-1 to keep searching left. For the right boundary, set l = m+1 to keep searching right. Swapping these finds the wrong boundary.
Fix: isFindingMostLeft=true → r = m-1. isFindingMostLeft=false → l = m+1.
✕03
Integer overflow in midpoint calculation
(l + r) / 2 can overflow if l and r are both large positive integers (though Swift's Int is 64-bit on modern hardware, making this unlikely in practice).
Fix: Use l + (r - l) / 2 for a safer midpoint if targeting 32-bit environments.
