Search Suggestions System

ArrayStringBinary SearchTrieSorting
01

Problem Overview

You are given an array of strings products and a string searchWord.

Design a system that suggests at most three product names from products after each character of searchWord is typed. Suggested products should have a common prefix with searchWord. If there are more than three products with a common prefix, return the three lexicographically minimum products.

Return a list of lists of the suggested products after each character of searchWord is typed.

Constraints:

  • 1 <= products.length <= 1000
  • 1 <= products[i].length <= 3000
  • 1 <= sum(products[i].length) <= 2 * 10^4
  • All the strings of products are unique.
  • 1 <= searchWord.length <= 1000
  • products[i] and searchWord consist of lowercase English letters only.

04

Approach

Sort products lexicographically first. Then insert each product into the trie one character at a time. At every node along the path, call appendResult(product) — since products are inserted in sorted order, each node naturally accumulates the first (smallest) 3 that share its prefix.

For the query phase, traverse the trie following searchWord characters. At each step, the current node's results array is the answer for that prefix. If the trie path breaks (a character is missing), all remaining prefixes return an empty list.

Why sort first?

Sorting before insertion guarantees that appendResult fills each node's results array with the lexicographically smallest products — no sorting is needed during lookups.

Guard on appendResult

appendResult only appends when results.count less than 3. This caps memory and ensures O(1) per append — no priority queue needed.

06

Step-by-Step Solution

Sort products —

Sort products lexicographically so that insertion order determines lexicographic priority.

Build trie —

For each product, walk the trie character by character, creating nodes as needed. At each node, call appendResult(product) to store it if fewer than 3 results are already cached.

Query searchWord —

Traverse from the root following each character of searchWord. At each step, append the current node's results array (or [] if the path breaks) to the output.

Handle missing prefix —

If any character is not found in the trie (curr?.children[char] is nil), set curr = nil. Subsequent iterations will always produce [] via the ?? [] fallback.

07

Code Implementation

class Solution {
  let root = Node()

  func suggestedProducts(_ products: [String], _ searchWord: String) -> [[String]] {
    let sortedProducts = products.sorted()

    for product in sortedProducts {
      var current = root
      for char in product {
        if current.children[char] == nil {
          current.children[char] = Node()
        }
        current = current.children[char]!
        current.appendResult(product)
      }
    }

    var results = [[String]]()
    var curr: Node? = root
    for char in searchWord {
      let products = curr?.children[char]?.results ?? []
      results.append(products)
      curr = curr?.children[char]
    }

    return results
  }
}

class Node {
  var children = [Character: Node]()
  private(set) var results = [String]()

  func appendResult(_ string: String) {
    guard results.count < 3 else { return }
    results.append(string)
  }
}

09

Common Pitfalls

✕01
Forgetting to sort before insertion
If products are inserted in arbitrary order, nodes accumulate non-lexicographic results, producing wrong suggestions.
Fix: Sort products before the insertion loop so the first 3 appended to any node are the lexicographically smallest.
✕02
Not handling a broken trie path
Once a character of searchWord is missing from the trie, all subsequent prefixes must return []. Without a nil guard, accessing children of a nil node crashes.
Fix: Use optional chaining (curr?.children[char]) and a nil-coalescing fallback (?? []) so a broken path silently returns empty arrays.
✕03
Appending after 3 results are stored
Without the count guard in appendResult, nodes accumulate all matching products instead of just 3, wasting memory and time.
Fix: Guard results.count less than 3 inside appendResult and return early once 3 are stored.
