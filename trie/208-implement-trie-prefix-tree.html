Implement Trie (Prefix Tree)

Hash TableStringDesignTrie
01

Problem Overview

A trie (also called a prefix tree) is a tree data structure used to efficiently store and retrieve keys in a dataset of strings.

Implement the Trie class with the following operations:

  • Trie() — Initialises the trie object.
  • void insert(String word) — Inserts the string word into the trie.
  • boolean search(String word) — Returns true if the string word is in the trie (i.e. was previously inserted), and false otherwise.
  • boolean startsWith(String prefix) — Returns true if there is a previously inserted string that has the prefix prefix, and false otherwise.

Constraints:

  • 1 <= word.length, prefix.length <= 2000
  • word and prefix consist only of lowercase English letters.
  • At most 3 * 10^4 calls in total will be made to insert, search, and startsWith.

Each Node holds a dictionary of children ([Character: Node]) and an isEndOfWord flag. Insert walks the trie character by character, creating nodes as needed, then sets the final node's flag. Search and startsWith both walk the same path — search returns curr.isEndOfWord, while startsWith returns true as long as the path exists.

isEndOfWord vs path existence

Without the end-of-word flag, searching for "app" would return true even if only "apple" was inserted — the path a→p→p exists but "app" was never fully inserted as a word.

Step-by-Step Solution

Define the Node class —

Each node has var children = [Character: Node]() and var isEndOfWord = false.

insert — walk and create —

For each character: if a child exists, move into it; otherwise create a new node. After the loop, set curr.isEndOfWord = true.

search — walk and check flag —

For each character: if the child does not exist, return false. After the loop, return curr.isEndOfWord.

startsWith — walk and check existence —

Identical traversal to search. Return false if any child is missing; return true if the full prefix path exists (ignore isEndOfWord).

07

Code Implementation

class Node {
  var children = [Character: Node]()
  var isEndOfWord = false
}

class Trie {
  let root = Node()

  init() {}

  func insert(_ word: String) {
    var curr = root
    for char in word {
      if let existing = curr.children[char] {
        curr = existing
      } else {
        let node = Node()
        curr.children[char] = node
        curr = node
      }
    }
    curr.isEndOfWord = true
  }

  func search(_ word: String) -> Bool {
    var curr = root
    for char in word {
      guard let existing = curr.children[char] else { return false }
      curr = existing
    }
    return curr.isEndOfWord
  }

  func startsWith(_ prefix: String) -> Bool {
    var curr = root
    for char in prefix {
      guard let existing = curr.children[char] else { return false }
      curr = existing
    }
    return true
  }
}

09

Common Pitfalls

✕01
Returning true from search when only a prefix was inserted
If only 'apple' was inserted, searching for 'app' walks to the third 'p' node successfully — but isEndOfWord is false there.
Fix: Always check curr.isEndOfWord at the end of search. Only startsWith skips this check.
✕02
Confusing search and startsWith return conditions
Both traverse identically. The difference is a single character: search returns curr.isEndOfWord; startsWith returns true unconditionally if the path exists.
Fix: Keep the traversal logic identical in both methods — only the final return statement differs.
✕03
Using a class-level curr variable instead of a local one
If curr is stored on the Trie rather than declared locally in each method, concurrent or sequential calls corrupt each other's traversal state.
Fix: Always declare var curr = root at the start of each method.
