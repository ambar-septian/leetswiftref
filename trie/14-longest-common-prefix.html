Longest Common Prefix

ArrayStringTrie
01

Problem Overview

Write a function to find the longest common prefix string amongst an array of strings.

If there is no common prefix, return an empty string "".

Constraints

  • 1 <= strs.length <= 200
  • 0 <= strs[i].length <= 200
  • strs[i] consists of only lowercase English letters.
Core Insight

Use the first string as the reference. For each character position, check whether every other string has the same character at that position. Stop as soon as any string is too short or has a different character — everything up to that point is the common prefix.

02

Examples

Example 1​

Input:  strs = ["flower","flow","flight"]
Output: "fl"
Explanation: "fl" is the longest prefix shared by all three strings.

Example 2​

Input:  strs = ["dog","racecar","car"]
Output: ""
Explanation: There is no common prefix among the input strings.

04

Approach

Iterate through each character index of strs[0]. For that index, check every other string in the array. If a string is shorter than the current index, or its character at that index differs from strs[0]'s character, break out of both loops immediately and return the prefix collected so far. Otherwise append the character to the result.

Why iterate over the first string

The common prefix is bounded by the length of the shortest string. Iterating over the first string and using early exit on length or mismatch is equivalent to — and avoids computing — the minimum length upfront.

06

Step-by-Step Solution

Handle empty input —

If strs is empty return "" immediately.

Iterate over first string's characters —

Use strs.first!.enumerated() to get each (index, char) pair.

Check every other string at this index —

For each string from index 1 onward: if firstCharIndex >= nextWord.count or nextWord[nextWordIndex] != char, break out of both loops via a labeled break.

Accumulate matching character —

If all strings match at this position, append String(char) to results.

Return joined result —

results.joined() concatenates the collected characters into the prefix string.

07

Code Implementation

class Solution {
  func longestCommonPrefix(_ strs: [String]) -> String {
    if strs.isEmpty {
      return ""
    }

    var results = [String]()
    outerLoop: for (firstCharIndex, char) in strs.first!.enumerated() {
      for stringIndex in 1..<strs.count {
        let nextWord = strs[stringIndex]
        guard firstCharIndex < nextWord.count else { break outerLoop }
        let nextWordIndex = nextWord.index(nextWord.startIndex, offsetBy: firstCharIndex)
        if nextWord[nextWordIndex] != char {
          return results.joined()
        }
      }
      results.append(String(char))
    }

    return results.joined()
  }
}

09

Common Pitfalls

✕01
Not breaking out of both loops on mismatch
When a mismatch is found inside the inner loop, a plain break only exits the inner loop. The outer loop continues to the next character, producing a wrong result.
Fix: Use a labeled break (outerLoop: / break outerLoop) or return results.joined() directly from inside the inner loop.
✕02
Forgetting to check string length before index access
If a string is shorter than the current index, subscripting it will crash. The guard must check length before accessing the character.
Fix: Guard firstCharIndex < nextWord.count before computing the String.Index and accessing the character.
✕03
Using String subscript with integers directly
Swift Strings are not integer-indexed. Using an Int offset requires converting via String.Index, otherwise the code will not compile.
Fix: Use nextWord.index(nextWord.startIndex, offsetBy: firstCharIndex) to get the correct String.Index before subscripting.
