Asteroid Collision

ArrayStackSimulation
01

Problem Overview

We are given an array asteroids of integers representing asteroids in a row. For each asteroid, the absolute value represents its size, and the sign represents its direction (positive = right, negative = left). Each asteroid moves at the same speed.

Find the state of the asteroids after all collisions. If two asteroids meet, the smaller one explodes. If both are the same size, both explode. Two asteroids moving in the same direction will never meet.

Constraints:

  • 2 <= asteroids.length <= 10^4
  • -1000 <= asteroids[i] <= 1000
  • asteroids[i] != 0
02

Examples

Example 1​

Input:  asteroids = [5,10,-5]
Output: [5,10]
Explanation: The 10 and -5 collide. 10 is bigger so 10 survives. The 5 and 10 never collide.

Example 2​

Input:  asteroids = [8,-8]
Output: []
Explanation: The 8 and -8 collide. Same size — both explode.

Example 3​

Input:  asteroids = [10,2,-5]
Output: [10]
Explanation: 2 and -5 collide: -5 wins (abs(-5)=5 > 2). Then 10 and -5 collide: 10 wins.
03

Pattern Recognition

→ Pattern Decision Tree
?
Does the outcome depend only on the most recent surviving asteroid?
✓ Yes — A negative asteroid can only collide with the top of the stack (most recent positive). A stack naturally models this last-in comparison.
?
Can a single negative asteroid destroy multiple positives?
✓ Yes — A large negative asteroid keeps popping stack tops until it hits something equal or larger, or the stack runs out of positives.
?
Do two positives or two negatives ever collide?
✗ No — Same-direction asteroids move apart — only a positive followed by a negative can collide.

04

Approach

Maintain a stack. For each asteroid ast: while the stack top is positive and ast is negative, a collision occurs. If |ast| > top, pop the top (top destroyed). If |ast| < top, set ast = 0 (ast destroyed). If equal, pop top and set ast = 0 (both destroyed). After the loop, if ast != 0, push it.

When does no collision happen?

No collision when: ast is positive (moving right, cannot catch stack top), stack is empty, or stack top is negative (both moving left — they move away from each other). Only a positive-on-stack-top collides with an incoming negative.

06

Step-by-Step Solution

Initialise stack —

var stack = [Int]().

Process each asteroid —

var ast = asteroids[i]. Enter collision loop: while !stack.isEmpty, stack.last! > 0, and ast < 0.

Resolve collision —

If |ast| > |top|: pop top (top destroyed). If |top| > |ast|: ast = 0 (ast destroyed). If equal: pop top and ast = 0 (both destroyed).

Push survivor —

If ast != 0, push to stack.

Return stack —

The stack holds all surviving asteroids in order.

07

Code Implementation

class Solution {
  func asteroidCollision(_ asteroids: [Int]) -> [Int] {
    var stack: [Int] = []

    for i in 0..<asteroids.count {
      var ast = asteroids[i]
      while let top = stack.last, top > 0, ast < 0 {
        if abs(ast) > abs(top) {
          stack.removeLast()
        } else if abs(top) > abs(ast) {
          ast = 0
        } else {
          stack.removeLast()
          ast = 0
        }
      }
      if ast != 0 {
        stack.append(ast)
      }
    }

    return stack
  }
}

09

Common Pitfalls

✕01
Forgetting the equal-size case
If |ast| == |top|, both are destroyed. Failing to handle this leaves one survivor incorrectly.
Fix: The equal branch must pop the stack top AND set ast=0, so neither asteroid is pushed.
✕02
Colliding same-direction asteroids
Two positives or two negatives never collide. Triggering the collision loop for positive-vs-positive produces wrong results.
Fix: The while condition must check top > 0 AND ast < 0 — only this combination represents a collision.
✕03
Pushing after ast is already zero
If ast becomes 0 (asteroid was destroyed), pushing 0 to the stack corrupts the result.
Fix: Always guard with `if ast != 0` before pushing.
