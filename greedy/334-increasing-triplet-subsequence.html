Increasing Triplet Subsequence

#334MediumGreedy Two-VariableLeetCode ↗

Increasing Triplet Subsequence

ArrayGreedy
01

Problem Overview

Given an integer array nums, return true if there exists a triple of indices (i, j, k) such that i < j < k and nums[i] < nums[j] < nums[k]. If no such indices exist, return false.

You must solve it in O(n) time and O(1) space.

Constraints:

  • 1 <= nums.length <= 5 * 10^5
  • -2^31 <= nums[i] <= 2^31 - 1
02

Examples

Example 1​

Input: nums = [1,2,3,4,5]
Output: true
Explanation: Any triplet where i < j < k works.

Example 2​

Input: nums = [5,4,3,2,1]
Output: false

Example 3​

Input: nums = [2,1,5,0,4,6]
Output: true
Explanation: (1,4,6) with indices (1,4,5).
03

Pattern Recognition

→ Pattern Decision Tree
?
Can we track just two candidates instead of the full subsequence?
✓ Yes — We only need to know: what is the smallest first element, and what is the smallest valid second element given that first. If a third element exceeds both, we have a triplet.
?
Does updating first to a smaller value invalidate a previously found second?
✗ No — Even if first updates to something smaller (logically before second), the fact that second was reached means a valid pair existed. A new third element larger than second still forms a triplet.
?
Must we find the actual indices?
✗ No — Only existence matters — the greedy variables track the smallest achievable pair, not the actual indices.
Tap to reveal pattern →
04

Approach

Maintain first = Int.max and second = Int.max. For each num:

  • If num is less than first, update first.
  • Else if num is greater than first but less than second, update second.
  • Else if num is greater than both first and second, a triplet exists — return true.
The subtle invariant

When first gets updated to a smaller value, second still holds a value that was legitimately larger than an even earlier first. So if any subsequent number is larger than second, a valid triplet (old_first, second, num) exists — even if old_first's index is before the current first's index.

Step-by-Step Solution

Initialise two sentinels —

var first = Int.max, var second = Int.max.

Scan the array —

For each num: check in order — update first, then second, then check for triplet.

Update first —

If num less than first: first = num.

Update second —

Else if num greater than first and num less than second: second = num.

Return true —

Else if num greater than first and num greater than second: a triplet exists — return true.

Return false —

After the loop, no triplet found — return false.

07

Code Implementation

class Solution {
  func increasingTriplet(_ nums: [Int]) -> Bool {
    var first  = Int.max
    var second = Int.max

    for num in nums {
      if num > first, num < second {
        second = num
      } else if num < first {
        first = num
      } else if num > first, num > second {
        return true
      }
    }

    return false
  }
}

09

Common Pitfalls

✕01
Returning true when num equals first or second
The triplet requires strictly increasing values. A value equal to first or second does not extend the subsequence.
Fix: Use strict inequalities: num > first and num > second — never >=.
✕02
Misunderstanding first after it updates to a smaller value
When first is updated to a smaller value, second's original pair partner is 'lost' as far as indices go — but the invariant guarantees a valid (old first, second) pair still exists somewhere earlier in the array.
Fix: Trust the greedy invariant: if any num is greater than second, a valid triplet exists regardless of what first currently holds.
✕03
Checking num less than or equal to second before num less than first
Order of the conditions matters. Checking num less than first after the second check means an element that should update first might accidentally update second instead.
Fix: Check the second update condition first (num > first and num less than second), then the first update (num less than first), then the triplet check (num > second). Or arrange conditions as mutually exclusive branches.
