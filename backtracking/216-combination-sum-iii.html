Combination Sum III

ArrayBacktracking
01

Problem Overview

Find all valid combinations of k numbers that sum up to n such that:

  • Only numbers 1 through 9 are used.
  • Each number is used at most once.

Return a list of all possible valid combinations. The list must not contain the same combination twice, and the combinations may be returned in any order.

Constraints:

  • 2 <= k <= 9
  • 1 <= n <= 60

Output: []

Approach

valid result (sum==n, len==k)
pruned (sum > n)
in-progress path
result: [[1,4],[2,3]]

Use a recursive DFS over digits 1–9. At each level, choose the next digit (from startIndex to 9), add it to path and sum, recurse with startIndex = i + 1, then undo the choice. Record path when sum == n and path.count == k. Prune when sum > n.

Why startIndex prevents duplicates

By always starting the next loop at i + 1, we never pick the same digit twice and we always build combinations in ascending order. This eliminates the need for deduplication post-hoc.

Step-by-Step Solution

Initialise candidates —

nums = [1,2,3,4,5,6,7,8,9], path = [Int](), sum = 0, results = [[Int]]().

Base cases —

If sum == n and path.count == k, append path to results and return. If sum > n, return (prune).

Choose and recurse —

For i from startIndex to nums.count - 1: append nums[i], add to sum, recurse with i + 1, then undo.

Return results —

Return results after the initial dfs(0) call.

07

Code Implementation

class Solution {
  func combinationSum3(_ k: Int, _ n: Int) -> [[Int]] {
    var results = [[Int]]()
    let nums = Array(1...9)
    var path = [Int]()
    var sum  = 0

    func dfs(_ startIndex: Int) {
      if sum == n, path.count == k {
        results.append(path)
        return
      } else if sum > n {
        return
      }
      for i in startIndex..<nums.count {
        path.append(nums[i])
        sum += nums[i]
        dfs(i + 1)
        path.removeLast()
        sum -= nums[i]
      }
    }

    dfs(0)
    return results
  }
}

09

Common Pitfalls

✕01
Pruning only on sum > n but not on path.count > k
A path can reach the right sum with too many elements, or the right count with the wrong sum. Both conditions must be met simultaneously.
Fix: Only record a result when both sum == n AND path.count == k. Add an early return when path.count == k and sum is still wrong.
✕02
Recursing with i instead of i + 1
Passing i instead of i+1 allows the same number to be selected multiple times, producing combinations with repeated digits.
Fix: Always recurse with dfs(i + 1) to advance past the chosen index.
✕03
Not pruning when path.count already equals k
If path.count == k but sum != n, continuing the loop tries to add more elements beyond k.
Fix: Add if path.count == k { return } (or equivalently, check inside the base-case block) to stop early when the path is full.
