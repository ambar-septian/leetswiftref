Permutations

ArrayBacktracking
01

Problem Overview

Given an array nums of distinct integers, return all the possible permutations. You can return the answer in any order.

A permutation is an arrangement of all elements in every possible order. For n distinct elements there are n! permutations.

Constraints

  • 1 <= nums.length <= 6
  • -10 <= nums[i] <= 10
  • All the integers of nums are unique.
Core Insight

Unlike combination problems there is no startIndex — every position can pick any unused element. Use a boolean used[] array to track which elements are in the current path. When path.count == nums.count we have a complete permutation.

04

Approach

Maintain a path array and a used: [Bool] array of the same length as nums. The recursive function backTrack() (no index parameter) iterates over every index i from 0 to nums.count - 1. Skip i if used[i] is true. Otherwise: mark used[i] = true, append nums[i] to path, recurse, then undo — path.removeLast(), used[i] = false.

Base case: when path.count == nums.count, append a copy of path to results.

Why no startIndex is needed

Combination problems use startIndex to avoid counting [1,2] and [2,1] as different results. Permutations treat them as different — order matters. So every recursive call scans from index 0, relying solely on used[] to avoid picking the same element twice within one path.

06

Step-by-Step Solution

Initialise shared state —

results = [[Int]](), path = [Int](), used = Array(repeating: false, count: nums.count).

Base case —

When path.count == nums.count, append a copy of path to results and return.

Iterate all indices —

For i from 0 to nums.count - 1: if used[i], skip. Otherwise mark used[i] = true, append nums[i], call backTrack().

Backtrack —

After recursive call: path.removeLast(), used[i] = false.

07

Code Implementation

class Solution {
  func permute(_ nums: [Int]) -> [[Int]] {
    var results = [[Int]]()
    var path = [Int]()
    var used = Array(repeating: false, count: nums.count)

    func backTrack() {
      if path.count == nums.count {
        results.append(path)
        return
      }

      for i in 0..<nums.count {
        guard !used[i] else { continue }

        used[i] = true
        path.append(nums[i])

        backTrack()

        path.removeLast()
        used[i] = false
      }
    }

    backTrack()
    return results
  }
}

09

Common Pitfalls

✕01
Using startIndex like combination problems
Adding a startIndex parameter causes the loop to start past the current index, so elements before the current position are never considered for later positions — generating an incomplete result set.
Fix: Always iterate from 0 to nums.count - 1 and rely on used[] to exclude already-picked elements.
✕02
Forgetting to reset used[i] after backtracking
If used[i] stays true after the recursive call returns, future branches at the same level will incorrectly skip index i.
Fix: After path.removeLast(), always set used[i] = false to restore state before the next iteration.
✕03
Checking path.count == nums.count inside the loop
The base case must be checked at the top of backTrack(), before the loop. Checking inside the loop means we enter the loop even when the path is full.
Fix: Place the base case check at the very start of backTrack(), before any iteration.
