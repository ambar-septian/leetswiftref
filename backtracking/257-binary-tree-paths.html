Binary Tree Paths

StringBacktrackingTreeDepth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, return all root-to-leaf paths in any order.

A leaf is a node with no children.

Constraints:

  • The number of nodes in the tree is in the range [1, 100].
  • -100 <= Node.val <= 100
02

Examples

Example 1​

Input: root = [1,2,3,null,5]
Output: ["1->2->5","1->3"]

Example 2​

Input: root = [1]
Output: ["1"]
03

Pattern Recognition

→ Pattern Decision Tree
?
Does the path need to be tracked across recursive calls?
✓ Yes — Each DFS call extends the current path by one node; the path must be maintained as a mutable array shared across the recursion.
?
When should we record a path?
✓ Yes — A root-to-leaf path is complete when both children are nil — record then return.
?
Must we undo path changes on backtrack?
✓ Yes — After returning from a subtree, the node just explored must be popped so the path is clean for the next branch.
Tap to reveal pattern →
04

Approach

DFS backtracking · build path string at leaf · pop on return
"1->2->5"
"1->3"
DFS trace (click row):
path=[1]visit 1, push to path
path=[1,2]go left, visit 2, push
path=[1,2,5]go left, visit 5 (leaf) → record "1->2->5"
path=[1,2]backtrack: pop 5, pop 2
path=[1]backtrack to root
path=[1,3]go right, visit 3 (leaf) → record "1->3"
path=[]backtrack: pop 3, pop 1 — done

Maintain a path array. Push the current node's value as a string. If it is a leaf, join path with "->" and append to results. Otherwise recurse into left and right children. After all recursion from this node, pop the last element to restore the path for sibling branches.

Pop at the end vs pop after each child

Popping once at the end of the function (after both children) is equivalent to popping after each child — but the single-pop placement at the bottom is cleaner. It restores the path for the caller regardless of which branch was taken.

06

Step-by-Step Solution

Guard nil root —

Return [] if the root is nil.

Push current node value —

Append String(root.val) to path.

At leaf — record path —

If root.left == nil && root.right == nil, append path.joined(separator: "->") to results and pop, then return.

Recurse into children —

Call dfs(root.left) and dfs(root.right).

Pop on return —

path.removeLast() after both children have been explored.

07

Code Implementation

class Solution {
  func binaryTreePaths(_ root: TreeNode?) -> [String] {
    guard let root else { return [] }
    var results = [String]()
    var path    = [String]()

    func dfs(_ root: TreeNode?) {
      guard let root else { return }
      path.append(String(root.val))

      if root.left == nil, root.right == nil {
        results.append(path.joined(separator: "->"))
        path.removeLast()
        return
      }

      dfs(root.left)
      dfs(root.right)
      path.removeLast()
    }

    dfs(root)
    return results
  }
}

09

Common Pitfalls

✕01
Not popping after the leaf case
The leaf branch has an early return. Without a path.removeLast() before that return, the leaf node stays in path when control passes back to the parent.
Fix: Pop before returning at the leaf: results.append(...); path.removeLast(); return.
✕02
Recording at every node instead of only at leaves
A root-to-leaf path ends at a leaf. Recording at every node creates partial paths.
Fix: Only record when root.left == nil && root.right == nil.
✕03
Using string concatenation instead of a path array
Building a String directly (e.g. path + '->' + val) copies the entire string on each call — O(n×h) extra allocations. It also makes backtracking harder since strings are value types.
Fix: Use a mutable [String] array and join only at the leaf.
