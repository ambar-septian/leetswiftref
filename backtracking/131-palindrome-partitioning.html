Palindrome Partitioning

StringDynamic ProgrammingBacktracking
01

Problem Overview

Given a string s, partition s such that every substring of the partition is a palindrome. Return all possible palindrome partitionings of s.

A palindrome is a string that reads the same forward and backward.

Constraints:

  • 1 <= s.length <= 16
  • s consists of only lowercase English letters.
02

Examples

Example 1​

Input: s = "aab"
Output: [["a","a","b"],["aa","b"]]

Example 2​

Input: s = "a"
Output: [["a"]]

Use a dfs(i) function that tries every substring s[i...j] for j from i to s.count - 1. If the substring is a palindrome, append it to the current partition and recurse with dfs(j + 1). On return, remove the last element. When i >= s.count, record a copy of the current partition.

A cache [[Int]: String] maps [i, j] to the precomputed substring so that repeated checks on the same range are O(1).

Why backtracking works here

Each position in the string is either a cut point or not. By trying every palindromic prefix and recursing on the remainder, we guarantee that every combination is explored without repetition.

06

Step-by-Step Solution

Convert to character array —

var chars = Array(s) for O(1) index access in the palindrome check.

Define dfs(i) —

Base case: if i >= chars.count, append a copy of part to results and return.

Try every ending index j —

For j in i..<chars.count: check the cache [[i,j]] first, else call isPalindrome(i, j, chars).

Append and recurse —

If palindrome: build the substring, store in cache, append to part, recurse dfs(j+1), then part.removeLast().

isPalindrome helper —

Two-pointer check: expand inward from l and r, return false on the first mismatch.

07

Code Implementation

class Solution {
  func partition(_ s: String) -> [[String]] {
    var results = [[String]]()
    var part    = [String]()
    let chars   = Array(s)
    var cache   = [[Int]: String]()

    func dfs(_ i: Int) {
      if i >= chars.count {
        results.append(part)
        return
      }

      for j in i..<chars.count {
        if let exist = cache[[i, j]] {
          part.append(exist)
          dfs(j + 1)
          part.removeLast()
        } else if isPalindrome(i, j, chars) {
          let sub = String(chars[i...j])
          cache[[i, j]] = sub
          part.append(sub)
          dfs(j + 1)
          part.removeLast()
        }
      }
    }

    dfs(0)
    return results
  }

  private func isPalindrome(_ l: Int, _ r: Int, _ s: [Character]) -> Bool {
    var l = l, r = r
    while l <= r {
      if s[l] != s[r] { return false }
      l += 1; r -= 1
    }
    return true
  }
}

09

Common Pitfalls

✕01
Forgetting to removeLast after recursion
Not removing the last element after the recursive call leaves stale substrings in the part array and produces incorrect results.
Fix: Always call part.removeLast() immediately after the recursive dfs(j+1) call — both in the cache branch and the fresh-check branch.
✕02
Appending part without copying
In Swift, results.append(part) copies the array (value semantics), so this is safe. In Python or Java, you must write results.append(list(part)).
Fix: In Swift no extra step is needed; in reference-semantics languages always copy before appending.
✕03
Cache key collision for different (i, j) pairs
Using a flat integer key like i * n + j can collide if n is not chosen carefully. The [[Int]: String] dictionary with a two-element array key is unambiguous.
Fix: Use the two-element array key [i, j] as shown, or encode as i * (s.count + 1) + j if using an integer key.
