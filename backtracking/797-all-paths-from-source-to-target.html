All Paths From Source to Target

BacktrackingDepth-First SearchBreadth-First SearchGraph
01

Problem Overview

Given a directed acyclic graph (DAG) of n nodes labelled from 0 to n - 1, find all possible paths from node 0 to node n - 1 and return them in any order.

The graph is given as follows: graph[i] is a list of all nodes you can visit from node i (i.e., there is a directed edge from node i to each node in graph[i]).

Constraints:

  • n == graph.length
  • 2 <= n <= 15
  • 0 <= graph[i][j] < n
  • graph[i][j] != i (no self-loops)
  • All the values of graph[i] are unique.
  • The input graph is guaranteed to be a DAG.

04

Approach

Start DFS from node 0 with list = [0]. At each node i: if i == n-1 (target), append a copy of list to result and return. Otherwise, for each neighbor in graph[i]: append neighbor, recurse, remove neighbor. Because it is a DAG, no visited set is needed.

No visited set needed

The graph is guaranteed to be a DAG (no cycles). DFS on a DAG always terminates — every path reaches the target or a dead end. Adding a visited set would be unnecessary overhead.

Step-by-Step Solution

Initialise —

var result = [[Int]](), var list = [0], let n = graph.count - 1.

Define DFS —

func dfs(_ i: Int): if i == n, append list to result and return.

Explore neighbors —

For each neighbor in graph[i]: list.append(neighbor); dfs(neighbor); list.removeLast().

Call and return —

dfs(0). Return result.

07

Code Implementation

class Solution {
  func allPathsSourceTarget(_ graph: [[Int]]) -> [[Int]] {
    var result = [[Int]]()
    var list = [0]
    let n = graph.count - 1

    func dfs(_ i: Int) {
      if i == n {
        result.append(list)
        return
      }
      for neighbor in graph[i] {
        list.append(neighbor)
        dfs(neighbor)
        list.removeLast()
      }
    }

    dfs(0)
    return result
  }
}

09

Common Pitfalls

✕01
Appending list reference instead of a copy
If you append list directly (by reference), all recorded paths will reflect the final state of list when iteration ends.
Fix: In Swift, arrays are value types — appending list copies it automatically. In languages with reference semantics (Python list), use list[:] or list.copy().
✕02
Adding a visited set unnecessarily
The graph is a DAG — cycles are impossible. Adding a visited set adds complexity and can incorrectly skip valid paths if a node is reachable by multiple routes.
Fix: Remove the visited set entirely for DAG problems. DFS terminates naturally when it reaches the target or a node with no neighbors.
✕03
Checking i == graph.count instead of i == n-1
The target node label is n-1 (0-indexed), not n. Off-by-one misses the target or checks out-of-bounds.
Fix: Set n = graph.count - 1 and check i == n as the base case.
