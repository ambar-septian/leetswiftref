Combination Sum II

ArrayBacktracking
01

Problem Overview

Given a collection of candidate numbers (candidates) and a target number (target), find all unique combinations in candidates where the candidate numbers sum to target.

Each number in candidates may only be used once in the combination. The solution set must not contain duplicate combinations.

Constraints

  • 1 <= candidates.length <= 100
  • 1 <= candidates[i] <= 50
  • 1 <= target <= 30
Core Insight

Sort the candidates first. Use dfs(i+1) to prevent reusing the same element. Skip duplicates at the same recursion level with if i > index && candidates[i] == candidates[i-1] — this prevents generating identical combinations from different positions.

Output: [[1,1,6],[1,2,5],[1,7],[2,6]]

Example 2​

Input:  candidates = [2,5,2,1,2], target = 5
Output: [[1,2,2],[5]]

04

Approach

Sort candidates so that duplicates are adjacent. Maintain a path array and a sum running total. The recursive function dfs(_ index) iterates from index to the end:

  • For each i, if i > index and candidates[i] == candidates[i-1], skip — this is a duplicate at the same recursion level.
  • Otherwise, append, recurse with dfs(i+1) (advance past the current element — no reuse), then backtrack.

Why the skip condition uses i > index

When i == index we are picking the first element at this level — always allowed. The skip only fires when i > index, meaning we have already processed candidates[i-1] at this same level and are about to pick an identical value again, which would produce a duplicate combination.

Step-by-Step Solution

Sort the candidates —

var cn = candidates.sorted() — groups duplicates together so the skip rule works.

Base cases —

sum == target → append copy of path and return. sum > target → prune and return.

Iterate from index, skipping duplicates at the same level —

For i from index to cn.count - 1: if i > index && cn[i] == cn[i-1], skip. Otherwise append, add to sum, recurse dfs(i+1).

Backtrack —

After each recursive call: path.removeLast(), sum -= cn[i].

07

Code Implementation

class Solution {
  func combinationSum2(_ candidates: [Int], _ target: Int) -> [[Int]] {
    var results = [[Int]]()
    var path = [Int]()
    var sum = 0
    var cn = candidates.sorted()

    func dfs(_ index: Int) {
      if sum == target {
        results.append(path)
        return
      } else if sum > target {
        return
      }

      for i in index..<cn.count {
        let c = cn[i]
        if i > index, c == cn[i - 1] {
          continue               // skip duplicate at same level
        }

        path.append(c)
        sum += c
        dfs(i + 1)              // i+1: each element used at most once

        path.removeLast()
        sum -= c
      }
    }

    dfs(0)
    return results
  }
}

09

Common Pitfalls

✕01
Using dfs(i) instead of dfs(i+1)
Passing i instead of i+1 allows the same element to be reused, producing invalid combinations that exceed the single-use constraint.
Fix: Always pass i+1 to prevent reusing the element at position i.
✕02
Wrong skip condition: i > startIndex vs i > index
The condition must use the parameter name that represents the start of the current loop — not a fixed 0. Using 0 would skip the second 1 even at the very first call level.
Fix: Use i > index (the parameter passed into dfs), not i > 0.
✕03
Forgetting to sort before the skip rule
The duplicate-skip rule relies on identical values being adjacent. Without sorting, candidates[i] == candidates[i-1] may fire incorrectly or miss real duplicates.
Fix: Always sort candidates before entering the DFS.
