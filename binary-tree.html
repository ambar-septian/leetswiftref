Binary Tree

A binary tree is a recursive structure where each node has at most two children. Almost every binary tree problem reduces to a traversal — DFS (inorder, preorder, postorder) or BFS (level order) — combined with a local computation at each node. The recursive structure of the tree mirrors the recursive structure of the algorithm.

01

What is a Binary Tree?

A binary tree is defined recursively: a node with a value, an optional left subtree, and an optional right subtree. The base case is nil (empty tree). This structure makes recursive algorithms natural — the algorithm for the whole tree is the same as the algorithm for each subtree.

PropertyDescription
HeightLongest path from root to a leaf — O(log n) for balanced, O(n) worst case
Depth of nodeDistance from root to that node
LeafNode with no children
Complete treeAll levels full except last; last level filled left to right
Perfect treeAll internal nodes have two children; all leaves at same depth
BalancedHeight difference between left and right subtrees is at most 1 (AVL definition)
Key Constraint

Always handle the nil / empty tree base case first in recursive functions. In Swift, the pattern is guard let node = root else { return /* base value */ }. Forgetting the base case causes a crash on nil input or an infinite recursion on a leaf node. For problems that return a numeric value, the empty-tree base case is usually 0, -1, Int.min, or true depending on the problem semantics.

Rule of Thumb

If the problem requires processing nodes in a specific order (sorted, level by level, parent before children, children before parent) — choose the traversal that naturally gives that order. For problems that need to combine information from both subtrees (diameter, LCA, path sum) — use postorder so left and right results are available when the root is processed.
