Search in a Binary Search Tree

TreeBinary Search TreeBinary Tree
01

Problem Overview

You are given the root of a binary search tree (BST) and an integer val.

Find the node in the BST that the node's value equals val and return the subtree rooted with that node. If such a node does not exist, return null.

Constraints:

  • The number of nodes in the tree is in the range [1, 5000].
  • 1 <= Node.val <= 10^7
  • root is a binary search tree.
  • 1 <= val <= 10^7
02

Examples

Example 1​

Input:  root = [4,2,7,1,3], val = 2
Output: [2,1,3]
Explanation: Return the subtree rooted at node 2.

Example 2​

Input:  root = [4,2,7,1,3], val = 5
Output: []
Explanation: Node with val 5 does not exist.

04

Approach

Start with next = root. Each iteration: if curr.val == val, return curr. If curr.val > val, go left (next = curr.left). If curr.val < val, go right (next = curr.right). If next becomes nil, the node was not found — return nil.

BST invariant

At every node, all values in the left subtree are strictly less than the node value, and all values in the right subtree are strictly greater. This means we never need to explore the wrong subtree — one comparison eliminates half the remaining tree.

Step-by-Step Solution

Initialise pointer —

var next = root.

Loop while node exists —

while let curr = next.

Compare and navigate —

If curr.val == val, return curr. If curr.val > val, next = curr.left. Else next = curr.right.

Not found —

Return next (which is nil when the loop exits).

07

Code Implementation

class Solution {
  func searchBST(_ root: TreeNode?, _ val: Int) -> TreeNode? {
    var next = root

    while let curr = next {
      if curr.val == val {
        return curr
      }
      if curr.val > val {
        next = curr.left
      } else {
        next = curr.right
      }
    }

    return next
  }
}

09

Common Pitfalls

✕01
Returning nil instead of the matching node
Forgetting to return curr when the value matches exits the loop and returns nil.
Fix: Check curr.val == val first inside the loop and return curr immediately.
✕02
Going right when val is smaller
Confusing the direction (left vs right) wastes O(n) time or misses the node entirely.
Fix: Go left when curr.val is greater than val (target is in the smaller side), right when curr.val is less than val.
✕03
Using DFS recursion unnecessarily
A recursive solution works but uses O(h) call-stack space. The iterative approach is strictly better here.
Fix: Use the iterative while-loop — same logic, O(1) extra space.
