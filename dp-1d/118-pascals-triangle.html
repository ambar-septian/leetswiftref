Pascal's Triangle

ArrayDynamic Programming
01

Problem Overview

Given an integer numRows, return the first numRows rows of Pascal's triangle.

In Pascal's triangle, each number is the sum of the two numbers directly above it. The first and last element of every row are always 1.

Constraints:

  • 1 <= numRows <= 30
Key Constraint

numRows can be as small as 1 — the single-row case must be handled. The first and last element of every row are always 1. All values fit comfortably in a 32-bit integer for numRows up to 30.

Output: [[1],[1,1],[1,2,1],[1,3,3,1],[1,4,6,4,1]]

Example 2​

Output: [[1]]

Rule of Thumb

Pascal's triangle is a pure DP recurrence: each interior element is the sum of the two elements directly above. Initialise rows as all-1s and only overwrite interior positions — boundaries handle themselves.

04

Approach

Core Insight

Each row is initialised as an array of 1s. Interior elements at index j are computed as result[i-1][j-1] + result[i-1][j] — reading from the completed previous row. The first and last positions stay 1, so no boundary special-casing is needed.

Initialise each new row as an array of 1s of length i + 1. Iterate over the interior positions j from 1 to rs.count - 2 and set rs[j] = result[i-1][j-1] + result[i-1][j]. Append the completed row to result.

Why initialise with all 1s

The first and last element of every row are always 1. By starting with all 1s, the boundary conditions are handled automatically and the loop only needs to touch interior elements.

06

Step-by-Step Solution

Handle single row —

If numRows == 1, return [[1]] directly.

Seed row 0 —

result[0] = [1].

Build each row —

For i from 1 to numRows - 1, create rs = Array(repeating: 1, count: i + 1).

Fill interior —

For j from 1 to rs.count - 2, set rs[j] = result[i-1][j-1] + result[i-1][j].

Append row —

result[i] = rs.

07

Code Implementation

class Solution {
  func generate(_ numRows: Int) -> [[Int]] {
    guard numRows > 1 else { return [[1]] }

    var result = Array(repeating: [Int](), count: numRows)
    result[0] = [1]

    for i in 1..<numRows {
      var rs = Array(repeating: 1, count: i + 1)
      for j in 1..<rs.count - 1 {
        rs[j] = result[i - 1][j - 1] + result[i - 1][j]
      }
      result[i] = rs
    }

    return result
  }
}
Swift Notes

Array(repeating: 1, count: i + 1) creates each row pre-filled with 1s — both the boundary values and the interior defaults. result[i] = rs assigns the completed row by value copy. Pre-allocating result with Array(repeating: [Int](), count: numRows) lets rows be assigned by index rather than appended.

08

Complexity Analysis

Time Complexity — O(n²)

O(n²) — the total number of elements across all n rows is 1 + 2 + ... + n = n(n+1)/2. Each element is computed in O(1), giving O(n²) total work.

Space Complexity — O(n²)

O(n²) for the output — all n rows must be returned and stored. If only the last row were needed, space could be reduced to O(n) by keeping only the previous row, but the problem requires the full triangle.

09

Common Pitfalls

✕01
Off-by-one on the interior loop
The interior loop should run from index 1 to rs.count - 2 inclusive. Using j less than rs.count would overwrite the trailing 1.
Fix: Use for j in 1..<rs.count - 1 to skip the first and last positions.
✕02
Reading from the current row instead of the previous row
If you update in-place using the same array, computing rs[j] = rs[j-1] + rs[j] would read an already-modified value for rs[j-1].
Fix: Always read from result[i-1] (the previous completed row), not from rs which is being built.
✕03
Missing the numRows == 1 guard
The loop starts at i = 1, so if numRows == 1 the loop body never executes and the pre-allocated result[0] would be empty.
Fix: Seed result[0] = [1] before the loop and guard for numRows == 1 to return early.
