Largest Rectangle in Histogram

ArrayStackMonotonic Stack
01

Problem Overview

Given an array of integers heights representing the histogram's bar height where the width of each bar is 1, return the area of the largest rectangle in the histogram.

A rectangle must be formed by consecutive bars and its height is limited by the shortest bar in the span. You must find the maximum area over all possible contiguous spans.

Constraints

  • 1 <= heights.length <= 10^5
  • 0 <= heights[i] <= 10^4
Core Insight

Maintain a monotonically increasing stack of (startIndex, height) pairs. When a shorter bar arrives, every taller bar on the stack can no longer extend rightward — pop each one and compute its rectangle. The popped bar's startIndex also becomes the new bar's leftward reach, since the new shorter bar could have started there.

04

Approach

Iterate left to right. Maintain a stack of (startIndex, height) pairs in strictly increasing height order. For each bar at index i with height h:

  • Set start = i.
  • While the stack top has height > h: pop it, compute area = popped.height * (i - popped.index), update maxArea, and set start = popped.index (the new bar can reach back to where the taller bar started).
  • Push (start, h) onto the stack.

After the loop, drain remaining stack entries: each spans from its index to the end of the array, so area = item.height * (heights.count - item.index).

Why start is updated when popping

When bar at index 2 (height=5) is popped because bar at index 4 (height=2) arrived, the height-2 bar could have extended all the way back to index 2 — because everything from index 2 onward was at least height 2. By setting start=2, the new entry (2, 2) correctly represents this wider rectangle.

Step-by-Step Solution

Initialise stack and maxArea —

var maxArea = 0, var stack = [(index: Int, height: Int)]().

Iterate bars, pop taller entries —

For each i: set start = i. While stack.last.height > heights[i]: pop, compute area, update maxArea, set start = popped.index. Push (start, heights[i]).

Drain remaining stack —

After the loop, for each entry in stack: area = item.height * (heights.count - item.index). Update maxArea.

Return maxArea —

Return the accumulated maximum.

07

Code Implementation

class Solution {
  func largestRectangleArea(_ heights: [Int]) -> Int {
    var maxArea = 0
    var stack = [(index: Int, height: Int)]()

    for i in 0..<heights.count {
      var start = i
      let height = heights[i]

      while let top = stack.last, top.height > height {
        let (index, h) = stack.removeLast()
        maxArea = max(maxArea, h * (i - index))
        start = index   // new bar can extend back to popped bar's start
      }

      stack.append((start, height))
    }

    // Drain: remaining bars extend to the right edge
    for item in stack {
      maxArea = max(maxArea, item.height * (heights.count - item.index))
    }

    return maxArea
  }
}

Common Pitfalls

✕01
Forgetting to drain the stack after the loop
Bars that were never popped during the loop (because no shorter bar arrived to their right) represent rectangles that extend all the way to the right edge. Skipping the drain misses these.
Fix: After the main loop, iterate remaining stack entries and compute area = item.height * (heights.count - item.index).
✕02
Not updating start when popping
When a taller bar is popped, the current (shorter) bar can extend back to where the taller bar originated. Forgetting to update start leaves the new bar's left boundary too narrow.
Fix: Inside the while-pop loop, set start = popped.index after each pop, then push (start, height) at the end.
✕03
Computing area as height * i instead of height * (i - index)
The width of the rectangle is not i itself but the distance from the bar's (possibly extended) start index to the current index.
Fix: Area = popped.height * (i - popped.index). For drain: area = item.height * (heights.count - item.index).
