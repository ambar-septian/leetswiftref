Next Greater Element I

ArrayHash TableStackMonotonic Stack
01

Problem Overview

The next greater element of some element x in an array is the first greater element that is to its right in the same array.

You are given two distinct integer arrays nums1 and nums2, where nums1 is a subset of nums2.

For each element in nums1, find the next greater element in nums2 and return an array of the answers. If there is no next greater element, output -1 for that position.

Constraints:

  • 1 <= nums1.length <= nums2.length <= 1000
  • 0 <= nums1[i], nums2[i] <= 10^4
  • All integers in nums1 and nums2 are unique.
  • All the integers of nums1 also appear in nums2.
02

Examples

Example 1​

Input:  nums1 = [4,1,2], nums2 = [1,3,4,2]
Output: [-1,3,-1]
Explanation:
  4 → no element to its right is greater → -1
  1 → next greater in nums2 is 3 → 3
  2 → no element to its right is greater → -1

Example 2​

Input:  nums1 = [2,4], nums2 = [1,2,3,4]
Output: [3,-1]
Explanation:
  2 → next greater is 3 → 3
  4 → no next greater → -1

04

Approach

First, build a hash map dict from each value in nums1 to its index in the result array. Initialise result with -1 for every position. Iterate through nums2 with a decreasing monotonic stack. When the current element curr is greater than the stack's top, pop from the stack: if the popped value is in dict, record curr as its next-greater answer. At the end, every element not popped kept its default -1.

Why only push nums1 values onto the stack

We only need next-greater answers for values that appear in nums1. Pushing only those values means the stack never holds irrelevant elements, keeping both space and pop-check logic minimal.

06

Step-by-Step Solution

Build index map for nums1 —

Map each nums1[i] to index i in a dictionary dict.

Initialise result array —

var result = Array(repeating: -1, count: nums1.count).

Scan nums2 with a monotonic stack —

For each curr in nums2: while stack.last exists and curr > stack.last, pop the top value. If it is in dict, set result[dict[val]!] = curr.

Push only nums1 values —

After popping, if dict[curr] != nil, push curr onto the stack.

Return result —

Elements never popped retain -1 from initialisation.

07

Code Implementation

class Solution {
  func nextGreaterElement(_ nums1: [Int], _ nums2: [Int]) -> [Int] {
    var dict = [Int: Int]()
    for i in 0..<nums1.count {
      dict[nums1[i]] = i
    }

    var stack = [Int]()
    var result = Array(repeating: -1, count: nums1.count)

    for curr in nums2 {
      while let last = stack.last, curr > last {
        let val = stack.removeLast()
        if let index = dict[val] {
          result[index] = curr
        }
      }
      if dict[curr] != nil {
        stack.append(curr)
      }
    }

    return result
  }
}

09

Common Pitfalls

✕01
Pushing all nums2 values onto the stack
Pushing values not in nums1 wastes stack space and requires an extra dict-check on every pop. It still gives correct results but pollutes the stack.
Fix: Only push a value if it appears in nums1 (dict[curr] != nil). Non-nums1 values have no corresponding result slot.
✕02
Returning 0 instead of -1 for elements with no next greater
Initialising result with zeros means values without a next-greater element get 0 instead of the required -1.
Fix: Initialise result with Array(repeating: -1, count: nums1.count).
✕03
Using value index in nums2 instead of nums1
The result array is indexed by position in nums1, not position in nums2. Using the nums2 index produces out-of-bounds or wrong results.
Fix: Use dict[val] — which maps the popped value to its index in nums1 — to place the answer in the correct result slot.
