Online Stock Span

#901MediumMonotonic StackLeetCode ↗

Online Stock Span

StackDesignMonotonic StackData Stream
01

Problem Overview

Design an algorithm that collects daily price quotes for some stock and returns the span of that stock's price for the current day.

The span of the stock's price in one day is the maximum number of consecutive days (starting from that day and going backward) for which the stock price was less than or equal to today's price.

Implement the StockSpanner class:

  • StockSpanner() — Initializes the object of the class.
  • int next(int price) — Returns the span of the stock's price given that today's price is price.

Constraints:

  • 1 <= price <= 10^5
  • At most 10^4 calls will be made to next.
02

Examples

Example 1​

Input:  ["StockSpanner","next","next","next","next","next","next","next"]
        [[],[100],[80],[60],[70],[60],[75],[85]]
Output: [null,1,1,1,2,1,4,6]

Explanation:
  next(100) → 1. next(80) → 1. next(60) → 1. next(70) → 2 (70≥60).
  next(60) → 1. next(75) → 4 (75≥60,70,60). next(85) → 6.
03

Pattern Recognition

→ Pattern Decision Tree
?
Does a new price absorb all previous prices that are smaller or equal?
✓ Yes — When a higher price arrives, it can span over all consecutive days with prices no greater than itself — those days' spans merge into the new span.
?
Can we store (price, cumulativeSpan) pairs on a stack?
✓ Yes — Instead of re-scanning all past days, store the pre-accumulated span at each stack entry. Pop and add spans when the current price is larger.
?
Is this a monotonically decreasing stack?
✓ Yes — The stack remains non-increasing in price — we only pop entries whose price is less than or equal to the current price.
Tap to reveal pattern →
04

Approach

Maintain a stack of (price, span) tuples. For each new price: start span = 1. While the stack is not empty and price >= stack.last.price: pop and add stack.last.span to span. Push (price, span) and return span. The stack acts as a compressed representation — each entry covers the span of all days it absorbed.

Why store cumulative span?

When a high price absorbs several lower prices, we do not want to re-examine every individual day. Storing the cumulative span in each stack entry lets us merge O(1) per pop — instead of O(k) where k is the number of absorbed days.

06

Step-by-Step Solution

Initialise stack —

var stack = [(Int, Int)]() — each element is (price, span).

next(price) —

var span = 1.

Pop smaller or equal prices —

While !stack.isEmpty and price >= stack.last!.0: span += stack.removeLast().1.

Push and return —

stack.append((price, span)). Return span.

07

Code Implementation

class StockSpanner {
  var stack = [(Int, Int)]()  // (price, span)

  init() {}

  func next(_ price: Int) -> Int {
    var span = 1
    while !stack.isEmpty, let last = stack.last, price >= last.0 {
      span += last.1
      stack.removeLast()
    }
    stack.append((price, span))
    return span
  }
}

09

Common Pitfalls

✕01
Using strictly greater than for the pop condition
The span includes days with prices strictly less than or equal to today. Using price > stack.last.price misses equal-price days.
Fix: Use price >= stack.last.0 to pop days with equal prices.
✕02
Not accumulating the popped span
If you pop stack entries but forget to add their span to the running total, you lose the days they represented.
Fix: span += last.1 before (or immediately after) the pop — add the popped entry's span to the running span.
✕03
Re-scanning all past prices naively
A naive loop over all previous prices to count the span is O(n) per call and O(n^2) total.
Fix: The stack with cumulative spans amortises this to O(1) per call — no re-scanning needed.
