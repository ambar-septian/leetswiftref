Bit Manipulation Patterns

Core patterns using bitwise operators: parity check (n & 1), integer halving (n >> 1), lowest-set-bit clear (n & (n-1)), and counting steps to zero by alternately halving (even) or decrementing (odd). All operations are O(1) per step, with O(log n) steps total proportional to the number of bits.

01

Overview

Bit manipulation replaces division, modulo, and flag-checking with single-instruction bitwise operations. The key patterns appear repeatedly:

  • n & 1 — check if n is odd (last bit is 1).
  • n >> 1 — divide by 2, equivalent to n / 2 for non-negative integers.
  • n & (n - 1) — clear the lowest set bit (used to count set bits in O(popcount) steps).
  • n ^ n == 0 — XOR of a value with itself is 0 (finds the unique element among duplicates).
  • 1 << k — the integer with only bit k set (used as a mask).
Core Insight

For "steps to reduce n to zero": if n is even, dividing by 2 is a right shift (1 step). If n is odd, subtracting 1 clears the last bit (1 step). Counting these steps is equivalent to counting the number of bits in n plus the number of set bits (1-bits): each 0 bit requires one right-shift step, and each 1 bit (except the leading 1) requires one subtract-1 step plus one right-shift step.

Key Constraint

Swift does not allow shifting by a value that exceeds or equals the bit width — 1 << 64 on a 64-bit Int is undefined behaviour and will crash in debug mode. Always guard shifts: k &lt; MemoryLayout&lt;Int&gt;.size * 8. For LeetCode problems with 32-bit constraints, use Int32 explicitly or mask with & 0xFFFFFFFF when needed.

Implementation

Number of Steps to Reduce to Zero — parity-driven simulation:

func numberOfSteps(_ num: Int) -> Int {
    var n = num
    var steps = 0
    while n > 0 {
        if n & 1 == 1 {
            n -= 1      // odd: subtract 1 (clears last bit)
        } else {
            n >>= 1     // even: divide by 2 (right shift)
        }
        steps += 1
    }
    return steps
}

Concise version using bit counting:

func numberOfStepsConcise(_ num: Int) -> Int {
    guard num > 0 else { return 0 }
    // Each bit position contributes 1 step for the shift.
    // Each 1-bit (set bit) contributes an extra step for the subtract.
    // The leading 1 only needs one subtract (no extra shift after it).
    let bits = Int.bitWidth - num.leadingZeroBitCount  // number of bits
    let setBits = num.nonzeroBitCount                  // number of 1-bits
    return (bits - 1) + (setBits - 1) + 1             // simplifies to:
    // return bits + setBits - 1
}

Common bit tricks — reference:

// Check even/odd
let isOdd = n & 1 == 1

// Divide by 2 (non-negative integers)

// Count set bits (1-bits)
let ones = n.nonzeroBitCount

// Clear lowest set bit (Brian Kernighan's algorithm)
var x = n
var count = 0
while x > 0 { x &= x - 1; count += 1 }
// count == n.nonzeroBitCount

// XOR to find unique element (all others appear twice)
let unique = nums.reduce(0, ^)

// Check if n is a power of 2
let isPowerOf2 = n > 0 && n & (n - 1) == 0
Swift Notes

Swift provides built-in bit intrinsics on integer types: n.nonzeroBitCount (popcount), n.leadingZeroBitCount, n.trailingZeroBitCount. Use &+, &-, &* for overflow-safe arithmetic when the problem works with values near Int.max. The ^ operator in Swift is XOR — use reduce(0, ^) on an array to XOR all elements together cleanly.

05

Complexity Analysis

Time Complexity — O(log n)

The loop in numberOfSteps runs once per bit of n. n has floor(log₂ n) + 1 bits, so the loop runs at most 2 * floor(log₂ n) + 1 times (worst case: a number like 0b111...1 needs one subtract and one shift per bit). Total time is O(log n). The concise version using nonzeroBitCount and leadingZeroBitCount is O(1) — these are single CPU instructions (popcount, lzcnt).

Space Complexity — O(1)

Only a constant number of integer variables are used — no arrays or recursion. All bit operations work in-place on the integer value.

When to Use

Rule of Thumb

Use bit manipulation when the problem works with integers at the binary level: checking even/odd, dividing by 2, counting 1-bits, or toggling flags. The key benefit is constant-time O(1) operations that replace O(log n) or O(n) arithmetic. When you see "reduce to zero by halving or decrement", "count bits", or "find unique among duplicates" — reach for the bit toolkit.
