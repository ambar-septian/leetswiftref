<SectionHeader n={1} title="Problem Overview" />

Given an array of integers `temperatures` representing the daily temperatures, return an array `answer` such that `answer[i]` is the number of days you have to wait after the `i`-th day to get a warmer temperature. If there is no future day for which this is possible, keep `answer[i] == 0` instead.

**Constraints:**
- `1 <= temperatures.length <= 10^5`
- `30 <= temperatures[i] <= 100`

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  temperatures = [73,74,75,71,69,72,76,73]
Output: [1,1,4,2,1,1,0,0]
```

#### Example 2

```
Input:  temperatures = [30,40,50,60]
Output: [1,1,1,0]
```

#### Example 3

```
Input:  temperatures = [30,60,90]
Output: [1,1,0]
```

<SectionHeader n={4} title="Approach" />

Maintain a stack of indices (temperatures stored implicitly). For each index `i`: while the stack is not empty and `temperatures[i] > temperatures[stack.last!]`, pop `topIndex` and set `results[topIndex] = i - topIndex`. Push `i` onto the stack. Indices remaining in the stack at the end have no warmer future day — they keep `results[i] = 0` (the array default).

<Callout type="cyan" label="Store indices, not temperatures">
Storing indices on the stack lets us compute the day gap directly as currentIndex - poppedIndex. If we stored temperatures instead we would still need to map back to indices.
</Callout>

  </StepItem>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func dailyTemperatures(_ temperatures: [Int]) -> [Int] {
    var stack = [Int]()
    var results = Array(repeating: 0, count: temperatures.count)

    for i in 0..<temperatures.count {
      while !stack.isEmpty, let topIndex = stack.last,
            temperatures[i] > temperatures[topIndex] {
        stack.removeLast()
        results[topIndex] = i - topIndex
      }
      stack.append(i)
    }

    return results
  }
}
```
