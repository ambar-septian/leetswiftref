<SectionHeader n={1} title="What is a Monotonic Stack?" />

A monotonic stack is a regular stack with one extra invariant: elements are always kept in sorted order (either all increasing or all decreasing from bottom to top). The trick is that this invariant is maintained lazily — when a new element arrives, we pop everything that violates the order before pushing.

| Property | Monotonic Increasing Stack | Monotonic Decreasing Stack |
|---|---|---|
| Stack order | bottom → top is increasing | bottom → top is decreasing |
| When to pop | new element is smaller than top | new element is larger than top |
| What pop reveals | next smaller element for popped item | next greater element for popped item |
| Typical use | next smaller, trapping rain water | next greater, daily temperatures |

<Callout type="amber" label="Key Constraint">
  Each element is pushed and popped at most once, so the total work across all pops is O(n) even though there is a nested loop structure. Never mistake the nested `while` for O(n²) — the amortised cost is O(1) per element.
</Callout>

<Callout type="green" label="Rule of Thumb">
  If the problem asks for the next (or previous) element that is greater or smaller than the current one — reach for a monotonic stack. If it involves areas between bars or water levels, a monotonic stack is almost certainly the right tool.
</Callout>
