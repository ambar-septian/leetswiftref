<SectionHeader n={1} title="Problem Overview" />

You are given an integer array `prices` where `prices[i]` is the price of the `i`-th item in a shop.

There is a special discount for items in the shop. If you buy the `i`-th item, then you will receive a discount equivalent to `prices[j]` where `j` is the minimum index such that `j > i` and `prices[j] <= prices[i]`. Otherwise, you will not receive any discount at all.

Return an integer array `answer` where `answer[i]` is the final price you will pay for the `i`-th item of the shop considering the special discount.

**Constraints:**
- `1 <= prices.length <= 500`
- `1 <= prices[i] <= 10^3`

<Callout type="amber" label="Key Constraint">
  The discount applies when prices[j] &lt;= prices[i] — less than or equal, not strictly less than. Missing equality causes wrong answers for equal-price pairs. The pop condition must use &lt;= not &lt;.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  prices = [8,4,6,2,3]
Output: [4,2,4,2,3]
Explanation:
  8 → next price <=8 is 4 (index 1) → 8-4=4
  4 → next price <=4 is 2 (index 3) → 4-2=2
  6 → next price <=6 is 2 (index 3) → 6-2=4
  2 → no next price <=2 after index 3 → 2
  3 → no next price → 3
```

#### Example 2

```
Input:  prices = [1,2,3,4,5]
Output: [1,2,3,4,5]
Explanation: No item has a cheaper or equal item after it.
```

#### Example 3

```
Input:  prices = [10,1,1,6]
Output: [9,0,1,6]
```

  "Find the next smaller-or-equal element to the right" is the monotonic stack trigger. Maintain a non-decreasing stack of (value, index) pairs; pop and apply the discount when the current price is &lt;= the stack top.

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Maintain a stack of (value, index) pairs in non-decreasing order. When a new price arrives that is &lt;= the stack top, it is the discount for all entries at the top — pop and apply. Entries remaining after the full loop received no discount and keep their original price.
</Callout>

Maintain a stack of `(value, index)` pairs in non-decreasing order. When `prices[i] <= stack.top.value`, the top entry's item receives a discount of `prices[i]`. Pop and apply. Push the current item.

<Callout type="green" label="Pop condition includes equality">
The discount applies when `prices[j] <= prices[i]` — note the `<=`, not just `less than`. Pop when the current price is less than or equal to the stack top.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise result as a copy of prices">
    `var result = prices` — items with no discount keep their original price.
  </StepItem>
  <StepItem title="Iterate and maintain stack">
    For each `i`, while `stack.last?.value >= prices[i]`: pop the top, apply discount `result[top.index] = top.value - prices[i]`.
  </StepItem>
  <StepItem title="Push current item">
    After popping, push `(value: prices[i], index: i)` onto the stack.
  </StepItem>
  <StepItem title="Return result">
    Any indices still in the stack at the end received no discount — their result values are unchanged.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func finalPrices(_ prices: [Int]) -> [Int] {
    var stack = [(value: Int, index: Int)]()
    var result = prices

    for i in 0..<prices.count {
      while let top = stack.last, prices[i] <= top.value {
        stack.removeLast()
        result[top.index] = top.value - prices[i]
      }
      stack.append((value: prices[i], index: i))
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `var stack = [(value: Int, index: Int)]()` — a named-tuple array for readable .value/.index access without a custom struct. `var result = prices` initialises from prices so items with no discount already hold their correct value — no post-loop cleanup needed. `stack.removeLast()` is O(1), and `while let top = stack.last` safely pattern-matches the optional top without force-unwrap.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  Each index is pushed onto the stack exactly once and popped at most once. The while loop pops happen across the full iteration, not per element — total pops across all iterations is at most n. Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
  The stack holds at most n entries (when prices are strictly increasing and nothing is ever popped). The result array is O(n) but is required output. Auxiliary stack: O(n).
</Callout>
