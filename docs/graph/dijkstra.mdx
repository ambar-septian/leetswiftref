<SectionHeader n={1} title="Overview" />

Dijkstra's algorithm is the go-to for **single-source shortest path** when edge weights are non-negative. It works by maintaining a set of "settled" nodes (shortest distance confirmed) and greedily settling the closest unsettled node next.

The key insight: once a node is settled, its shortest distance is final. This greedy choice is valid *only* because all edge weights are non-negative — a future edge can never make an already-settled path shorter.

<Callout type="amber" label="Key Constraint">
  Dijkstra requires **non-negative edge weights**. If any edge weight is negative, use Bellman-Ford instead (O(V·E)). Negative weights break the greedy invariant — a settled node's distance could later be improved through a negative edge, making the algorithm incorrect.
</Callout>

<SectionHeader n={2} title="Key Concepts" />

| Concept | Description |
|---|---|
| **Relaxation** | Updating `dist[v]` if going through `u` gives a shorter path: `dist[v] = min(dist[v], dist[u] + weight(u,v))` |
| **Settled / Visited** | A node whose shortest distance has been confirmed and won't change |
| **Min-heap** | Efficiently retrieves the unsettled node with the smallest known distance |
| **dist[]** | Array of shortest known distances from source; initialised to ∞ except source = 0 |
| **Lazy deletion** | Stale entries in the heap are skipped with `if d > dist[u] { continue }` |
| **Single-source** | All distances computed from one fixed starting node |

<SectionHeader n={4} title="Implementation" />

The standard Swift implementation uses **lazy deletion** — stale heap entries are skipped rather than removed. This avoids the complexity of a decrease-key operation.

```swift
import Collections // for Heap

func dijkstra(_ n: Int, _ edges: [[Int]], _ src: Int) -> [Int] {
    // Build adjacency list: adj[u] = [(v, weight)]
    var adj = [[(Int, Int)]](repeating: [], count: n)
    for e in edges {
        let (u, v, w) = (e[0], e[1], e[2])
        adj[u].append((v, w))
        adj[v].append((u, w)) // undirected — omit for directed graphs
    }

    // Initialise distances
    var dist = [Int](repeating: Int.max, count: n)
    dist[src] = 0

    // Min-heap ordered by distance: (distance, node)
    var heap = Heap<(Int, Int)>(comparator: { $0.0 < $1.0 })
    heap.insert((0, src))

    while let (d, u) = heap.popMin() {
        // Lazy deletion: skip if we've already found a shorter path
        guard d <= dist[u] else { continue }

        for (v, w) in adj[u] {
            let newDist = dist[u] + w
            if newDist < dist[v] {
                dist[v] = newDist
                heap.insert((newDist, v))
            }
        }
    }

    return dist // dist[i] = shortest distance from src to i; Int.max = unreachable
}
```

**Without `swift-collections`** — use a sorted array (acceptable for small graphs):

```swift
func dijkstraSimple(_ n: Int, _ adj: [[(Int, Int)]], _ src: Int) -> [Int] {
    var dist = [Int](repeating: Int.max, count: n)
    dist[src] = 0
    var heap: [(Int, Int)] = [(0, src)] // (dist, node)

    while !heap.isEmpty {
        heap.sort { $0.0 < $1.0 }      // O(k log k) — replace with real heap for production
        let (d, u) = heap.removeFirst()
        guard d <= dist[u] else { continue }

        for (v, w) in adj[u] {
            let nd = dist[u] + w
            if nd < dist[v] {
                dist[v] = nd
                heap.append((nd, v))
            }
        }
    }

    return dist
}
```

<Callout type="green" label="Swift Notes">
  Use `swift-collections`' `Heap` when available — it's a proper min-heap with O(log n) insert and popMin. The **lazy deletion** guard (`guard d <= dist[u] else { continue }`) is essential: when you find a shorter path to a node, you push a new entry without removing the old one. The old (stale) entry will eventually be popped — the guard discards it instantly. Make the heap comparator compare on the **distance** (first element), not the node ID. When the problem asks for shortest path to a *specific* destination, add `if u == dest { return dist[dest] }` after the guard for early exit.
</Callout>

<SectionHeader n={5} title="Common Patterns" />

**Single-source, all destinations:**

```swift
let distances = dijkstra(n, edges, src)
// distances[dest] = shortest path from src to dest
```

**Shortest path to a specific destination with early exit:**

```swift
while let (d, u) = heap.popMin() {
    if u == dest { return d }              // ← early exit
    guard d <= dist[u] else { continue }
    // ... relaxation
}
```

**Path reconstruction** — track predecessors:

```swift
var prev = [Int](repeating: -1, count: n)
// In relaxation: if nd < dist[v] { dist[v] = nd; prev[v] = u; heap.insert(...) }

// Reconstruct path from src to dest:
func reconstructPath(_ prev: [Int], _ dest: Int) -> [Int] {
    var path: [Int] = []
    var node = dest
    while node != -1 { path.append(node); node = prev[node] }
    return path.reversed()
}
```

<SectionHeader n={6} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O((V+E) log V)">
  Each vertex is settled once (V pops from heap). Each edge is relaxed at most once (E insertions into heap). Each heap operation (insert/popMin) costs O(log V) — in the worst case we push one entry per edge relaxation, giving O(E log V) for all heap operations. Adding the V pops: O((V+E) log V). With the dense-graph optimisation (array instead of heap), it's O(V²) — better when E ≈ V².
</Callout>

<Callout type="green" label="Space Complexity — O(V+E)">
  The distance array and predecessor array are O(V). The adjacency list is O(V+E). The heap holds at most O(E) entries (one per edge relaxation with lazy deletion). Total: O(V+E).
</Callout>

<SectionHeader n={7} title="Limitations and When to Use" />

<Callout type="purple" label="Dijkstra vs BFS vs Bellman-Ford — which to choose?">
  **Dijkstra:** weighted graph, non-negative edges, single source. The default for shortest path in LeetCode problems with weights.

  **BFS:** unweighted graph (or all weights equal). Always O(V+E) — faster than Dijkstra. If the problem says "minimum number of steps/hops", use BFS.

  **Bellman-Ford:** weighted graph with possible negative edges. O(V·E) — much slower, only use when negative weights exist.
</Callout>

| Problem signal | Algorithm |
|---|---|
| "Shortest path, weighted, non-negative" | Dijkstra |
| "Minimum hops / fewest steps" | BFS |
| "Network delay time" | Dijkstra (single source, all destinations) |
| "Cheapest flight within k stops" | Modified Dijkstra / BFS with level constraint |
| "Negative edge weights possible" | Bellman-Ford |
| "All-pairs shortest path" | Floyd-Warshall O(V³) |
| "Can two nodes reach each other?" | BFS/DFS — Dijkstra is overkill |
