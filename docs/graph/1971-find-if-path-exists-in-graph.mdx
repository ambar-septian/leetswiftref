<SectionHeader n={1} title="Problem Overview" />

There is a **bi-directional** graph with `n` vertices, where each vertex is labeled from `0` to `n - 1` (inclusive). The edges in the graph are represented as a 2D integer array `edges`, where each `edges[i] = [u_i, v_i]` denotes a bi-directional edge between vertex `u_i` and vertex `v_i`. Every vertex pair is connected by **at most one** edge, and no vertex has an edge to itself.

You want to determine if there is a **valid path** that exists from vertex `source` to vertex `destination`.

Given `edges` and the integers `n`, `source`, and `destination`, return `true` if there is a valid path from `source` to `destination`, or `false` otherwise.

**Constraints:**
- `1 <= n <= 2 * 10^5`
- `0 <= edges.length <= 2 * 10^5`
- `edges[i].length == 2`
- `0 <= u_i, v_i <= n - 1`
- `u_i != v_i`
- `1 <= source, destination <= n - 1`
- There are no duplicate edges.
- There are no self-edges.

<Callout type="amber" label="Key Constraint">
  The graph is bidirectional (undirected) — each edge must be stored in both directions in the adjacency list. source == destination is a valid edge case that should return true immediately before any traversal.
</Callout>

  For simple reachability (does a path exist?), DFS with a visited array from source is sufficient — O(V+E). Only upgrade to Union-Find when you need to answer many independent reachability queries on the same graph.

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Build an undirected adjacency list (each edge stored both directions). DFS from source: if any neighbor equals destination, return true immediately. Mark nodes visited before recursing to prevent infinite cycles. If DFS exhausts all reachable nodes, return false.
</Callout>

Build an undirected adjacency list. DFS from `source`. At each step, if a neighbor equals `destination`, return `true`. Otherwise, mark it visited and recurse. If the DFS exhausts all reachable nodes without finding `destination`, return `false`.

<Callout type="green" label="Array vs HashSet for visited">
Using `visited = Array(repeating: false, count: n)` is faster than a Set for large n — O(1) lookup with better cache performance. Only use a HashSet when n is unknown or very large.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Early exit">
    If `source == destination`, return `true` immediately.
  </StepItem>
  <StepItem title="Build adjacency list">
    For each edge `[u, v]`, add `v` to `list[u]` and `u` to `list[v]`.
  </StepItem>
  <StepItem title="DFS from source">
    For each unvisited neighbor: if it equals `destination`, return `true`. Otherwise mark visited and recurse.
  </StepItem>
  <StepItem title="Return false">
    If DFS completes without reaching `destination`, return `false`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func validPath(_ n: Int, _ edges: [[Int]], _ source: Int, _ destination: Int) -> Bool {
    var visited = Array(repeating: false, count: n)
    var list = Array(repeating: [Int](), count: n)
    guard source != destination else { return true }

    for edge in edges {
      list[edge[0]].append(edge[1])
      list[edge[1]].append(edge[0])
    }

    func dfs(_ curr: Int) -> Bool {
      for neighbour in list[curr] {
        if neighbour == destination { return true }
        if !visited[neighbour] {
          visited[neighbour] = true
          if dfs(neighbour) { return true }
        }
      }
      return false
    }

    return dfs(source)
  }
}
```

<Callout type="green" label="Swift Notes">
  `var visited = Array(repeating: false, count: n)` is faster than Set&lt;Int&gt; — O(1) array lookup with better cache performance for large n. `guard source != destination else { return true }` handles the trivial case before any allocation. The nested `func dfs(_ curr: Int) -> Bool` captures `visited` and `list` by reference from the outer scope — both mutate correctly without inout.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(V + E)">
  Building the adjacency list takes O(E). DFS visits each vertex at most once and traverses each edge at most twice (undirected). Total: O(V + E).
</Callout>

<Callout type="green" label="Space Complexity — O(V + E)">
  The adjacency list stores O(V + E) entries (2E for undirected edges). The visited array is O(V). The call stack is O(V) in the worst case (a linear graph). Total: O(V + E).
</Callout>
