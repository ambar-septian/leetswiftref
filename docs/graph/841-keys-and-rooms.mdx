<SectionHeader n={1} title="Problem Overview" />

There are `n` rooms labelled from `0` to `n - 1` and all the rooms are locked except for room `0`. Your goal is to visit all the rooms. However, you cannot enter a locked room without having its key.

When you visit a room, you may find a set of **distinct keys** in it. Each key has a number on it, denoting which room it unlocks, and you can take all of them with you to unlock the other rooms.

Given an array `rooms` where `rooms[i]` is the list of keys in room `i`, return `true` if you can visit **all** the rooms, or `false` otherwise.

**Constraints:**
- `n == rooms.length`
- `2 <= n <= 1000`
- `0 <= rooms[i].length <= 1000`
- `1 <= sum(rooms[i].length) <= 3000`
- `0 <= rooms[i][j] < n`
- All the values in `rooms[i]` are **unique**.

<Callout type="amber" label="Key Constraint">
  Room 0 is the only room that starts unlocked — it is the sole DFS starting point. Multiple rooms may contain the same key; the visited set prevents re-entering rooms that have already been processed, avoiding infinite loops.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  rooms = [[1],[2],[3],[]]
Output: true
Explanation: Visit room 0 (key 1) → room 1 (key 2) → room 2 (key 3) → room 3. All visited.
```

#### Example 2

```
Input:  rooms = [[1,3],[3,0],[2],[0]]
Output: false
Explanation: We can never enter room 2.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Model rooms as graph nodes and keys as directed edges. DFS from room 0 with a visited set explores every reachable room exactly once. After DFS, comparing visited.count to rooms.count tells you whether full reachability was achieved — O(N + K) total.
</Callout>

DFS from room `0`. Maintain a `visited` set. At each room `i`: insert `i` into visited, then recurse into each key (neighbor) that has not yet been visited. After DFS completes, check if `visited.count == rooms.count`.

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise visited set">
    `var visited = Set(Int)()`.
  </StepItem>
  <StepItem title="Define DFS">
    `func dfs(_ i: Int)`: guard `!visited.contains(i)` else return. Insert `i`. For each `key` in `rooms[i]`: `dfs(key)`.
  </StepItem>
  <StepItem title="Start from room 0">
    `dfs(0)`.
  </StepItem>
  <StepItem title="Check all rooms visited">
    Return `visited.count == rooms.count`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func canVisitAllRooms(_ rooms: [[Int]]) -> Bool {
    var visited = Set<Int>()

    func dfs(_ i: Int) {
      guard !visited.contains(i) else { return }
      visited.insert(i)
      for key in rooms[i] {
        dfs(key)
      }
    }

    dfs(0)
    return visited.count == rooms.count
  }
}
```

<Callout type="green" label="Swift Notes">
  `Set&lt;Int&gt;()` is the idiomatic Swift visited set — `insert` and `contains` are O(1) average. The nested `func dfs(_ i: Int)` captures `visited` and `rooms` from the enclosing scope. Since `visited` is a `var`, mutations inside the nested function are reflected in the outer scope — no `inout` is needed for captured variables.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(N + K)">
  N = number of rooms, K = total number of keys across all rooms. Each room is visited at most once (visited set guard). For each visited room, we iterate over its keys. Total work is proportional to rooms + keys — O(N + K).
</Callout>

<Callout type="green" label="Space Complexity — O(N)">
  The visited set holds at most N room indices. The DFS call stack depth is at most N in the worst case (a chain of rooms each with one key). O(N) auxiliary space total.
</Callout>
