<SectionHeader n={1} title="Problem Overview" />

Given a reference of a node in a **connected undirected graph**, return a **deep copy** (clone) of the graph.

Each node in the graph contains a value (`val`) and a list of its neighbors (`neighbors`). The graph is represented in the test as an adjacency list where `adjList[i]` is a list of all nodes adjacent to node `i + 1` (1-indexed).

Your clone must be an entirely new set of node objects. Changing the original graph must not affect the clone and vice versa.

**Constraints:**
- The number of nodes in the graph is in the range `[0, 100]`.
- `1 <= Node.val <= 100`
- `Node.val` is unique for each node.
- There are no repeated edges and no self-loops.
- The graph is connected, and all nodes can be visited starting from the given node.

<Callout type="amber" label="Key Constraint">
Node values are unique (1 to 100) — use val as the clone dictionary key. The graph is guaranteed connected, so a single DFS from the given node reaches all nodes. The empty graph (nil input) must return nil.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: adjList = [[2,4],[1,3],[2,4],[1,3]]
Output: [[2,4],[1,3],[2,4],[1,3]]
Explanation: 4-node cycle: 1↔2↔3↔4↔1. The returned graph is a deep copy — entirely new node objects.
```

#### Example 2

```
Input: adjList = [[]]
Output: [[]]
Explanation: Single node with no neighbors.
```

#### Example 3

```
Input: adjList = []
Output: []
Explanation: Empty graph.
```

<Callout type="green" label="Rule of Thumb">
When cloning a cyclic structure, store the clone in a map immediately after creation — before recursing into neighbors. This single rule prevents both infinite recursion and duplicate node objects.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
A DFS clone of each node is stored in a dictionary immediately after creation — before recursing into its neighbors. When a neighbor is already in the dict (cycle or shared edge), the existing clone is reused. This guarantees each node is cloned exactly once and all edges are correctly reproduced.
</Callout>

<CloneGraphDiagram />

Use a `dict [Int: Node]` mapping old node values to their clones. When DFS visits a node: create a new node, store it in `dict`, then iterate over neighbors. For each neighbor, if its clone already exists in `dict` (cycle or shared neighbor), append that clone. Otherwise recurse into the neighbor to create its clone.

<Callout type="cyan" label="Store before recursing">
The clone must be stored in <code>dict</code> immediately after creation — before iterating neighbors. This prevents infinite recursion when a neighbor points back to the current node.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard nil">
    Return nil if the input node is nil.
  </StepItem>
  <StepItem title="Initialise dict">
    `var dict = [Int: Node]()` — maps old node val to its clone.
  </StepItem>
  <StepItem title="DFS: create and store clone">
    Create `let newNode = Node(oldNode.val)` and immediately store `dict[oldNode.val] = newNode`.
  </StepItem>
  <StepItem title="Clone neighbors">
    For each neighbor: if `dict[neighbor.val]` exists, append that clone. Else call `dfs(neighbor)` and append the returned clone.
  </StepItem>
  <StepItem title="Return the clone">
    Return `newNode` from DFS. The top-level call returns the clone of the entry node.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func cloneGraph(_ node: Node?) -> Node? {
    guard let node else { return nil }
    var dict = [Int: Node]()

    func dfs(_ oldNode: Node?) -> Node? {
      guard let oldNode else { return nil }
      let newNode = Node(oldNode.val)
      dict[oldNode.val] = newNode

      for neighbor in oldNode.neighbors {
        guard let neighbor else { continue }
        if let existing = dict[neighbor.val] {
          newNode.neighbors.append(existing)
        } else {
          newNode.neighbors.append(dfs(neighbor))
        }
      }

      return newNode
    }

    return dfs(node)
  }
}
```

<Callout type="green" label="Swift Notes">
`guard let neighbor else { continue }` handles optional elements in `[Node?]`. The nested `dfs` closure captures `dict` by reference so the dictionary is shared across all recursive calls without being passed as `inout`. Using `oldNode.val` as the dictionary key works because val is guaranteed unique per the problem constraints.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(V + E)">
O(V + E) — every node is visited exactly once (the dict prevents revisits) and every edge is traversed once when iterating each node's neighbors. V is the number of nodes, E the number of edges.
</Callout>

<Callout type="green" label="Space Complexity — O(V)">
O(V) for the clone dictionary, which stores one entry per node. The DFS call stack is O(V) in the worst case (a chain graph). The cloned graph itself is not counted as auxiliary space.
</Callout>

<Callout type="amber" label="When to prefer DFS vs BFS">
Both DFS and BFS produce identical asymptotic complexity. Use **DFS** for simpler recursive code that directly mirrors the problem structure. Use **BFS** if the graph is very deep (risking a stack overflow) or when you need level-by-level processing order.
</Callout>
