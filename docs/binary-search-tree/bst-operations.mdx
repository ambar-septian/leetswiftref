<SectionHeader n={1} title="Overview" />

A BST provides O(h) lookup by using the ordering invariant to eliminate half the tree at each step — analogous to binary search on a sorted array. The key insight for each operation:

- **Search**: go left if target is smaller, right if larger.
- **Validate**: pass tightening bounds (`min`, `max`) down the recursion — a node is valid only if `min < node.val < max`.
- **Kth smallest**: inorder traversal (sorted order) — decrement a counter, stop when counter hits 0.
- **Sorted array → BST**: choose the midpoint as root to guarantee height balance.
- **Delete**: three cases — no children (nil out), one child (replace with child), two children (replace value with inorder successor, then delete successor).

<Callout type="cyan" label="Core Insight">
  The BST invariant means inorder traversal always produces elements in sorted ascending order. This single fact underlies kth-smallest (inorder + count), validate (check invariant at every node with bounds), and convert-sorted-array (reverse of inorder — build from sorted sequence by choosing midpoints). Whenever a BST problem feels unclear, ask: what does inorder traversal look like here?
</Callout>

<SectionHeader n={2} title="Key Concepts" />

| Concept | Description |
|---|---|
| **BST invariant** | `left.val < node.val < right.val` — applies recursively to all subtrees, not just direct children |
| **Min/max bounds** | `validate(node, min, max)` — every node must satisfy `min < node.val < max` |
| **Inorder counter** | Decrement k on each visit in inorder; return when k reaches 0 |
| **Inorder successor** | Leftmost node in right subtree — used as replacement in delete |
| **Mid-point construction** | `root = mid`, recurse left on `[lo..mid-1]`, right on `[mid+1..hi]` |

<Callout type="amber" label="Key Constraint">
  In the **validate** problem, use `Int.min` and `Int.max` as the initial bounds — not `nil`. When passing bounds down: going left tightens the upper bound (`max = node.val`); going right tightens the lower bound (`min = node.val`). Using the parent value only (not the inherited bound) breaks for cases like a left subtree with a right child that exceeds the grandparent.
</Callout>

<SectionHeader n={4} title="Implementation" />

**Search — navigate using BST property:**

```swift
func searchBST(_ root: TreeNode?, _ val: Int) -> TreeNode? {
    guard let n = root else { return nil }
    if n.val == val { return n }
    return val < n.val ? searchBST(n.left, val) : searchBST(n.right, val)
}
```

**Validate — pass min/max bounds down:**

```swift
func isValidBST(_ root: TreeNode?) -> Bool {
    func validate(_ node: TreeNode?, _ min: Int, _ max: Int) -> Bool {
        guard let n = node else { return true }
        guard n.val > min && n.val < max else { return false }
        return validate(n.left, min, n.val) && validate(n.right, n.val, max)
    }
    return validate(root, Int.min, Int.max)
}
```

**Kth Smallest — inorder traversal with counter:**

```swift
func kthSmallest(_ root: TreeNode?, _ k: Int) -> Int {
    var remaining = k
    var result = 0

    func inorder(_ node: TreeNode?) {
        guard let n = node, remaining > 0 else { return }
        inorder(n.left)
        remaining -= 1
        if remaining == 0 { result = n.val; return }
        inorder(n.right)
    }

    inorder(root)
    return result
}
```

**Sorted Array to BST — mid-point recursion:**

```swift
func sortedArrayToBST(_ nums: [Int]) -> TreeNode? {
    func build(_ lo: Int, _ hi: Int) -> TreeNode? {
        guard lo <= hi else { return nil }
        let mid = lo + (hi - lo) / 2
        let node = TreeNode(nums[mid])
        node.left  = build(lo, mid - 1)
        node.right = build(mid + 1, hi)
        return node
    }
    return build(0, nums.count - 1)
}
```

**Delete Node — three cases:**

```swift
func deleteNode(_ root: TreeNode?, _ key: Int) -> TreeNode? {
    guard let n = root else { return nil }
    if key < n.val {
        n.left  = deleteNode(n.left,  key)
    } else if key > n.val {
        n.right = deleteNode(n.right, key)
    } else {
        // Found the node to delete
        if n.left  == nil { return n.right }  // no left child
        if n.right == nil { return n.left  }  // no right child
        // Two children: replace with inorder successor (min of right subtree)
        var successor = n.right!
        while let left = successor.left { successor = left }
        n.val   = successor.val                          // copy successor value
        n.right = deleteNode(n.right, successor.val)    // delete successor
    }
    return n
}
```

<Callout type="green" label="Swift Notes">
  Swift's `TreeNode` is a class (reference type), so mutating `n.left` or `n.val` modifies the actual node — no copy is made. The delete function returns the (possibly new) root of each subtree; callers assign it back (`n.left = deleteNode(n.left, key)`). For kth smallest, capturing `remaining` and `result` as `var` in the outer scope and modifying them in the nested function is idiomatic Swift — no need for `inout` parameters since nested functions capture outer variables by reference.
</Callout>

<SectionHeader n={5} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(h)">
  All single-value operations (search, insert, delete) navigate from root to a leaf along one path — O(h) comparisons. For a balanced BST (height = log n), this is O(log n). For a degenerate BST (e.g. input was sorted — all nodes in a line), height = n and time is O(n). Kth smallest requires visiting k nodes in inorder — O(k + h). Sorted array to BST visits all n elements — O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(h)">
  Recursive implementations use O(h) call stack space. For balanced trees O(log n); for degenerate trees O(n). Sorted array to BST also uses O(h) stack space in addition to the O(n) output tree.
</Callout>

<SectionHeader n={6} title="When to Use" />

<Callout type="purple" label="BST vs Sorted Array — which to choose?">
  A **sorted array** gives O(1) access by index and O(log n) binary search, but O(n) insertion and deletion. A **BST** gives O(h) search, insert, and delete — O(log n) when balanced. Use a sorted array when the data is static. Use a BST (or a self-balancing BST like Swift's `SortedSet` from swift-collections) when you need dynamic insertions and deletions alongside fast lookups.
</Callout>

<Callout type="green" label="Rule of Thumb">
  When a problem says "BST" and asks for a specific value — navigate left/right using the ordering property. When it asks to validate — pass min/max bounds. When it asks for the kth element — inorder traversal with a counter. When it asks to build from sorted data — mid-point recursion guarantees balance.
</Callout>

| Problem shape | Algorithm |
|---|---|
| Search for value | Navigate L if smaller, R if larger |
| Validate a BST | Recursive with min/max bounds |
| Kth smallest/largest | Inorder (or reverse-inorder) with counter |
| Build balanced BST from sorted array | Mid-point recursion |
| Delete a node | Three cases: leaf, one child, two children (successor) |
