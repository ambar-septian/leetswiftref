<SectionHeader n={1} title="Problem Overview" />

Given a root node reference of a BST and a key, delete the node with the given key in the BST. Return the root node reference (possibly updated) of the BST.

Basically, the deletion can be divided into two stages:
1. Search for the node to remove.
2. If the node is found, delete the node.

**Constraints:**
- The number of nodes in the tree is in the range `[0, 10^4]`.
- `-10^5 <= Node.val <= 10^5`
- Each node has a **unique** value.
- `root` is a valid binary search tree.
- `-10^5 <= key <= 10^5`

<Callout type="amber" label="Key Constraint">
The key may not exist in the BST. The base case `guard let root else { return root }` handles this by returning nil when the search falls off the tree — no node is modified and the tree is returned unchanged.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  root = [5,3,6,2,4,null,7], key = 3
Output: [5,4,6,2,null,null,7]
Explanation: Node 3 has two children. Its inorder successor (min of right subtree) is 4.
             Replace 3 with 4, then delete 4 from the right subtree.
```

#### Example 2

```
Input:  root = [5,3,6,2,4,null,7], key = 0
Output: [5,3,6,2,4,null,7]
Explanation: Key 0 is not found — return unchanged tree.
```

#### Example 3

```
Input:  root = [], key = 0
Output: []
```

<Callout type="green" label="Rule of Thumb">
BST deletion always has three cases based on child count: zero children (return nil), one child (return that child), two children (replace value with inorder successor, then recursively delete the successor). Returning the (possibly updated) root at every level rewires the parent's pointer automatically.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Navigate using BST ordering to find the target, then handle three cases: leaf (return nil), one child (return that child), two children (replace value with the inorder successor — the leftmost node of the right subtree — then recursively delete the successor). Returning `root` at every call level rewires the parent's pointer to the (possibly updated) subtree root.
</Callout>

Recurse using the BST property:
- If `key > root.val`: problem is in the right subtree — `root.right = deleteNode(root.right, key)`.
- If `key < root.val`: problem is in the left subtree — `root.left = deleteNode(root.left, key)`.
- If `key == root.val`: found the node. Handle three cases:
  - No left child → return `root.right`.
  - No right child → return `root.left`.
  - Both children → find minimum of right subtree (inorder successor), copy its value into root, then delete that minimum from the right subtree.

Return `root` at the end to rewire the parent's pointer.

<DeleteNodeBSTDiagram />

<Callout type="cyan" label="Finding the inorder successor">
Traverse left as far as possible in the right subtree. The leftmost node in the right subtree is the smallest value greater than the deleted node — it can safely replace the deleted node while preserving BST ordering.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Base case — node not found">
    `guard let root else { return root }`. Returns nil without modification if key exceeds the tree.
  </StepItem>
  <StepItem title="Navigate using BST ordering">
    If `key > root.val`: `root.right = deleteNode(root.right, key)`. Else if `key < root.val`: `root.left = deleteNode(root.left, key)`.
  </StepItem>
  <StepItem title="Found — leaf or one child">
    If `root.left == nil`, return `root.right`. If `root.right == nil`, return `root.left`.
  </StepItem>
  <StepItem title="Found — two children: locate inorder successor">
    Traverse left from `root.right` until `curr?.left == nil`. `curr` is the minimum node in the right subtree.
  </StepItem>
  <StepItem title="Replace value and delete successor">
    `root.val = curr!.val`. Then `root.right = deleteNode(root.right, root.val)`.
  </StepItem>
  <StepItem title="Return root">
    `return root` rewires the parent pointer to the (potentially updated) subtree root.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func deleteNode(_ root: TreeNode?, _ key: Int) -> TreeNode? {
    guard let root else {
      return root
    }

    if key > root.val {
      root.right = deleteNode(root.right, key)
    } else if key < root.val {
      root.left = deleteNode(root.left, key)
    } else {
      if root.left == nil {
        return root.right
      } else if root.right == nil {
        return root.left
      }

      // Find the inorder successor (min of right subtree)
      var curr = root.right
      while let left = curr?.left {
        curr = left
      }

      root.val = curr?.val ?? -1
      root.right = deleteNode(root.right, root.val)

      return root
    }

    return root
  }
}
```

<Callout type="green" label="Swift Notes">
`guard let root else { return root }` binds the non-optional and returns the original `TreeNode?` (nil) in one step. `while let left = curr?.left { curr = left }` traverses to the leftmost node using optional binding — idiomatic for BST min-finding. The `curr?.val ?? -1` fallback is safe because the two-children branch guarantees `root.right` is non-nil.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(h)">
Each recursive call descends one level of the tree. Navigation to the target takes O(h) steps. Finding the inorder successor (leftmost of right subtree) also takes O(h). The recursive deletion of the successor adds one more O(h) traversal. Total: O(h), where h = O(log n) for a balanced BST and O(n) worst case for a skewed tree.
</Callout>

<Callout type="green" label="Space Complexity — O(h)">
The recursion call stack reaches depth h — one frame per level descended. No auxiliary data structures are allocated. Stack depth is O(log n) for balanced trees and O(n) for skewed trees.
</Callout>
