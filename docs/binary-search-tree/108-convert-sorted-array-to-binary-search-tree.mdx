<SectionHeader n={1} title="Problem Overview" />

Given an integer array `nums` where the elements are sorted in **ascending order**, convert it to a **height-balanced** binary search tree.

A **height-balanced** binary tree is a binary tree in which the depth of the two subtrees of every node never differs by more than one.

The sorted array property guarantees that the BST ordering invariant is maintained when the midpoint is used as the root.

**Constraints**
- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` is sorted in a **strictly increasing** order.

<Callout type="green" label="Core Insight">
Always pick the midpoint of the current subarray as the root. This splits the remaining elements equally between left and right subtrees, guaranteeing height balance. Recurse on the left half for the left subtree and the right half for the right subtree.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  nums = [-10,-3,0,5,9]
Output: [0,-3,9,-10,null,5]  (one valid answer)
```

#### Example 2
```
Input:  nums = [1,3]
Output: [3,1]  or  [1,null,3]  (both valid)
```

<SectionHeader n={4} title="Approach" />

Define `recursive(l, r)`. Base case: `l > r` returns nil. Compute `m = (l + r) / 2`. Create `TreeNode(nums[m])`. Set `tree.left = recursive(l, m - 1)`, `tree.right = recursive(m + 1, r)`. Return `tree`.

<Callout type="cyan" label="Why floor division gives a valid balanced result">
For even-length subarrays there are two valid midpoints. Floor division (l+r)/2 consistently picks the lower-middle element. The resulting tree is still height-balanced — both choices are accepted by LeetCode since the problem allows any valid height-balanced BST.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Base case">
    If `l > r`, return nil — the subarray is empty.
  </StepItem>
  <StepItem title="Pick midpoint as root">
    `let m = (l + r) / 2`. Create `TreeNode(nums[m])`.
  </StepItem>
  <StepItem title="Recurse on halves">
    `tree.left = recursive(l: l, r: m - 1)`. `tree.right = recursive(l: m + 1, r: r)`.
  </StepItem>
  <StepItem title="Return">
    Return `tree`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func sortedArrayToBST(_ nums: [Int]) -> TreeNode? {
    func recursive(l: Int, r: Int) -> TreeNode? {
      guard l <= r else { return nil }
      let m = (l + r) / 2
      let tree = TreeNode(nums[m])
      tree.left  = recursive(l: l,     r: m - 1)
      tree.right = recursive(l: m + 1, r: r)
      return tree
    }
    return recursive(l: 0, r: nums.count - 1)
  }
}
```
