<SectionHeader n={1} title="Problem Overview" />

Given an array of integers `nums` and an integer `target`, return the **indices** of the two numbers that add up to `target`.

You may assume that each input has **exactly one solution**, and you may not use the same element twice. You can return the answer in any order.

**Constraints**
- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- Exactly one valid answer exists.

<Callout type="amber" label="Key Constraint">
Exactly one valid answer is guaranteed — you never need to handle zero or multiple solutions. You may not use the same element twice, so the complement must come from a different index.
</Callout>

<Callout type="green" label="Core Insight">
For every number `x`, we need `target - x`. Instead of scanning the whole array each time, store numbers you have already seen in a hash map so the lookup is O(1).
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  nums = [2, 7, 11, 15], target = 9
Output: [0, 1]
Explanation: nums[0] + nums[1] = 2 + 7 = 9
```

#### Example 2
```
Input:  nums = [3, 2, 4], target = 6
Output: [1, 2]
Explanation: nums[1] + nums[2] = 2 + 4 = 6
```

#### Example 3
```
Input:  nums = [3, 3], target = 6
Output: [0, 1]
Explanation: nums[0] + nums[1] = 3 + 3 = 6
```

<Callout type="green" label="Rule of Thumb">
When a problem asks for two indices that sum to a target, reach for a hash map — store each value and its index as you scan, then look up the complement in O(1).
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
For every element x, compute complement = target - x. Storing previously seen values in a hash map lets you check for the complement in O(1). A single left-to-right pass suffices — when the second element of any valid pair is reached, the first is already in the map.
</Callout>

Use a single pass with a hash map that maps each value to its index. For every element, compute the complement (`target - nums[i]`). If the complement is already in the map, we have our answer. Otherwise, insert the current element.

<Callout type="cyan" label="Why one pass works">
When we reach index `i` and the complement was at index `j` (j less than i), it is already in the map. We return immediately without ever needing a second scan.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise an empty dictionary">
    `dict: [Int: Int]` maps a value to its index in `nums`.
  </StepItem>
  <StepItem title="Iterate through nums">
    For each index `i`, compute `complement = target - nums[i]`.
  </StepItem>
  <StepItem title="Check for complement">
    If `complement` exists in `dict`, return `[dict[complement]!, i]` — we are done.
  </StepItem>
  <StepItem title="Store current value">
    Otherwise, store `dict[nums[i]] = i` and move to the next element.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func twoSum(_ nums: [Int], _ target: Int) -> [Int] {
    var dict = [Int: Int]()  // value → index

    for i in 0..<nums.count {
      let complement = target - nums[i]
      if let existingIndex = dict[complement] {
        return [existingIndex, i]
      }
      dict[nums[i]] = i
    }

    return []
  }
}
```

<Callout type="green" label="Swift Notes">
`[Int: Int]()` creates an empty dictionary — no import required. `if let existingIndex = dict[complement]` safely unwraps the optional lookup without force-unwrap. Looking up before inserting ensures an element is never matched against itself even when `nums[i] * 2 == target`.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
Each of the n elements is visited exactly once. Hash map lookup and insertion are both O(1) on average, so total time is O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
The hash map stores at most n entries — one per element seen — giving O(n) auxiliary space in the worst case (no match found until the last element).
</Callout>
