<SectionHeader n={1} title="Problem Overview" />

Given an integer array `nums`, return `true` if there exists a triple of indices `(i, j, k)` such that `i < j < k` and `nums[i] < nums[j] < nums[k]`. If no such indices exist, return `false`.

You must solve it in O(n) time and O(1) space.

**Constraints:**
- `1 <= nums.length <= 5 * 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: nums = [1,2,3,4,5]
Output: true
Explanation: Any triplet where i < j < k works.
```

#### Example 2

```
Input: nums = [5,4,3,2,1]
Output: false
```

#### Example 3

```
Input: nums = [2,1,5,0,4,6]
Output: true
Explanation: (1,4,6) with indices (1,4,5).
```

<SectionHeader n={4} title="Approach" />

Maintain `first = Int.max` and `second = Int.max`. For each `num`:
- If `num` is less than `first`, update `first`.
- Else if `num` is greater than `first` but less than `second`, update `second`.
- Else if `num` is greater than both `first` and `second`, a triplet exists — return `true`.

<Callout type="cyan" label="The subtle invariant">
When first gets updated to a smaller value, second still holds a value that was legitimately larger than an even earlier first. So if any subsequent number is larger than second, a valid triplet (old_first, second, num) exists — even if old_first's index is before the current first's index.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise two sentinels">
    `var first = Int.max`, `var second = Int.max`.
  </StepItem>
  <StepItem title="Scan the array">
    For each `num`: check in order — update first, then second, then check for triplet.
  </StepItem>
  <StepItem title="Update first">
    If `num less than first`: `first = num`.
  </StepItem>
  <StepItem title="Update second">
    Else if `num greater than first and num less than second`: `second = num`.
  </StepItem>
  <StepItem title="Return true">
    Else if `num greater than first and num greater than second`: a triplet exists — return `true`.
  </StepItem>
  <StepItem title="Return false">
    After the loop, no triplet found — return `false`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func increasingTriplet(_ nums: [Int]) -> Bool {
    var first  = Int.max
    var second = Int.max

    for num in nums {
      if num > first, num < second {
        second = num
      } else if num < first {
        first = num
      } else if num > first, num > second {
        return true
      }
    }

    return false
  }
}
```
