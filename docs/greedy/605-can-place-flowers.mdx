<SectionHeader n={1} title="Problem Overview" />

You have a long flowerbed in which some of the plots are planted and some are not. However, flowers cannot be planted in adjacent plots.

Given an integer array `flowerbed` containing `0`s and `1`s, where `0` means empty and `1` means not empty, and an integer `n`, return `true` if `n` new flowers can be planted in the flowerbed without violating the no-adjacent-flowers rule and `false` otherwise.

**Constraints:**
- `1 <= flowerbed.length <= 2 * 10^4`
- `flowerbed[i]` is `0` or `1`.
- There are no two adjacent flowers in `flowerbed`.
- `0 <= n <= flowerbed.length`

<Callout type="amber" label="Key Constraint">
The input guarantees no two adjacent flowers already exist. This means we never need to validate the existing state — we only need to check the immediate left and right neighbours before planting.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  flowerbed = [1,0,0,0,1], n = 1
Output: true
Explanation: Plant at index 2 → [1,0,1,0,1].
```

#### Example 2

```
Input:  flowerbed = [1,0,0,0,1], n = 2
Output: false
Explanation: Only one spot is available — index 2.
```

<Callout type="green" label="Rule of Thumb">
When placing items with spacing constraints along a linear sequence, scanning left to right and planting at the first eligible slot is always optimal — placing early never blocks a valid later placement.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
A slot is eligible when it is empty and both immediate neighbours are also empty (or absent at the boundary). After planting, set `flowerbed[i] = 1` immediately — this automatically blocks the next position from qualifying without any explicit skip. Break as soon as `n` reaches zero to avoid unnecessary scanning.
</Callout>

Iterate through the flowerbed. A slot `i` is eligible if:
1. `flowerbed[i] == 0`
2. The left neighbour is `0` or does not exist (`i == 0`)
3. The right neighbour is `0` or does not exist (`i == lastIndex`)

When eligible, plant (`flowerbed[i] = 1`), decrement `n`, and break early if `n == 0`.

<Callout type="cyan" label="Modifying the input array">
Planting at index i sets flowerbed[i] = 1. This automatically blocks index i+1 from being eligible (its left neighbour is now 1), so we never need to explicitly skip i+1.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Short-circuit if n == 0">
    If `n == 0`, return `true` immediately — nothing to plant.
  </StepItem>
  <StepItem title="Iterate through each slot">
    For each index `i` from `0` to `lastIndex`:
  </StepItem>
  <StepItem title="Skip occupied slots">
    If `flowerbed[i] != 0`, continue.
  </StepItem>
  <StepItem title="Check left neighbour">
    If `i > 0` and `flowerbed[i - 1] == 1`, continue (left is occupied).
  </StepItem>
  <StepItem title="Check right neighbour">
    If `i + 1 less than or equal to lastIndex` and `flowerbed[i + 1] == 1`, continue (right is occupied).
  </StepItem>
  <StepItem title="Plant and decrement">
    `flowerbed[i] = 1; n -= 1`. If `n == 0`, break early.
  </StepItem>
  <StepItem title="Return">
    Return `n == 0`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func canPlaceFlowers(_ flowerbed: [Int], _ n: Int) -> Bool {
    guard n != 0 else { return true }
    let lastIndex = flowerbed.count - 1
    var n = n
    var flowerbed = flowerbed

    for i in 0...lastIndex {
      guard flowerbed[i] == 0 else { continue }

      if i > 0, flowerbed[i - 1] == 1 { continue }
      if lastIndex >= i + 1, flowerbed[i + 1] == 1 { continue }

      flowerbed[i] = 1
      n -= 1
      if n == 0 { break }
    }

    return n == 0
  }
}
```

<Callout type="green" label="Swift Notes">
`var flowerbed = flowerbed` creates a mutable local copy — Swift arrays are value types, so the caller's array is unmodified. `guard n != 0 else { return true }` short-circuits before the loop for the trivial case. `lastIndex >= i + 1` guards the right-boundary access cleanly without recomputing `flowerbed.count - 1` inside the loop.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(m)">
A single left-to-right pass over all m positions in the flowerbed. Early termination when n reaches zero means the worst case (n = 0 or no valid slot) scans all m elements once — O(m).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
The local copy of `flowerbed` is created for mutability (value-type semantics), but no additional data structures grow with input size. All state is held in two scalar variables: `n` and loop index `i`.
</Callout>
