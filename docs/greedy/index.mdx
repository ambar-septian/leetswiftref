<SectionHeader n={1} title="What is a Greedy Algorithm?" />

A greedy algorithm builds a solution by always making the best available choice at the current step — without reconsidering earlier choices. This contrasts with dynamic programming (which considers all choices) and backtracking (which undoes bad choices).

| Aspect | Greedy | Dynamic Programming |
|---|---|---|
| Choice | Locally optimal at each step | Considers all options |
| Revisit | Never — once chosen, always kept | Yes — memoises all subproblems |
| Correctness proof | Greedy-choice property + optimal substructure | Optimal substructure sufficient |
| Speed | Usually O(n) | Usually O(n²) or O(n·k) |

<Callout type="amber" label="Key Constraint">
  Greedy algorithms are only correct when the **greedy-choice property** holds: making the locally optimal choice at each step never prevents reaching the global optimum. This must be proved (or strongly intuited). A greedy approach that is wrong will often pass easy test cases but fail on carefully constructed edge cases — always verify with the problem's examples and extremes.
</Callout>

<Callout type="green" label="Rule of Thumb">
  If the problem asks to maximise or satisfy a condition with a single forward pass — try greedy first. The tell is that making the "best" local choice (place a flower, track the minimum, give minimum candies) never conflicts with future choices. If the greedy is wrong you will see it in the constraints or examples.
</Callout>
