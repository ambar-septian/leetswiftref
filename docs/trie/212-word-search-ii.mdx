<SectionHeader n={1} title="Problem Overview" />

Given an `m x n` board of characters and a list of strings `words`, return all words on the board. Each word must be constructed from letters of sequentially adjacent cells (horizontally or vertically neighbouring). The same letter cell may not be used more than once in a word.

**Constraints:**
- `m == board.length`
- `n == board[i].length`
- `1 <= m, n <= 12`
- `board[i][j]` is a lowercase English letter.
- `1 <= words.length <= 3 * 10^4`
- `1 <= words[i].length <= 10`
- `words[i]` consists of lowercase English letters.
- All strings in `words` are unique.

<Callout type="amber" label="Key Constraint">
Up to 30,000 words on a 12×12 board. Running Word Search I independently for each word is O(W × m × n × 4^L) — too slow. A trie enables all words to share prefix traversal, pruning the DFS the moment the board path diverges from every word prefix simultaneously.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:
board = [["o","a","a","n"],
         ["e","t","a","e"],
         ["i","h","k","r"],
         ["i","f","l","v"]]
words = ["oath","pea","eat","rain"]
Output: ["eat","oath"]
```

#### Example 2

```
Input: board = [["a","b"],["c","d"]], words = ["abcb"]
Output: []
```

<Callout type="green" label="Rule of Thumb">
When searching for multiple words on a board, build a trie from all target words and run a single DFS pass — the trie allows early pruning whenever the current board path diverges from all word prefixes simultaneously.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Build a trie from all words, storing the full word string at each end-of-word node. DFS each board cell, advancing the trie cursor alongside. If no trie child matches the current character, prune immediately. When an end-of-word node is reached, collect the word and nil it to prevent duplicates. After DFS, prune childless nodes bottom-up to stop revisiting exhausted branches.
</Callout>

<WordSearchIIDiagram />

Build a trie from all target words, storing the full word string at each end-of-word node. Run DFS from every cell; at each step, look up `board[r][c]` in the current trie node's children. If a child exists, mark the cell visited (`board[r][c] = "#"`), recurse in four directions, then restore the cell. When a node with a non-nil `word` is reached, collect it and nil the word to prevent duplicates. After collecting, prune leaf trie nodes bottom-up to stop revisiting exhausted branches.

<Callout type="cyan" label="Why prune the trie">
Once a word is found and its trie path has no other words branching off, those nodes are dead weight. Removing them speeds up subsequent DFS calls by cutting branches immediately.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Build the trie">
    For each word, walk character-by-character creating nodes; store the word string on the final node.
  </StepItem>
  <StepItem title="DFS from every cell">
    For each `(r, c)`, call `dfs(r, c, root)`. The root node's children represent the first characters of all words.
  </StepItem>
  <StepItem title="Advance trie node by board character">
    Look up `board[r][c]` in the current node's children. If no child exists, return immediately (pruning).
  </StepItem>
  <StepItem title="Mark and restore visited cells">
    Set `board[r][c] = "#"` before recursing; restore the original character afterward.
  </StepItem>
  <StepItem title="Collect found words and prune">
    When `nextNode.word != nil`, collect it and set it to nil. After all four recursive calls, if the node has no word and no children, remove it from the parent's children map.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class TrieNode {
  var children = [Character: TrieNode]()
  var word: String?
}

class Solution {
  func findWords(_ board: [[Character]], _ words: [String]) -> [String] {
    let root = TrieNode()
    for word in words {
      var curr = root
      for char in word {
        if curr.children[char] == nil { curr.children[char] = TrieNode() }
        curr = curr.children[char]!
      }
      curr.word = word
    }

    let rows = board.count, cols = board[0].count
    var board = board
    var results = [String]()

    func dfs(_ r: Int, _ c: Int, _ node: TrieNode?) {
      guard let node else { return }
      if let word = node.word {
        node.word = nil
        results.append(word)
      }
      guard r >= 0, c >= 0, r < rows, c < cols else { return }
      guard let nextNode = node.children[board[r][c]] else { return }

      let temp = board[r][c]
      board[r][c] = "#"
      dfs(r+1, c, nextNode); dfs(r-1, c, nextNode)
      dfs(r, c+1, nextNode); dfs(r, c-1, nextNode)
      board[r][c] = temp

      if nextNode.word == nil, nextNode.children.isEmpty {
        node.children[board[r][c]] = nil
      }
    }

    for r in 0..<rows {
      for c in 0..<cols { dfs(r, c, root) }
    }
    return results
  }
}
```

<Callout type="green" label="Swift Notes">
Storing `word: String?` on `TrieNode` avoids reconstructing the word from the DFS path — collect it directly when the node is reached. Setting `node.word = nil` after collection deduplicates without an extra Set. Marking cells with `"#"` reuses the `[[Character]]` board as a visited map — no additional O(m×n) memory needed. Swift's `var board = board` at the top creates a local mutable copy in O(m×n).
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(m×n×3^L)">
DFS from each of the m×n cells explores at most 3^(L-1) paths (cannot backtrack to the prior cell). L = max word length. Trie pruning significantly reduces constant-factor work in practice.
</Callout>

<Callout type="green" label="Space Complexity — O(W×L)">
The trie stores O(W × L) nodes for W words of max length L. The DFS call stack is at most O(L) deep. The `var board = board` copy is O(m×n).
</Callout>

<Callout type="amber" label="When to prefer Trie+DFS vs Word Search I × W">
Prefer **Trie + DFS** when W is large (hundreds or thousands of words) — shared prefix traversal and early pruning make it vastly faster. Use **Word Search I × W** only for tiny word lists (single digits) where the trie build overhead is not justified.
</Callout>
