<SectionHeader n={1} title="What is a Trie?" />

A **trie** (prefix tree) stores strings character by character. Each node represents one character position; children represent possible next characters. A node marked `isEnd = true` signals that the path from root to that node forms a complete word.

**Key properties:**

| Property | Detail |
|---|---|
| **Insert** | Follow / create one node per character — O(L) |
| **Search (exact)** | Traverse character by character; check `isEnd` at the last node — O(L) |
| **StartsWith (prefix)** | Same traversal without the `isEnd` check — O(L) |
| **Space** | O(SIGMA · N · L) worst case — SIGMA = alphabet size, N = word count, L = avg length |
| **Advantage over hash map** | Supports prefix queries and autocomplete in O(L); hash map needs O(N) scan |

<Callout type="amber" label="Key Constraint">
  A trie node needs one child slot per possible character. For lowercase English only, a `[Character: TrieNode]` dictionary or a fixed 26-element array works. Using a dictionary is safer for arbitrary alphabets; the fixed array is faster in practice.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **Root** | The top-level node; stores no character itself |
| **Edge** | The character label on the path from parent to child |
| **Terminal / word-end** | A node where `isEnd = true` — a complete word ends here |
| **Prefix** | Any path from root that does not require `isEnd` — all words sharing a prefix pass through the same nodes |
| **TrieNode** | The building block: a dictionary (or array) of children + an `isEnd` flag |
| **SIGMA** | The alphabet size — 26 for lowercase English |

<Callout type="green" label="Rule of Thumb">
  Use a trie whenever the problem involves **common prefixes**, **autocomplete**, **word lookup by prefix**, or **prefix pruning during search**. If every query is exact lookup with no prefix structure, a hash set is simpler.
</Callout>
