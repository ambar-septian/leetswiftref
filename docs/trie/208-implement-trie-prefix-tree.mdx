<SectionHeader n={1} title="Problem Overview" />

A **trie** (also called a prefix tree) is a tree data structure used to efficiently store and retrieve keys in a dataset of strings.

Implement the `Trie` class with the following operations:

- `Trie()` — Initialises the trie object.
- `void insert(String word)` — Inserts the string `word` into the trie.
- `boolean search(String word)` — Returns `true` if the string `word` is in the trie (i.e. was previously inserted), and `false` otherwise.
- `boolean startsWith(String prefix)` — Returns `true` if there is a previously inserted string that has the prefix `prefix`, and `false` otherwise.

**Constraints:**
- `1 <= word.length, prefix.length <= 2000`
- `word` and `prefix` consist only of lowercase English letters.
- At most `3 * 10^4` calls in total will be made to `insert`, `search`, and `startsWith`.

<Callout type="amber" label="Key Constraint">
Words and prefixes consist only of lowercase English letters (26 possible characters). This makes both hash-map children (`[Character: Node]`) and 26-slot fixed-array children viable — the choice affects constant-factor performance, not correctness.
</Callout>

<Callout type="green" label="Rule of Thumb">
When a problem involves fast prefix lookups over a string dictionary, reach for a trie. Traversal depth equals the word length regardless of dictionary size — unlike a hash map, which must hash the full string on every lookup.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Each `Node` stores a dictionary of children and an `isEndOfWord` flag. Insert walks character by character, creating nodes as needed, then marks the final node. Both `search` and `startsWith` traverse identically — the only difference is the return value: `search` checks `isEndOfWord` at the final node while `startsWith` returns `true` as soon as the full path exists.
</Callout>

Search and `startsWith` both walk the same path — `search` returns `curr.isEndOfWord`, while `startsWith` returns `true` as long as the path exists.

<Callout type="cyan" label="isEndOfWord vs path existence">
Without the end-of-word flag, searching for "app" would return true even if only "apple" was inserted — the path a→p→p exists but "app" was never fully inserted as a word.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Define the Node class">
    Each node has `var children = [Character: Node]()` and `var isEndOfWord = false`.
  </StepItem>
  <StepItem title="insert — walk and create">
    For each character: if a child exists, move into it; otherwise create a new node. After the loop, set `curr.isEndOfWord = true`.
  </StepItem>
  <StepItem title="search — walk and check flag">
    For each character: if the child does not exist, return `false`. After the loop, return `curr.isEndOfWord`.
  </StepItem>
  <StepItem title="startsWith — walk and check existence">
    Identical traversal to search. Return `false` if any child is missing; return `true` if the full prefix path exists (ignore `isEndOfWord`).
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Node {
  var children = [Character: Node]()
  var isEndOfWord = false
}

class Trie {
  let root = Node()

  init() {}

  func insert(_ word: String) {
    var curr = root
    for char in word {
      if let existing = curr.children[char] {
        curr = existing
      } else {
        let node = Node()
        curr.children[char] = node
        curr = node
      }
    }
    curr.isEndOfWord = true
  }

  func search(_ word: String) -> Bool {
    var curr = root
    for char in word {
      guard let existing = curr.children[char] else { return false }
      curr = existing
    }
    return curr.isEndOfWord
  }

  func startsWith(_ prefix: String) -> Bool {
    var curr = root
    for char in prefix {
      guard let existing = curr.children[char] else { return false }
      curr = existing
    }
    return true
  }
}
```

<Callout type="green" label="Swift Notes">
`[Character: Node]` for children leverages `Character`'s `Hashable` conformance. The `let root = Node()` stored property uses `let` safely because `Node` is a reference type — the constant reference never changes, but the node's `children` dictionary and `isEndOfWord` flag are freely mutated.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(m) per operation">
Each operation traverses at most m characters where m is the word or prefix length. Each character lookup in the `[Character: Node]` dictionary is O(1) amortised. All three operations (`insert`, `search`, `startsWith`) are O(m).
</Callout>

<Callout type="green" label="Space Complexity — O(m × n)">
In the worst case, no two inserted words share any prefix. The trie stores m nodes per word for n total words: O(m × n) nodes. Each node holds a dictionary with up to 26 entries for lowercase English.
</Callout>

<Callout type="amber" label="When to prefer Hash Map vs 26-Slot Array children">
Prefer a **hash map `[Character: Node]`** for any character set or when memory per node matters — it only allocates entries for characters actually inserted. Prefer a **26-slot fixed array** for purely lowercase input when lookup speed is critical; array indexing by ASCII offset is faster than dictionary hashing.
</Callout>
