<SectionHeader n={1} title="What is a Math Problem?" />

Math problems on LeetCode rely on properties of integers rather than on data-structure traversal. They are usually O(log n) or O(n) in complexity, but the insight is purely mathematical.

**Four recurring primitives:**

| Primitive | Swift idiom | Typical use |
|---|---|---|
| **Modulo / remainder** | `n % k` | FizzBuzz, cycle detection, circular indexing |
| **Digit extraction** | `n % 10`, `n / 10` | Palindrome number, digit sum, reverse integer |
| **GCD (Euclidean)** | `gcd(a, b)` via `while b != 0` | GCD of strings, coprime checks |
| **Integer palindrome** | Reverse half, compare | Palindrome Number without string conversion |

<Callout type="amber" label="Key Constraint">
  Most math problems have an **integer overflow trap**. In Swift, `Int` is 64-bit on all modern platforms, so `Int.max ≈ 9.2 × 10¹⁸` — this covers the vast majority of LeetCode constraints. When a problem says values can exceed `2³¹ - 1`, check whether the intermediate computation overflows `Int` before reaching the result.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **Modulo** | `a % b` — the remainder after dividing a by b; always in `[0, b-1]` for positive b |
| **Integer division** | `a / b` — truncates toward zero in Swift (`-7 / 2 == -3`, not `-4`) |
| **GCD** | Greatest common divisor — largest integer dividing both a and b with no remainder |
| **Digit extraction** | Repeatedly taking `n % 10` to read the last digit, then `n / 10` to shift right |
| **Overflow** | Result exceeds the type's representable range; Swift traps on overflow by default |

<Callout type="green" label="Rule of Thumb">
  If the problem works entirely on **integer values** with no array or string structure — modulo, digit sums, divisibility, GCD — reach for a math primitive before considering any data structure.
</Callout>
