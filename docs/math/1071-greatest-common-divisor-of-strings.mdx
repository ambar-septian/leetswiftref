<SectionHeader n={1} title="Problem Overview" />

For two strings `str1` and `str2`, we say `t` divides `str1` if `str1 = t + t + ... + t` for some number of concatenations of `t`. Given two strings `str1` and `str2`, return the largest string `t` that divides both `str1` and `str2`.

**Constraints:**
- `1 <= str1.length, str2.length <= 1000`
- `str1` and `str2` consist of English uppercase letters only.

<Callout type="amber" label="Key Constraint">
  A common divisor string only exists if `str1 + str2 == str2 + str1`. This commutativity check is both necessary and sufficient — if it fails, return `""` immediately without computing GCD.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  str1 = "ABCABC", str2 = "ABC"
Output: "ABC"
```

#### Example 2

```
Input:  str1 = "ABABAB", str2 = "ABAB"
Output: "AB"
```

#### Example 3

```
Input:  str1 = "LEET", str2 = "CODE"
Output: ""
```

<Callout type="green" label="Rule of Thumb">
  When a string problem involves repeating units, check commutativity of concatenation first. If `str1 + str2 == str2 + str1`, a common tiling unit exists and its length equals the GCD of the two string lengths.

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Two strings have a common divisor if and only if `str1 + str2 == str2 + str1`. This single check is both necessary and sufficient. Once confirmed, the longest common divisor has length equal to `gcd(str1.count, str2.count)` — take that prefix from str1 as the answer.
</Callout>

Two strings share a common divisor if and only if their concatenations commute: `str1 + str2 == str2 + str1`. This is because any repeating unit `t` that tiles both strings must also tile their concatenation regardless of order.

Once we confirm commutativity, the length of the largest divisor must divide both `str1.count` and `str2.count`. The mathematical GCD gives us exactly that length, and `str1.prefix(gcd)` is our answer.

<Callout type="green" label="Key Insight">
The commutativity check `str1 + str2 == str2 + str1` is both necessary and sufficient. If it passes, the GCD-length prefix of `str1` is guaranteed to divide both strings — no further verification needed.
</Callout>

<Callout type="cyan" label="Why GCD works">
If `t` of length `k` tiles both strings, then `k` divides `len(str1)` and `k` divides `len(str2)`, so `k` divides `gcd(len(str1), len(str2))`. The GCD itself gives the maximum valid `k`.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Commutativity check">
    If `str1 + str2 != str2 + str1`, no common divisor exists — return `""`.
  </StepItem>
  <StepItem title="Compute GCD of lengths">
    Use the Euclidean algorithm: repeatedly compute `result % curr` until `curr == 0`. The final `result` is the GCD.
  </StepItem>
  <StepItem title="Return prefix">
    Return `String(str1.prefix(gcdResult))`. This is the largest string that tiles both inputs.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func gcdOfStrings(_ str1: String, _ str2: String) -> String {
    guard str1 + str2 == str2 + str1 else { return "" }
    let gcdResult = gcd(str1.count, str2.count)
    return String(str1.prefix(gcdResult))
  }

  func gcd(_ x: Int, _ y: Int) -> Int {
    var curr = y
    var result = x
    while curr != 0 {
      let temp = curr
      curr = result % temp
      result = temp
    }
    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `str1 + str2` creates a new String — O(n + m). The `guard` short-circuits immediately if the check fails. `String(str1.prefix(gcdResult))` slices using Swift's `prefix` method which is O(gcdResult). The custom `gcd` helper uses the Euclidean algorithm — Swift's standard library does not expose a GCD function in the language core (it is available via `Foundation`'s `__gcd` but defining your own is cleaner).
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n + m)">
  The commutativity check concatenates and compares two strings: O(n + m). The GCD computation via the Euclidean algorithm is O(log(min(n, m))). Taking the prefix is O(gcd). Dominant cost: O(n + m).
</Callout>

<Callout type="green" label="Space Complexity — O(n + m)">
  The two concatenated strings `str1 + str2` and `str2 + str1` each take O(n + m) space. The result prefix is O(gcd). Total auxiliary space: O(n + m).
</Callout>
