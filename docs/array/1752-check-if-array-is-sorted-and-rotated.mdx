<SectionHeader n={1} title="Problem Overview" />

Given an array `nums`, return `true` if the array was originally sorted in non-decreasing order and then **rotated** some number of positions (including zero). Otherwise, return `false`.

There may be **duplicates** in the original array.

Note: An array `A` rotated by `x` positions results in an array `B` where `A[i] == B[(i+x) % A.length]`.

**Constraints:**
- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`

<Callout type="amber" label="Key Constraint">
  Duplicates are allowed — the original array is non-decreasing (not strictly increasing), so equal adjacent values are valid. Any array of length 1 or 2 is trivially a valid sorted rotation.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  nums = [3,4,5,1,2]
Output: true
Explanation: [1,2,3,4,5] rotated 3 positions.
```

#### Example 2

```
Input:  nums = [2,1,3,4]
Output: false
Explanation: Not a rotation of any sorted array.
```

#### Example 3

```
Input:  nums = [1,2,3]
Output: true
Explanation: Already sorted — rotated 0 positions.
```

<Callout type="green" label="Rule of Thumb">
  A sorted-and-rotated array has at most one circular descent — one position where nums[i] &gt; nums[(i+1) % n]. Count circular descents; if the count exceeds 1, return false.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  A sorted-and-rotated array has at most one position where nums[i] &lt; nums[i-1] (the rotation point). After that point, every element must also be &lt;= nums[0] to satisfy the circular non-decreasing property. Two violations in either condition mean the array is not a valid rotation.
</Callout>

Scan for descents (positions where `nums[i] < nums[i-1]`). A valid sorted-and-rotated array has at most one such descent. Additionally, at any descent point, `nums[i]` must be less than or equal to `nums[0]` to maintain the circular non-decreasing property.

After the rotation point, the sequence must remain non-decreasing AND each element must be less than or equal to `nums[0]`.

<Callout type="cyan" label="Simpler alternative">
Count the number of descents (positions where nums[i] greater than nums[i+1], treating the array as circular). A valid result has at most 1 such descent in the circular sense.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Handle edge cases">
    If `count <= 2`, any array of 1 or 2 elements is valid.
  </StepItem>
  <StepItem title="Scan for descents">
    For each `i` from 1 to n-1: if `nums[i] >= nums[i-1]`, check the post-rotation constraint (`!reset` means not yet past the rotation point); if `!reset`, continue. If `reset` is already true and the current value exceeds `nums[0]`, return false.
  </StepItem>
  <StepItem title="Handle descent">
    If `nums[i] < nums[i-1]` and `reset` is already true, return false (two descents). Check that the last element satisfies `nums[i] <= nums[0]`. Set `reset = true`.
  </StepItem>
  <StepItem title="Return true">
    If the loop completes, return true.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func check(_ nums: [Int]) -> Bool {
    let count = nums.count
    if count <= 2 { return true }
    var reset = false

    for i in 1..<count {
      let curr = nums[i]
      let prev = nums[i - 1]

      if curr >= prev {
        if !reset { continue }
        if curr > nums[0] { return false }  // post-rotation value exceeds start
      } else {
        if reset { return false }           // second descent
        if i == count - 1 && curr > nums[0] { return false }
        reset = true
      }
    }

    return true
  }
}
```

<Callout type="green" label="Swift Notes">
  `let curr = nums[i]` and `let prev = nums[i-1]` improve readability over inline indexing. The `reset` boolean tracks whether a descent has been seen — a single Bool is all the state needed. The early return `if count &lt;= 2 { return true }` avoids out-of-bounds when accessing nums[0] in the wrap check.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  A single pass over all n elements. Each element requires one comparison and at most one additional check — O(1) per element. Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
  Only a boolean flag (reset) and a loop index are maintained. No auxiliary data structures. O(1) extra space.
</Callout>
