<SectionHeader n={1} title="Problem Overview" />

Given an integer array `arr`, return `true` if there are three consecutive odd numbers in the array. Otherwise, return `false`.

**Constraints:**
- `1 <= arr.length <= 1000`
- `1 <= arr[i] <= 1000`

<Callout type="amber" label="Key Constraint">
  All values are positive (arr[i] &gt;= 1), so there are no negative numbers to worry about. The streak counter must not reset once it reaches 3 — guard with else if current &lt; 3 to preserve found streaks.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  arr = [2,6,4,1]
Output: false
Explanation: No three consecutive odd numbers.
```

#### Example 2

```
Input:  arr = [1,2,34,3,4,5,7,23,12]
Output: true
Explanation: [5,7,23] are three consecutive odd numbers.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  A streak counter is all that's needed: increment on odd, reset on even (guarded by else if current &lt; 3 to preserve found streaks), and return current &gt;= 3 at the end. Single pass, O(1) space.
</Callout>

Maintain a counter `current`. Increment on odd numbers; reset to 0 on even numbers. Return `true` as soon as `current >= 3`.

<Callout type="yellow" label="Early exit">
Once `current >= 3`, we can return immediately. The loop in the implementation does not early-exit, but once `current >= 3` any further even elements would only be skipped by the `current less than 3` guard.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise counter">
    `var current = 0`.
  </StepItem>
  <StepItem title="Scan the array">
    For each element: if odd, increment `current`; if even and `current less than 3`, reset `current = 0`.
  </StepItem>
  <StepItem title="Return result">
    Return `current >= 3`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func threeConsecutiveOdds(_ arr: [Int]) -> Bool {
    var current = 0
    for num in arr {
      if num % 2 != 0 {
        current += 1
      } else if current < 3 {
        current = 0
      }
    }
    return current >= 3
  }
}
```

<Callout type="green" label="Swift Notes">
  `num % 2 != 0` works correctly for all Swift integers including negatives — preferred over `num % 2 == 1` which would fail for negative odd numbers. The `else if current &lt; 3` guard prevents resetting the streak after it reaches 3, preserving the result without needing an early return.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  A single pass over all n elements. Each element requires one modulo check and one counter update — O(1) per element. Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
  Only one integer counter (current) is maintained. No auxiliary data structures. O(1) extra space.
</Callout>
