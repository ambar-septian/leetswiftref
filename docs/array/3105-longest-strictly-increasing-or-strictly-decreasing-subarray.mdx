<SectionHeader n={1} title="Problem Overview" />

You are given an array of integers `nums`.

Return the length of the **longest subarray** of `nums` which is either **strictly increasing** or **strictly decreasing**.

**Constraints:**
- `1 <= nums.length <= 50`
- `1 <= nums[i] <= 50`

<Callout type="amber" label="Key Constraint">
  Constraints are tiny (n, values ≤ 50), but the key correctness requirement is handling equal adjacent elements — they break both strict conditions and must reset both streak counters to 1.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  nums = [1,4,3,3,2]
Output: 2
Explanation:
  Strictly increasing subarrays: [1,4] (length 2).
  Strictly decreasing subarrays: [4,3], [3,2] (length 2).
  Maximum is 2.
```

#### Example 2

```
Input:  nums = [3,3,3,3]
Output: 1
Explanation: All elements equal — no adjacent pair is strictly increasing or decreasing.
```

#### Example 3

```
Input:  nums = [3,2,1]
Output: 3
Explanation: The whole array is strictly decreasing.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Maintain two independent streak counters (`currentInc`, `currentDec`). On each step: if the direction matches, extend one and reset the other to 1. On equality, reset both. The answer is the maximum streak seen in either counter.
</Callout>

Initialise `currentInc = 1`, `currentDec = 1`, `maxInc = 1`, `maxDec = 1`. Iterate from index 1. For each pair `(prev, curr)`:
- If `curr > prev`: extend `currentInc`, reset `currentDec = 1`.
- If `curr < prev`: extend `currentDec`, reset `currentInc = 1`.
- If `curr == prev`: reset both to 1.

Update the running maxima. Return `max(maxInc, maxDec)`.

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise counters">
    `var currentInc = 1, currentDec = 1, maxInc = 1, maxDec = 1`.
  </StepItem>
  <StepItem title="Iterate from index 1">
    For each `i` in `1..<nums.count`: compare `nums[i]` with `nums[i-1]`.
    - Greater: `currentInc += 1`, `maxInc = max(maxInc, currentInc)`, `currentDec = 1`.
    - Less: `currentDec += 1`, `maxDec = max(maxDec, currentDec)`, `currentInc = 1`.
    - Equal: `currentInc = 1`, `currentDec = 1`.
  </StepItem>
  <StepItem title="Return max">
    Return `max(maxInc, maxDec)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func longestMonotonicSubarray(_ nums: [Int]) -> Int {
    var currentInc = 1, currentDec = 1
    var maxInc = 1, maxDec = 1

    for i in 1..<nums.count {
      let curr = nums[i], prev = nums[i - 1]
      if curr > prev {
        currentInc += 1
        maxInc = max(maxInc, currentInc)
        currentDec = 1
      } else if curr < prev {
        currentDec += 1
        maxDec = max(maxDec, currentDec)
        currentInc = 1
      } else {
        currentInc = 1
        currentDec = 1
      }
    }

    return max(maxInc, maxDec)
  }
}
```

<Callout type="green" label="Swift Notes">
  Using four named variables (`currentInc`, `maxInc`, `currentDec`, `maxDec`) is clearer than an array of 2 and avoids indexing errors. `let curr = nums[i], prev = nums[i-1]` caches both values once, keeping all three branches readable.
</Callout>

<Callout type="cyan" label="Time Complexity — O(n)">

<Callout type="green" label="Space Complexity — O(1)">
