<SectionHeader n={1} title="What are Array Patterns?" />

Most array problems fall into a small number of recurring shapes:

| Pattern | Description | Examples |
|---|---|---|
| **Linear scan** | Single pass with a running counter or flag | Three Consecutive Odds, Special Array I |
| **Index mapping** | Output at index i is computed from another index | Build Array from Permutation |
| **Simulation** | Follow instructions step by step, maintain state | Plus One, Baseball Game |
| **Subarray check** | Sliding window or two-pointer over subarrays | Subarray Count with Condition |
| **Rotation / sorted check** | Detect structure in a transformed array | Check if Sorted and Rotated |

<Callout type="amber" label="Key Constraint">
  Watch for **off-by-one errors** in index arithmetic and **integer overflow** when summing or multiplying array values. In Swift, `Int` is 64-bit on all Apple platforms, so overflow is rare — but be careful with `num * 10 + digit` style digit extraction for very large numbers. Also note that Swift's `Array` is a **value type**: assigning `var copy = array` makes a full copy — useful for in-place problems but a hidden cost if done inside a loop.
</Callout>

<Callout type="green" label="Rule of Thumb">
  If the problem involves a single pass over an array and the answer is a count, a flag, or an element — reach for a linear scan. If the answer at index i is computed from another index — reach for index mapping. Only reach for a second data structure when the single-pass approach cannot maintain the necessary state.
</Callout>
