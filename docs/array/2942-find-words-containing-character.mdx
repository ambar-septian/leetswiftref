<SectionHeader n={1} title="Problem Overview" />

You are given a **0-indexed** array of strings `words` and a character `x`.

Return an array of **indices** representing the words that contain the character `x`.

**Note** that the returned array may be in **any** order.

**Constraints:**
- `1 <= words.length <= 50`
- `1 <= words[i].length <= 50`
- `x` is a lowercase English letter.
- `words[i]` consists only of lowercase English letters.

<Callout type="amber" label="Key Constraint">
  Both `words.length` and `words[i].length` are at most 50 — the O(n × m) brute-force scan is at most 2,500 operations. No preprocessing or hash structure is needed.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  words = ["leet","code"], x = "e"
Output: [0,1]
Explanation: "leet" contains 'e'. "code" contains 'e'.
```

#### Example 2

```
Input:  words = ["abc","bcd","aaaa","cbc"], x = "a"
Output: [0,2]
Explanation: "abc" contains 'a'. "aaaa" contains 'a'. "bcd" and "cbc" do not.
```

#### Example 3

```
Input:  words = ["abc","bcd","aaaa","cbc"], x = "z"
Output: []
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Iterate words with `enumerated()` to get both the index and the word in one expression. For each word, `word.contains(x)` does a linear scan — O(m) per word. Collect the index on a match. One pass, no extra data structure.
</Callout>

Iterate `words` with their indices. For each word, check `word.contains(x)`. If true, append the index to the result array.

<Callout type="green" label="Swift contains">
`String.contains(Character)` scans the string linearly — O(m) per word. Total: O(n * m) where n = number of words, m = max word length.
</Callout>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func findWordsContaining(_ words: [String], _ x: Character) -> [Int] {
    var result = [Int]()
    for (i, word) in words.enumerated() {
      if word.contains(x) {
        result.append(i)
      }
    }
    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `words.enumerated()` yields `(offset: Int, element: String)` tuples — destructured as `(i, word)` in the for-in binding. `word.contains(x)` where `x` is a `Character` does a linear scan. For inputs this small (n, m ≤ 50) there is no benefit to building a `Set&lt;Character&gt;` per word.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n · m)">
  n words, each `contains` call is O(m) where m is the word length. With n, m ≤ 50 this is at most 2,500 operations. Total: O(n · m).
</Callout>

<Callout type="green" label="Space Complexity — O(k)">
  The result array holds k matching indices where k ≤ n. No auxiliary data structures beyond the output array.
</Callout>
