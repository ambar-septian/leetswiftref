<SectionHeader n={1} title="Problem Overview" />

Given two strings `s` and `t`, return `true` if `s` is a subsequence of `t`, or `false` otherwise.

A subsequence of a string is a new string that is formed from the original string by deleting some (can be none) of the characters without disturbing the relative order of the remaining characters. For example, `"ace"` is a subsequence of `"abcde"` while `"aec"` is not.

**Constraints:**
- `0 <= s.length <= 100`
- `0 <= t.length <= 10^4`
- `s` and `t` consist only of lowercase English letters.

**Follow-up:** If there are lots of incoming `s` values (e.g., `s1, s2, ... sk` where k is large), how would you change your code? (Binary search with pre-built index lists.)

<Callout type="amber" label="Key Constraint">
`s` can be empty (length 0), which is a valid subsequence of any `t`. The two-pointer return condition `i == s.count` handles this correctly without special-casing — when `s` is empty, `i` starts at 0 which already equals `s.count`.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  s = "abc", t = "ahbgdc"
Output: true
Explanation: a_h_b_g_d_c — a, b, c all appear in order.
```

#### Example 2

```
Input:  s = "axc", t = "ahbgdc"
Output: false
Explanation: 'x' does not appear in t after 'a'.
```

<Callout type="green" label="Rule of Thumb">
When checking subsequence membership, advance the subsequence pointer only on a match and always advance the text pointer. The subsequence is confirmed when its pointer exhausts all characters.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Use pointer `i` for `s` and pointer `j` for `t`. Advance `i` only when characters match; always advance `j`. When `j` reaches the end of `t`, check if `i` reached the end of `s`. This scans `t` exactly once — O(|t|) — regardless of how many characters matched.
</Callout>

Convert both strings to character arrays. Use pointer `i` for `s` and pointer `j` for `t`. While `j` is within `t` and `i` has not exhausted `s`: if the characters match, advance `i`. Always advance `j`. At the end, if `i == s.count` every character of `s` was matched in order — return `true`.

<Callout type="cyan" label="Why always advance j">
We scan t completely to find all characters of s in order. Advancing j on every iteration ensures we check every potential match position in t without double-counting.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Convert to character arrays">
    `let schars = s.map { $0 }`, `let tchars = t.map { $0 }`.
  </StepItem>
  <StepItem title="Initialise two pointers">
    `var i = 0` (tracks position in s), `var j = 0` (tracks position in t).
  </StepItem>
  <StepItem title="Scan t">
    While `j less than t.count` and `i less than s.count`: if `schars[i] == tchars[j]`, increment `i`. Always increment `j`.
  </StepItem>
  <StepItem title="Check completion">
    Return `i == s.count`. If `i` reached the end, every character of `s` was matched in order.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func isSubsequence(_ s: String, _ t: String) -> Bool {
    let schars = s.map { $0 }
    let tchars = t.map { $0 }
    let tcount = tchars.count
    let scount = schars.count

    var i = 0
    var j = 0

    while j < tcount, i < scount {
      if schars[i] == tchars[j] {
        i += 1
      }
      j += 1
    }

    return i == scount
  }
}
```

<Callout type="green" label="Swift Notes">
`s.map { $0 }` converts `String` to `[Character]` for O(1) integer indexing — Swift strings are not directly integer-subscriptable. The `while j < tcount, i < scount` condition exits early once all of `s` is matched, avoiding unnecessary scans of the rest of `t`. The empty-string case (`s = ""`) is handled correctly: `i == 0 == scount` is true immediately.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
`j` advances on every iteration and never goes backward, so the while loop runs at most n = `t.length` times. `i` advances at most `s.length` times. Total: O(s.length + t.length) = O(n) where n dominates.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
Converting both strings to `[Character]` arrays takes O(s.length + t.length) extra space. If the strings were iterable without conversion (e.g., using `String.unicodeScalars`), space could be reduced to O(1) — only two index variables needed.
</Callout>
