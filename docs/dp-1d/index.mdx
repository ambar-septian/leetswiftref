<SectionHeader n={1} title="What is 1D Dynamic Programming?" />

Dynamic programming solves problems by breaking them into overlapping subproblems and storing each result so it is computed only once. In the 1D case, the subproblem state is a single index — the answer for index i depends only on answers for indices less than i.

| Aspect | Description |
|---|---|
| **State** | `dp[i]` = answer for subproblem at index i |
| **Recurrence** | Express `dp[i]` in terms of earlier `dp` values |
| **Base case** | The smallest subproblems that can be answered directly |
| **Answer** | Usually `dp[n]` or `max(dp)` or `dp[n-1]` |
| **Space opt.** | When `dp[i]` depends only on `dp[i-1]` (and `dp[i-2]`), keep only the last 1–2 values |

<Callout type="amber" label="Key Constraint">
  Identify the **recurrence relation** before writing any code. If you cannot express `dp[i]` as a function of earlier `dp` values, the subproblem decomposition is wrong. Common recurrences: `dp[i] = dp[i-1] + dp[i-2]` (Fibonacci/stairs), `dp[i] = max(dp[i-1], ...)` (best-so-far), `dp[i] = dp[i-1] + val[i]` (running sum / Pascal).
</Callout>

<Callout type="green" label="Rule of Thumb">
  If the answer for position i depends only on answers for positions before i, and the same subproblems recur across different values of i — that is 1D DP. The tell is overlapping subproblems: naive recursion would recompute the same value many times.
</Callout>
