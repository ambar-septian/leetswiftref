<SectionHeader n={1} title="Problem Overview" />

Given `n` non-negative integers representing an elevation map where the width of each bar is `1`, compute how much water it can trap after raining.

The amount of water trapped above position `i` is determined by the minimum of the tallest bar to its left and the tallest bar to its right, minus the height at `i`.

**Constraints**
- `n == height.length`
- `1 <= n <= 2 * 10^4`
- `0 <= height[i] <= 10^5`

<Callout type="green" label="Core Insight">
Maintain a left pointer and a right pointer with their respective running maximums. The side with the smaller maximum is the bottleneck — advance that pointer inward and accumulate `max - height[pointer]` as trapped water. No extra arrays needed.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  height = [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6
```

#### Example 2
```
Input:  height = [4,2,0,3,2,5]
Output: 9
```

<SectionHeader n={4} title="Approach" />

<TrappingRainWaterDiagram />

Initialize `l=0`, `r=n-1`, `leftMax=height[0]`, `rightMax=height[n-1]`, `result=0`.

While `l < r`:
- If `leftMax < rightMax`, the left side is the bottleneck. Advance `l` inward, update `leftMax`, and add `leftMax - height[l]` to `result`.
- Otherwise, the right side is the bottleneck. Advance `r` inward, update `rightMax`, and add `rightMax - height[r]`.

The key invariant: when processing the left pointer, we know `rightMax >= leftMax`, so the water level is capped by `leftMax` and the right side is guaranteed to hold it.

<Callout type="cyan" label="Why leftMax - height[l] is always non-negative">
We update leftMax before computing the difference: `leftMax = max(leftMax, height[l])`. This means leftMax is always at least height[l], so the subtraction is always >= 0. We only add to result — never subtract.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard empty input">
    Return 0 immediately if `height` is empty.
  </StepItem>
  <StepItem title="Initialize two pointers and running maxima">
    `l=0`, `r=height.count-1`, `leftMax=height[0]`, `rightMax=height[r]`, `result=0`.
  </StepItem>
  <StepItem title="Advance the weaker side">
    While `l < r`: if `leftMax < rightMax`, increment `l`, update `leftMax`, add `leftMax - height[l]`. Else decrement `r`, update `rightMax`, add `rightMax - height[r]`.
  </StepItem>
  <StepItem title="Return result">
    The accumulated `result` is the total trapped water.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func trap(_ height: [Int]) -> Int {
    guard !height.isEmpty else { return 0 }

    var l = 0
    var r = height.count - 1
    var leftMax = height[l]
    var rightMax = height[r]
    var result = 0

    while l < r {
      if leftMax < rightMax {
        l += 1
        leftMax = max(leftMax, height[l])
        result += leftMax - height[l]
        
      } else {
        r -= 1
        rightMax = max(rightMax, height[r])
        result += rightMax - height[r]
      }
    }

    return result
  }
}
```
