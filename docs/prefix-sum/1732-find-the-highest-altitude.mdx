<SectionHeader n={1} title="Problem Overview" />

There is a biker going on a road trip. The road trip consists of `n + 1` points at different altitudes. The biker starts his trip on point `0` with altitude equal to `0`.

You are given an integer array `gain` of length `n` where `gain[i]` is the **net gain in altitude** between points `i` and `i+1` for all `0 <= i < n`. Return the **highest altitude** of a point.

**Constraints:**
- `n == gain.length`
- `1 <= n <= 100`
- `-100 <= gain[i] <= 100`

<Callout type="amber" label="Key Constraint">
  gain[i] can be negative, so altitudes can decrease at any step. The biker starts at altitude 0, which must be included as a candidate for the maximum — initialise maxAlt = 0, not Int.min.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  gain = [-5,1,5,0,-7]
Output: 1
Explanation: Altitudes: [0,-5,-4,1,1,-6]. Maximum = 1.
```

#### Example 2

```
Input:  gain = [-4,-3,-2,-1,4,3,2]
Output: 0
Explanation: Altitudes: [0,-4,-7,-9,-10,-6,-3,-1]. Maximum = 0 (the start).
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  altitude[i] = altitude[i-1] + gain[i-1]. Start with sum = 0 and maxAlt = 0 (to include the starting altitude). After each addition, update maxAlt = max(sum, maxAlt). A single accumulator replaces a full prefix array.
</Callout>

Start with `sum = 0` (altitude at point 0) and `maxAlt = 0`. For each gain value, add it to `sum` and update `maxAlt = max(sum, maxAlt)`.

<Callout type="yellow" label="Include starting altitude">
The biker starts at altitude 0. Initialise `maxAlt = 0` to ensure the starting point is considered even if all subsequent altitudes are negative.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise">
    `var sum = 0`, `var maxAlt = 0`.
  </StepItem>
  <StepItem title="Accumulate and track">
    For each gain: `sum += gain[i]`, `maxAlt = max(sum, maxAlt)`.
  </StepItem>
  <StepItem title="Return maxAlt">
    Return the highest altitude reached.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func largestAltitude(_ gain: [Int]) -> Int {
    var sum = 0
    var maxAlt = 0
    for g in gain {
      sum += g
      maxAlt = max(sum, maxAlt)
    }
    return maxAlt
  }
}
```

<Callout type="green" label="Swift Notes">
  `max(sum, maxAlt)` uses Swift's standard library global function — no import needed. Iterating `for g in gain` (value-type copy) is clean and avoids index arithmetic. An alternative: `gain.reduce(into: (sum: 0, max: 0)) { acc, g in acc.sum += g; acc.max = Swift.max(acc.sum, acc.max) }.max` — but the plain loop is more readable.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  A single pass over all n gain values. Each step requires one addition and one max call — both O(1). Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
  Only two variables (sum, maxAlt) are maintained. No auxiliary array or data structure. O(1) extra space.
</Callout>
