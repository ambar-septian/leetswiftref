<SectionHeader n={1} title="What is a Prefix Sum?" />

A **prefix sum array** stores the cumulative sum of elements from the start of the array up to each index. Given `nums`, the prefix array is defined as:

```
prefix[0] = 0
prefix[i] = prefix[i-1] + nums[i-1]   (1-indexed from input)
```

This one-off O(n) build lets you answer "what is the sum of `nums[l..r]`?" in O(1):

```
rangeSum(l, r) = prefix[r + 1] - prefix[l]
```

**Key properties:**

| Property | Value |
|---|---|
| Build time | O(n) |
| Range query time | O(1) per query |
| Extra space | O(n) for the prefix array |
| Works for | Sum, product, XOR, count — any invertible aggregate |

<Callout type="amber" label="Key Constraint">
  The prefix sum trick requires that the **aggregate operation is invertible** (addition is; max/min are not). For range minimum/maximum queries you need a Sparse Table or Segment Tree instead.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **Prefix sum** | `prefix[i]` = sum of `nums[0..i-1]`; `prefix[0] = 0` |
| **Range query** | "What is the sum of elements from index l to r (inclusive)?" |
| **Off-by-one sentinel** | `prefix[0] = 0` avoids a special case for queries starting at index 0 |
| **Running sum** | Simpler variant: compute prefix in-place, overwriting the input array |
| **2D prefix sum** | Extends the idea to matrices for rectangle sum queries |
| **Difference array** | The inverse: stores deltas so range updates are O(1) and point queries are O(n) after prefix reconstruction |

<Callout type="green" label="Rule of Thumb">
  If the problem asks for **range sums**, **subarray sums equal to a target**, or mentions **multiple queries over the same array** — build a prefix sum first.
</Callout>
