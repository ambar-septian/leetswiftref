<SectionHeader n={1} title="Problem Overview" />

Given an integer array `nums` and an integer `k`, return the `k`th largest element in the array. Note that it is the `k`th largest element in sorted order, not the `k`th distinct element.

You must solve it without sorting.

**Constraints:**
- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

<Callout type="amber" label="Key Constraint">
The problem explicitly forbids sorting. This steers toward a heap or Quickselect approach. The heap builds in O(n) and retrieves the kth maximum in O(k log n) — O(n + k log n) total, beating the O(n log n) sort.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: nums = [3,2,1,5,6,4], k = 2
Output: 5
```

#### Example 2

```
Input: nums = [3,2,3,1,2,4,5,5,6], k = 4
Output: 4
```

<Callout type="green" label="Rule of Thumb">
When asked for the kth largest (or smallest) element without sorting, reach for a heap. A max-heap lets you pop the global maximum in O(log n) — k pops give the kth largest in O(n + k log n) total.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Heapify all elements into a max-heap in O(n). Then pop the maximum k times — each pop is O(log n). The value returned on the kth pop is the kth largest. This avoids a full sort and stops as soon as the answer is reached.
</Callout>

Build a max-heap from all elements. Call `popMax()` k times; the value returned on the kth pop is the kth largest.

<Callout type="cyan" label="Swift Heap">
Swift's <code>Heap</code> type (from the swift-collections package) is a min-heap by default. To get max-heap behaviour, wrap values in a struct that reverses the comparator, or call <code>popMax()</code> which is provided on <code>Heap</code> for the maximum element.
</Callout>

<SectionHeader n={7} title="Code Implementation" />

<Callout type="green" label="Swift Notes">
`Heap` comes from the [swift-collections](https://github.com/apple/swift-collections) package. Initialise with `Heap(nums)` to heapify in O(n). It exposes both `popMin()` and `popMax()` — use `popMax()` here for the largest element. `var k = k` shadows the parameter to make it mutable; Swift function parameters are constants by default.
</Callout>

```swift
import Collections

class Solution {
  func findKthLargest(_ nums: [Int], _ k: Int) -> Int {
    var heap = Heap(nums)  // Swift Collections Heap; popMax() available
    var k = k
    var result = -1

    while k > 0 {
      guard let max = heap.popMax() else { continue }
      result = max
      k -= 1
    }

    return result
  }
}
```

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n + k log n)">
Building the heap via heapify is O(n). Each of the k `popMax()` calls restores the heap property in O(log n). Total O(n + k log n) — better than O(n log n) sort when k is small.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
The heap stores all n elements. No additional data structures are allocated beyond the heap itself.
</Callout>

<Callout type="amber" label="When to prefer Max-Heap vs Min-Heap (size k) vs Quickselect">
Prefer **Max-Heap** for clarity when k is not too small. Prefer **Min-Heap (size k)** when memory is tight — it uses O(k) space and is better when k is much smaller than n. Prefer **Quickselect** for O(n) average with O(1) space when the input is not adversarial and the constraint allows in-place mutation.
</Callout>
