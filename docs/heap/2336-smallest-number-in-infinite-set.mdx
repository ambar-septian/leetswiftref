<SectionHeader n={1} title="Problem Overview" />

You have a set which contains all positive integers `[1, 2, 3, 4, 5, ...]`.

Implement the `SmallestInfiniteSet` class:
- `SmallestInfiniteSet()` — initialises the `SmallestInfiniteSet` object to contain **all** positive integers.
- `int popSmallest()` — **removes** and returns the smallest integer contained in the infinite set.
- `void addBack(int num)` — **adds** a positive integer `num` back into the infinite set, if it is not already in the set.

**Constraints:**
- `1 <= num <= 1000`
- At most `1000` calls will be made in total to `popSmallest` and `addBack`.

<SectionHeader n={2} title="Examples" />

#### Example 1

```
SmallestInfiniteSet obj = new SmallestInfiniteSet();
obj.addBack(2);       // 2 is already in the set, does nothing
obj.popSmallest();    // returns 1, set = [2, 3, 4, 5, ...]
obj.popSmallest();    // returns 2, set = [3, 4, 5, ...]
obj.popSmallest();    // returns 3, set = [4, 5, ...]
obj.addBack(1);       // adds 1 back, set = [1, 4, 5, ...]
obj.popSmallest();    // returns 1, set = [4, 5, ...]
obj.popSmallest();    // returns 4, set = [5, 6, ...]
```

<SectionHeader n={4} title="Approach" />

Represent the infinite set with two structures:

1. **`nextAvailable`** — an integer pointer to the smallest number in the untouched range `[nextAvailable, ∞)`. Initially `1`.
2. **Min-heap + hash set** — store only numbers that were popped and then added back, in sorted order.

**`popSmallest`:** If the heap is non-empty, its minimum is smaller than `nextAvailable` (it was previously removed and re-added). Pop and return it, also removing it from the set. Otherwise, return `nextAvailable` and increment it.

**`addBack(num)`:** Skip if `num >= nextAvailable` (it was never removed — still in the unmodified range). Skip if `num` is already in the hash set (already re-added). Otherwise insert into both the heap and the set.

<Callout type="cyan" label="Why the counter approach is efficient">
The naive approach pre-fills a heap with 1..1000 at init (O(n) space, O(n log n) build). The counter approach uses O(M) space where M is the number of elements currently added back — typically a tiny fraction of 1000.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise">
    `var heap = Heap[Int]()` (min-heap). `var uniqueSet = Set[Int]()`. `var nextAvailable = 1`.
  </StepItem>
  <StepItem title="popSmallest">
    If heap is empty: capture `val = nextAvailable`, increment `nextAvailable`, return `val`. Otherwise: `val = heap.popMin()`, remove `val` from `uniqueSet`, return `val`.
  </StepItem>
  <StepItem title="addBack(num)">
    If `num >= nextAvailable`, return (still in unmodified range). If `uniqueSet.contains(num)`, return (already present). Otherwise: `uniqueSet.insert(num)`, `heap.insert(num)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
import Collections

class SmallestInfiniteSet {
  var heap: Heap<Int>
  var uniqueSet: Set<Int>
  var nextAvailable: Int

  init() {
    heap = Heap()
    uniqueSet = Set()
    nextAvailable = 1
  }

  func popSmallest() -> Int {
    if heap.isEmpty {
      let val = nextAvailable
      nextAvailable += 1
      return val
    }
    let min = heap.popMin()!
    uniqueSet.remove(min)
    return min
  }

  func addBack(_ num: Int) {
    guard num < nextAvailable else { return }
    guard !uniqueSet.contains(num) else { return }
    uniqueSet.insert(num)
    heap.insert(num)
  }
}
```
