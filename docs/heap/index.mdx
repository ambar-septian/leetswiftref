<SectionHeader n={1} title="What is a Heap?" />

A heap is stored as a plain array — no pointers needed. The tree structure is implicit from the index relationships:

```
parent(i)      = (i - 1) / 2
leftChild(i)   = 2 * i + 1
rightChild(i)  = 2 * i + 2
```

**Min-heap vs Max-heap:**

| Property | Min-Heap | Max-Heap |
|---|---|---|
| Root element | Smallest | Largest |
| Heap property | `parent <= children` | `parent >= children` |
| Use case | Kth largest, Dijkstra | Kth smallest, scheduling |
| Priority function | `{ $0 < $1 }` | `{ $0 > $1 }` |

<Callout type="amber" label="Key Constraint">
  Swift's standard library does **not** include a heap or priority queue. For LeetCode you have two options: implement a lightweight `Heap` struct (shown below — straightforward and interview-ready), or add the **Swift Collections** package (`import Collections`) which provides `Heap` with a modern API but requires a package dependency. The custom implementation is recommended for interviews.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **Complete binary tree** | All levels full except possibly the last, which is filled left to right |
| **Heap property** | Every node has higher priority than its children (min or max depending on type) |
| **Sift up** | After insert: bubble the new element up until the heap property is restored |
| **Sift down** | After remove-top: push the replacement element down until property is restored |
| **Build heap** | Convert an arbitrary array into a heap in O(n) by sifting down from the middle |
| **Priority function** | Closure `(Element, Element) -> Bool` that defines which element has higher priority |

<Callout type="green" label="Rule of Thumb">
  Reach for a heap whenever a problem repeatedly asks for the current minimum or maximum of a changing set. If the problem says "k largest", "k smallest", "kth element", or requires a greedy always-pick-the-best approach — that is a heap.
</Callout>
