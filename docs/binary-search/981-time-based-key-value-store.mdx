<SectionHeader n={1} title="Problem Overview" />

Design a time-based key-value data structure that can store multiple values for the same key at different time stamps and retrieve the key's value at a certain timestamp.

Implement the `TimeMap` class:
- `TimeMap()` — Initializes the object of the data structure.
- `void set(String key, String value, int timestamp)` — Stores the key `key` with the value `value` at the given time `timestamp`.
- `String get(String key, int timestamp)` — Returns a value such that `set` was called previously with `key = key`, `value = value`, and `timestamp_prev <= timestamp`. If there are multiple such values, it returns the value associated with the **largest** `timestamp_prev`. If there are no values, it returns `""`.

It is guaranteed that, for all calls to `set`, the timestamps are strictly increasing.

**Constraints:**
- `1 <= key.length, value.length <= 100`
- `key` and `value` consist of lowercase English letters and digits.
- `1 <= timestamp <= 10^7`
- All the timestamps of `set` are strictly increasing.
- At most `2 * 10^5` calls will be made to `set` and `get`.

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:
["TimeMap","set","get","get","set","get","get"]
[[],["foo","bar",1],["foo",1],["foo",3],["foo","bar2",4],["foo",4],["foo",5]]

Output: [null,null,"bar","bar",null,"bar2","bar2"]

Explanation:
  set("foo","bar",1)  → entries["foo"] = [{value:"bar",timestamp:1}]
  get("foo",1)        → ts<=1: "bar"
  get("foo",3)        → ts<=3: largest is 1 → "bar"
  set("foo","bar2",4) → entries["foo"] = [{1,"bar"},{4,"bar2"}]
  get("foo",4)        → ts<=4: largest is 4 → "bar2"
  get("foo",5)        → ts<=5: largest is 4 → "bar2"
```

<SectionHeader n={4} title="Approach" />

`set`: append `Entry(value, timestamp)` to `entries[key]`. `get`: retrieve the entry list for `key`. Binary search: maintain `l = 0`, `r = count - 1`, `result = ""`. Each iteration: compute `mid`. If `entries[mid].timestamp <= timestamp`, save `result = entries[mid].value` and move `l = mid + 1`. Else move `r = mid - 1`. Return `result`.

<Callout type="cyan" label="Upper bound binary search">
This is the classic find-last-less-than-or-equal pattern: when the midpoint's timestamp is valid (within range), save the value and try to find a later entry by moving left forward. When the midpoint is too large, move right backward.
</Callout>

<StepList>
  <StepItem title="Define Entry struct">
    `struct Entry { let value: String; let timestamp: Int }`.
  </StepItem>
  <StepItem title="set()">
    `entries[key, default: []].append(Entry(value: value, timestamp: timestamp))`.
  </StepItem>
  <StepItem title="get() — binary search">
    `var l = 0, r = existingEntries.count - 1, result = ""`.
  </StepItem>
  <StepItem title="Upper-bound search">
    While `l <= r`: `mid = (l+r)/2`. If `entries[mid].timestamp <= timestamp`: `result = entries[mid].value; l = mid + 1`. Else `r = mid - 1`.
  </StepItem>
  <StepItem title="Return">
    Return `result` (`""` if no valid timestamp found).
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class TimeMap {
  struct Entry {
    let value: String
    let timestamp: Int
  }

  var entries: [String: [Entry]] = [:]

  init() {}

  func set(_ key: String, _ value: String, _ timestamp: Int) {
    entries[key, default: []].append(Entry(value: value, timestamp: timestamp))
  }

  func get(_ key: String, _ timestamp: Int) -> String {
    guard let existingEntries = entries[key] else { return "" }

    var l = 0
    var r = existingEntries.count - 1
    var result = ""

    while l <= r {
      let mid = (l + r) / 2
      if existingEntries[mid].timestamp <= timestamp {
        result = existingEntries[mid].value
        l = mid + 1
      } else {
        r = mid - 1
      }
    }

    return result
  }
}
```
