<SectionHeader n={1} title="Problem Overview" />

Given an array of integers `nums` sorted in non-decreasing order, find the starting and ending position of a given `target` value.

If `target` is not found in the array, return `[-1, -1]`.

You must write an algorithm with **O(log n)** runtime complexity.

**Constraints**
- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `nums` is sorted in non-decreasing order.
- `-10^9 <= target <= 10^9`

<Callout type="amber" label="Key Constraint">
O(log n) runtime is explicitly required — a linear scan is not acceptable. The target may not exist; return [-1, -1] in that case. The array can be empty.
</Callout>

<Callout type="green" label="Core Insight">
Run binary search twice — once biased left (when target is found, keep searching left by setting r = m-1) and once biased right (keep searching right by setting l = m+1). Each pass records the last index where target was found.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  nums = [5,7,7,8,8,10], target = 8
Output: [3, 4]
Explanation: 8 first appears at index 3 and last at index 4.
```

#### Example 2
```
Input:  nums = [5,7,7,8,8,10], target = 6
Output: [-1, -1]
Explanation: 6 is not in the array.
```

#### Example 3
```
Input:  nums = [], target = 0
Output: [-1, -1]
```

<Callout type="green" label="Rule of Thumb">
When finding the leftmost or rightmost occurrence of a value in a sorted array, think biased binary search — on each hit, record the index and keep narrowing in the target direction instead of returning immediately.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Run binary search twice with a bias flag. When the target is found, record the index but keep narrowing: set r = m-1 to find the leftmost occurrence, or l = m+1 to find the rightmost. The last recorded index after the loop terminates is the boundary.
</Callout>

A single `binarySearch` helper takes an `isFindingMostLeft` flag. Standard binary search runs with `l=0, r=nums.count-1`. When `nums[m] == target`: record `result = m`, then bias the search — if finding the left boundary set `r = m - 1`; if finding the right boundary set `l = m + 1`. Call this helper twice and return both results.

<Callout type="cyan" label="Why record result and continue">
A normal binary search returns immediately on finding target. Here we record the index and keep shrinking the window in the biased direction. This guarantees we land on the extreme occurrence, not an arbitrary middle one.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Define binarySearch helper">
    Parameters: `nums`, `target`, `isFindingMostLeft: Bool`. Returns the boundary index or -1.
  </StepItem>
  <StepItem title="Standard binary search loop">
    `l=0, r=nums.count-1, result=-1`. While `l <= r`, compute `m = (l+r)/2`.
  </StepItem>
  <StepItem title="Handle the three cases">
    `nums[m] < target` → `l = m+1`. `nums[m] > target` → `r = m-1`. Equal → `result = m`, then bias: left search sets `r = m-1`, right search sets `l = m+1`.
  </StepItem>
  <StepItem title="Call twice and return">
    `[binarySearch(nums, target, true), binarySearch(nums, target, false)]`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func searchRange(_ nums: [Int], _ target: Int) -> [Int] {
    let firstItem = binarySearch(nums, target, isFindingMostLeft: true)
    let lastItem  = binarySearch(nums, target, isFindingMostLeft: false)
    return [firstItem, lastItem]

    func binarySearch(_ nums: [Int], _ target: Int, isFindingMostLeft: Bool) -> Int {
      var l = 0
      var r = nums.count - 1
      var result = -1

      while l <= r {
        let m = (l + r) / 2
        if nums[m] < target {
          l = m + 1

        } else if nums[m] > target {
          r = m - 1
          
        } else {
          result = m
          if isFindingMostLeft { r = m - 1 }
          else                 { l = m + 1 }
        }
      }

      return result
    }
  }
}
```

<Callout type="green" label="Swift Notes">
The nested `func binarySearch` is defined inside `searchRange` after its call sites — Swift resolves nested function names at compile time so declaration order within the enclosing function does not matter. Passing `isFindingMostLeft: Bool` as a labeled parameter makes call sites self-documenting: `binarySearch(nums, target, isFindingMostLeft: true)`.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(log n)">
Each biased binary search pass halves the search space per iteration, running in O(log n). Two independent passes give O(2 log n) = O(log n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
Only four integer variables per pass — l, r, m, result. No auxiliary data structures are needed. Space is O(1).
</Callout>

<Callout type="amber" label="When to prefer Biased Binary Search over Linear Scan">
Always prefer biased binary search when O(log n) is required or n can be large (up to 10^5 here). Linear scan is only acceptable for very small arrays or when the sorted guarantee cannot be relied upon.
</Callout>
