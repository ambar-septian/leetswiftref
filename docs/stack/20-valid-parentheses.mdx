<SectionHeader n={1} title="Problem Overview" />

Given a string `s` containing just the characters `'('`, `')'`, `'{'`, `'}'`, `'['` and `']'`, determine if the input string is **valid**.

An input string is valid if:
1. Open brackets must be closed by the same type of brackets.
2. Open brackets must be closed in the correct order.
3. Every close bracket has a corresponding open bracket of the same type.

**Constraints**
- `1 <= s.length <= 10^4`
- `s` consists of parentheses only `'()[]{}'`.

<Callout type="green" label="Core Insight">
Push every opening bracket onto a stack. When a closing bracket is encountered, the top of the stack must be the matching opener — pop it. If the stack is empty at the end, all brackets were matched and nested correctly.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  s = "()"
Output: true
```

#### Example 2
```
Input:  s = "()[]{}"
Output: true
```

#### Example 3
```
Input:  s = "(]"
Output: false
```

<SectionHeader n={4} title="Approach" />

Iterate through each character. For opening brackets (`{`, `[`, `(`), push onto the stack. For closing brackets, check if the stack's last element is the matching opener — if so, pop it; otherwise push the closer. After processing all characters, the string is valid if and only if the stack is empty.

<Callout type="cyan" label="Why push the closer on mismatch">
Pushing a mismatched closer (rather than returning false immediately) ensures the stack ends up non-empty, which correctly returns false at the end. Both approaches are equivalent; the push approach avoids an extra early-return path.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard odd-length strings">
    A valid bracket string always has even length. Return false immediately if `s.count % 2 != 0` or `s.count < 2`.
  </StepItem>
  <StepItem title="Iterate through characters">
    Use a switch on each character to decide push or pop.
  </StepItem>
  <StepItem title="Push opening brackets">
    For `{`, `[`, `(` — append the character to the stack.
  </StepItem>
  <StepItem title="Pop on matching closer">
    For each closer, if `stack.last` is the corresponding opener, call `stack.removeLast()`. Otherwise push the closer onto the stack.
  </StepItem>
  <StepItem title="Return stack.isEmpty">
    An empty stack means all brackets were matched correctly.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func isValid(_ s: String) -> Bool {
    var stack = [String]()
    guard s.count > 1, s.count % 2 == 0 else { return false }

    for char in s {
      switch char {
      case "{": 
        stack.append("{")
        
      case "}":
        if !stack.isEmpty && stack.last == "{" { stack.removeLast() }
        else { stack.append("}") }

      case "[": 
        stack.append("[")
      
      case "]":
        if !stack.isEmpty && stack.last == "[" { stack.removeLast() }
        else { stack.append("]") }

      case "(": 
        stack.append("(")

      case ")":
        if !stack.isEmpty && stack.last == "(" { stack.removeLast() }
        else { stack.append(")") }

      default: break
      }
    }

    return stack.isEmpty
  }
}
```
