<SectionHeader n={1} title="Problem Overview" />

We are given an array `asteroids` of integers representing asteroids in a row. For each asteroid, the absolute value represents its size, and the sign represents its direction (positive = right, negative = left). Each asteroid moves at the same speed.

Find the state of the asteroids after all collisions. If two asteroids meet, the smaller one explodes. If both are the same size, both explode. Two asteroids moving in the same direction will never meet.

**Constraints:**
- `2 <= asteroids.length <= 10^4`
- `-1000 <= asteroids[i] <= 1000`
- `asteroids[i] != 0`

<Callout type="amber" label="Key Constraint">
  A collision only happens between a positive asteroid on the stack and an incoming negative asteroid. Two positives or two negatives never collide — they move in the same direction. The equal-size case destroys both, and neither is pushed to the stack.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  asteroids = [5,10,-5]
Output: [5,10]
Explanation: The 10 and -5 collide. 10 is bigger so 10 survives. The 5 and 10 never collide.
```

#### Example 2

```
Input:  asteroids = [8,-8]
Output: []
Explanation: The 8 and -8 collide. Same size — both explode.
```

#### Example 3

```
Input:  asteroids = [10,2,-5]
Output: [10]
Explanation: 2 and -5 collide: -5 wins (abs(-5)=5 > 2). Then 10 and -5 collide: 10 wins.
```

  When the outcome of each element depends only on the most recent surviving element, use a stack. Here, a negative asteroid only interacts with the positive directly in front of it — exactly what the stack top represents.

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Model the asteroid row as a stack. A collision can only occur between the rightmost surviving positive asteroid (stack top) and an incoming negative asteroid. Pop and compare sizes in a loop — the larger survives, equal sizes both die. Any asteroid that makes it through the loop is pushed onto the stack.
</Callout>

Maintain a stack. For each asteroid `ast`: while the stack top is positive and `ast` is negative, a collision occurs. If `|ast| > top`, pop the top (top destroyed). If `|ast| < top`, set `ast = 0` (ast destroyed). If equal, pop top and set `ast = 0` (both destroyed). After the loop, if `ast != 0`, push it.

<Callout type="cyan" label="When does no collision happen?">
No collision when: ast is positive (moving right, cannot catch stack top), stack is empty, or stack top is negative (both moving left — they move away from each other). Only a positive-on-stack-top collides with an incoming negative.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise stack">
    `var stack = [Int]()`.
  </StepItem>
  <StepItem title="Process each asteroid">
    `var ast = asteroids[i]`. Enter collision loop: while `!stack.isEmpty`, `stack.last! > 0`, and `ast < 0`.
  </StepItem>
  <StepItem title="Resolve collision">
    If `|ast| > |top|`: pop top (top destroyed). If `|top| > |ast|`: `ast = 0` (ast destroyed). If equal: pop top and `ast = 0` (both destroyed).
  </StepItem>
  <StepItem title="Push survivor">
    If `ast != 0`, push to stack.
  </StepItem>
  <StepItem title="Return stack">
    The stack holds all surviving asteroids in order.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func asteroidCollision(_ asteroids: [Int]) -> [Int] {
    var stack: [Int] = []

    for i in 0..<asteroids.count {
      var ast = asteroids[i]
      while let top = stack.last, top > 0, ast < 0 {
        if abs(ast) > abs(top) {
          stack.removeLast()
        } else if abs(top) > abs(ast) {
          ast = 0
        } else {
          stack.removeLast()
          ast = 0
        }
      }
      if ast != 0 {
        stack.append(ast)
      }
    }

    return stack
  }
}
```

<Callout type="green" label="Swift Notes">
  Use `var ast = asteroids[i]` as a local mutable copy — iterating with `for ast in asteroids` gives a `let` binding. `stack.last` returns an optional; pattern match with `if let top = stack.last` or use `stack.last!` inside a non-empty guard. The stack is an `[Int]` array — `removeLast()` and `append()` are O(1) amortised.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  Each asteroid is pushed onto the stack at most once and popped at most once. The inner while loop does not cause O(n²) — the total number of pops across the entire loop is bounded by n, giving O(n) total work.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
  The stack holds surviving asteroids — in the worst case (all positives or all negatives, no collisions) all n asteroids are on the stack. O(n) auxiliary space.
</Callout>
