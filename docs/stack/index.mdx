<SectionHeader n={1} title="What is a Stack?" />

A stack supports two primary operations: **push** (add to the top) and **pop** (remove from the top). Swift's `Array` provides both as O(1) amortised `append` and `removeLast`.

| Property | Value |
|---|---|
| Order | Last In, First Out (LIFO) |
| Push | `array.append(x)` — O(1) amortised |
| Pop | `array.removeLast()` — O(1) |
| Peek | `array.last` — O(1) |
| Empty check | `array.isEmpty` — O(1) |

<Callout type="amber" label="Key Constraint">
  Always **peek before popping**. Use `guard let top = stack.last else { /* handle empty */ }` to avoid a crash on an empty stack. In Swift, `array.last` returns an `Optional`; `array.removeLast()` crashes if the array is empty — never call it without an emptiness check.
</Callout>

<Callout type="green" label="Rule of Thumb">
  Reach for a stack whenever you need to process something and then "undo" or revisit it in reverse order. Brackets need their matching close-bracket to appear later in LIFO order. Nested strings need their outer context restored after the inner part is processed. That reversal pattern is the signature of a stack problem.
</Callout>
