<SectionHeader n={1} title="Problem Overview" />

Given an absolute path for a Unix-style file system, which begins with a slash `'/'`, transform it into its **simplified canonical path**.

In a Unix-style file system:
- A single period `'.'` refers to the current directory.
- A double period `'..'` refers to the parent directory (go up one level).
- Multiple consecutive slashes such as `'//'` or `'///'` are treated as a single slash `'/'`.
- Any other sequence of characters is a valid directory or file name.

The simplified canonical path must:
- Start with a single slash `'/'`.
- Separate directories with a single slash.
- Not end with a trailing slash (unless it is the root `/`).
- Not contain any `.` or `..` components.

**Constraints**
- `1 <= path.length <= 3000`
- `path` consists of English letters, digits, period `.`, slash `/`, or underscore `_`.
- `path` is a valid absolute Unix path.

<Callout type="green" label="Core Insight">
Scan the path character by character. When a slash is encountered, process the accumulated token: push valid directory names onto a stack, pop on '..', and ignore '.' and empty tokens. Reconstruct the canonical path from the stack with '/' separators.
</Callout>

```

#### Example 2

```

#### Example 3
```
Input:  path = "/home/user/Documents/../Pictures"
Output: "/home/user/Pictures"
```

#### Example 4
```
Input:  path = "/../"

```

#### Example 5
```
Input:  path = "/.../a/../b/c/../d/./"
Output: "/.../b/d"
Explanation: "..." is a valid directory name.
```

<SectionHeader n={4} title="Approach" />

Iterate through each character. If the character is not `'/'`, accumulate it into `current`. When a `'/'` is hit (or at the end of the string), call `process(&current, &stack)`:
- `'..'` → pop the stack if non-empty
- `'.'` or empty → do nothing
- anything else → push onto the stack

After the loop, call `process` once more for the final accumulated token. Reconstruct: `"/" + stack.joined(separator: "/")`.

<Callout type="cyan" label="Why process is called both on '/' and after the loop">
The final token (e.g., "foo" in "/home/foo") is not followed by a slash, so the loop exits with the last token still in current. A final process call handles it. This is simpler than checking the last character separately.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise stack and current accumulator">
    `var stack = [String]()`, `var current = ""`.
  </StepItem>
  <StepItem title="Scan character by character">
    For each `char` in `path`: if `char == "/"`, call `process(&current, &stack)`. Else `current.append(char)`.
  </StepItem>
  <StepItem title="Flush the last token">
    Call `process(&current, &stack)` after the loop to handle the final segment.
  </StepItem>
  <StepItem title="Reconstruct">
    Return `"/" + stack.joined(separator: "/")`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func simplifyPath(_ path: String) -> String {
    var stack = [String]()
    var current = ""

    for char in path {
      if char == "/" {
        process(&current, &stack)
      } else {
        current.append(char)
      }
    }

    process(&current, &stack)
    return "/" + stack.joined(separator: "/")
  }

  func process(_ current: inout String, _ stack: inout [String]) {
    if current == ".." {
      if !stack.isEmpty {
        stack.removeLast()
      }
    } else if !current.isEmpty, current != "." {
      stack.append(current)
    }
    current.removeAll()
  }
}
```
