<SectionHeader n={1} title="Problem Overview" />

You are given a string `s`, which contains stars `*`.

In one operation, you choose a star in `s`, remove the **closest non-star character** to its **left**, as well as remove the star itself.

Return the string after **all** stars have been removed.

**Note:**
- The input is generated such that the operation is always possible.
- It can be shown that the resulting string will always be unique.

**Constraints:**
- `1 <= s.length <= 10^5`
- `s` consists of lowercase English letters and stars `*`.
- The operation above can be always performed on the input.

<Callout type="amber" label="Key Constraint">
  The input guarantees the operation is always possible — there are never more stars than preceding non-star characters. `removeLast()` on the character stack is always safe without an empty-check.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  s = "leet**cod*e"
Output: "lecoe"
Explanation:
  - Remove 't' (closest left of first *), and the first *.  → "lee*cod*e"
  - Remove 'e' (closest left of next *), and the second *. → "lecod*e"
  - Remove 'd' (closest left of last *), and the third *.  → "lecoe"
```

#### Example 2

```
Input:  s = "erase*****"
Output: ""
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Model the result as a character stack. A star always removes the most recently added letter — exactly the LIFO behaviour of a stack. One left-to-right scan: push on letter, pop on star, join at the end.
</Callout>

Use a character array as a stack. Iterate left to right:
- If the character is `*`, call `removeLast()` to undo the most recently added letter.
- Otherwise, append the character.

Join the array into a string at the end.

<Callout type="green" label="Character array as stack">
Swift's `[Character]` with `append` / `removeLast` is O(1) amortised — faster than building a String directly because String mutation involves UTF-8 re-encoding overhead.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise stack">
    `var result = [Character]()` — pre-reserve capacity with `result.reserveCapacity(s.count)` to avoid repeated reallocations.
  </StepItem>
  <StepItem title="Iterate characters">
    For each `char` in `s`: if `char == "*"`, call `result.removeLast()`; else call `result.append(char)`.
  </StepItem>
  <StepItem title="Return result">
    `return String(result)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func removeStars(_ s: String) -> String {
    var result = [Character]()
    result.reserveCapacity(s.count)

    for char in s {
      if char == "*" {
        result.removeLast()
      } else {
        result.append(char)
      }
    }

    return String(result)
  }
}
```

<Callout type="green" label="Swift Notes">
  `result.reserveCapacity(s.count)` pre-allocates the backing buffer to avoid repeated O(log n) reallocations during `append`. `String(result)` at the end is a single O(n) encoding pass — faster than building a `String` character-by-character during the loop.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  Each character is visited once. Each `append` and `removeLast` is O(1) amortised. The final `String(result)` join is O(n). Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
  The character array holds at most n characters (when there are no stars). `reserveCapacity(s.count)` preallocates this worst-case size upfront to eliminate reallocations.
</Callout>
