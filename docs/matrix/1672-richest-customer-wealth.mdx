<SectionHeader n={1} title="Problem Overview" />

You are given an `m x n` integer grid `accounts` where `accounts[i][j]` is the amount of money the `i`-th customer has in the `j`-th bank. Return the **wealth** that the richest customer has.

A customer's wealth is the amount of money they have in all their bank accounts. The richest customer is the customer that has the maximum wealth.

**Constraints:**
- `m == accounts.length`
- `n == accounts[i].length`
- `1 <= m, n <= 50`
- `1 <= accounts[i][j] <= 100`

<Callout type="amber" label="Key Constraint">
  All values are positive (accounts[i][j] &gt;= 1), so wealth is always positive. m and n are both at least 1 — no empty-row or empty-column guard needed. The maximum wealth fits comfortably in a 32-bit integer (50 * 50 * 100 = 250,000).
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  accounts = [[1,2,3],[3,2,1]]
Output: 6
Explanation: Customer 0 wealth = 6. Customer 1 wealth = 6. Both are richest.
```

#### Example 2

```
Input:  accounts = [[1,5],[7,3],[3,5]]
Output: 10
Explanation: Customer 0 wealth=6, Customer 1 wealth=10, Customer 2 wealth=8. Richest is 10.
```

#### Example 3

```
Input:  accounts = [[2,8,7],[7,1,3],[1,9,5]]
Output: 17
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Wealth = sum of one row. Maximum wealth = max of all row sums. Map each row to its sum with reduce(0, +), then call max() — two chained higher-order functions replace nested loops. The result is idiomatic, readable, and O(m·n).
</Callout>

Use Swift's `map` + `max()` idiom: map each row to its sum with `reduce(0, +)`, then call `max()` on the resulting array.

<Callout type="green" label="One-liner in Swift">
accounts.map `{` $0.reduce(0, +) `}`.max() ?? 0 — clean, readable, and O(m*n).
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Map each customer row to its sum">
    For each row, compute the sum of all bank account values.
  </StepItem>
  <StepItem title="Return the maximum sum">
    Find and return the largest sum across all customers.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func maximumWealth(_ accounts: [[Int]]) -> Int {
    return accounts.map { $0.reduce(0, +) }.max() ?? 0
  }
}
```

<Callout type="green" label="Swift Notes">
  `accounts.map` with `reduce(0, +)` then `.max() ?? 0` — `reduce(0, +)` passes the `+` operator directly as the combining function (no explicit closure needed). `.max()` returns Optional since the array could be empty; `?? 0` provides a safe fallback. The entire expression is a single chain — no intermediate variables needed.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(m·n)">
  map iterates over m rows, and each reduce sums n values — O(m·n) total work. The final max() scans the m row sums — O(m). Total: O(m·n).
</Callout>

<Callout type="green" label="Space Complexity — O(m)">
  The intermediate array from map holds m row sums. No other auxiliary data structures. O(m) extra space.
</Callout>
