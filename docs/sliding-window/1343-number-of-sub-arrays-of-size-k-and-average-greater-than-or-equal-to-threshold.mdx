<SectionHeader n={1} title="Problem Overview" />

Given an array of integers `arr` and two integers `k` and `threshold`, return the number of sub-arrays of size `k` and average greater than or equal to `threshold`.

**Constraints:**
- `1 <= arr.length <= 10^5`
- `1 <= arr[i] <= 10^4`
- `1 <= k <= arr.length`
- `0 <= threshold <= 10^4`

<Callout type="amber" label="Key Constraint">
  k is always valid (1 &lt;= k &lt;= arr.length), so the first window always exists — no empty-array guard is needed. Compare sum &gt;= threshold * k instead of avg &gt;= threshold to avoid integer division rounding entirely.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  arr = [2,2,2,2,5,5,5,8], k = 3, threshold = 4
Output: 3
Explanation: Sub-arrays [2,5,5],[5,5,5],[5,5,8] have averages 4, 5, 6 — all >= 4.
```

#### Example 2

```
Input:  arr = [11,13,17,23,29,31,7,5,2,3], k = 3, threshold = 5
Output: 6
```

<Callout type="green" label="Rule of Thumb">
  When the window size is fixed and you need to test every subarray of that size, use a fixed-size sliding window: O(k) initial fill followed by O(1) per slide instead of O(k) per window. Replace avg &gt;= threshold with sum &gt;= threshold * k to avoid division.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Build the initial window of size k in O(k), then slide: add the incoming element and remove the outgoing element in O(1) per step. Replace avg &gt;= threshold with sum &gt;= threshold * k to avoid integer division and floating-point concerns entirely.
</Callout>

Build the initial window of size `k`. Then slide one element at a time: add `arr[i]`, remove `arr[i-k]`, and check if `sum / k >= threshold`. Using integer division is safe here since both `sum` and `threshold * k` are integers.

<Callout type="green" label="Avoid division when possible">
Instead of computing `sum / k >= threshold`, compare `sum >= threshold * k`. This avoids any integer division rounding and is slightly faster.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Build first window">
    Sum `arr[0..k-1]`. Check if `sum / k >= threshold`. Initialise `result` accordingly.
  </StepItem>
  <StepItem title="Slide the window">
    For `i` from `k` to `arr.count - 1`: `sum = sum - arr[i-k] + arr[i]`.
  </StepItem>
  <StepItem title="Check threshold">
    If `sum / k >= threshold`, increment `result`.
  </StepItem>
  <StepItem title="Return result">
    Return the count of qualifying windows.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func numOfSubarrays(_ arr: [Int], _ k: Int, _ threshold: Int) -> Int {
    var sum = 0
    for i in 0..<k { sum += arr[i] }

    var result = sum / k >= threshold ? 1 : 0

    for i in k..<arr.count {
      sum = sum - arr[i - k] + arr[i]
      if sum / k >= threshold { result += 1 }
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `sum / k >= threshold` uses integer division — safe here since both operands are positive integers, but `sum >= threshold * k` avoids division entirely. The slide loop starts at index `k` (not `k-1`) since the initial window covers indices `0..<k`. The initial check `sum / k >= threshold ? 1 : 0` handles the first window inline before the loop.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  The initial window takes O(k) to build; each subsequent slide is O(1) — one addition and one subtraction. Total: O(k) + O(n-k) = O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
  Only sum and result counters are maintained — no auxiliary array or buffer. O(1) extra space.
</Callout>
