<SectionHeader n={1} title="What is Sliding Window?" />

A **sliding window** is a contiguous range `[L, R]` inside the array. As `R` advances right, the window grows; when the window violates a constraint, `L` advances right to shrink it. The key insight is that each element enters and leaves the window at most once, so the total work across all iterations is O(n).

**Two flavours:**

| Variant | Window size | L moves | When to use |
|---|---|---|---|
| **Fixed** | Constant k | Every step | "Subarray of exactly k elements" |
| **Variable** | Grows and shrinks | When constraint violated | "Shortest/longest subarray satisfying a condition" |

<Callout type="amber" label="Key Constraint">
  Sliding window only works when the **window property is monotone**: adding an element can only make the window more or less valid (not toggle back and forth). If the property is non-monotone (e.g., the sum must equal exactly k on an unsorted array with negatives), use a prefix sum + hash map instead.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **L (left)** | The left boundary of the window — inclusive |
| **R (right)** | The right boundary of the window — inclusive |
| **Window property** | The condition the window must satisfy (e.g., sum ≥ k, all unique chars) |
| **Frequency map** | `[T: Int]` dictionary tracking element counts inside the current window |
| **Shrink phase** | Moving L right when the window violates the constraint |
| **Expand phase** | Moving R right to grow the window and include more elements |
| **Valid window** | A window that satisfies the constraint — candidate for best/worst answer |

<Callout type="green" label="Rule of Thumb">
  Fixed window when the problem says "exactly k". Variable window when the problem says "at most", "at least", "longest", or "shortest". If the window needs a frequency map (characters, distinct elements), it's still sliding window — just with O(k) extra space.
</Callout>
