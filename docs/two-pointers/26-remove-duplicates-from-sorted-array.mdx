<SectionHeader n={1} title="Problem Overview" />

Given an integer array `nums` sorted in **non-decreasing order**, remove the duplicates **in-place** such that each unique element appears only once. The **relative order** of the elements should be kept the same. Then return the number of unique elements in `nums`.

Consider the number of unique elements of `nums` to be `k`. To get accepted, you need to:
- Change the array `nums` such that the first `k` elements of `nums` contain the unique elements in the order they were present in `nums` initially.
- The remaining elements of `nums` are not important, as is the size of `nums`.
- Return `k`.

**Constraints**
- `1 <= nums.length <= 3 * 10^4`
- `-100 <= nums[i] <= 100`
- `nums` is sorted in **non-decreasing** order.

<Callout type="amber" label="Key Constraint">
The array is sorted — duplicates are always adjacent. The modification must be done in-place with O(1) extra space; only the first k elements of the result matter, the rest can be anything.
</Callout>

<Callout type="green" label="Core Insight">
Because the array is sorted, duplicates are always adjacent. A slow pointer marks the boundary of the unique prefix. A fast pointer scans ahead — whenever it finds a value different from slow, copy it to slow+1 and advance slow.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  nums = [1,1,2]
Output: 2, nums = [1,2,_]
Explanation: Two unique elements. First 2 positions hold [1,2].
```

#### Example 2
```
Input:  nums = [0,0,1,1,1,2,2,3,3,4]
Output: 5, nums = [0,1,2,3,4,_,_,_,_,_]
Explanation: Five unique elements.
```

<Callout type="green" label="Rule of Thumb">
When removing duplicates in-place from a sorted array, think slow/fast two pointers — slow marks the end of the unique prefix, fast scans forward and copies each new value to slow+1 when it differs.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Sorted order guarantees duplicates are adjacent, so comparing only nums[slow] with nums[fast] is sufficient to detect every new unique value. The slow pointer marks the tail of the unique prefix; when fast finds a new value, copy it to slow+1 and advance slow. The result is a compacted unique prefix written in-place.
</Callout>

Start `slow = 0`, `fast = 1`. Walk `fast` through the array. When `nums[fast] != nums[slow]` a new unique value is found: advance `slow` and copy `nums[fast]` into `nums[slow]`. When equal, just advance `fast`. Return `slow + 1` (the count of unique elements written).

<Callout type="cyan" label="Why sorted order is required">
If the array were unsorted, duplicates could appear anywhere and a simple adjacent comparison would miss them. Sorting ensures every duplicate is adjacent to its original, so comparing only slow and fast is sufficient.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard empty array">
    Return `0` immediately if `nums` is empty.
  </StepItem>
  <StepItem title="Initialise pointers">
    `slow = 0`, `fast = 1`, `result = 1` (the first element is always unique).
  </StepItem>
  <StepItem title="Scan with fast">
    While `fast < nums.count`: if `nums[slow] == nums[fast]`, advance `fast`. Otherwise advance `slow`, write `nums[fast]` into `nums[slow]`, increment `result`, then advance `fast`.
  </StepItem>
  <StepItem title="Return result">
    `result` equals the number of unique elements written to the front of the array.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func removeDuplicates(_ nums: inout [Int]) -> Int {
    guard !nums.isEmpty else { return 0 }

    var slow = 0
    var fast = 1
    var result = 1

    while fast < nums.count {
      if nums[slow] == nums[fast] {
        fast += 1
      } else {
        slow += 1
        nums[slow] = nums[fast]
        result += 1
      }
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
The `inout` parameter (`_ nums: inout [Int]`) passes the array by reference using copy-in/copy-out semantics — mutations are written back to the caller's variable on return. `guard !nums.isEmpty else { return 0 }` is idiomatic Swift for an early exit without nesting the entire function body in an `if`.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
Each element is visited exactly once by the fast pointer. The slow pointer only advances when a new unique value is found. Total time is O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
Only two index variables (slow and fast) are maintained. No additional array is allocated — unique values are written directly into the input array. Space is O(1).
</Callout>
