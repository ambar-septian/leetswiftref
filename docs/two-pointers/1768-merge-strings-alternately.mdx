<SectionHeader n={1} title="Problem Overview" />

You are given two strings `word1` and `word2`. Merge the strings by adding letters in alternating order, starting with `word1`. If a string is longer than the other, append the additional letters onto the end of the merged string.

Return the merged string.

**Constraints:**
- `1 <= word1.length, word2.length <= 100`
- `word1` and `word2` consist of lowercase English letters.

<Callout type="amber" label="Key Constraint">
  Both strings are non-empty (length &gt;= 1), so no empty-string guard needed. The longer string's tail is appended verbatim — handle this by looping to max(len1, len2) with per-string bounds checks.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  word1 = "abc", word2 = "pqr"
Output: "apbqcr"
```

#### Example 2

```
Input:  word1 = "ab", word2 = "pqrs"
Output: "apbqrs"
Explanation: word2 is longer; "rs" appended at end.
```

#### Example 3

```
Input:  word1 = "abcd", word2 = "pq"
Output: "apbqcd"
```

<Callout type="green" label="Rule of Thumb">
  When interleaving two sequences of unequal length, loop to max(len1, len2) with per-sequence bounds checks — a single loop handles both the interleave phase and the longer tail, eliminating a separate trailing append.

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  One index i, two bounds checks: append chars1[i] if i &lt; len1, then chars2[i] if i &lt; len2. Loop to max(len1, len2). This handles both the interleave phase and the longer-string tail in a single loop — no separate append needed.
</Callout>

Convert both strings to character arrays for O(1) index access. Use a single loop up to `max(len1, len2)`. At each iteration, append `chars1[i]` if `i < len1`, then `chars2[i]` if `i < len2`. A pre-allocated result array avoids repeated string concatenation.

<Callout type="green" label="Pre-allocate the result">
Allocate result with capacity `len1 + len2` upfront. Appending to a pre-sized array is O(1) amortised, while string concatenation in a loop can be O(n²).
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Convert to char arrays">
    `let chars1 = Array(word1)`, `let chars2 = Array(word2)`. Enables O(1) subscript access.
  </StepItem>
  <StepItem title="Initialise result">
    Allocate `result = Array(repeating: Character(" "), count: len1 + len2)` with a write index `j = 0`.
  </StepItem>
  <StepItem title="Interleave">
    Loop `i` from 0 to `max(len1, len2) - 1`. If `i < len1`, write `chars1[i]` and advance `j`. If `i < len2`, write `chars2[i]` and advance `j`.
  </StepItem>
  <StepItem title="Return">
    `return String(result)`
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func mergeAlternately(_ word1: String, _ word2: String) -> String {
    let chars1 = Array(word1)
    let chars2 = Array(word2)
    let count = max(chars1.count, chars2.count)
    var result = Array(repeating: Character(" "), count: chars1.count + chars2.count)
    var j = 0
    for i in 0..<count {
      if i < chars1.count { result[j] = chars1[i]; j += 1 }
      if i < chars2.count { result[j] = chars2[i]; j += 1 }
    }
    return String(result)
  }
}
```

<Callout type="green" label="Swift Notes">
  `Array(word)` converts String to [Character] for O(1) subscript — Swift String subscript is O(n) due to Unicode. `Array(repeating: Character(" "), count: len1 + len2)` pre-allocates the exact output size, avoiding reallocations. `String(result)` converts [Character] back to String in O(n). Using a write index `j` instead of append avoids the overhead of capacity checks.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n + m)">
  A single loop up to max(n, m) iterations, each doing at most two array writes — O(1) per step. Two Array(word) conversions add O(n) + O(m). Total: O(n + m).
</Callout>

<Callout type="green" label="Space Complexity — O(n + m)">
  The result array of size n+m is required output. The two character arrays (chars1, chars2) add O(n) + O(m). Total auxiliary: O(n + m).
</Callout>
