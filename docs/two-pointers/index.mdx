<SectionHeader n={1} title="What is Two Pointers?" />

Two Pointers replaces O(n²) brute-force search with an O(n) single-pass scan by exploiting the structure of the problem — usually **sorted order** or **symmetric structure**.

**The two main variants:**

| Variant | Pointers start | Move toward | Typical trigger |
|---|---|---|---|
| **Opposite ends** | Left = 0, Right = n-1 | Each other (converging) | Palindrome, sorted-array pair sum, max area |
| **Same direction** | Both at left | Same direction (fast races ahead) | In-place removal, partitioning, merge |

<Callout type="amber" label="Key Constraint">
  The **opposite-ends variant** requires the array to be **sorted** (or have symmetric structure like a palindrome). Moving a pointer only makes sense if you can predict which direction brings you closer to the answer. On an unsorted array, use a hash map instead.
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **Left pointer (L)** | Starts at index 0; advances when the current element is processed |
| **Right pointer (R)** | Starts at index n-1 (opposite ends) or just ahead of L (same direction) |
| **Slow pointer** | The write cursor in same-direction problems — marks where the next valid value should go |
| **Fast pointer** | The read cursor — scans the entire array looking for valid values |
| **In-place** | Modifying the array without allocating a separate output array |
| **Invariant** | The property that holds before and after each pointer move (e.g. "everything left of slow is unique") |

<Callout type="green" label="Rule of Thumb">
  Opposite ends for **symmetric** or **sorted-pair** problems; same direction for **in-place write / partition** problems where you need to skip or overwrite elements.
</Callout>
