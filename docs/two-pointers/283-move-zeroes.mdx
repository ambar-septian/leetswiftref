<SectionHeader n={1} title="Problem Overview" />

Given an integer array `nums`, move all `0`s to the end of it while maintaining the relative order of the non-zero elements.

You must do this **in-place** without making a copy of the array.

**Constraints:**
- `1 <= nums.length <= 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`

<Callout type="amber" label="Key Constraint">
The operation must be done in-place without copying the array, and the relative order of non-zero elements must be preserved. This is a stable partition — not just any rearrangement.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  nums = [0,1,0,3,12]
Output: [1,3,12,0,0]
```

#### Example 2

```
Input:  nums = [0]
Output: [0]
```

<Callout type="green" label="Rule of Thumb">
When partitioning an array in-place with order preserved, use a slow write pointer that only advances when a qualifying element is swapped into it. A single pass handles placement and zero-filling simultaneously.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
`insertPosition` marks where the next non-zero element belongs. When `nums[i]` is non-zero, swapping it with `nums[insertPosition]` moves the non-zero forward and pushes the zero back in one step. A single forward pass handles placement and implicit zero-filling simultaneously — no second pass needed.
</Callout>

Maintain an `insertPosition` pointer starting at 0. Scan `i` from 0 to n-1. Whenever `nums[i] != 0`, swap `nums[i]` with `nums[insertPosition]` and advance `insertPosition`. After the scan, all non-zeros occupy the front in original order; all zeros have been pushed to the back.

<Callout type="cyan" label="Why swap instead of overwrite">
Swapping is equivalent to overwriting non-zeros forward and zeros backward simultaneously — no second pass is needed to fill the zeros at the end.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise write pointer">
    `var insertPosition = 0`.
  </StepItem>
  <StepItem title="Scan with read pointer">
    For `i` in 0 to `nums.count - 1`.
  </StepItem>
  <StepItem title="Swap non-zero to front">
    If `nums[i] != 0`: `nums.swapAt(i, insertPosition); insertPosition += 1`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func moveZeroes(_ nums: inout [Int]) {
    var insertPosition = 0
    for i in 0..<nums.count {
      if nums[i] != 0 {
        nums.swapAt(i, insertPosition)
        insertPosition += 1
      }
    }
  }
}
```

<Callout type="green" label="Swift Notes">
Swift arrays are value types, so `inout` is required for in-place modification — without it, changes inside the function would not affect the caller's array. The call site uses `&nums`. `nums.swapAt(i, insertPosition)` is an O(1) built-in that is more idiomatic than a manual three-step swap.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
A single forward pass visits all n elements. Each non-zero element triggers at most one swap. Total work is proportional to the number of elements — O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
Only the `insertPosition` counter and loop index are used alongside the input array. All modifications are in-place with no auxiliary allocations.
</Callout>
