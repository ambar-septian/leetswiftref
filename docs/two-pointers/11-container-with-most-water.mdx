<SectionHeader n={1} title="Problem Overview" />

You are given an integer array `height` of length `n`. There are `n` vertical lines drawn such that the two endpoints of the `i`th line are at `(i, 0)` and `(i, height[i])`.

Find two lines that together with the x-axis form a container that holds the most water. Return the **maximum amount of water** a container can store.

**Note:** You may not slant the container.

**Constraints**
- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

<Callout type="amber" label="Key Constraint">
Water volume is always limited by the shorter of the two lines — you cannot slant the container. Heights can be zero, so a line may contribute nothing to any container it forms.
</Callout>

<Callout type="green" label="Core Insight">
Start with the widest possible container (pointers at both ends). The area is always limited by the shorter line. Moving the taller line inward can only decrease width without increasing the height limit — so always move the shorter pointer inward to have any chance of finding a larger area.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation: Lines at index 1 (height 8) and index 8 (height 7).
             Area = min(8, 7) * (8 - 1) = 7 * 7 = 49.
```

#### Example 2
```
Input:  height = [1,1]
Output: 1
Explanation: Only two lines. Area = min(1,1) * 1 = 1.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Start with the widest possible container (pointers at both ends). Area is always capped by the shorter line. Moving the taller pointer inward shrinks width without any possibility of raising the height cap — so always move the shorter pointer, which is the only way to potentially find a taller boundary.
</Callout>

<ContainerWithMostWaterDiagram />

Place `left` at index `0` and `right` at index `n - 1`. At each step compute `area = min(height[left], height[right]) * (right - left)` and update the running maximum. Then move whichever pointer points to the shorter line — if they are equal, move either (here we move `left`). Repeat until the pointers meet.

<Callout type="cyan" label="Why moving the taller pointer is always wrong">
Suppose left is shorter. Moving right inward shrinks the width AND keeps the same height limit (still capped by the shorter left). The area can only decrease or stay the same. Moving left is the only way to possibly increase the height cap.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise pointers and result">
    `left = 0`, `right = height.count - 1`, `total = 0`.
  </StepItem>
  <StepItem title="Compute area at current window">
    `minHeight = min(height[left], height[right])`, `width = right - left`, `area = minHeight * width`. Update `total = max(total, area)`.
  </StepItem>
  <StepItem title="Move the shorter pointer">
    If `height[left] > height[right]` move `right -= 1`, else move `left += 1`.
  </StepItem>
  <StepItem title="Repeat until pointers meet">
    Loop while `left != right`. Return `total`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func maxArea(_ height: [Int]) -> Int {
    var left = 0
    var right = height.count - 1
    var total = 0

    while left != right {
      let minHeight = min(height[left], height[right])
      let width = right - left
      total = max(width * minHeight, total)

      if height[left] > height[right] {
        right -= 1
      } else {
        left += 1
      }
    }

    return total
  }
}
```

<Callout type="green" label="Swift Notes">
`min(_:_:)` and `max(_:_:)` are free functions in Swift's standard library — no import needed. Using `left != right` as the loop condition is safe because the two pointers strictly converge toward each other and are guaranteed to meet without crossing.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
The two pointers together traverse the array at most n steps — left only moves right and right only moves left, each advancing by 1 per iteration. The loop runs at most n times.
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
Only three integer variables are used — left, right, and total. No auxiliary data structures are needed.
</Callout>

Always prefer two pointers for this problem. Brute force is O(n²) and only useful as a baseline to verify the greedy claim. The two-pointer approach is strictly better in time with identical space.
