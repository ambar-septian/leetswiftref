<SectionHeader n={1} title="What is Backtracking?" />

Backtracking is a systematic way to enumerate all solutions to a problem by building candidates incrementally and abandoning a candidate (backtracking) as soon as it is determined that the candidate cannot lead to a valid solution.

The three-step template is the same for every backtracking problem:

| Step | Action |
|---|---|
| **Choose** | Pick one candidate from the available options |
| **Explore** | Recurse with the updated state |
| **Unchoose** | Undo the choice to restore state before moving to the next candidate |

<Callout type="amber" label="Key Constraint">
  The constraint that distinguishes backtracking problems from pure DFS is that the **state must be mutable and restored** after each recursive call. If state is immutable (e.g. passing a new array copy each time), that is plain DFS — correct but O(n) extra space per level. Backtracking modifies state in-place and undoes changes, keeping space proportional to the recursion depth, not the branching factor.
</Callout>

<Callout type="green" label="Rule of Thumb">
  If the problem asks to enumerate all valid combinations, permutations, or subsets — or to find whether a path exists on a grid — that is backtracking. The branching factor and depth of the decision tree determine the time complexity.
</Callout>
