<SectionHeader n={1} title="Problem Overview" />

Given a string containing digits from `2` to `9` inclusive, return all possible letter combinations that the number could represent. Return the answer in **any order**.

A mapping of digits to letters (just like on the telephone buttons) is given below. Note that `1` does not map to any letters.

```
2 → a b c       3 → d e f
4 → g h i       5 → j k l
6 → m n o       7 → p q r s
8 → t u v       9 → w x y z
```

**Constraints**
- `0 <= digits.length <= 4`
- `digits[i]` is a digit in the range `['2', '9']`.

<Callout type="amber" label="Key Constraint">
An empty input string must return an empty array — not an array containing an empty string. The maximum of 4 digits bounds the output to at most 4^4 = 256 combinations, so exhaustive backtracking is efficient.
</Callout>

<Callout type="green" label="Core Insight">
Build each combination one character at a time. At each recursive call, append one letter mapped from the current digit, recurse to the next digit, then remove the letter (backtrack). When the path length equals the number of digits, record the combination.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  digits = "23"
Output: ["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

#### Example 2
```
Input:  digits = ""
Output: []
```

#### Example 3
```
Input:  digits = "2"
Output: ["a","b","c"]
```

<Callout type="green" label="Rule of Thumb">
When asked to enumerate all combinations where each step picks from a small known set, think backtracking — build the path one choice at a time, recurse, then undo the last choice before trying the next.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Each digit maps to 2–4 letters. At each recursion level, try every letter for the current digit: append it to the path, recurse to the next digit, then remove it (backtrack). When the path length equals the number of digits, the combination is complete — record it and return.
</Callout>

Pre-build a dictionary mapping each digit character to its letter array. Use a recursive `backTrack(_ index: Int)` function with a shared `path` array. At each call, look up the letters for `digs[index]`, iterate over them: append the letter, recurse with `index + 1`, then remove the last element. The base case is `path.count == digits.count` — join the path and append to results.

<LetterCombinationsDiagram />

<Callout type="cyan" label="Why backtracking not iteration">
An iterative approach would need nested loops, one per digit — impossible when the number of digits is variable. Backtracking handles arbitrary depth with a single recursive function.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Build the digit-to-letters map">
    A dictionary `[Character: [Character]]` maps '2'→['a','b','c'] through '9'→['w','x','y','z'].
  </StepItem>
  <StepItem title="Define the backtracking function">
    `backTrack(_ index: Int)` uses shared `path: [Character]` and `results: [String]`.
  </StepItem>
  <StepItem title="Base case — complete combination">
    If `path.count == digits.count`, append `String(path)` to results and return.
  </StepItem>
  <StepItem title="Recursive case — try each letter">
    For each letter in `consts[digs[index]]`: append letter → recurse with `index + 1` → remove last.
  </StepItem>
  <StepItem title="Kick off and return">
    Call `backTrack(0)`. Return `results` (empty if `digits` is empty since the base case is never reached).
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func letterCombinations(_ digits: String) -> [String] {
    let consts: [Character: [Character]] = [
      "2": ["a","b","c"], "3": ["d","e","f"],
      "4": ["g","h","i"], "5": ["j","k","l"],
      "6": ["m","n","o"], "7": ["p","q","r","s"],
      "8": ["t","u","v"], "9": ["w","x","y","z"],
    ]
    var results = [String]()
    var path = [Character]()
    let digs = Array(digits)

    func backTrack(_ index: Int) {
      if path.count == digits.count {
        results.append(String(path))
        return
      }
      let chars = consts[digs[index], default: []]
      for char in chars {
        path.append(char)
        backTrack(index + 1)
        path.removeLast()
      }
    }

    backTrack(0)
    return results
  }
}
```

<Callout type="green" label="Swift Notes">
The nested `func backTrack` captures `path`, `results`, `digs`, and `consts` by reference from the enclosing function — Swift nested functions capture variables automatically, no explicit capture list needed. `path.removeLast()` is O(1) on Array since it only decrements the count without reallocation.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(4^n * n)">
There are at most 4^n complete combinations (digit 7 and 9 map to 4 letters each). Building each combination string takes O(n) time. With n capped at 4, the practical maximum is 256 combinations of length 4.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
The recursion stack goes at most n levels deep and the path array holds at most n characters. Excluding the output array, auxiliary space is O(n).
</Callout>
