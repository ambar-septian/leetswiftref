<SectionHeader n={1} title="Problem Overview" />

Given a string `s`, partition `s` such that every substring of the partition is a **palindrome**. Return **all possible palindrome partitionings** of `s`.

A **palindrome** is a string that reads the same forward and backward.

**Constraints:**
- `1 <= s.length <= 16`
- `s` consists of only lowercase English letters.

<Callout type="amber" label="Key Constraint">
The string length is at most 16 — the exponential 2^n cost is bounded in practice. Each position can be a cut point or not, giving at most 2^15 possible partitions. A single character is always a palindrome, so the base case is always reachable.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: s = "aab"
Output: [["a","a","b"],["aa","b"]]
```

#### Example 2

```
Input: s = "a"
Output: [["a"]]
```

<Callout type="green" label="Rule of Thumb">
When you need all partitions of a string where each part satisfies a property, use backtracking: try every prefix, recurse only on valid ones, backtrack after each recursion. Cache repeated substring checks to avoid O(n) recomputation per branch.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Try every prefix starting at the current index. If the prefix is a palindrome, append it to the path and recurse on the remainder. On return, remove the prefix (backtrack). A cache from [i,j] → substring avoids re-checking the same range and re-slicing the string on repeated visits.
</Callout>

<PalindromePartitioningDiagram />

Use a `dfs(i)` function that tries every substring `s[i...j]` for `j` from `i` to `s.count - 1`. If the substring is a palindrome, append it to the current partition and recurse with `dfs(j + 1)`. On return, remove the last element. When `i >= s.count`, record a copy of the current partition.

A cache `[[Int]: String]` maps `[i, j]` to the precomputed substring so that repeated checks on the same range are O(1).

<Callout type="cyan" label="Why backtracking works here">
Each position in the string is either a cut point or not. By trying every palindromic prefix and recursing on the remainder, we guarantee that every combination is explored without repetition.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Convert to character array">
    `var chars = Array(s)` for O(1) index access in the palindrome check.
  </StepItem>
  <StepItem title="Define dfs(i)">
    Base case: if `i >= chars.count`, append a copy of `part` to `results` and return.
  </StepItem>
  <StepItem title="Try every ending index j">
    For `j in i..<chars.count`: check the cache `[[i,j]]` first, else call `isPalindrome(i, j, chars)`.
  </StepItem>
  <StepItem title="Append and recurse">
    If palindrome: build the substring, store in cache, append to `part`, recurse `dfs(j+1)`, then `part.removeLast()`.
  </StepItem>
  <StepItem title="isPalindrome helper">
    Two-pointer check: expand inward from `l` and `r`, return false on the first mismatch.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func partition(_ s: String) -> [[String]] {
    var results = [[String]]()
    var part    = [String]()
    let chars   = Array(s)
    var cache   = [[Int]: String]()

    func dfs(_ i: Int) {
      if i >= chars.count {
        results.append(part)
        return
      }

      for j in i..<chars.count {
        if let exist = cache[[i, j]] {
          part.append(exist)
          dfs(j + 1)
          part.removeLast()
        } else if isPalindrome(i, j, chars) {
          let sub = String(chars[i...j])
          cache[[i, j]] = sub
          part.append(sub)
          dfs(j + 1)
          part.removeLast()
        }
      }
    }

    dfs(0)
    return results
  }

  private func isPalindrome(_ l: Int, _ r: Int, _ s: [Character]) -> Bool {
    var l = l, r = r
    while l <= r {
      if s[l] != s[r] { return false }
      l += 1; r -= 1
    }
    return true
  }
}
```

<Callout type="green" label="Swift Notes">
`[[Int]: String]` uses a two-element array as a dictionary key — arrays are `Hashable` in Swift when their elements are Hashable. `results.append(part)` is safe without explicit copying because Swift arrays use value semantics — each append captures the current state of `part`. The nested `dfs` closure captures `chars`, `cache`, `part`, and `results` by reference, allowing mutation without `inout` parameters.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n · 2^n)">
O(n · 2^n) — there are at most 2^(n-1) ways to place cut points in a string of length n. For each valid partition we spend O(n) copying the path. The palindrome cache reduces repeated checks to O(1), eliminating an additional O(n) factor per branch compared to the uncached version.
</Callout>

<Callout type="green" label="Space Complexity — O(n²)">
O(n²) for the cache — up to n²/2 distinct (i,j) pairs each storing a substring of average length n/2. The recursion stack is O(n) deep (at most n cuts). The result array holds all valid partitions but is excluded from auxiliary space analysis.
</Callout>

<Callout type="amber" label="When to prefer cached vs uncached">
Always use the **cached version** — the cache reduces each palindrome check from O(n) to O(1) and each substring slice from O(n) to O(1) on repeated visits. For n=16 the uncached version does up to 16 × 2^16 = 1M extra operations. The space cost of O(n²) is negligible for n ≤ 16.
</Callout>
