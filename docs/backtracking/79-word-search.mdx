<SectionHeader n={1} title="Problem Overview" />

Given an `m x n` grid of characters `board` and a string `word`, return `true` if `word` exists in the grid.

The word can be constructed from letters of sequentially adjacent cells, where adjacent cells are horizontally or vertically neighboring. The same letter cell may not be used more than once.

**Constraints**
- `m == board.length`
- `n == board[0].length`
- `1 <= m, n <= 6`
- `1 <= word.length <= 15`
- `board` and `word` consist of only lowercase and uppercase English letters.

<Callout type="amber" label="Key Constraint">
The same cell may not be used more than once in a single path. The board is at most 6×6 = 36 cells and the word is at most 15 characters — backtracking with in-place visited marking is efficient enough.
</Callout>

<Callout type="green" label="Core Insight">
DFS from every starting cell. At each step check bounds, character match, and whether the cell has already been visited. Mark a visited cell by replacing it with '#' in-place, then restore after backtracking — no separate visited set needed.
</Callout>

<Callout type="green" label="Rule of Thumb">
Grid path problems with no-reuse constraints pattern directly to DFS backtracking with in-place visited marking — replacing a cell with '#' eliminates a separate visited set and is impossible to accidentally leave un-restored.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
From every cell matching word[0], launch DFS. At each step validate bounds, character match, and '#' (already visited). Mark the cell '#' before recursing into four neighbors; restore it after all four recursive calls return. Return true as soon as charIndex reaches word.count — no further exploration needed.
</Callout>

Find all cells where `board[r][c] == word[0]`. For each starting cell call `dfs(r, c, 0)`. The DFS:
1. If `charIndex == word.count` — all characters matched, return `true`.
2. If out of bounds, or `board[r][c] != word[charIndex]` — return `false`.
3. Mark: `board[r][c] = "#"`.
4. Recurse in four directions with `charIndex + 1`. If any returns `true`, propagate `true`.
5. Restore: `board[r][c] = original`.

An optional frequency pruning step verifies that the board contains at least as many of each character as the word requires, eliminating hopeless inputs before DFS begins.

<WordSearchDiagram />

<Callout type="cyan" label="Why '#' works as a visited marker">
'#' is not a letter and will never match any character in word. So if DFS reaches a cell already in the current path, the character check board[r][c] != chars[charIndex] will fail immediately, preventing reuse — without any extra data structure.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Frequency pruning (optional optimisation)">
    Count character frequencies in word and board. If any word character is more frequent than in the board, return false immediately.
  </StepItem>
  <StepItem title="Find starting positions">
    Collect all (r, c) where board[r][c] == word.first.
  </StepItem>
  <StepItem title="DFS with in-place marking">
    For each start: call dfs(r, c, 0). Base case: charIndex == count → return true. Guard: out of bounds, mismatch, or '#' → return false. Mark, recurse 4 directions, restore.
  </StepItem>
  <StepItem title="Return false if no path found">
    If no starting position leads to a full match, return false.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func exist(_ board: [[Character]], _ word: String) -> Bool {
    var chars = Array(word)
    var board = board
    let row = board.count
    let col = board[0].count
    let count = chars.count

    guard row * col >= count else { return false }

    let firstCharPositions = firstCharInGrid(board, word)
    guard !firstCharPositions.isEmpty else { return false }

    func dfs(_ r: Int, _ c: Int, _ charIndex: Int) -> Bool {
      if charIndex == count { return true }

      if r < 0 || c < 0 || r >= row || c >= col || board[r][c] != chars[charIndex] {
        return false
      }

      let temp = board[r][c]
      board[r][c] = "#"          // mark visited

      let next = charIndex + 1
      let result = dfs(r+1,c,next) || dfs(r-1,c,next) || dfs(r,c+1,next) || dfs(r,c-1,next)

      board[r][c] = temp         // restore

      return result
    }

    for (r, c) in firstCharPositions {
      if dfs(r, c, 0) { return true }
    }
    return false
  }

}
```

<Callout type="green" label="Swift Notes">
`var board = board` creates a mutable local copy since Swift 2D arrays are value types. The nested `dfs` closure captures this `var` by reference, so `board[r][c] = "#"` and the restore mutate the same array across all recursion frames without extra copying.
</Callout>

```swift
extension Solution {
  func firstCharInGrid(_ board: [[Character]], _ word: String) -> [(Int, Int)] {
    var freq = [Character: Int]()
    var positions = [(Int, Int)]()

    for char in word { freq[char, default: 0] += 1 }

    for i in 0..<board.count {
      for j in 0..<board[0].count {
        let char = board[i][j]
        if char == word.first { positions.append((i, j)) }
        if freq[char] != nil {
          freq[char]! -= 1
          if freq[char]! <= 0 { freq[char] = nil }
        }
      }
    }

    return freq.isEmpty ? positions : []
  }
}
```

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(N × 3^L)">
O(N × 3^L) where N is the total number of cells and L is the word length. DFS is launched from each of the N starting cells. At each step there are at most 3 directions to explore (cannot revisit the cell we came from), giving 3^L paths per start. The optional frequency pruning can eliminate many starting cells before DFS begins.
</Callout>

<Callout type="green" label="Space Complexity — O(N + L)">
O(L) for the recursion stack depth (one frame per matched character). O(N) if frequency pruning builds a character-count map over the board. No separate visited structure is needed — the '#' in-place marker uses the existing board array.
</Callout>
