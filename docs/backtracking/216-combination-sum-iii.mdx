<SectionHeader n={1} title="Problem Overview" />

Find all valid combinations of `k` numbers that sum up to `n` such that:
- Only numbers `1` through `9` are used.
- Each number is used at most once.

Return a list of all possible valid combinations. The list must not contain the same combination twice, and the combinations may be returned in any order.

**Constraints:**
- `2 <= k <= 9`
- `1 <= n <= 60`

<Callout type="amber" label="Key Constraint">
Only digits 1–9 are available, each used at most once. This tiny fixed search space (9 candidates, at most k slots) bounds the total combinations by C(9, k), making exhaustive backtracking completely feasible.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: k = 3, n = 7
Output: [[1,2,4]]
Explanation: 1 + 2 + 4 = 7, using 3 distinct numbers.
```

#### Example 2

```
Input: k = 3, n = 9
Output: [[1,2,6],[1,3,5],[2,3,4]]
```

#### Example 3

```
Input: k = 4, n = 1
Output: []
```

<Callout type="green" label="Rule of Thumb">
When choosing k elements from a small fixed set without repetition, use backtracking with a `startIndex` — always start the next loop from the element after the one just chosen to keep combinations in ascending order and avoid duplicates.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Recurse over digits 1–9 starting from `startIndex`. Append each digit to `path` and `sum`, recurse with `startIndex = i + 1` (so each digit is used at most once), then undo both additions. Record `path` when `sum == n` and `path.count == k`. Prune immediately when `sum > n`.
</Callout>

<CombinationSumIIIDiagram />

Use a recursive DFS over digits 1–9. At each level, choose the next digit (from `startIndex` to 9), add it to `path` and `sum`, recurse with `startIndex = i + 1`, then undo the choice. Record `path` when `sum == n` and `path.count == k`. Prune when `sum > n`.

<Callout type="cyan" label="Why startIndex prevents duplicates">
By always starting the next loop at <code>i + 1</code>, we never pick the same digit twice and we always build combinations in ascending order. This eliminates the need for deduplication post-hoc.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise candidates">
    `nums = [1,2,3,4,5,6,7,8,9]`, `path = [Int]()`, `sum = 0`, `results = [[Int]]()`.
  </StepItem>
  <StepItem title="Base cases">
    If `sum == n` and `path.count == k`, append `path` to results and return. If `sum > n`, return (prune).
  </StepItem>
  <StepItem title="Choose and recurse">
    For `i` from `startIndex` to `nums.count - 1`: append `nums[i]`, add to `sum`, recurse with `i + 1`, then undo.
  </StepItem>
  <StepItem title="Return results">
    Return `results` after the initial `dfs(0)` call.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func combinationSum3(_ k: Int, _ n: Int) -> [[Int]] {
    var results = [[Int]]()
    let nums = Array(1...9)
    var path = [Int]()
    var sum  = 0

    func dfs(_ startIndex: Int) {
      if sum == n, path.count == k {
        results.append(path)
        return
      } else if sum > n {
        return
      }
      for i in startIndex..<nums.count {
        path.append(nums[i])
        sum += nums[i]
        dfs(i + 1)
        path.removeLast()
        sum -= nums[i]
      }
    }

    dfs(0)
    return results
  }
}
```

<Callout type="green" label="Swift Notes">
The nested `dfs` closure captures `path`, `sum`, and `results` as `var` from the outer scope — Swift closures capture `var` variables by reference, allowing mutation. `path.removeLast()` and `sum -= nums[i]` form the backtracking undo step. `Array(1...9)` creates the candidates array from a closed range in one expression.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(C(9,k) × k)">
At most C(9, k) valid combinations are found. Each valid leaf requires copying a k-element path. The sum/prune condition eliminates many branches before reaching depth k. Total work is O(C(9, k) × k).
</Callout>

<Callout type="green" label="Space Complexity — O(k)">
The recursion depth equals k (one level per chosen element). The `path` array holds at most k elements. Only O(k) stack space is needed.
</Callout>
