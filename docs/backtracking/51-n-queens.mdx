<SectionHeader n={1} title="Problem Overview" />

The **n-queens** puzzle is the problem of placing `n` queens on an `n x n` chessboard such that no two queens attack each other.

Given an integer `n`, return all distinct solutions to the n-queens puzzle. You may return the answer in any order.

Each solution contains a distinct board configuration of the n-queens' placement, where `'Q'` and `'.'` both indicate a queen and an empty cell, respectively.

A queen attacks any cell in the same row, column, positive diagonal (top-left to bottom-right), or negative diagonal (top-right to bottom-left).

**Constraints**
- `1 <= n <= 9`

<Callout type="amber" label="Key Constraint">
n is at most 9, so the board is at most 9×9. A queen attacks any cell in the same row, column, or either diagonal — three independent constraints must all be satisfied for every placement.
</Callout>

<Callout type="green" label="Core Insight">
Place exactly one queen per row using DFS row by row. Track three sets — colSet (columns used), posDigSet (r+c values), negDigSet (r-c values) — to check in O(1) whether a column is safe. When row r == n, all queens are placed validly.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  n = 4
Output: [[".Q..","...Q","Q...","..Q."],["..Q.","Q...","...Q",".Q.."]]
```

#### Example 2
```
Input:  n = 1
Output: [["Q"]]
```

<Callout type="green" label="Rule of Thumb">
When asked to place N non-attacking pieces row-by-row on a grid, backtracking with O(1) conflict sets — one per constraint dimension — is the go-to pattern. Each set encodes a different attack direction.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Place exactly one queen per row, recursing row by row. Three O(1) set lookups cover all attack directions: a column set, a positive-diagonal set (r+c), and a negative-diagonal set (r-c). Any column that fires any of the three sets is skipped, pruning entire subtrees before recursing.
</Callout>

Represent the board as `[[String]]` initialised to `"."`. The recursive function `dfs(_ r: Int)` iterates over all `n` columns for row `r`. For each column `c`, check:
- `colSet.contains(c)` — column already has a queen
- `posDigSet.contains(r + c)` — positive diagonal (top-left to bottom-right) occupied
- `negDigSet.contains(r - c)` — negative diagonal (top-right to bottom-left) occupied

If none fire, place `"Q"`, insert into all three sets, recurse `dfs(r+1)`, then restore.

<NQueensDiagram />

<Callout type="cyan" label="Why r+c and r-c identify diagonals">
All cells on the same top-left-to-bottom-right diagonal share the same (row - col) value. All cells on the same top-right-to-bottom-left diagonal share the same (row + col) value. These invariants let us check diagonal conflicts with a single integer lookup.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise board and constraint sets">
    `board = [[String]](repeating: [String](repeating: ".", count: n), count: n)`. Three empty sets: `colSet`, `posDigSet`, `negDigSet`.
  </StepItem>
  <StepItem title="Base case">
    When `r == n`, convert `board` to `[String]` (each row joined) and append to `results`.
  </StepItem>
  <StepItem title="Try each column in row r">
    For `c` in `0..<n`: if any set contains the conflict key, skip. Otherwise place `"Q"`, update sets, recurse `dfs(r+1)`.
  </StepItem>
  <StepItem title="Backtrack">
    Restore `board[r][c] = "."`, remove from all three sets.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func solveNQueens(_ n: Int) -> [[String]] {
    var results = [[String]]()
    var board = Array(repeating: Array(repeating: ".", count: n), count: n)
    var colSet = Set<Int>()
    var posDigSet = Set<Int>()  // r + c
    var negDigSet = Set<Int>()  // r - c

    func dfs(_ r: Int) {
      if r == n {
        results.append(board.map { $0.joined() })
        return
      }

      for c in 0..<n {
        if colSet.contains(c) || posDigSet.contains(r + c) || negDigSet.contains(r - c) {
          continue
        }

        board[r][c] = "Q"
        colSet.insert(c)
        posDigSet.insert(r + c)
        negDigSet.insert(r - c)

        dfs(r + 1)

        board[r][c] = "."
        colSet.remove(c)
        posDigSet.remove(r + c)
        negDigSet.remove(r - c)
      }
    }

    dfs(0)
    return results
  }
}
```

<Callout type="green" label="Swift Notes">
`board.map { $0.joined() }` at the base case converts the 2D `[[String]]` board into `[String]` rows in one expression without a for loop. `Array(repeating: Array(repeating: ".", count: n), count: n)` correctly initialises an independent 2D array — each inner array is a separate copy, so mutating one row does not affect others.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(N!)">
O(N!) in the worst case. The first row has N choices, the second at most N-1 (one column eliminated), and so on — giving N × (N-1) × ... × 1 = N! leaves. In practice, diagonal pruning eliminates far more branches, making actual runtime significantly lower for small N.
</Callout>

<Callout type="green" label="Space Complexity — O(N²)">
O(N²) for the board itself. The three conflict sets and the recursion stack are each O(N). Since N ≤ 9, total memory usage is negligible.
</Callout>
