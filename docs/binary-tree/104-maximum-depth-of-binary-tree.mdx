<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree, return its **maximum depth**.

A binary tree's maximum depth is the number of nodes along the longest path from the root node down to the farthest leaf node.

**Constraints**
- The number of nodes in the tree is in the range `[0, 10^4]`.
- `-100 <= Node.val <= 100`

<Callout type="green" label="Core Insight">
The maximum depth is 1 (for the current node) plus the greater of the depths of its left and right subtrees. Base case: a nil node has depth 0.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  root = [3,9,20,null,null,15,7]
Output: 3
```

#### Example 2
```
Input:  root = [1,null,2]
Output: 2
```

<SectionHeader n={4} title="Approach" />

Recursive DFS `dfs(_ node: TreeNode?) -> Int`. If node is nil return 0. Recursively compute `left = dfs(node.left)` and `right = dfs(node.right)`. Return `1 + max(left, right)`.

<MaxDepthDiagram />

<Callout type="cyan" label="BFS alternative">
Level-order BFS also works: count how many levels are processed. Each completed level increments a depth counter. This is O(n) time and O(n) space (queue can hold a full level).
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Base case">
    If node is nil, return 0.
  </StepItem>
  <StepItem title="Recurse both subtrees">
    `let left = dfs(node.left)`, `let right = dfs(node.right)`.
  </StepItem>
  <StepItem title="Return combined depth">
    Return `1 + max(left, right)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func maxDepth(_ root: TreeNode?) -> Int {
    guard let root else { return 0 }
    return dfs(root)

    func dfs(_ node: TreeNode?) -> Int {
      guard let node else { return 0 }
      let left = dfs(node.left)
      let right = dfs(node.right)
      return 1 + max(left, right)
    }
  }
}
```
