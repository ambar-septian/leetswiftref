<SectionHeader n={1} title="Problem Overview" />

Given two integer arrays `preorder` and `inorder` where `preorder` is the preorder traversal of a binary tree and `inorder` is the inorder traversal of the same tree, construct and return the binary tree.

- **Preorder** traversal visits: root → left subtree → right subtree
- **Inorder** traversal visits: left subtree → root → right subtree

These two traversals together uniquely determine a binary tree (assuming all values are distinct).

**Constraints**
- `1 <= preorder.length <= 3000`
- `inorder.length == preorder.length`
- `-3000 <= preorder[i], inorder[i] <= 3000`
- `preorder` and `inorder` consist of **unique** values.
- Each value of `inorder` also appears in `preorder`.
- `preorder` is **guaranteed** to be the preorder traversal of the tree.
- `inorder` is **guaranteed** to be the inorder traversal of the tree.

<Callout type="green" label="Core Insight">
preorder[0] is always the root. Find that value's position (mid) in inorder — everything left of mid is the left subtree and everything right is the right subtree. A shared preIndex advances through preorder as each root is consumed; a hash map gives O(1) inorder lookups.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  preorder = [3,9,20,15,7], inorder = [9,3,15,20,7]
Output: [3,9,20,null,null,15,7]
```

#### Example 2
```
Input:  preorder = [-1], inorder = [-1]
Output: [-1]
```

<SectionHeader n={4} title="Approach" />

Build `hash[inorder[i]] = i` for all `i`. Use a shared `preIndex = 0` variable. Define `recursive(inLeft, inRight)`:
1. If `inLeft > inRight` return nil.
2. Root value = `preorder[preIndex]`; increment `preIndex`.
3. `mid = hash[rootVal]`.
4. `root.left = recursive(inLeft, mid - 1)`, `root.right = recursive(mid + 1, inRight)`.

<BuildTreeDiagram />

<Callout type="cyan" label="Why recurse left before right">
preorder is root → left → right. After consuming the root at preIndex, the next values in preorder belong to the left subtree. Recursing left first naturally consumes those values in order before moving to the right subtree.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Build inorder hash map">
    `var hash = [Int: Int]()`. For `i` in `0..<inorder.count`: `hash[inorder[i]] = i`. O(n) preprocessing.
  </StepItem>
  <StepItem title="Shared preIndex">
    `var preIndex = 0`. This advances globally as each root is consumed — no need to pass subarray slices.
  </StepItem>
  <StepItem title="Recursive build">
    `recursive(inLeft, inRight)`: base case `inLeft > inRight` → nil. Consume `preorder[preIndex++]` as root. Split at `mid = hash[rootVal]`. Recurse left, then right.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func buildTree(_ preorder: [Int], _ inorder: [Int]) -> TreeNode? {
    var hash = [Int: Int]()
    for i in 0..<inorder.count { hash[inorder[i]] = i }

    var preIndex = 0

    func recursive(_ inLeft: Int, _ inRight: Int) -> TreeNode? {
      if inLeft > inRight { return nil }

      let rootVal = preorder[preIndex]
      preIndex += 1
      let root = TreeNode(rootVal)

      let mid = hash[rootVal]!
      root.left  = recursive(inLeft,    mid - 1)
      root.right = recursive(mid + 1,   inRight)

      return root
    }

    return recursive(0, inorder.count - 1)
  }
}
```
