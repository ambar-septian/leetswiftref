<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree, check whether it is a mirror of itself (i.e., symmetric around its center).

A tree is symmetric if the left subtree is a mirror reflection of the right subtree. Two trees are mirrors of each other if:
- Their roots have the same value.
- The right subtree of each tree is a mirror of the left subtree of the other tree.

**Constraints**
- The number of nodes in the tree is in the range `[1, 1000]`.
- `-100 <= Node.val <= 100`

<Callout type="amber" label="Key Constraint">
The tree always has at least one node. Symmetry is not just adjacent children matching — at every depth, l1.left must mirror l2.right and l1.right must mirror l2.left. A structurally asymmetric nil placement breaks symmetry even if values happen to match.
</Callout>

<Callout type="green" label="Core Insight">
Use BFS with a queue that stores mirror pairs. Start with (root.left, root.right). At each step dequeue two nodes (l1, l2) that should mirror each other. Verify their values match, then enqueue the next mirror pairs: (l1.left, l2.right) and (l1.right, l2.left).
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  root = [1,2,2,3,4,4,3]
Output: true
```

#### Example 2
```
Input:  root = [1,2,2,null,3,null,3]
Output: false
```

<Callout type="green" label="Rule of Thumb">
Symmetry checks require comparing mirror-image node pairs. Queue pairs of nodes that should reflect each other — always cross-pair children: (l1.left, l2.right) and (l1.right, l2.left).
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
BFS processes mirror pairs level by level. Seed the queue with (root.left, root.right). Each dequeued pair (l1, l2) must have equal values; enqueue their children as the next mirror pairs: (l1.left, l2.right) and (l1.right, l2.left) — always cross-pairing across the axis.
</Callout>

Initialise the queue with `[root.left, root.right]`. While the queue is not empty: dequeue `l1` and `l2` (always in pairs). If both nil, continue. If one is nil or values differ, return `false`. Enqueue `l1.left, l2.right` and `l1.right, l2.left` — the next level's mirror pairs.

<SymmetricTreeDiagram />

<Callout type="cyan" label="Why enqueue l1.left with l2.right (not l2.left)">
For the tree to be symmetric, l1 and l2 are reflections of each other across the vertical axis. l1's left child should match l2's right child (they are on opposite sides of the axis at the next level). Enqueueing l1.left with l2.left would check the wrong pairing.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise queue with root's children">
    `var queue = [root?.left, root?.right]`.
  </StepItem>
  <StepItem title="Dequeue mirror pairs">
    While queue not empty: `let l1 = queue.removeFirst()`, `let l2 = queue.removeFirst()`.
  </StepItem>
  <StepItem title="Validate pair">
    Both nil → `continue`. One nil → `return false`. Values differ → `return false`.
  </StepItem>
  <StepItem title="Enqueue children as mirror pairs">
    `queue.append(contentsOf: [l1.left, l2.right, l1.right, l2.left])`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func isSymmetric(_ root: TreeNode?) -> Bool {
    var queue = [root?.left, root?.right]

    while !queue.isEmpty {
      let l1 = queue.removeFirst()
      let l2 = queue.removeFirst()

      if l1 == nil && l2 == nil { continue }
      if l1?.val != l2?.val { return false }

      queue.append(contentsOf: [l1?.left, l2?.right, l1?.right, l2?.left])
    }

    return true
  }
}
```

<Callout type="green" label="Swift Notes">
`queue.append(contentsOf: [l1?.left, l2?.right, l1?.right, l2?.left])` appends four optional TreeNode? elements in one call. Optional chaining on `l1?.left` is safe — nil children are appended as nil and handled by the both-nil continue on the next dequeue.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
O(n) — each node is enqueued and dequeued exactly once. The queue processes all n nodes before concluding. Short-circuit returns false immediately on the first mismatch, making typical runtime lower than worst case.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
O(n) for the queue, which holds at most one full level of nodes at a time. For a complete binary tree the widest level has n/2 nodes — O(n) total. The recursion-based alternative uses O(h) stack space where h is the tree height.
</Callout>
