<SectionHeader n={1} title="Problem Overview" />

Given a binary tree `root`, a node X in the tree is named **good** if in the path from root to X there are no nodes with a value **greater than** X's value.

Return the number of good nodes in the binary tree.

**Constraints:**
- The number of nodes in the binary tree is in the range `[1, 10^5]`.
- Each node's value is between `-10^4` and `10^4`.

<Callout type="amber" label="Key Constraint">
  Node values can be negative (down to -10^4). Initialise maxSoFar as Int.min, not 0 — otherwise a negative root node is incorrectly marked non-good. The root is always a good node.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  root = [3,1,4,3,null,1,5]
Output: 4
Explanation: Good nodes are marked with (G):
  [3(G)] → [1] → [3(G)]
  [3(G)] → [4(G)] → [5(G)]
  Node 1 (left child of root) is NOT good since 3 > 1.
  Node 1 (left child of 4) is NOT good since 4 > 1.
```

#### Example 2

```
Input:  root = [3,3,null,4,2]
Output: 3
```

#### Example 3

```
Input:  root = [1]
Output: 1
```

<Callout type="green" label="Rule of Thumb">
  When each node needs context from all its ancestors (path max, path sum, path product), use top-down DFS and pass that state as a parameter — not bottom-up, which only aggregates from leaves upward.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  A node is good if its value is &gt;= the maximum on the root-to-node path. Pass this running max top-down via DFS parameters, updating it at each level with max(node.val, prev). Initialize with Int.min so the root is always counted as good.
</Callout>

Perform a DFS from the root, passing the maximum value seen so far on the current root-to-node path. At each node: if `node.val >= maxSoFar`, it is a good node (count 1); otherwise count 0. Update `maxSoFar = max(node.val, maxSoFar)` before recursing into children.

<Callout type="green" label="Start with -infinity">
Initialise maxSoFar with `Int.min`. This ensures the root is always counted as good, since any integer is >= Int.min.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Define recursive DFS">
    `func dfs(_ node: TreeNode?, _ prev: Int) -> Int` — `prev` is the max value seen from root to the parent of this node.
  </StepItem>
  <StepItem title="Base case">
    If `node` is nil, return 0.
  </StepItem>
  <StepItem title="Check if current node is good">
    `let current = node.val >= prev ? 1 : 0`
  </StepItem>
  <StepItem title="Update max and recurse">
    `let maxVal = max(node.val, prev)`. Return `current + dfs(node.left, maxVal) + dfs(node.right, maxVal)`.
  </StepItem>
  <StepItem title="Start DFS">
    Call `dfs(root, Int.min)` to begin.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func goodNodes(_ root: TreeNode?) -> Int {
    func dfs(_ root: TreeNode?, _ prev: Int) -> Int {
      guard let root else { return 0 }
      let current = root.val >= prev ? 1 : 0
      let maxVal = max(root.val, prev)
      return current + dfs(root.left, maxVal) + dfs(root.right, maxVal)
    }
    return dfs(root, Int.min)
  }
}
```

<Callout type="green" label="Swift Notes">
  `func dfs(_ root: TreeNode?, _ prev: Int) -> Int` is a nested function capturing nothing from the outer scope — clean and avoids a class-level variable. `guard let root else { return 0 }` handles the nil base case idiomatically. The single return `current + dfs(root.left, maxVal) + dfs(root.right, maxVal)` computes both subtrees inline. `Int.min` as the initial prev ensures the root node always passes the &gt;= check.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  DFS visits every node exactly once. At each node, constant work is done — one comparison, one max call, two recursive calls. Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(h)">
  The call stack depth equals the tree height h. For a balanced tree h = O(log n); for a skewed tree h = O(n). No auxiliary data structures beyond the call stack.
</Callout>
