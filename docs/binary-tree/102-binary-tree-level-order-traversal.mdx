<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree, return the **level order traversal** of its nodes' values (i.e., from left to right, level by level).

Level order traversal visits all nodes at depth 0 (root), then all nodes at depth 1 (root's children), then depth 2, and so on. The output is a list of lists, one inner list per level.

**Constraints**
- The number of nodes in the tree is in the range `[0, 2000]`.
- `-1000 <= Node.val <= 1000`

<Callout type="amber" label="Key Constraint">
The result must group nodes by level — the inner list order within each level is left-to-right. The tree can be empty (return []). Snapshotting the queue size before each level is the key to correctly separating levels.
</Callout>

<Callout type="green" label="Core Insight">
BFS with a queue. At the start of each while-loop iteration, snapshot the current queue size — that is exactly the number of nodes at the current level. Process only that many nodes, appending their children. Collect values into a level array, then append to the result.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  root = [3,9,20,null,null,15,7]
Output: [[3],[9,20],[15,7]]
```

#### Example 2
```
Input:  root = [1]
Output: [[1]]
```

#### Example 3
```
Input:  root = []
Output: []
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
BFS naturally visits nodes level by level. The key is snapshotting count = queue.count before the inner loop — this number is exactly the current level's size. Process only that many nodes, collect their values, then append to the result. Children added during the inner loop belong to the next level.
</Callout>

Initialise `queue = [root]`. Each while-loop iteration: record `count = queue.count`. Loop exactly `count` times: dequeue, skip nils, append value to `curr`, enqueue both children. After the inner loop, if `curr` is non-empty append it to `result`.

<LevelOrderDiagram />

<Callout type="cyan" label="Why snapshot count before the inner loop">
During the inner loop, new children are added to the queue. If you check queue.count dynamically inside the inner loop, you will process next-level nodes too. Snapshotting count = queue.count at the start isolates exactly the current level's nodes.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard empty root">
    Return `[]` if root is nil.
  </StepItem>
  <StepItem title="Initialise queue and result">
    `var queue: [TreeNode?] = [root]`, `var result = [[Int]]()`.
  </StepItem>
  <StepItem title="BFS loop with level snapshot">
    While queue not empty: `let count = queue.count`. Inner loop `count` times: dequeue, guard non-nil, append value, enqueue children. Append `curr` to `result` if non-empty.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func levelOrder(_ root: TreeNode?) -> [[Int]] {
    guard let root else { return [] }

    var queue: [TreeNode?] = [root]
    var result = [[Int]]()

    while !queue.isEmpty {
      let count = queue.count      // snapshot current level size
      var curr = [Int]()

      for _ in 0..<count {
        guard let item = queue.removeFirst() else { continue }
        curr.append(item.val)
        queue.append(item.left)
        queue.append(item.right)
      }

      if !curr.isEmpty { result.append(curr) }
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
`queue.removeFirst()` on a Swift Array is O(n) — acceptable here since n ≤ 2000, but for large inputs use an index pointer (`var head = 0; queue[head]; head += 1`) to make dequeue O(1). `guard let item = queue.removeFirst() else { continue }` cleanly skips nil children without a separate if-let.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
O(n) — each of the n nodes is enqueued and dequeued exactly once. The level-snapshot and inner loop add no extra traversal: every node is processed in exactly one inner-loop iteration.
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
O(n) for the queue, which holds at most one complete level at a time. For a perfect binary tree the last level has n/2 nodes — the maximum queue size is O(n). The result array is also O(n) total.
</Callout>
