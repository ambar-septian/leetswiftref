<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree, invert the tree (mirror it), and return its root. Inverting means every node's left and right children are swapped at every level of the tree.

**Constraints:**
- The number of nodes in the tree is in the range `[0, 100]`.
- `-100 <= Node.val <= 100`

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: root = [4,2,7,1,3,6,9]
Output: [4,7,2,9,6,3,1]
```

#### Example 2

```
Input: root = [2,1,3]
Output: [2,3,1]
```

#### Example 3

```
Input: root = []
Output: []
```

<SectionHeader n={4} title="Approach" />

<InvertBinaryTreeDiagram />

Recurse into the left subtree and right subtree. Once both return their inverted roots, assign `node.left = invertedRight` and `node.right = invertedLeft`. Return the node. The base case returns nil for a nil node.

<Callout type="cyan" label="Post-order vs pre-order">
Swapping children before recursing (pre-order) also works — the swap just happens in a different relative order. Post-order is more common in explanations because it mirrors the idea of "invert children first, then attach them."
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Base case">
    If `node` is nil, return nil.
  </StepItem>
  <StepItem title="Recurse into both subtrees">
    `let left = invert(node.left)`, `let right = invert(node.right)`.
  </StepItem>
  <StepItem title="Swap children">
    `node.left = right`, `node.right = left`.
  </StepItem>
  <StepItem title="Return node">
    Return the current node (its children are now swapped).
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func invertTree(_ root: TreeNode?) -> TreeNode? {
    return invert(root)

    func invert(_ node: TreeNode?) -> TreeNode? {
      guard let node else { return nil }
      let left  = invert(node.left)
      let right = invert(node.right)
      node.left  = right
      node.right = left
      return node
    }
  }
}
```
