<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree, return the **postorder traversal** of its node values.

Postorder traversal visits nodes in the order: **left subtree → right subtree → root**.

**Constraints:**
- The number of nodes in the tree is in the range `[0, 100]`.
- `-100 <= Node.val <= 100`
- **Follow-up:** Recursive solution is trivial. Can you do it iteratively?

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input: root = [1,null,2,3]
Output: [3,2,1]
```

#### Example 2

```
Input: root = []
Output: []
```

#### Example 3

```
Input: root = [1]
Output: [1]
```

<SectionHeader n={4} title="Approach" />

<PostorderTraversalDiagram />

Return `postorderTraversal(root.left) + postorderTraversal(root.right) + [root.val]`. Both subtrees are fully processed before the root is appended, which is exactly what postorder requires.

<Callout type="cyan" label="Postorder use case">
Postorder is naturally suited to problems where a node must be processed only after all its descendants — such as deleting a tree, computing subtree properties, or flattening a tree (problem 114).
</Callout>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func postorderTraversal(_ root: TreeNode?) -> [Int] {
    guard let root else { return [] }
    return postorderTraversal(root.left)
      + postorderTraversal(root.right)
      + [root.val]
  }
}
```
