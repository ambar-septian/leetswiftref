<SectionHeader n={1} title="Problem Overview" />

Given the `root` of a binary tree and an integer `targetSum`, return `true` if the tree has a **root-to-leaf path** such that adding up all the values along the path equals `targetSum`.

A **leaf** is a node with no children. The path must start at the root and end at a leaf — paths to internal nodes do not count.

**Constraints**
- The number of nodes in the tree is in the range `[0, 5000]`.
- `-1000 <= Node.val <= 1000`
- `-1000 <= targetSum <= 1000`

<Callout type="amber" label="Key Constraint">
The path must start at the root and end at a leaf — a node with no children. Paths to internal nodes do not count. An empty tree has no root-to-leaf path, so return false even if targetSum is 0.
</Callout>

<Callout type="green" label="Core Insight">
Accumulate a running sum down each branch via DFS. When a leaf is reached, check if the accumulated sum equals targetSum. Return true as soon as any root-to-leaf path matches — no need to explore further.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  root = [5,4,8,11,null,13,4,7,2,null,null,null,1], targetSum = 22
Output: true
Explanation: Path 5→4→11→2 = 22
```

#### Example 2
```
Input:  root = [1,2,3], targetSum = 5
Output: false
```

#### Example 3
```
Input:  root = [], targetSum = 0
Output: false
```

<Callout type="green" label="Rule of Thumb">
Tree path-existence problems (does any root-to-leaf path satisfy condition X?) map directly to DFS with a running value passed down. Check the condition only at leaves; short-circuit with || means the first matching path ends the search.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Accumulate a running sum as you recurse down. At every leaf, check if the sum matches targetSum and return true immediately if it does. Use || when recursing both branches — the first true terminates the entire search without exploring remaining paths.
</Callout>

Define `calculate(_ root: TreeNode?, _ sum: Int, _ targetSum: Int) -> Bool`. If `root` is nil return `false`. Compute `sm = sum + root.val`. If root is a leaf (`left == nil && right == nil`), return `sm == targetSum`. Otherwise return `calculate(left, sm, targetSum) || calculate(right, sm, targetSum)`.

<PathSumDiagram />

<Callout type="cyan" label="Why check at leaves only">
Internal nodes on a root-to-internal path are not valid path endpoints. If the sum matches at an internal node, we must continue to a leaf. Only when both children are nil (leaf) do we commit to the sum check.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard nil">
    If `root` is nil, return `false` — no path exists.
  </StepItem>
  <StepItem title="Accumulate running sum">
    `let sm = sum + root.val`.
  </StepItem>
  <StepItem title="Check at leaf">
    If `root.left == nil && root.right == nil`: return `sm == targetSum`.
  </StepItem>
  <StepItem title="Recurse both branches">
    Return `calculate(root.left, sm, targetSum) || calculate(root.right, sm, targetSum)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func hasPathSum(_ root: TreeNode?, _ targetSum: Int) -> Bool {
    return calculate(root, 0, targetSum)
  }

  func calculate(_ root: TreeNode?, _ sum: Int, _ targetSum: Int) -> Bool {
    guard let root else { return false }

    let sm = sum + root.val
    if root.left == nil && root.right == nil {
      return sm == targetSum
    }
    return calculate(root.left,  sm, targetSum)
        || calculate(root.right, sm, targetSum)
  }
}
```

<Callout type="green" label="Swift Notes">
Swift's `||` short-circuits: `calculate(root.left, sm, targetSum)` is evaluated first, and `calculate(root.right, ...)` is only called if the left branch returned false. This means the right subtree is skipped entirely once a valid path is found in the left subtree.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
O(n) in the worst case — all nodes are visited when no path matches. Short-circuit evaluation with || means the right subtree is skipped as soon as a matching path is found in the left subtree, making typical runtime much lower.
</Callout>

<Callout type="green" label="Space Complexity — O(h)">
O(h) for the recursion call stack where h is the tree height. O(log n) for a balanced tree, O(n) for a skewed tree. No additional data structures are needed — the running sum is passed as a parameter.
</Callout>
