<SectionHeader n={1} title="Problem Overview" />

Given the root of a binary tree, the level of its root is `1`, the level of its children is `2`, and so on.

Return the **smallest** level `x` such that the sum of all the values of nodes at level `x` is maximal.

**Constraints:**
- The number of nodes in the tree is in the range `[1, 10^4]`.
- `-10^5 <= Node.val <= 10^5`

<Callout type="amber" label="Key Constraint">
  Node values can be negative, so a level with fewer nodes may have a larger sum than a full level. The result must be the **smallest** level number when multiple levels share the maximum sum — update only on strict improvement (`sum > maxVal`), not on ties.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  root = [1,7,0,7,-8,null,null]
Output: 2
Explanation:
  Level 1 sum = 1
  Level 2 sum = 7 + 0 = 7  ← maximum
  Level 3 sum = 7 + (-8) = -1
```

#### Example 2

```
Input:  root = [989,null,10250,98693,-89388,null,null,null,-32127]
Output: 2
```

<Callout type="green" label="Rule of Thumb">
  When you need to aggregate a per-level statistic across a binary tree, BFS with a level snapshot (capture queue.count before the inner loop) gives you clean level boundaries. Use strict `>` for max comparison to keep the smallest level on ties.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  BFS with a level snapshot (capture `queue.count` at the start of each iteration) cleanly separates levels. Sum all node values in the current level, then compare with the running maximum using strict `>` — this automatically picks the smallest level number when ties occur.
</Callout>

BFS processes the tree level by level. For each level, we sum all node values and compare against the running maximum. We update the result only on a strict improvement (`sum > maxVal`), which automatically gives the smallest level in case of ties.

<Callout type="green" label="Strict greater-than for tie-breaking">
Using `sum > maxVal` (not `>=`) means we update the answer only when we find a strictly larger sum. The first (smallest-numbered) level with the maximum sum wins — which is exactly what the problem requires.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Guard nil root">
    Return 0 immediately if root is nil (empty tree).
  </StepItem>
  <StepItem title="Initialise BFS">
    Push root into the queue. Set `currentLevel = 0`, `result = 0`, `maxVal = Int.min`.
  </StepItem>
  <StepItem title="Process each level">
    Snapshot the queue count at the start of each iteration. Dequeue exactly that many nodes, accumulating their values into `sum`. Enqueue their non-nil children.
  </StepItem>
  <StepItem title="Increment level and compare">
    After processing the full level, increment `currentLevel`. If `sum > maxVal`, update `maxVal = sum` and `result = currentLevel`.
  </StepItem>
  <StepItem title="Return result">
    When the queue is empty, return `result` — the 1-indexed level with the highest sum.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func maxLevelSum(_ root: TreeNode?) -> Int {
    guard let root else { return 0 }

    var queue = [root]
    var currentLevel = 0
    var result = 0
    var maxVal = Int.min

    while !queue.isEmpty {
      let count = queue.count
      var sum = 0

      for _ in 0..<count {
        let node = queue.removeFirst()
        sum += node.val
        if let left  = node.left  { queue.append(left)  }
        if let right = node.right { queue.append(right) }
      }

      currentLevel += 1
      if sum > maxVal {
        maxVal = sum
        result = currentLevel
      }
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `var queue = [root]` initialises with the unwrapped root (already guarded above). `queue.removeFirst()` is O(n) — for large trees, use an index pointer instead. `Int.min` as the initial `maxVal` handles trees where all node values are negative. Initialise `currentLevel = 0` and increment inside the loop so the first level becomes 1.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  BFS visits every node exactly once. For each node, we add its value to the level sum and enqueue at most 2 children — constant work per node. Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
  The queue holds at most one full level of the tree at a time. In the worst case (a perfect binary tree), the last level has n/2 nodes — O(n) queue space. Total auxiliary space: O(n).
</Callout>
