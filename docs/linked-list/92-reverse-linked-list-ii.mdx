<SectionHeader n={1} title="Problem Overview" />

Given the head of a singly linked list and two integers `left` and `right` where `left <= right`, reverse the nodes of the list from position `left` to position `right`, and return the reversed list.

Positions are 1-indexed. Nodes outside the range `[left, right]` are left untouched. The reversal must be done in one pass.

**Constraints**
- The number of nodes in the list is `n`.
- `1 <= n <= 500`
- `-500 <= Node.val <= 500`
- `1 <= left <= right <= n`

<Callout type="green" label="Core Insight">
Walk to the node just before position left (lastNodeBeforeSublist) and the node at position left (lastNodeOnSublist — it becomes the tail after reversal). Reverse exactly (right - left + 1) nodes in-place using the standard iterative reversal. Re-attach the reversed sublist by connecting lastNodeBeforeSublist to the new head and lastNodeOnSublist to the first node after the sublist.
</Callout>

<SectionHeader n={4} title="Approach" />

**Phase 1 — Navigate to sublist start**: Walk `left - 1` steps to position `prev` just before the sublist and `curr` at the sublist start. Save `lastNodeBeforeSublist = prev` and `lastNodeOnSublist = curr`.

**Phase 2 — Reverse sublist**: Run the standard iterative reversal for exactly `target = right - left + 1` iterations using `prevReversed` and `curr`.

**Phase 3 — Reconnect**:
- If `lastNodeBeforeSublist != nil`: `lastNodeBeforeSublist.next = prevReversed` (the new sublist head).
- Else: `head = prevReversed` (reversal started from position 1 — update the overall head).
- `lastNodeOnSublist.next = curr` (connect old sublist head — now tail — to the node after position right).

<Callout type="cyan" label="Why lastNodeOnSublist is saved before reversal">
After iterative reversal, curr points to the first node past the sublist (right+1). lastNodeOnSublist is the node that was originally at position left — it becomes the reversed tail and must point to curr. Saving it before reversal is the only way to hold this reference.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Navigate to sublist start">
    Walk `left - 1` steps: `prev = curr`, `curr = curr.next`. Save `lastNodeBeforeSublist = prev`, `lastNodeOnSublist = curr`.
  </StepItem>
  <StepItem title="Reverse target range">
    `target = right - left + 1`. Repeat `target` times: save `next = curr.next`, set `curr.next = prevReversed`, advance `prevReversed = curr`, `curr = next`.
  </StepItem>
  <StepItem title="Reconnect to surrounding list">
    If `lastNodeBeforeSublist != nil`: `lastNodeBeforeSublist.next = prevReversed`. Else: `head = prevReversed`. Then `lastNodeOnSublist.next = curr`.
  </StepItem>
  <StepItem title="Return head">
    Return the (possibly updated) `head`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func reverseBetween(_ head: ListNode?, _ left: Int, _ right: Int) -> ListNode? {
    var head = head
    var curr = head
    var prev: ListNode?

    let target = right - left + 1

    // Phase 1: walk to sublist start
    for _ in 1..<left {
      prev = curr
      curr = curr?.next
    }

    let lastNodeBeforeSublist = prev
    let lastNodeOnSublist = curr   // becomes the tail after reversal

    // Phase 2: reverse target nodes
    var prevReversed: ListNode?
    for _ in 0..<target {
      let next = curr?.next
      curr?.next = prevReversed
      prevReversed = curr
      curr = next
    }

    // Phase 3: reconnect
    if lastNodeBeforeSublist != nil {
      lastNodeBeforeSublist?.next = prevReversed
    } else {
      head = prevReversed   // reversal started at position 1
    }

    lastNodeOnSublist?.next = curr

    return head
  }
}
```
