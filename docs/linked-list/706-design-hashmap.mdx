<SectionHeader n={1} title="Problem Overview" />

Design a HashMap without using any built-in hash table libraries.

Implement `MyHashMap`:
- `MyHashMap()` — Initializes the object with an empty map.
- `void put(int key, int value)` — Inserts a `(key, value)` pair into the HashMap. If the `key` already exists in the map, update the corresponding `value`.
- `int get(int key)` — Returns the `value` to which the specified `key` is mapped, or `-1` if this map contains no mapping for the key.
- `void remove(int key)` — Removes the `key` and its corresponding `value` if the map contains the mapping for the `key`.

**Constraints:**
- `0 <= key, value <= 10^6`
- At most `10^4` calls will be made to `put`, `get`, and `remove`.

<Callout type="amber" label="Key Constraint">
Values range `0 <= value <= 10^6`, so `0` is a valid stored value. The `get` sentinel of `-1` is safe because `-1` falls outside the valid value range. Key `0` is also valid — do not treat it as a missing-key indicator.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:
["MyHashMap","put","put","get","get","put","get","remove","get"]
[[],[1,1],[2,2],[1],[3],[2,1],[2],[2],[2]]

Output: [null,null,null,1,-1,null,1,null,-1]

Explanation:
MyHashMap myHashMap = new MyHashMap();
myHashMap.put(1, 1);  // The map is now [[1,1]]
myHashMap.put(2, 2);  // The map is now [[1,1], [2,2]]
myHashMap.get(1);     // return 1
myHashMap.get(3);     // return -1 (not found)
myHashMap.put(2, 1);  // update: [[1,1], [2,1]]
myHashMap.get(2);     // return 1
myHashMap.remove(2);  // [[1,1]]
myHashMap.get(2);     // return -1
```

<Callout type="green" label="Rule of Thumb">
In a chaining hash map, `put` must scan the chain before appending — if the key already exists, update its value in-place and return immediately. Appending unconditionally creates duplicate keys that cause stale `get` results.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Identical structure to HashSet (705) but each node stores both `key` and `value`. The critical difference in `put`: scan the chain — if the key already exists, update its value in-place and return immediately; only append a new node if the key is not present. This enforces the unique-key invariant without remove-and-reinsert.
</Callout>

Allocate 10,000 buckets. For `put(key, value)`: hash to bucket, scan chain for an existing node with the same key — if found, update value; otherwise append a new node. For `get(key)`: scan and return the value or -1. For `remove(key)`: find the predecessor and splice out the node.

<DesignHashMapDiagram />

<Callout type="cyan" label="In-place update vs insert">
When put() finds an existing key in the chain it updates the value in place and returns immediately — no removal and re-insertion needed. This keeps the chain order stable and avoids unnecessary allocations.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise buckets">
    `array = (0..10_000).map { _ in nil }` — 10,000 optional ListNode slots.
  </StepItem>
  <StepItem title="put(key, value)">
    Hash to bucket. If nil, create dummy → node(key, value). Else scan; if `current.next.key == key`, update `current.next.val = value` and return; else advance. Append new node at tail.
  </StepItem>
  <StepItem title="get(key)">
    Hash to bucket. If nil, return `-1`. Walk chain; if `current.key == key`, return `current.val`. Return `-1`.
  </StepItem>
  <StepItem title="remove(key)">
    Hash to bucket. If nil, return. Walk chain; when `current.next.key == key`, set `current.next = current.next.next`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class ListNode {
  var val: Int?
  var key: Int?
  var next: ListNode?

  init(_ key: Int?, _ val: Int?) {
    self.val = val
    self.key = key
  }
}

class MyHashMap {
  var array: [ListNode?]

  init() {
    array = (0..<10_000).map { _ in nil }
  }

  func put(_ key: Int, _ value: Int) {
    let hashKey = hash(key)
    if let listNode = array[hashKey] {
      var current: ListNode? = listNode
      while current?.next != nil {
        if current?.next?.key == key {
          current?.next?.val = value
          return
        }
        current = current?.next
      }
      current?.next = ListNode(key, value)
    } else {
      let listNode = ListNode(nil, nil)  // dummy
      listNode.next = ListNode(key, value)
      array[hashKey] = listNode
    }
  }

  func get(_ key: Int) -> Int {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return -1 }
    var current: ListNode? = listNode
    while current != nil {
      if current?.key == key { return current?.val ?? -1 }
      current = current?.next
    }
    return -1
  }

  func remove(_ key: Int) {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return }
    var current: ListNode? = listNode
    while current != nil {
      if current?.next?.key == key {
        current?.next = current?.next?.next
        break
      }
      current = current?.next
    }
    array[hashKey] = listNode
  }

  private func hash(_ key: Int) -> Int { key % 10_000 }
}
```

<Callout type="green" label="Swift Notes">
Both `key` and `val` on `ListNode` are `Int?` — `nil` serves as the sentinel for the dummy head. `current?.val ?? -1` in `get` returns -1 if `current.val` is nil (dummy head matched — impossible in practice, but defensive). `private func hash` uses `private` access control to prevent accidental external use.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(1) avg">
Same analysis as HashSet (705): O(1) hash computation, O(n/B) chain traversal on average. With 10,000 buckets and keys up to 10^6, maximum chain length is 100 — effectively O(1) per operation.
</Callout>

<Callout type="green" label="Space Complexity — O(B + n)">
B = 10,000 bucket slots plus one `ListNode` per stored key-value pair. Total: O(B + n) = O(10,000 + n).
</Callout>
