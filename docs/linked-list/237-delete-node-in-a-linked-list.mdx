<SectionHeader n={1} title="Problem Overview" />

There is a singly-linked list and you are given access to **only** a specific node to delete (not the head). You are guaranteed that the node to delete is not the tail.

Delete the given node such that:
- The value of the given node is no longer in the linked list.
- The number of nodes in the linked list decreases by one.
- All values before the given node remain in the same order.
- All values after the given node remain in the same order.

**Constraints:**
- The number of nodes in the list is in the range `[2, 1000]`.
- `-1000 <= Node.val <= 1000`
- The value of each node in the list is unique.
- The node to be deleted is in the list and is not a tail node.

<SectionHeader n={4} title="Approach" />

Since we cannot access the previous node, we cannot redirect a predecessor's `next` pointer. Instead, copy the value of `node.next` into `node`, then set `node.next = node.next.next`. The current node now impersonates its successor, and the real successor is unlinked.

<Callout type="cyan" label="Why this works">
Observers of the list see the node's value as whatever is stored in its memory. By overwriting the value and re-linking, the node physically remains in the list but logically represents the next node. The old next node is skipped and garbage-collected.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Copy successor value">
    `node.val = node.next.val` — overwrite this node's value with the next node's value.
  </StepItem>
  <StepItem title="Skip the successor">
    `node.next = node.next.next` — unlink the now-duplicate successor node.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func deleteNode(_ node: ListNode?) {
    node?.val  = node?.next?.val ?? 0
    node?.next = node?.next?.next
  }
}
```
