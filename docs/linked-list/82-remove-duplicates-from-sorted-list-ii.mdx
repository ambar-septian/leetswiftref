<SectionHeader n={1} title="Problem Overview" />

Given the head of a sorted linked list, delete all nodes that have duplicate numbers, leaving only distinct numbers from the original list. Return the linked list sorted as well.

Unlike problem 83 (which keeps one copy of each duplicate), this problem removes **all** occurrences of any value that appears more than once.

**Constraints**
- The number of nodes in the list is in the range `[0, 300]`.
- `-100 <= Node.val <= 100`
- The list is guaranteed to be **sorted** in ascending order.

<Callout type="amber" label="Key Constraint">
All occurrences of any duplicated value must be removed ÔÇö not just the extras. The list can be empty or consist entirely of duplicates. Sorted order guarantees all copies of a value are adjacent.
</Callout>

<Callout type="green" label="Core Insight">
Use a dummy head and a fast pointer. When a consecutive pair of equal values is detected, record the duplicate value. Skip the fast pointer forward past all nodes with that value. Only nodes whose value does not match the recorded duplicate value are appended to the result list.
</Callout>

<Callout type="green" label="Rule of Thumb">
To remove all nodes of any duplicated value from a sorted list, a dummy head and a fast pointer that records the duplicate value let you skip entire runs in a single pass ÔÇö no second pass needed.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Use a dummy head so the original head can be removed without special-casing. Advance a fast pointer one step at a time: on consecutive equal values, record the duplicate value and skip fast forward; on a node whose value matches the recorded duplicate, skip it; otherwise build a new node and attach to the result tail.
</Callout>

Use a `dummy` node pointing to `head` and a `result` pointer tracking the tail of the output list. Advance `fast` one node at a time:

- If `fast.val == fast.next.val`, record `duplValue = fast.val` and advance `fast` (skipping this duplicate run).
- If `fast.val == duplValue`, skip this node too (it is part of the duplicate run).
- Otherwise, create a new node with `fast.val` and attach to `result`.

Return `dummy.next`.

<Callout type="cyan" label="Why a new node is built rather than re-linking">
Building new nodes avoids mutating the original list's next pointers mid-traversal, which can cause cycles or skipped nodes. For competitive coding a re-link approach also works, but new-node construction is easier to reason about.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Create dummy head">
    `dummy = ListNode(0)`, `result = dummy`, `fast = head`, `duplValue: Int? = nil`.
  </StepItem>
  <StepItem title="Detect duplicate run">
    If `fast.val == fast.next.val`, record `duplValue = fast.val` and advance `fast` without appending.
  </StepItem>
  <StepItem title="Skip nodes matching duplValue">
    If `fast.val == duplValue`, advance `fast` without appending.
  </StepItem>
  <StepItem title="Append unique node">
    Otherwise, create `ListNode(fast.val)`, attach to `dummy.next`, advance both `dummy` and `fast`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func deleteDuplicates(_ head: ListNode?) -> ListNode? {
    var fast = head
    var dummy: ListNode? = ListNode(0)
    var result = dummy
    var duplValue: Int?

    while fast != nil {
      if let f = fast?.val, let f2 = fast?.next?.val, f2 == f {
        duplValue = f
        fast = fast?.next
        continue
      }

      if duplValue == fast?.val {
        fast = fast?.next
      } else {
        dummy?.next = ListNode(fast!.val)
        dummy = dummy?.next
        fast = fast?.next
      }
    }

    return result?.next
  }
}
```

<Callout type="green" label="Swift Notes">
Building new `ListNode` instances instead of re-linking avoids mutating `.next` pointers mid-traversal, which can create cycles. Swift's optional chaining (`fast?.val`, `fast?.next?.val`) cleanly handles nil-termination without explicit nil guards on every access.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity ÔÇö O(n)">
O(n) ÔÇö each node is visited exactly once by the fast pointer. The duplicate-detection and skip logic are both O(1) per node, so the single pass over all n nodes dominates.
</Callout>

<Callout type="green" label="Space Complexity ÔÇö O(n)">
O(n) for the new nodes built in the output list ÔÇö one new node per unique-value node retained. The four pointer variables (dummy, result, fast, duplValue) are O(1) auxiliary space. The input list nodes are left unmodified.
</Callout>
