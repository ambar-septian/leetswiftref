<SectionHeader n={1} title="Problem Overview" />

Given the head of a linked list where every node's value is a positive integer, insert a new node between every pair of adjacent nodes so that the new node's value is the **greatest common divisor (GCD)** of the two adjacent nodes.

Return the `head` of the modified linked list.

The **GCD** of two integers is the largest positive integer that evenly divides both numbers.

**Constraints:**
- The number of nodes in the list is in the range `[1, 5000]`.
- `1 <= Node.val <= 1000`

<Callout type="amber" label="Key Constraint">
  The list has at most 5000 nodes and Node.val ≤ 1000. Each GCD call is O(log 1000) ≈ 10 steps, so the full O(n log V) pass is at most ~50,000 operations — no optimisation beyond the Euclidean algorithm is needed.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  head = [18,6,10,3]
Output: [18,6,6,2,10,1,3]
Explanation:
  GCD(18,6)=6  → insert 6 between 18 and 6.
  GCD(6,10)=2  → insert 2 between 6 and 10.
  GCD(10,3)=1  → insert 1 between 10 and 3.
```

#### Example 2

```
Input:  head = [7]
Output: [7]
Explanation: Single node — no adjacent pairs to process.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Walk with `curr` and `curr.next`. For each pair, compute GCD, splice a new node between them (`curr.next = gcdNode`, `gcdNode.next = next`), then advance `curr` by two to skip the inserted node and land on the original next.
</Callout>

Walk `curr` from `head`. While `curr.next != nil`: save `next = curr.next`, compute `gcd = GCD(curr.val, next.val)`, create `gcdNode = ListNode(gcd)`, set `curr.next = gcdNode` and `gcdNode.next = next`. Advance `curr = curr.next.next` (skipping over the inserted node to reach the original next node).

<Callout type="cyan" label="Euclidean GCD">
GCD(a, b): while b != 0, swap (a,b) = (b, a%b). Return a. This is O(log min(a,b)) per call.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Walk adjacent pairs">
    `var current = head`. While `current?.next != nil`: capture `let next = current!.next`.
  </StepItem>
  <StepItem title="Compute GCD and splice">
    Compute `gcdNode = findGCD(current!.val, next!.val)` (returns a `ListNode`). Set `current!.next = gcdNode`, `gcdNode.next = next`.
  </StepItem>
  <StepItem title="Advance past the inserted node">
    `current = current?.next?.next` — this skips the inserted GCD node and lands on the original `next`.
  </StepItem>
  <StepItem title="Return head">
    Return the original `head` (unmodified reference).
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func insertGreatestCommonDivisors(_ head: ListNode?) -> ListNode? {
    var current = head

    while current?.next != nil {
      let next = current!.next
      let gcdNode = findGCD(current!.val, next!.val)
      current!.next = gcdNode
      gcdNode.next = next
      current = current?.next?.next  // skip inserted node
    }

    return head
  }

  func findGCD(_ a: Int, _ b: Int) -> ListNode {
    var x = a, y = b
    while y != 0 {
      let temp = y
      y = x % y
      x = temp
    }
    return ListNode(x)
  }
}
```

<Callout type="green" label="Swift Notes">
  `current?.next?.next` advances past the inserted GCD node in one expression — two levels of optional chaining. `findGCD` returns a `ListNode` directly, keeping the splice logic concise. The original `head` reference is unaffected by the pointer mutations, so returning `head` is safe.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n log V)">
  n-1 adjacent pairs are processed, each requiring one GCD call. The Euclidean GCD is O(log min(a, b)) = O(log V) where V is the max node value (1000). Total: O(n log V) ≈ O(n × 10).
</Callout>

<Callout type="green" label="Space Complexity — O(n)">
  n-1 new ListNode objects are allocated, one per adjacent pair. No auxiliary data structures — the pointer manipulation is in place. Output list length is 2n-1.
</Callout>
