<SectionHeader n={1} title="Problem Overview" />

Given the `head` of a singly linked list, reverse the list, and return the reversed list's head.

**Constraints:**
- The number of nodes in the list is in the range `[0, 5000]`.
- `-5000 <= Node.val <= 5000`

<SectionHeader n={2} title="Examples" />

<ReverseLinkedListDiagram />

#### Example 1

```
Input: head = [1,2,3,4,5]
Output: [5,4,3,2,1]
```

#### Example 2

```
Input: head = [1,2]
Output: [2,1]
```

#### Example 3

```
Input: head = []
Output: []
```

<SectionHeader n={4} title="Approach" />

Maintain two pointers: `curr` (starts at `head`) and `prev` (starts as `nil`). At each step:
1. Save `temp = curr.next` before overwriting it.
2. Redirect `curr.next = prev` (reverse the pointer).
3. Advance `prev = curr`, then `curr = temp`.

When `curr` becomes `nil`, `prev` holds the new head.

<Callout type="cyan" label="Why save next first">
Once we set <code>curr.next = prev</code>, the original next node is unreachable unless we saved it in <code>temp</code>. Always capture <code>temp</code> before the redirect.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise pointers">
    `var curr = head`, `var prev: ListNode? = nil`.
  </StepItem>
  <StepItem title="Loop until curr is nil">
    While `curr != nil`: save `let temp = curr?.next`.
  </StepItem>
  <StepItem title="Redirect pointer">
    Set `curr?.next = prev` to reverse the arrow.
  </StepItem>
  <StepItem title="Advance both pointers">
    `prev = curr`, then `curr = temp`.
  </StepItem>
  <StepItem title="Return new head">
    Return `prev` — it points to the last node visited, which is the new head.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func reverseList(_ head: ListNode?) -> ListNode? {
    var curr = head
    var prev: ListNode?

    while curr != nil {
      let temp = curr?.next
      curr?.next = prev
      prev = curr
      curr = temp
    }

    return prev
  }
}
```
