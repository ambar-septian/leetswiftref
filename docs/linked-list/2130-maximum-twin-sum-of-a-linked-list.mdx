<SectionHeader n={1} title="Problem Overview" />

In a linked list of size `n`, where `n` is **even**, the `i`-th node (**0-indexed**) of the linked list is known as the **twin** of the `(n-1-i)`-th node, if `0 <= i <= (n/2) - 1`.

For example, if `n = 4`, then node 0 is the twin of node 3, and node 1 is the twin of node 2.

The **twin sum** is defined as the sum of a node and its twin.

Given the `head` of a linked list with even length, return the **maximum twin sum** of the linked list.

**Constraints:**
- The number of nodes in the list is an **even** integer in the range `[2, 10^5]`.
- `1 <= Node.val <= 10^5`

<Callout type="amber" label="Key Constraint">
  n is always even (guaranteed) — twin i pairs with n-1-i, meaning the first half pairs with the second half in reverse order. The list has at least 2 nodes, so there is always at least one twin pair.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  head = [5,4,2,1]
Output: 6
Explanation: Twins: (5,1)=6 and (4,2)=6. Max=6.
```

#### Example 2

```
Input:  head = [4,2,2,3]
Output: 7
Explanation: Twins: (4,3)=7 and (2,2)=4. Max=7.
```

#### Example 3

```
Input:  head = [1,100000]
Output: 100001
```

<Callout type="green" label="Rule of Thumb">
  When pairing front and back halves of a linked list, push the first half onto a stack and pop while walking the second half — LIFO order gives the correct reverse pairing without random access or list mutation.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Fast/slow pointers locate the midpoint in O(n). Push the first half onto a stack (LIFO reversal). Walk the second half, popping each twin value — the stack's reverse order perfectly pairs node i with node n-1-i. Track the maximum twin sum across all pairs.
</Callout>

Use fast/slow pointers to locate the midpoint. Walk from head to the node after slow, pushing values onto a stack. Then walk the second half, popping from the stack to get the twin value, computing `curr.val + popped` and tracking the maximum.

<Callout type="green" label="Stack as a reversal tool">
Pushing first-half values onto a stack and popping while walking the second half gives us twin i paired with twin n-1-i in order — no random access into the list needed.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Find midpoint with fast/slow">
    While `fast.next != nil` and `fast.next.next != nil`: advance `slow = slow.next`, `fast = fast.next.next`. After the loop, `slow` is the last node of the first half.
  </StepItem>
  <StepItem title="Push first half onto stack">
    Walk from `head` up to (but not including) `slow.next`, pushing each value.
  </StepItem>
  <StepItem title="Walk second half and compare">
    Start at `slow.next`. For each node, pop from the stack, compute the twin sum, and update the running max.
  </StepItem>
  <StepItem title="Return max">
    Return the maximum twin sum found.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func pairSum(_ head: ListNode?) -> Int {
    var slow = head
    var fast = head

    // Find midpoint
    while fast?.next != nil && fast?.next?.next != nil {
      slow = slow?.next
      fast = fast?.next?.next
    }

    // Push first half onto stack
    var curr = head
    var stack = [Int]()
    var result = 0
    var passMiddle = false

    while curr != nil {
      if !passMiddle && curr === slow?.next {
        passMiddle = true
      }
      if passMiddle {
        result = max(stack.removeLast() + curr!.val, result)
      } else {
        stack.append(curr!.val)
      }
      curr = curr?.next
    }

    return result
  }
}
```

<Callout type="green" label="Swift Notes">
  `curr === slow?.next` uses reference identity (===) to detect when curr enters the second half — safer than value equality, which fails on duplicate node values. `stack.removeLast()` is O(1) for [Int] arrays. `passMiddle` is a flag that avoids re-checking the identity condition every iteration after the midpoint is passed.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n)">
  Fast/slow midpoint traversal is O(n). The combined push-and-pop pass visits each node exactly once — O(n). Total: O(n).
</Callout>

<Callout type="green" label="Space Complexity — O(n/2)">
  The stack holds exactly n/2 values (the first half). No other auxiliary data structures. O(n/2) = O(n) in asymptotic terms, but the constant factor is half that of storing the full list.
</Callout>
