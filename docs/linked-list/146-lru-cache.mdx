<SectionHeader n={1} title="Problem Overview" />

Design a data structure that follows the constraints of a **Least Recently Used (LRU) cache**.

Implement the `LRUCache` class:

- `LRUCache(int capacity)` — initialises the LRU cache with positive size `capacity`.
- `int get(int key)` — returns the value of the `key` if it exists, otherwise returns `-1`. Getting a key counts as a use, making it the most recently used.
- `void put(int key, int value)` — updates the value of the `key` if it exists, or inserts the key-value pair. If the number of keys exceeds `capacity`, evict the **least recently used** key.

**Both `get` and `put` must run in O(1) average time complexity.**

**Constraints:**
- `1 <= capacity <= 3000`
- `0 <= key <= 10^4`
- `0 <= value <= 10^5`
- At most `2 * 10^5` calls will be made to `get` and `put`.

<Callout type="amber" label="Key Constraint">
Both get and put must run in O(1) average time — an O(n) scan on any operation is too slow for 2*10^5 calls. The LRU key to evict is the one whose most recent access (get or put) was furthest in the past.
</Callout>

#### Example 1

```
Input:
["LRUCache","put","put","get","put","get","put","get","get","get"]
[[2],       [1,1],[2,2],[1],  [3,3],[2],  [4,4],[1],  [3],  [4]]

Output: [null,null,null,1,null,-1,null,-1,3,4]

Explanation:
LRUCache(2)  — capacity=2
put(1,1)     — {1:1}
put(2,2)     — {1:1, 2:2}
get(1) → 1   — 1 is MRU now: {2:2, 1:1}
put(3,3)     — capacity exceeded, evict LRU(2): {1:1, 3:3}
get(2) → -1  — 2 was evicted
put(4,4)     — evict LRU(1): {3:3, 4:4}
get(1) → -1  — evicted
get(3) → 3
get(4) → 4
```

<Callout type="green" label="Rule of Thumb">
For any LRU cache: O(1) lookup requires a hash map; O(1) reordering and eviction require a doubly linked list. Sentinel nodes at both ends eliminate all boundary checks on insert and remove.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
A hash map gives O(1) access to any node by key. A doubly linked list maintains recency order: the LRU node is always adjacent to the left sentinel, the MRU node adjacent to the right sentinel. Two dummy sentinels eliminate all edge-case null checks — insert always wires just before right, remove always stitches prev and next.
</Callout>

Maintain a doubly linked list between two dummy sentinels: `left` (LRU end) and `right` (MRU end). Nodes adjacent to `right` are most recently used; nodes adjacent to `left` are least recently used.

- **`get`**: find node via map, remove and reinsert adjacent to `right`, return value.
- **`put`**: if key exists, update value and move to MRU. If new, create node, insert at MRU end, add to map. If over capacity, remove node adjacent to `left` (LRU) and delete from map.

<Callout type="cyan" label="Why dummy sentinels">
Sentinels eliminate edge-case checks for empty list or removing the first/last node. <code>insert</code> always places a node just before <code>right</code>; <code>remove</code> always stitches <code>prev</code> and <code>next</code> together.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise sentinels and map">
    Create `left` and `right` dummy nodes. `left.next = right; right.prev = left`. `cache = [Int: Node]()`.
  </StepItem>
  <StepItem title="insert(node) helper">
    Insert `node` just before `right`: wire `right.prev ↔ node ↔ right`.
  </StepItem>
  <StepItem title="remove(node) helper">
    Stitch `node.prev.next = node.next` and `node.next.prev = node.prev`.
  </StepItem>
  <StepItem title="get(key)">
    If key missing, return -1. Else: `remove(node); insert(node)` — move to MRU. Return `node.value`.
  </StepItem>
  <StepItem title="put(key, value)">
    If key exists: update value, remove and reinsert. If new: create node, insert, add to map. If `cache.count > capacity`: let `lru = left.next`; `cache[lru.key] = nil`; `remove(lru)`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Node {
  let key: Int
  var value: Int
  var next: Node?
  var prev: Node?
  init(key: Int, value: Int) { self.key = key; self.value = value }
}

class LRUCache {
  let capacity: Int
  let left  = Node(key: -1, value: -1)   // LRU sentinel
  let right = Node(key: -1, value: -1)   // MRU sentinel
  var cache = [Int: Node]()

  init(_ capacity: Int) {
    self.capacity = capacity
    left.next  = right
    right.prev = left
  }

  func get(_ key: Int) -> Int {
    guard let node = cache[key] else { return -1 }
    remove(node); insert(node)
    return node.value
  }

  func put(_ key: Int, _ value: Int) {
    if let node = cache[key] {
      node.value = value
      remove(node); insert(node)
    } else {
      let node = Node(key: key, value: value)
      cache[key] = node
      insert(node)
      if cache.count > capacity, let lru = left.next {
        cache[lru.key] = nil
        remove(lru)
      }
    }
  }

  private func insert(_ node: Node) {
    let prev = right.prev
    prev?.next = node;  node.prev = prev
    node.next  = right; right.prev = node
  }

  private func remove(_ node: Node) {
    node.prev?.next = node.next
    node.next?.prev = node.prev
  }
}
```

<Callout type="green" label="Swift Notes">
`Node` is a class (reference type) — `cache[key] = node` stores a reference, not a copy. Mutating `var` properties on `let` sentinel nodes is valid in Swift because `let` on a class instance means the reference is constant, not the object's state. `cache[lru.key] = nil` is the idiomatic Swift way to remove a dictionary entry.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(1)">
O(1) per operation — hash map lookup is O(1) average; doubly linked list insert and remove are O(1) because nodes have direct prev/next pointers. No traversal is ever needed.
</Callout>

<Callout type="green" label="Space Complexity — O(capacity)">
O(capacity) — the cache holds at most `capacity` nodes at any time. Each node stores a key, value, and two pointers. The hash map and doubly linked list together use O(capacity) space.
</Callout>
