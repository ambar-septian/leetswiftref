<SectionHeader n={1} title="Problem Overview" />

You are given the heads of two sorted linked lists `list1` and `list2`.

Merge the two lists into one **sorted** list. The list should be made by splicing together the nodes of the first two lists.

Return the head of the merged linked list.

**Constraints**
- The number of nodes in both lists is in the range `[0, 50]`.
- `-100 <= Node.val <= 100`
- Both `list1` and `list2` are sorted in **non-decreasing** order.

<Callout type="green" label="Core Insight">
Use a dummy head node so the first real node can be appended uniformly. At each step compare the current heads of both lists, attach the smaller one to the tail of the merged list, and advance that list's pointer. Append any remaining nodes directly once one list is exhausted.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  list1 = [1,2,4], list2 = [1,3,4]
Output: [1,1,2,3,4,4]
```

#### Example 2
```
Input:  list1 = [], list2 = []
Output: []
```

#### Example 3
```
Input:  list1 = [], list2 = [0]
Output: [0]
```

<SectionHeader n={4} title="Approach" />

Create a `dummy` sentinel node and keep a `tail` pointer at it. While both lists are non-nil, compare `list1.val` and `list2.val`. Attach the node with the smaller value to `tail.next` and advance that list's head pointer. Advance `tail` to its new last node. Once one list is exhausted, attach the remaining non-nil list directly to `tail.next`. Return `dummy.next` to skip the sentinel.

<Callout type="cyan" label="Why attach the remainder directly">
Once one list is exhausted the other is already sorted. There is no need to iterate over its remaining nodes — a single pointer assignment links the entire tail in O(1).
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Create dummy sentinel and tail pointer">
    `dummy = ListNode()`, `tail = dummy`. This avoids special-casing the first node.
  </StepItem>
  <StepItem title="Compare heads while both lists are non-nil">
    While `list1 != nil && list2 != nil`: attach the smaller-valued node to `tail.next`, advance that list's head, advance `tail`.
  </StepItem>
  <StepItem title="Attach remaining nodes">
    After the loop, exactly one list may still have nodes. Attach it directly: `tail.next = list1 ?? list2`.
  </StepItem>
  <StepItem title="Return dummy.next">
    Skip the sentinel node — `dummy.next` is the true head of the merged list.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func mergeTwoLists(_ list1: ListNode?, _ list2: ListNode?) -> ListNode? {
    var list1 = list1
    var list2 = list2
    var dummy: ListNode? = ListNode()
    var tail = dummy

    while list1 != nil && list2 != nil {
      if let l1 = list1, let l2 = list2 {

        if l1.val < l2.val {
          tail?.next = list1
          list1 = list1?.next

        } else {
          tail?.next = list2
          list2 = list2?.next
        }
        tail = tail?.next
      }
    }

    if let list1 { tail?.next = list1 }
    else if let list2 { tail?.next = list2 }

    return dummy?.next
  }
}
```
