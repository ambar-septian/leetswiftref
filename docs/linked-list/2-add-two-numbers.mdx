<SectionHeader n={1} title="Problem Overview" />

You are given two **non-empty** linked lists representing two non-negative integers. The digits are stored in **reverse order**, and each of their nodes contains a single digit. Add the two numbers and return the sum as a linked list.

You may assume the two numbers do not have leading zeros, except the number 0 itself.

**Constraints**
- The number of nodes in each list is in the range `[1, 100]`.
- `0 <= Node.val <= 9`
- It is guaranteed that the list represents a number that does not have leading zeros.

<Callout type="amber" label="Key Constraint">
Digits are stored in reverse order — the head of each list is the least significant digit. The lists can have different lengths, and a carry after the last digit must produce an extra node.
</Callout>

<Callout type="green" label="Core Insight">
Because the digits are stored in reverse order, the head of each list is the least significant digit — exactly where grade-school addition starts. Walk both lists simultaneously, summing digits and tracking the carry, then append a final carry node if needed.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1
```
Input:  l1 = [2,4,3], l2 = [5,6,4]
Output: [7,0,8]
Explanation: 342 + 465 = 807, stored reversed as [7,0,8].
```

#### Example 2
```
Input:  l1 = [0], l2 = [0]
Output: [0]
```

#### Example 3
```
Input:  l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
Output: [8,9,9,9,0,0,0,1]
Explanation: 9999999 + 9999 = 10009998, stored reversed.
```

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Because digits are in reverse order, the head of each list is the ones digit — exactly where addition starts. Simulate grade-school addition left to right: sum each pair of digits plus the running carry, output the unit digit (total % 10), propagate the carry (total / 10), and handle lists of different lengths by substituting 0 for missing digits.
</Callout>

Extract both lists into integer arrays. Walk through both arrays index by index, summing the values plus the running carry. Store `total % 10` as the current digit and set carry to `1` if `total >= 10`, else `0`. After the loop, if carry is still `1`, append it. Convert the integer array back into a linked list by mapping each value to a `ListNode` and linking them in order.

<Callout type="cyan" label="Why extract to arrays first">
Working with plain arrays simplifies index-based access and bounds checking compared to walking two list pointers simultaneously. The conversion back to a linked list is a single O(n) pass.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Extract both lists to arrays">
    Walk l1 and l2 simultaneously, appending each node's value to `l1Values` and `l2Values` until both pointers are nil.
  </StepItem>
  <StepItem title="Add digit by digit with carry">
    Loop while `index < l1Values.count || index < l2Values.count`. Use 0 for a missing digit. `total = l1[i] + l2[i] + mod`. Append `total % 10` to result.
  </StepItem>
  <StepItem title="Propagate final carry">
    If `mod != 0` after the loop, append `mod` (which is `1`) to the result array.
  </StepItem>
  <StepItem title="Convert array to linked list">
    Map each integer to a `ListNode`. Link each node to the next by setting `child.next = listNodes[index + 1]`. Return the first node.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func addTwoNumbers(_ l1: ListNode?, _ l2: ListNode?) -> ListNode? {
    guard let l1 = l1, let l2 = l2 else { return nil }
    let total = calculate(l1: l1, l2: l2)
    return Solution.convert(total)
  }

  func calculate(l1: ListNode, l2: ListNode) -> [Int] {
    var currentL1: ListNode? = l1
    var currentL2: ListNode? = l2
    var l1Values = [Int]()
    var l2Values = [Int]()

    while currentL1 != nil || currentL2 != nil {
      if let val = currentL1?.val { l1Values.append(val) }
      if let val = currentL2?.val { l2Values.append(val) }
      
      currentL1 = currentL1?.next
      currentL2 = currentL2?.next
    }

    var index = 0
    var result = [Int]()
    var mod = 0

    while index < l1Values.count || index < l2Values.count {
      let v1 = l1Values.indices.contains(index) ? l1Values[index] : 0
      let v2 = l2Values.indices.contains(index) ? l2Values[index] : 0
      let total = v1 + v2 + mod

      result.append(total % 10)
      mod = total >= 10 ? 1 : 0

      index += 1
    }

    if mod != 0 { result.append(mod) }
    return result
  }

  static func convert(_ values: [Int]) -> ListNode? {
    let listNodes = values.map(ListNode.init)

    for (index, child) in listNodes.enumerated() {
      if index + 1 != listNodes.count {
        child.next = listNodes[index + 1]
      }
    }
    return listNodes.first
  }
}
```

<Callout type="green" label="Swift Notes">
`values.map(ListNode.init)` passes the memberwise initialiser as a function reference — a clean Swift idiom that avoids a trailing closure. `indices.contains(index)` is O(1) on Array since indices is a CountableRange. Marking `convert` as `static` keeps it side-effect-free and callable without an instance.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(max(m,n))">
Both lists are extracted in one pass each, then the addition loop runs for max(m, n) iterations, then a final pass converts the result array to a linked list. All passes are O(max(m, n)) where m and n are the list lengths.
</Callout>

<Callout type="green" label="Space Complexity — O(max(m,n))">
Two arrays hold the extracted digits and the result array holds at most max(m, n) + 1 elements for the carry. Total auxiliary space is O(max(m, n)).
</Callout>
