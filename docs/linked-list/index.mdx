<SectionHeader n={1} title="What is a Linked List?" />

A linked list stores elements in nodes scattered throughout memory, connected by `next` pointers. The trade-off vs arrays:

| Operation | Array | Linked List |
|---|---|---|
| Access by index | O(1) | O(n) |
| Insert / delete at known node | O(n) shift | O(1) pointer update |
| Insert / delete at head | O(n) shift | O(1) |
| Memory layout | Contiguous | Scattered |
| Extra memory | O(1) | O(n) for pointers |

<Callout type="amber" label="Key Constraint">
  Linked list problems almost always require **in-place** pointer manipulation — no extra array is allowed. The two most useful tools are the **sentinel (dummy) head** (eliminates edge cases for head removal/insertion) and the **fast/slow pointer** (finds middle, detects cycles, removes nth-from-end in one pass).
</Callout>

<SectionHeader n={2} title="Terminology" />

| Term | Definition |
|---|---|
| **ListNode** | A node with `val: Int` and `next: ListNode?` |
| **Head** | The first node; the entry point to the list |
| **Tail** | The last node; `tail.next == nil` |
| **Sentinel / dummy head** | A fake node before the real head; simplifies insert/delete without special-casing the head |
| **Slow pointer** | Advances one step per iteration |
| **Fast pointer** | Advances two steps per iteration — reaches the end when slow is at the middle |
| **In-place** | Rearranging nodes by updating `next` pointers without allocating new nodes |

<Callout type="green" label="Rule of Thumb">
  Reach for the **sentinel head** when the problem modifies the list head (removal, insertion at front). Reach for **fast/slow pointers** when the problem asks for middle, cycle, or nth-from-end. Use **in-place reversal** when the problem asks you to reverse a sublist or alternate segments.
</Callout>
