<SectionHeader n={1} title="Problem Overview" />

Design a HashSet without using any built-in hash table libraries.

Implement `MyHashSet`:
- `MyHashSet()` — Initializes the object with an empty set.
- `void add(int key)` — Inserts the value `key` into the HashSet.
- `bool contains(int key)` — Returns whether the value `key` exists in the HashSet or not.
- `void remove(int key)` — Removes the value `key` in the HashSet. If `key` does not exist in the HashSet, do nothing.

**Constraints:**
- `0 <= key <= 10^6`
- At most `10^4` calls will be made to `add`, `remove`, and `contains`.

<Callout type="amber" label="Key Constraint">
Keys range up to 10^6. With 10,000 buckets (`key % 10_000`), the maximum chain length is 100 (10^6 / 10^4) — so every operation is effectively O(1) even in the worst case of a uniformly bad hash. The 10^4 call limit also ensures total work stays well within bounds.
</Callout>

```
[[], [1], [2], [1], [3], [2], [2], [2], [2]]

Output: [null, null, null, true, false, null, true, null, false]

Explanation:
MyHashSet myHashSet = new MyHashSet();
myHashSet.add(1);       // set = {1}
myHashSet.add(2);       // set = {1, 2}
myHashSet.contains(1);  // return true
myHashSet.contains(3);  // return false (not found)
myHashSet.add(2);       // set = {1, 2} (no duplicate)
myHashSet.contains(2);  // return true
myHashSet.remove(2);    // set = {1}
myHashSet.contains(2);  // return false
```

<Callout type="green" label="Rule of Thumb">
For a chaining hash set, always use a dummy head node per bucket — it eliminates the special case of removing the first real node. Every removal becomes identical: find the predecessor, set `predecessor.next = node.next`. The dummy is never removed.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Hash each key to a bucket index (`key % 10_000`). Each bucket is a linked list with a dummy head. `add` scans for duplicates before appending a new node; `remove` finds the predecessor and splices out the target node; `contains` scans the chain. The dummy head ensures a predecessor always exists for every real node.
</Callout>

Allocate 10,000 buckets. `hash(key) = key % 10_000`. For `add`: locate the bucket; if empty, create dummy → node(key); otherwise scan for duplicates before appending. For `remove`: scan for the predecessor of key, splice out. For `contains`: scan the chain for the key.

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise buckets">
    `array = (0..10_000).map { _ in nil }` — 10,000 optional ListNode slots.
  </StepItem>
  <StepItem title="add(key)">
    `hashKey = hash(key)`. If `array[hashKey] == nil`, create dummy and append node(key). Else scan chain; if key already exists return; else append at tail.
  </StepItem>
  <StepItem title="remove(key)">
    `hashKey = hash(key)`. If bucket is nil, return. Walk chain; when `current.next.val == key`, set `current.next = current.next.next`.
  </StepItem>
  <StepItem title="contains(key)">
    `hashKey = hash(key)`. If bucket is nil, return `false`. Walk chain; if `current.val == key`, return `true`. Return `false` at end.
  </StepItem>
  <StepItem title="Hash function">
    `private func hash(_ key: Int) -> Int { key % 10_000 }`.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class ListNode {
  var val: Int?
  var next: ListNode?
  init(_ val: Int?) { self.val = val }
}

class MyHashSet {
  var array: [ListNode?]

  init() {
    array = (0..<10_000).map { _ in nil }
  }

  func add(_ key: Int) {
    let hashKey = hash(key)
    if let listNode = array[hashKey] {
      var current: ListNode? = listNode
      while current?.next != nil {
        if current?.next?.val == key { return }
        current = current?.next
      }
      current?.next = ListNode(key)
    } else {
      let listNode = ListNode(nil)  // dummy
      listNode.next = ListNode(key)
      array[hashKey] = listNode
    }
  }

  func remove(_ key: Int) {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return }
    var current: ListNode? = listNode
    while current != nil {
      if current?.next?.val == key {
        current?.next = current?.next?.next
        break
      }
      current = current?.next
    }
    array[hashKey] = listNode
  }

  func contains(_ key: Int) -> Bool {
    let hashKey = hash(key)
    guard let listNode = array[hashKey] else { return false }
    var current: ListNode? = listNode
    while current != nil {
      if current?.val == key { return true }
      current = current?.next
    }
    return false
  }

  private func hash(_ key: Int) -> Int { key % 10_000 }
}
```

<Callout type="green" label="Swift Notes">
`ListNode` is defined at the top level (not inside `MyHashSet`) because LeetCode submission requires a clean namespace. `(0..<10_000).map { _ in nil }` creates the bucket array with 10,000 nil slots — Swift underscores in integer literals (`10_000`) are purely cosmetic. `current?.next?.val == key` uses optional chaining for safe traversal; any nil in the chain short-circuits to `false`.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(1) avg">
Each operation hashes the key in O(1), then traverses a chain of average length n/B (n = stored keys, B = 10,000 buckets). With keys uniformly distributed, the expected chain length is small and effectively constant. Worst case (all keys collide) is O(n), but the constraints cap the chain at 100.
</Callout>

<Callout type="green" label="Space Complexity — O(B + n)">
The bucket array of size B = 10,000 is allocated at init. Each stored key adds one `ListNode` to the heap, for O(n) node allocations. Total: O(B + n).
</Callout>
