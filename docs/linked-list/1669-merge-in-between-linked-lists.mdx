<SectionHeader n={1} title="Problem Overview" />

You are given two linked lists: `list1` and `list2` of sizes `n` and `m` respectively.

Remove `list1`'s nodes from the `a`-th node to the `b`-th node, and put `list2` in their place.

Build the result list and return its head.

**Constraints:**
- `3 <= list1.length <= 10^4`
- `1 <= a <= b < list1.length - 1`
- `1 <= list2.length <= 10^4`

<Callout type="amber" label="Key Constraint">
  a and b are guaranteed valid (1 &lt;= a &lt;= b &lt; list1.length - 1), so nodeBeforeA and nodeAfterB always exist within list1 — no nil guards needed for the traversal endpoints.
</Callout>

<SectionHeader n={2} title="Examples" />

#### Example 1

```
Input:  list1 = [10,1,13,6,9,5], a=3, b=4, list2 = [1000000,1000001,1000002]
Output: [10,1,13,1000000,1000001,1000002,5]
```

#### Example 2

```
Input:  list1 = [0,1,2,3,4,5,6], a=2, b=5, list2 = [1000000,1000001,1000002]
Output: [0,1,1000000,1000001,1000002,6]
```

<Callout type="green" label="Rule of Thumb">
  To splice a segment out of a linked list and insert another list, locate the node before the removal start (index a-1) and the node after the removal end (index b+1), find the insert list's tail, then make exactly two pointer assignments.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
  Reuse a single pointer: walk to index a-1 to get nodeBeforeA, continue walking from there to index b+1 to get nodeAfterB. Walk list2 to its tail. Then just two pointer assignments — nodeBeforeA.next = list2 and list2Tail.next = nodeAfterB — complete the splice.
</Callout>

Walk list1 to index `a-1` to get `nodeBeforeA`. Continue walking from there to index `b+1` to get `nodeAfterB`. Walk list2 to its tail. Connect: `nodeBeforeA.next = list2.head` and `list2Tail.next = nodeAfterB`.

<Callout type="green" label="Reuse the same pointer">
After reaching `nodeBeforeA` at index `a-1`, keep walking the same pointer `temp` forward `b - (a-1) + 1` steps to land at `nodeAfterB`. This avoids a separate second traversal from the head.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Walk to nodeBeforeA">
    Traverse list1 until `index == a - 1`. Store this node as `temp` (nodeBeforeA).
  </StepItem>
  <StepItem title="Walk to nodeAfterB">
    Continue advancing `temp` one step at a time, incrementing index, until `index > b`. Now `temp` is `nodeAfterB`.
  </StepItem>
  <StepItem title="Find list2 tail">
    Walk `temp2` from `list2` until `temp2.next == nil`.
  </StepItem>
  <StepItem title="Splice">
    `temp2.next = temp` (list2 tail points to nodeAfterB). `curr.next = list2` (nodeBeforeA points to list2 head).
  </StepItem>
  <StepItem title="Return head">
    Return the original `head` of list1.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func mergeInBetween(_ list1: ListNode?, _ a: Int, _ b: Int, _ list2: ListNode?) -> ListNode? {
    let head = list1
    var curr = list1
    var index = 0
    var nodeBeforeA: ListNode?

    // Walk to index a-1
    while curr?.next != nil {
      if index == a - 1 {
        nodeBeforeA = curr
        break
      }
      curr = curr?.next
      index += 1
    }

    // Walk to nodeAfterB (index b+1)
    var nodeAfterB = nodeBeforeA
    while index <= b {
      nodeAfterB = nodeAfterB?.next
      index += 1
    }

    // Find list2 tail
    var list2Tail = list2
    while list2Tail?.next != nil {
      list2Tail = list2Tail?.next
    }

    // Splice
    list2Tail?.next = nodeAfterB
    curr?.next = list2  // curr is still nodeBeforeA

    return head
  }
}
```

<Callout type="green" label="Swift Notes">
  Optional chaining `curr?.next` is used throughout since ListNode parameters arrive as ListNode?. `nodeBeforeA = curr` captures the reference before advancing — Swift's reference semantics ensure this points to the actual node. The second walk reuses `nodeAfterB = nodeBeforeA` and advances it, avoiding a separate second traversal from the head.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(n + m)">
  Walking list1 to index b+1 is O(b+1) ≤ O(n). Walking list2 to its tail is O(m). Two constant-time pointer assignments complete the splice. Total: O(n + m).
</Callout>

<Callout type="green" label="Space Complexity — O(1)">
  Only a handful of pointer variables are used — head, curr, nodeBeforeA, nodeAfterB, list2Tail. No extra allocation. O(1) auxiliary space.
</Callout>
