<SectionHeader n={1} title="Problem Overview" />

Given the heads of two singly linked lists `headA` and `headB`, return the node at which the two lists intersect. If the two linked lists have no intersection at all, return `null`.

The intersection is by **reference**, not by value — it is the node object (same memory address) that both lists share, not a node with the same value.

**Note:** The linked lists must **retain their original structure** after the function returns.

**Constraints:**
- The number of nodes of `listA` is `m`.
- The number of nodes of `listB` is `n`.
- `1 <= m, n <= 3 * 10^4`
- `1 <= Node.val <= 10^5`
- `0 <= skipA, skipB <= 3 * 10^4` (offset to intersection node)
- **Follow-up:** Can you solve it in O(n) time and use only O(1) memory?

<Callout type="amber" label="Key Constraint">
Intersection is by reference (same node object), not by value. The original list structures must not be modified after the function returns. If no intersection exists, return null — not a node with a matching value.
</Callout>

<Callout type="green" label="Rule of Thumb">
To find the intersection of two lists in O(1) space: redirect each pointer to the other list's head after reaching nil. Both pointers travel the same total distance and meet at the intersection — or both reach nil simultaneously if no intersection exists.
</Callout>

<SectionHeader n={4} title="Approach" />

<Callout type="cyan" label="Core Insight">
Each pointer walks its own list, then is redirected to the other list's head when it reaches nil. After the redirect, both pointers have walked the same total distance (|A| + |B|). They arrive at the intersection node simultaneously — or both reach nil simultaneously if no intersection exists.
</Callout>

Advance `tailA` and `tailB` simultaneously. When `tailA` reaches nil (and has not yet redirected), set `tailA = headB`. When `tailB` reaches nil, set `tailB = headA`. After the redirect, both pointers have effectively walked `|A| + |B|` total steps — they are synchronised. If an intersection exists, they land on the same node. If not, both land on nil simultaneously.

<Callout type="cyan" label="Why the paths equalise">
Pointer A travels |A-only| + |shared| steps, hits nil, then walks |B-only| + |shared| more = total |A| + |B|. Pointer B travels the same total in the opposite order. Both arrive at the intersection node — or nil — at the same step.
</Callout>

<SectionHeader n={6} title="Step-by-Step Solution" />

<StepList>
  <StepItem title="Initialise two pointers">
    `var tailA = headA`, `var tailB = headB`, `var hasLoopedToB = false`, `var hasLoopedToA = false`.
  </StepItem>
  <StepItem title="Loop while either pointer is non-nil">
    Check `tailA === tailB` — if true, return `tailA`.
  </StepItem>
  <StepItem title="Advance pointers">
    `tailA = tailA?.next`, `tailB = tailB?.next`.
  </StepItem>
  <StepItem title="Redirect on nil (once each)">
    If `tailA == nil` and not yet redirected: `tailA = headB; hasLoopedToB = true`. Same symmetrically for `tailB`.
  </StepItem>
  <StepItem title="No intersection">
    When both pointers are nil simultaneously, return nil.
  </StepItem>
</StepList>

<SectionHeader n={7} title="Code Implementation" />

```swift
class Solution {
  func getIntersectionNode(_ headA: ListNode?, _ headB: ListNode?) -> ListNode? {
    var hasLoopToB = false
    var hasLoopToA = false
    var tailA = headA
    var tailB = headB

    while tailA != nil || tailB != nil {
      if tailA === tailB { return tailA }

      tailA = tailA?.next
      tailB = tailB?.next

      if tailA == nil, !hasLoopToB {
        tailA = headB
        hasLoopToB = true
      }
      if tailB == nil, !hasLoopToA {
        tailB = headA
        hasLoopToA = true
      }
    }

    return nil
  }
}
```

<Callout type="green" label="Swift Notes">
`tailA === tailB` uses reference equality (===) to compare node identity — value equality (==) would produce false positives for nodes with the same val at different positions. `tailA?.next` uses optional chaining safely. The boolean flags `hasLoopToB` and `hasLoopToA` guard against infinite redirects when no intersection exists.
</Callout>

<SectionHeader n={8} title="Complexity Analysis" />

<Callout type="cyan" label="Time Complexity — O(m+n)">
O(m+n) — each pointer walks its own list (m or n steps) and then at most the other list (n or m steps). In total each pointer travels at most m+n steps before both either meet at the intersection or simultaneously reach nil.
</Callout>

<Callout type="green" label="Space Complexity — O(1) / O(m)">
O(1) for the two-pointer redirect — only four pointer variables and two boolean flags. O(m) for the hash set approach, which stores a reference for every node in list A.
</Callout>

<Callout type="amber" label="When to prefer two-pointer vs hash set">
Use the **two-pointer redirect** when O(1) space is required — it is the follow-up solution. Use the **hash set** when simplicity matters: store all of list A's nodes, then scan list B for the first node already in the set. The hash set is easier to reason about and has no boolean guard flags.
</Callout>
