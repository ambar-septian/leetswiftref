Graph

A graph is a collection of nodes (vertices) connected by edges. Unlike trees, graphs can have cycles, multiple paths between nodes, and no single root. Graph algorithms power everything from social networks and maps to dependency resolution and circuit analysis.

DFSBFSUnion-FindTopological SortDijkstraDAG
01

What is a Graph?

A graph G = (V, E) consists of a set of vertices V and a set of edges E connecting pairs of vertices.

Key properties to identify:

PropertyOptionsImpact
DirectionDirected / UndirectedDo edges go one way or both?
WeightWeighted / UnweightedAre edges free or have cost?
CyclesCyclic / AcyclicCan you revisit a node?
ConnectivityConnected / DisconnectedCan you reach every node?
Key Constraint

Always clarify whether the graph is directed and whether it can have cycles before choosing an algorithm. These two properties alone determine which of DFS, BFS, Union-Find, Topological Sort, or Dijkstra applies.

Representation in code:

The adjacency list is almost always preferred over the matrix for sparse graphs (most LeetCode problems), because it uses O(V + E) space instead of O(V²) and iterates only over real edges.

// Adjacency list — the standard for most graph problems
var adj = [[Int]](repeating: [], count: n)
for edge in edges {
    adj[edge[0]].append(edge[1])
    adj[edge[1]].append(edge[0]) // undirected
}

03

Algorithm Variants

Graph problems map to one of four core algorithms. Pick the right one based on what the problem asks:

DFS and BFS
Traverse or search a graph. DFS explores depth-first using a stack (or recursion); BFS explores level-by-level using a queue. Use for connectivity, path finding, flood fill, and cycle detection.
O(V+E)O(V)
→
Union-Find
Efficiently track which nodes belong to the same connected component. Supports union and find operations with near-constant time using path compression and union by rank.
O(a(N))O(N)
→
Topological Sort
Order nodes in a DAG such that every directed edge u to v has u before v. Use Kahn's algorithm (BFS + in-degree) for cycle detection, or DFS-based for the ordering.
O(V+E)O(V)
→
Dijkstra
Find the shortest path from a single source to all other nodes in a weighted graph with non-negative edge weights. Uses a min-heap priority queue for greedy relaxation.
O((V+E) log V)O(V)
→
Rule of Thumb

Unweighted reachability/connectivity: DFS or BFS. Component grouping: Union-Find. Dependency ordering: Topological Sort. Shortest weighted path: Dijkstra.
