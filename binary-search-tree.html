Binary Search Tree

A binary search tree (BST) is a binary tree with the ordering invariant: every node in the left subtree is strictly less than the root, and every node in the right subtree is strictly greater. This invariant enables O(h) search, insert, and delete, and guarantees that inorder traversal produces a sorted sequence.

01

What is a BST?

The BST invariant applies recursively to every subtree — not just the direct children:

PropertyRule
Left subtreeAll values strictly less than the root
Right subtreeAll values strictly greater than the root
Inorder traversalProduces values in strictly ascending sorted order
SearchGo left if target < current, go right if target > current — O(h)
HeightO(log n) for balanced BST, O(n) for degenerate (sorted input)
Key Constraint

The BST invariant is a global constraint, not just a local parent-child constraint. A common mistake is checking only that node.left.val &lt; node.val and node.right.val &gt; node.val. This misses cases like a left child having a right subtree with values that are greater than the root. Correct validation passes a min and max bound down the recursion.

Rule of Thumb

If the problem mentions "BST" or "sorted binary tree" and asks about a specific value (search, insert, delete, kth element) — the BST ordering property guides you left or right at each node, giving O(h) time. If the problem asks to validate or reconstruct, use the min/max bound pattern (validate) or the mid-point recursion (sorted array to BST).
