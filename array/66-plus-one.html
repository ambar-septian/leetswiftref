#66EasyArray TraversalLeetCode ↗

Plus One

ArrayMath
01

Problem Overview

You are given a large integer represented as an integer array digits, where each digits[i] is the i-th digit of the integer. The digits are ordered from most significant to least significant in left-to-right order. The large integer does not contain any leading zeros.

Increment the large integer by one and return the resulting array of digits.

Constraints

  • 1 <= digits.length <= 100
  • 0 <= digits[i] <= 9
  • digits does not contain any leading zeros.
Core Insight

Walk backward through the digits. If the last digit is not 9, simply increment it and stop. If it is 9, set it to 0 and carry — keep going left while digits are 9. If all digits were 9, every digit is now 0 and we need to insert a leading 1.

Does carry propagate from the least significant digit?
✓ Yes — Adding 1 to a 9 produces a 0 with carry-1; that carry may cascade left through consecutive 9s.
?
Can we stop early?
✓ Yes — As soon as we encounter a digit less than 9 we increment it and return — no carry to propagate.
?
Is a new array length ever needed?
✓ Yes — Only when all digits are 9 (e.g., [9,9,9] → [1,0,0,0]). This is the only edge case requiring an extra digit.

04

Approach

Iterate from the last index down to 0. If digits[last] != 9, increment it and immediately return. If it is 9, set it to 0 and continue left (carry propagates). After the loop exits — which means we fell off the left end, i.e., all digits were 9 — every digit is now 0, so insert a 1 at index 0.

Why insert(1, at: 0) is only reached in the all-nines case

The loop only finishes without returning when every iteration sets a digit to 0 (all were 9). In that case digits is now all zeros and we need a new most-significant digit of 1. For any other input the early return fires before the loop ends.

06

Step-by-Step Solution

Copy the array (Swift arrays are value types — mutate in place) —

var digits = digits. Initialise last = digits.count - 1.

Walk backward —

While last >= 0: if digits[last] != 9, increment and break. Else set digits[last] = 0 and last -= 1.

Handle all-nines edge case —

If last < 0 after the loop, insert 1 at index 0: digits.insert(1, at: 0).

Return —

Return digits.

07

Code Implementation

class Solution {
  func plusOne(_ digits: [Int]) -> [Int] {
    var digits = digits
    var last = digits.count - 1

    while last >= 0 {
      if digits[last] != 9 {
        digits[last] += 1
        break
      }
      digits[last] = 0
      last -= 1

      if last < 0 {
        digits.insert(1, at: 0)
      }
    }

    return digits
  }
}

09

Common Pitfalls

✕01
Converting to Int and adding 1
digits can represent numbers with up to 100 digits — far beyond Int64 range. Converting to a numeric type will overflow.
Fix: Operate digit-by-digit with carry propagation. Never convert the entire array to an integer.
✕02
Forgetting the all-nines edge case
If every digit is 9 the result needs one extra digit (e.g., [9,9] → [1,0,0]). A simple digits[last] += 1 at the end misses this.
Fix: After the backward loop, check if last < 0 and insert a leading 1 if so.
✕03
Using a modular arithmetic approach without break
Some implementations compute (digit+1) % 10 and continue the loop even when no carry is needed, wasting iterations.
Fix: Break immediately as soon as you increment a non-9 digit — no further digits need updating.
