Binary Tree Inorder Traversal

StackTreeDepth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, return the inorder traversal of its nodes' values.

Inorder traversal visits nodes in left → root → right order. For a valid Binary Search Tree this produces a sorted sequence. The traversal is defined recursively: visit the entire left subtree, then the current node, then the entire right subtree.

Constraints

  • The number of nodes in the tree is in the range [0, 100].
  • -100 <= Node.val <= 100
Core Insight

Recursively concatenate the inorder traversal of the left subtree, the root value, and the inorder traversal of the right subtree. The base case is a nil node returning an empty array.

Approach

The recursive solution is a direct translation of the definition: inorderTraversal(root) = inorderTraversal(root.left) + [root.val] + inorderTraversal(root.right). Guard against nil with an early return of [].

Iterative alternative

An iterative solution uses an explicit stack: push nodes left as far as possible, then pop, record the value, and shift to the right child. This runs in the same O(n) time but O(h) space (h = tree height) without the function-call overhead.

Step-by-Step Solution

Guard nil —

guard let root else { return [] } — nil node contributes nothing.

Recurse left —

inorderTraversal(root.left) — collect all values from the left subtree in order.

Add root —

Append [root.val] — the current node is visited after its entire left subtree.

Recurse right —

inorderTraversal(root.right) — collect all right subtree values.

07

Code Implementation

class Solution {
  func inorderTraversal(_ root: TreeNode?) -> [Int] {
    guard let root else { return [] }
    return inorderTraversal(root.left) + [root.val] + inorderTraversal(root.right)
  }
}

09

Common Pitfalls

✕01
Visiting root before left subtree (preorder mistake)
Returning [root.val] + left + right visits the root first — that is preorder, not inorder.
Fix: Order must be left + [root.val] + right for inorder.
✕02
Forgetting the nil guard produces a crash
Without the guard, accessing root.val on a nil TreeNode will crash at runtime.
Fix: Always guard at the top: guard let root else { return [] }.
✕03
Array concatenation in Swift creates new arrays at each call level
The + operator creates a new array. For very deep trees this is O(n) per level, making the total O(n log n) or O(n²) in pathological cases.
Fix: For large inputs, prefer an iterative solution with a pre-allocated result array and inout append.
