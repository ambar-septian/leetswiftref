Leaf-Similar Trees

TreeDepth-First SearchBinary Tree
01

Problem Overview

Consider all the leaves of a binary tree, from left to right order. The values of those leaves form a leaf value sequence.

Two binary trees are considered leaf-similar if their leaf value sequence is the same.

Return true if and only if the two given trees with head nodes root1 and root2 are leaf-similar.

Constraints:

  • The number of nodes in each tree will be in the range [1, 200].
  • Both of the given trees will have values in the range [0, 200].
02

Examples

Example 1​

Input:  root1 = [3,5,1,6,2,9,8,null,null,7,4], root2 = [3,5,1,6,7,4,2,null,null,null,null,null,null,9,8]
Output: true

Example 2​

Input:  root1 = [1,2,3], root2 = [1,3,2]
Output: false

Pattern Recognition

→ Pattern Decision Tree
?
Do we need leaf values in left-to-right order?
✓ Yes — DFS visiting left before right naturally produces leaves in left-to-right order — just collect them into an array.
?
Is a node a leaf when both children are nil?
✓ Yes — A leaf has no left or right child. When both are nil, append the node's value to the sequence.
?
Can we compare sequences after two separate DFS calls?
✓ Yes — Run dfs(root1) to get sequence 1, dfs(root2) to get sequence 2, then compare for equality.

04

Approach

Define dfs(_ root: TreeNode?) -> [Int]. If root is nil, return []. If both children are nil, return [root.val]. Otherwise return dfs(root.left) + dfs(root.right). Call dfs(root1) and dfs(root2) and compare the results.

Two identical trees [3,5,1,6,2]  |  DFS collects leaves left-to-right
Step 1 / 7
Run DFS on both trees to collect leaf sequences.
Leaf detection

A node is a leaf exactly when root.left == nil AND root.right == nil. A node with one child is not a leaf — it is an internal node with one subtree.

06

Step-by-Step Solution

Define DFS to collect leaves —

func dfs(_ root: TreeNode?) -> [Int]. Base: guard let root else { return [] }.

Leaf base case —

If root.left == nil && root.right == nil, return [root.val].

Internal node —

Return dfs(root.left) + dfs(root.right).

Compare —

Return dfs(root1) == dfs(root2).

07

Code Implementation

class Solution {
  func leafSimilar(_ root1: TreeNode?, _ root2: TreeNode?) -> Bool {
    func dfs(_ root: TreeNode?) -> [Int] {
      guard let root else { return [] }
      if root.left == nil, root.right == nil {
        return [root.val]
      }
      return dfs(root.left) + dfs(root.right)
    }

    return dfs(root1) == dfs(root2)
  }
}

09

Common Pitfalls

✕01
Treating a single-child node as a leaf
A node with one child is not a leaf — only nodes with both children nil are leaves.
Fix: Check root.left == nil AND root.right == nil (both conditions) before treating a node as a leaf.
✕02
Comparing trees structurally instead of by leaf sequences
Two trees with the same shape but different internal values can still have the same leaf sequence — structural comparison is stricter than the problem requires.
Fix: Only collect and compare the leaf arrays, not the full tree structure.
✕03
Collecting leaves in wrong order
Visiting right before left reverses the leaf sequence, producing an incorrect comparison.
Fix: Always recurse left before right: dfs(root.left) + dfs(root.right).
