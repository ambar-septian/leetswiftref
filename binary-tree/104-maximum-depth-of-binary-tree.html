Maximum Depth of Binary Tree

#104EasyDFSLeetCode ↗

Maximum Depth of Binary Tree

TreeDepth-First SearchBreadth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, return its maximum depth.

A binary tree's maximum depth is the number of nodes along the longest path from the root node down to the farthest leaf node.

Constraints

  • The number of nodes in the tree is in the range [0, 10^4].
  • -100 <= Node.val <= 100
Key Constraint

The tree can be empty (root = nil) — depth is 0. Maximum depth counts nodes along the path, not edges: a single-node tree has depth 1. The definition matches height of the root, not depth from a specific node.

Core Insight

The maximum depth is 1 (for the current node) plus the greater of the depths of its left and right subtrees. Base case: a nil node has depth 0.

02

Examples

Example 1​

Input:  root = [3,9,20,null,null,15,7]
Output: 3

Example 2​

Input:  root = [1,null,2]
Output: 2

Rule of Thumb

Tree depth or height problems almost always reduce to 1 + max(left, right) — a post-order DFS that bubbles up the answer from leaves to root. When you see "longest path from root to leaf", reach for this pattern.

04

Approach

Core Insight

Depth of a node = 1 + max(depth of left subtree, depth of right subtree). This post-order recursion computes depths bottom-up — leaves return 1, nil returns 0. The BFS alternative counts completed levels and is equally valid.

Recursive DFS dfs(_ node: TreeNode?) -> Int. If node is nil return 0. Recursively compute left = dfs(node.left) and right = dfs(node.right). Return 1 + max(left, right).

Post-order DFS · each node returns 1 + max(left, right) · answer = 3
root — max depth returned
subtree height
BFS alternative

Level-order BFS also works: count how many levels are processed. Each completed level increments a depth counter. This is O(n) time and O(n) space (queue can hold a full level).

06

Step-by-Step Solution

Base case —

If node is nil, return 0.

Recurse both subtrees —

let left = dfs(node.left), let right = dfs(node.right).

Return combined depth —

Return 1 + max(left, right).

07

Code Implementation

class Solution {
  func maxDepth(_ root: TreeNode?) -> Int {
    guard let root else { return 0 }
    return dfs(root)

    func dfs(_ node: TreeNode?) -> Int {
      guard let node else { return 0 }
      let left = dfs(node.left)
      let right = dfs(node.right)
      return 1 + max(left, right)
    }
  }
}
Swift Notes

The nested dfs function is defined after the return dfs(root) call — Swift allows using a nested function before its declaration within the same scope. guard let root else { return 0 } at the outer function handles the empty-tree case before the nested dfs is ever called.

08

Complexity Analysis

Time Complexity — O(n)

O(n) — every node is visited exactly once. Both the recursive DFS and BFS alternative are O(n) since no node can be skipped when computing the true maximum depth.

Space Complexity — O(h)

O(h) for the recursive call stack, where h is the tree height. O(log n) for a balanced tree and O(n) for a skewed tree. The BFS alternative uses O(n) space for the queue (widest level can have n/2 nodes).

09

Common Pitfalls

✕01
Returning 1 instead of 0 for nil nodes
If nil returns 1, every non-existent child adds 1 to the depth, making the result double the correct answer.
Fix: Nil nodes have no depth — return 0 from the base case.
✕02
Using + instead of max
depth(node) = 1 + left + right would sum both subtree depths instead of taking the longer path.
Fix: Use 1 + max(left, right) to take the deeper of the two subtrees.
✕03
Confusing depth with height
Depth of a node = distance from root. Height of a node = longest path to a leaf. maxDepth asks for the tree's height (longest root-to-leaf path), which equals the height of the root.
Fix: The recursive formula 1 + max(left, right) computes height, which is what the problem calls max depth. The naming is consistent with LeetCode's definition.
