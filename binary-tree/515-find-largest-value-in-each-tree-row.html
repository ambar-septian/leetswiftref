Find Largest Value in Each Tree Row

#515MediumBFS Level-Order TraversalLeetCode ↗

Find Largest Value in Each Tree Row

TreeDepth-First SearchBreadth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, return an array of the largest value in each row of the tree (0-indexed).

Constraints:

  • The number of nodes in the tree will be in the range [0, 10^4].
  • -2^31 <= Node.val <= 2^31 - 1
02

Examples

Example 1​

Input:  root = [1,3,2,5,3,null,9]
Output: [1,3,9]
Explanation:
  Level 0: [1] → max = 1
  Level 1: [3, 2] → max = 3
  Level 2: [5, 3, 9] → max = 9

Example 2​

Input:  root = [1,2,3]
Output: [1,3]
03

Pattern Recognition

→ Pattern Decision Tree
?
Do we need to process all nodes at the same depth before moving deeper?
✓ Yes — We need the maximum per level — BFS naturally groups nodes by level.
?
Can we track max while processing each level's nodes?
✓ Yes — Initialise maxValue = Int.min before each level's inner loop; update as each node is dequeued.
?
Does DFS work here too?
✓ Yes — Pass the level as a parameter and update results[level] = max(results[level], node.val). Both DFS and BFS are O(n).

04

Approach

Use a queue initialised with root. Each outer loop iteration processes one complete level: record count = queue.count, then dequeue exactly count nodes, tracking maxValue. After the inner loop, append maxValue to results. Enqueue left and right children as they are encountered.

Tree: [1, 3, 2, 5, 3, null, 9]  |  Find max value in each level
Level 0 — 1 node
Values: [1]
Max: 1
Result so far: [1]
Initialise maxValue to Int.min

Node values can be negative — the constraint allows values down to -2^31. Initialising maxValue to 0 would produce wrong results for an all-negative level. Use Int.min as the sentinel.

06

Step-by-Step Solution

Guard empty tree —

guard let root else { return [] }. Initialise var queue = [root].

Outer loop — one iteration per level —

While queue is not empty: record let count = queue.count, var maxValue = Int.min.

Inner loop — process this level —

Repeat count times: let item = queue.removeFirst(). Update maxValue = max(maxValue, item.val). Append left and right children if they exist.

Record level result —

After the inner loop, results.append(maxValue).

07

Code Implementation

class Solution {
  func largestValues(_ root: TreeNode?) -> [Int] {
    guard let root else { return [] }

    var queue = [root]
    var results = [Int]()

    while !queue.isEmpty {
      let count = queue.count
      var maxValue = Int.min

      for _ in 0..<count {
        let item = queue.removeFirst()
        maxValue = max(maxValue, item.val)

        if let left = item.left  { queue.append(left)  }
        if let right = item.right { queue.append(right) }
      }

      results.append(maxValue)
    }

    return results
  }
}

09

Common Pitfalls

✕01
Initialising maxValue to 0 instead of Int.min
If all nodes in a level are negative, maxValue stays 0 and the wrong value is appended to results.
Fix: Always initialise maxValue = Int.min at the start of each level's inner loop.
✕02
Not snapshotting the queue count before the inner loop
If count is not captured before the loop, newly enqueued children get processed in the same level as their parents.
Fix: Capture let count = queue.count before the inner for loop — process exactly that many nodes.
✕03
Using Swift's Array.removeFirst() without considering performance
Array.removeFirst() is O(n) due to element shifting. For large trees this makes BFS O(n²).
Fix: For correctness this is fine given the constraint (n up to 10^4). For production code, use an index-based queue or a deque structure.
