Invert Binary Tree

TreeDepth-First SearchBreadth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, invert the tree (mirror it), and return its root. Inverting means every node's left and right children are swapped at every level of the tree.

Constraints:

  • The number of nodes in the tree is in the range [0, 100].
  • -100 <= Node.val <= 100

Pattern Recognition

→ Pattern Decision Tree
?
Should we invert subtrees before or after swapping children?
✓ Yes — Post-order: invert left and right subtrees first, then swap node.left and node.right. The result is identical either way, but post-order makes the logic clearer.
?
Is a single pass enough?
✓ Yes — Every node is visited exactly once. The swap at each node is O(1).
?
Can BFS also work?
✓ Yes — Level-order BFS can swap children as nodes are dequeued — also O(n) time and O(n) space for the queue.

Recurse into the left subtree and right subtree. Once both return their inverted roots, assign node.left = invertedRight and node.right = invertedLeft. Return the node. The base case returns nil for a nil node.

Post-order vs pre-order

Swapping children before recursing (pre-order) also works — the swap just happens in a different relative order. Post-order is more common in explanations because it mirrors the idea of "invert children first, then attach them."

Step-by-Step Solution

Base case —

If node is nil, return nil.

Recurse into both subtrees —

let left = invert(node.left), let right = invert(node.right).

Swap children —

node.left = right, node.right = left.

Return node —

Return the current node (its children are now swapped).

07

Code Implementation

class Solution {
  func invertTree(_ root: TreeNode?) -> TreeNode? {
    return invert(root)

    func invert(_ node: TreeNode?) -> TreeNode? {
      guard let node else { return nil }
      let left  = invert(node.left)
      let right = invert(node.right)
      node.left  = right
      node.right = left
      return node
    }
  }
}

09

Common Pitfalls

✕01
Swapping node references before recursing without saving them
If you do node.left = node.right; node.right = node.left without a temp variable, the second line reassigns node.right to the already-swapped node.left.
Fix: Save both subtree results: let left = invert(node.left); let right = invert(node.right), then assign node.left = right; node.right = left.
✕02
Not returning the node after swapping
The function must return the modified node so callers can use it as a child. Returning nil silently disconnects subtrees.
Fix: Always return node at the end of the invert function.
✕03
Inverting only one level
Swapping just the root's children leaves all deeper nodes uninverted.
Fix: The recursion handles all levels automatically — invert must be called on both children before swapping at the current node.
