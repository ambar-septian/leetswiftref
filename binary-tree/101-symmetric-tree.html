Symmetric Tree

#101EasyBFSLeetCode ↗

Symmetric Tree

TreeDepth-First SearchBreadth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, check whether it is a mirror of itself (i.e., symmetric around its center).

A tree is symmetric if the left subtree is a mirror reflection of the right subtree. Two trees are mirrors of each other if:

  • Their roots have the same value.
  • The right subtree of each tree is a mirror of the left subtree of the other tree.

Constraints

  • The number of nodes in the tree is in the range [1, 1000].
  • -100 <= Node.val <= 100
Core Insight

Use BFS with a queue that stores mirror pairs. Start with (root.left, root.right). At each step dequeue two nodes (l1, l2) that should mirror each other. Verify their values match, then enqueue the next mirror pairs: (l1.left, l2.right) and (l1.right, l2.left).

Examples

Output: false

✓ Yes — Symmetry means l1.left mirrors l2.right and l1.right mirrors l2.left at every level.

✓ Yes — Queue pairs of mirrored nodes — dequeue two at a time, validate, enqueue their children as new mirror pairs.

✓ Yes — Both nil → symmetric (continue). One nil, one non-nil → asymmetric (return false).

04

Approach

Initialise the queue with [root.left, root.right]. While the queue is not empty: dequeue l1 and l2 (always in pairs). If both nil, continue. If one is nil or values differ, return false. Enqueue l1.left, l2.right and l1.right, l2.left — the next level's mirror pairs.

BFS mirror pairs — l1.left paired with l2.right, l1.right with l2.left
mirror pair — level 1
mirror pairs — level 2
Why enqueue l1.left with l2.right (not l2.left)

For the tree to be symmetric, l1 and l2 are reflections of each other across the vertical axis. l1's left child should match l2's right child (they are on opposite sides of the axis at the next level). Enqueueing l1.left with l2.left would check the wrong pairing.

Step-by-Step Solution

Initialise queue with root's children —

var queue = [root?.left, root?.right].

Dequeue mirror pairs —

While queue not empty: let l1 = queue.removeFirst(), let l2 = queue.removeFirst().

Validate pair —

Both nil → continue. One nil → return false. Values differ → return false.

Enqueue children as mirror pairs —

queue.append(contentsOf: [l1.left, l2.right, l1.right, l2.left]).

07

Code Implementation

class Solution {
  func isSymmetric(_ root: TreeNode?) -> Bool {
    var queue = [root?.left, root?.right]

    while !queue.isEmpty {
      let l1 = queue.removeFirst()
      let l2 = queue.removeFirst()

      if l1 == nil && l2 == nil { continue }
      if l1?.val != l2?.val { return false }

      queue.append(contentsOf: [l1?.left, l2?.right, l1?.right, l2?.left])
    }

    return true
  }
}

09

Common Pitfalls

✕01
Enqueuing l1.left with l2.left instead of l2.right
Checking l1.left against l2.left compares the same side of the tree, not mirror sides. This incorrectly validates asymmetric trees.
Fix: Enqueue (l1.left, l2.right) and (l1.right, l2.left) — always pair nodes that should be reflections of each other.
✕02
Not handling both-nil early with continue
When both l1 and l2 are nil, enqueueing their children (nil.left etc.) will crash. The both-nil case must be detected and skipped before the children enqueue step.
Fix: Add if l1 == nil && l2 == nil { continue } before any child access.
✕03
Checking values before nil
l1?.val != l2?.val is true when one is nil and the other is not (Optional(val) != nil). This works in Swift but relying on it silently can be confusing. It is cleaner to handle one-nil explicitly.
Fix: Rely on optional comparison for the one-nil case: l1?.val != l2?.val covers it. Just ensure the both-nil continue fires first.
