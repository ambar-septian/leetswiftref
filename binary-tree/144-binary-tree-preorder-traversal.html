Binary Tree Preorder Traversal

StackTreeDepth-First SearchBinary Tree
01

Problem Overview

Given the root of a binary tree, return the preorder traversal of its node values.

Preorder traversal visits nodes in the order: root → left subtree → right subtree.

Constraints:

  • The number of nodes in the tree is in the range [0, 100].
  • -100 <= Node.val <= 100
  • Follow-up: Recursive solution is trivial. Can you do it iteratively?
02

Examples

Example 1​

Input: root = [1,null,2,3]
Output: [1,2,3]

Example 2​

Input: root = []
Output: []

Example 3​

Input: root = [1]
Output: [1]

04

Approach

Preorder: root → left → right  ·  result = [1, 2, 3]
Call stack:
preorder(1)visit 1 → [1]
preorder(nil)left nil → []
preorder(2)visit 2 → [1,2]
preorder(3)visit 3 → [1,2,3]

The recursive solution directly encodes the preorder definition: if the root is non-nil, return [root.val] concatenated with the preorder result of the left subtree followed by the right subtree. The base case (nil root) returns an empty array.

Concatenation vs mutation

Swift's array concatenation with + creates a new array. For very deep trees this allocates O(n) intermediate arrays. An iterative approach using an explicit stack avoids this, but the recursive version is idiomatic and clear.

Step-by-Step Solution

Guard nil root —

guard let root else { return [] } — returns empty array for nil.

Build preorder array —

[root.val] + preorderTraversal(root.left) + preorderTraversal(root.right).

Return result —

The concatenated array is the preorder traversal.

07

Code Implementation

class Solution {
  func preorderTraversal(_ root: TreeNode?) -> [Int] {
    guard let root else { return [] }
    return [root.val]
      + preorderTraversal(root.left)
      + preorderTraversal(root.right)
  }
}

09

Common Pitfalls

✕01
Confusing preorder with inorder
Inorder is left → root → right; preorder is root → left → right. A common mistake is writing the left recursive call before appending root.val.
Fix: Put [root.val] first in the concatenation, before the left subtree result.
✕02
Iterative approach: pushing left before right onto the stack
A stack is LIFO. To process the left subtree first, the right child must be pushed first so the left child sits on top.
Fix: In the iterative version, push root.right first, then root.left, so left is popped (processed) first.
✕03
Not handling the nil root case
Accessing root.val on a nil root causes a crash.
Fix: Guard against nil at the start of the function and return an empty array.
