Count Good Nodes in Binary Tree

#1448MediumDFS — Path MaximumLeetCode ↗

Count Good Nodes in Binary Tree

TreeDepth-First SearchBreadth-First SearchBinary Tree
01

Problem Overview

Given a binary tree root, a node X in the tree is named good if in the path from root to X there are no nodes with a value greater than X's value.

Return the number of good nodes in the binary tree.

Constraints:

  • The number of nodes in the binary tree is in the range [1, 10^5].
  • Each node's value is between -10^4 and 10^4.
02

Examples

Example 1​

Input:  root = [3,1,4,3,null,1,5]
Output: 4
Explanation: Good nodes are marked with (G):
  [3(G)] → [1] → [3(G)]
  [3(G)] → [4(G)] → [5(G)]
  Node 1 (left child of root) is NOT good since 3 > 1.
  Node 1 (left child of 4) is NOT good since 4 > 1.

Example 2​

Input:  root = [3,3,null,4,2]
Output: 3

Example 3​

Input:  root = [1]
Output: 1

04

Approach

Perform a DFS from the root, passing the maximum value seen so far on the current root-to-node path. At each node: if node.val >= maxSoFar, it is a good node (count 1); otherwise count 0. Update maxSoFar = max(node.val, maxSoFar) before recursing into children.

Start with -infinity

Initialise maxSoFar with Int.min. This ensures the root is always counted as good, since any integer is >= Int.min.

06

Step-by-Step Solution

Define recursive DFS —

func dfs(_ node: TreeNode?, _ prev: Int) -> Int — prev is the max value seen from root to the parent of this node.

Base case —

If node is nil, return 0.

Check if current node is good —

let current = node.val >= prev ? 1 : 0

Update max and recurse —

let maxVal = max(node.val, prev). Return current + dfs(node.left, maxVal) + dfs(node.right, maxVal).

Start DFS —

Call dfs(root, Int.min) to begin.

07

Code Implementation

class Solution {
  func goodNodes(_ root: TreeNode?) -> Int {
    func dfs(_ root: TreeNode?, _ prev: Int) -> Int {
      guard let root else { return 0 }
      let current = root.val >= prev ? 1 : 0
      let maxVal = max(root.val, prev)
      return current + dfs(root.left, maxVal) + dfs(root.right, maxVal)
    }
    return dfs(root, Int.min)
  }
}

09

Common Pitfalls

✕01
Starting maxSoFar at 0 instead of Int.min
If the root has a negative value (e.g., -5), starting at 0 incorrectly marks the root as not-good.
Fix: Initialise with Int.min so any node value passes the first check.
✕02
Updating prev before checking
Updating maxVal before the current node check would compare the node against itself rather than its ancestors.
Fix: Check node.val >= prev first, then compute maxVal = max(node.val, prev) for children.
✕03
Using BFS instead of DFS
BFS level-order does not naturally carry path-specific state (the max on each root-to-node path). Each path can diverge.
Fix: Use DFS (preorder) and pass maxSoFar as a parameter to correctly track per-path state.
