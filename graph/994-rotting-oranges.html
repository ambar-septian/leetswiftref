Rotting Oranges

ArrayBreadth-First SearchMatrix
01

Problem Overview

You are given an m x n integer grid grid where each cell can have one of three values:

  • 0 representing an empty cell,
  • 1 representing a fresh orange,
  • 2 representing a rotten orange.

Every minute, any fresh orange that is 4-directionally adjacent to a rotten orange becomes rotten.

Return the minimum number of minutes that must elapse until no cell has a fresh orange. If this is impossible, return -1.

Constraints:

  • m == grid.length
  • n == grid[i].length
  • 1 <= m, n <= 10
  • grid[i][j] is 0, 1, or 2.
02

Examples

Example 1​

Input:  grid = [[2,1,1],[1,1,0],[0,1,1]]
Output: 4

Example 2​

Input:  grid = [[2,1,1],[0,1,1],[1,0,1]]
Output: -1
Explanation: The orange in the bottom-left corner can never rot.

Example 3​

Input:  grid = [[0,2]]
Output: 0

04

Approach

Seed the BFS queue with all initially rotten cells (encoded as r * cols + c). Count freshCount. Process BFS level by level: for each level, process all current queue entries, spreading rot to adjacent fresh oranges (mark as 2, decrement freshCount, enqueue). If any new rot occurred in a level, increment results. After BFS, return freshCount == 0 ? results : -1.

Seeding with all rotten cells simultaneously models the real problem: rot spreads from every rotten orange in parallel. BFS naturally processes all cells at distance 1 before distance 2, etc. — distance in BFS levels equals elapsed minutes.

06

Step-by-Step Solution

Seed queue and count fresh —

Scan grid: if grid[r][c] == 2, enqueue r * cols + c. If grid[r][c] == 1, freshCount += 1.

BFS level by level —

While queue is not empty: snapshot count = queue.count. Process count items. For each, spread to 4 neighbors: if neighbor is fresh, mark as 2, enqueue, decrement freshCount. If any neighbor was newly rotted, set rootenFound = true.

Count minutes —

After processing a full level, if rootenFound: results += 1.

Return —

return freshCount == 0 ? results : -1.

07

Code Implementation

class Solution {
  func orangesRotting(_ grid: [[Int]]) -> Int {
    guard !grid.isEmpty else { return 0 }
    var grid = grid
    let rows = grid.count, cols = grid[0].count
    let directions = [(0,1),(1,0),(0,-1),(-1,0)]
    var queue = [Int](), freshCount = 0, results = 0

    for r in 0..<rows {
      for c in 0..<cols {
        if grid[r][c] == 2 { queue.append(r * cols + c) }
        else if grid[r][c] == 1 { freshCount += 1 }
      }
    }

    while !queue.isEmpty {
      let count = queue.count
      var rootenFound = false
      for _ in 0..<count {
        let ev = queue.removeFirst()
        let rIdx = ev / cols, cIdx = ev % cols
        for dir in directions {
          let rr = rIdx + dir.0, cc = cIdx + dir.1
          guard rr >= 0, cc >= 0, rr < rows, cc < cols else { continue }
          if grid[rr][cc] == 1 {
            grid[rr][cc] = 2
            queue.append(rr * cols + cc)
            rootenFound = true
            freshCount -= 1
          }
        }
      }
      if rootenFound { results += 1 }
    }

    return freshCount == 0 ? results : -1
  }
}

09

Common Pitfalls

✕01
Not seeding all initial rotten cells
If only one rotten cell is seeded, single-source BFS misses the simultaneous spread from other rotten cells and over-counts minutes.
Fix: Scan the entire grid first and enqueue every cell with value 2 before starting BFS.
✕02
Counting minutes for empty BFS levels
If the BFS processes a level but no new orange rots (all neighbours are empty or already rotten), that level should not count as a minute.
Fix: Only increment results when rootenFound is true — i.e., at least one fresh orange became rotten in that BFS level.
✕03
Returning results when fresh oranges still remain
If isolated fresh oranges are unreachable from any rotten cell, the algorithm finishes BFS but there are still fresh oranges left.
Fix: Check freshCount == 0 after BFS. If fresh oranges remain, return -1.
