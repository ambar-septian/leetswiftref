01

Problem Overview

There are a total of numCourses courses you have to take, labelled from 0 to numCourses - 1. You are given an array prerequisites where prerequisites[i] = [a, b] indicates that you must take course b before course a.

Return the ordering of courses you should take to finish all courses. If there are multiple valid answers, return any of them. If it is impossible to finish all courses (i.e. a cycle exists), return an empty array.

Constraints:

  • 1 <= numCourses <= 2000
  • 0 <= prerequisites.length <= numCourses * (numCourses - 1)
  • prerequisites[i].length == 2
  • 0 <= a, b < numCourses
  • a != b
  • All the pairs [a, b] are distinct.
Key Constraint

Unlike Course Schedule I, a boolean is not enough — the full topological order must be returned. If a cycle exists, return an empty array []. This output format forces collecting the BFS order into a results array rather than just counting processed nodes.

Approach

Core Insight

The BFS body is identical to Course Schedule I. The only additions: collect each dequeued node into results, and return results if results.count == numCourses or [] otherwise. The same in-degree / adjacency-list construction and queue seeding apply unchanged.

Build an adjacency list and in-degree array. Seed a queue with all zero-in-degree courses. Process each course: append it to results, decrement each dependent course's in-degree, and enqueue dependents that hit zero. If results.count == numCourses, a valid topological order exists; otherwise return [].

Difference from Course Schedule I

Problem 207 only needs a boolean — cycle or no cycle. Problem 210 needs the actual order. The BFS body is identical; the only change is collecting nodes into results and returning the array instead of a count comparison.

06

Step-by-Step Solution

Build adjacency list and in-degree array —

For each [course, prereq]: adjList[prereq].append(course), inDegree[course] += 1.

Seed queue with zero-in-degree courses —

All courses with inDegree == 0 have no prerequisites and can be taken immediately.

BFS — collect topological order —

Dequeue a course, append to results, decrement each dependent's in-degree; enqueue dependents whose in-degree reaches 0.

Return order or empty —

If results.count == numCourses, return results. Otherwise return [].

07

Code Implementation

class Solution {
  func findOrder(_ numCourses: Int, _ prerequisites: [[Int]]) -> [Int] {
    var inDegree = Array(repeating: 0, count: numCourses)
    var results  = [Int]()
    var adjList  = [Int: [Int]]()

    for pairs in prerequisites {
      let course = pairs[0], prereq = pairs[1]
      adjList[prereq, default: []].append(course)
      inDegree[course] += 1
    }

    var queue = [Int]()
    for (index, count) in inDegree.enumerated() {
      if count == 0 { queue.append(index) }
    }

    while !queue.isEmpty {
      let node = queue.removeFirst()
      results.append(node)
      guard let neighbors = adjList[node] else { continue }
      for neighbor in neighbors {
        inDegree[neighbor] -= 1
        if inDegree[neighbor] == 0 { queue.append(neighbor) }
      }
    }

    return results.count == numCourses ? results : []
  }
}
Swift Notes

adjList[prereq, default: []].append(course) initialises missing dictionary keys in one expression — idiomatic Swift. The ternary results.count == numCourses ? results : [] relies on Swift's type inference to resolve [] as [Int] from the function's return type — no explicit annotation needed.

08

Complexity Analysis

Time Complexity — O(V + E)

Building the adjacency list and in-degree array is O(E). The BFS processes each of the V nodes once and traverses each of the E edges once. Total O(V + E) where V = numCourses and E = prerequisites.count.

Space Complexity — O(V + E)

The adjacency list stores all edges: O(E). The in-degree array, results array, and queue each grow to O(V). Total O(V + E).

When to prefer Kahn's BFS vs DFS

Prefer Kahn's BFS for its iterative implementation and intuitive count-based cycle check. Prefer DFS with 3-color marking when you need to detect AND report the actual cycle path — DFS tracks the current path in the call stack, making it easy to reconstruct.

09

Common Pitfalls

✕01
Returning results without checking the count
If a cycle exists, results will have fewer than numCourses entries. Returning it without checking produces a partial order that skips cyclic nodes.
Fix: Always guard: return results.count == numCourses ? results : [].
✕02
Confusing edge direction
prerequisite[i] = [a, b] means b unlocks a. The directed edge is b → a (b's adjacency list contains a), and a's in-degree is incremented.
Fix: adjList[prereq].append(course) and inDegree[course] += 1.
✕03
Not handling courses with no prerequisites
Courses not mentioned in prerequisites have in-degree 0 and must still be included in the output.
Fix: The in-degree array is initialised to all zeros for numCourses entries, so unconstrained courses are automatically seeded into the queue.
