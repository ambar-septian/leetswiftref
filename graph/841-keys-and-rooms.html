Keys and Rooms

Depth-First SearchBreadth-First SearchGraph
01

Problem Overview

There are n rooms labelled from 0 to n - 1 and all the rooms are locked except for room 0. Your goal is to visit all the rooms. However, you cannot enter a locked room without having its key.

When you visit a room, you may find a set of distinct keys in it. Each key has a number on it, denoting which room it unlocks, and you can take all of them with you to unlock the other rooms.

Given an array rooms where rooms[i] is the list of keys in room i, return true if you can visit all the rooms, or false otherwise.

Constraints:

  • n == rooms.length
  • 2 <= n <= 1000
  • 0 <= rooms[i].length <= 1000
  • 1 <= sum(rooms[i].length) <= 3000
  • 0 <= rooms[i][j] < n
  • All the values in rooms[i] are unique.
02

Examples

Example 1​

Input:  rooms = [[1],[2],[3],[]]
Output: true
Explanation: Visit room 0 (key 1) → room 1 (key 2) → room 2 (key 3) → room 3. All visited.

Example 2​

Input:  rooms = [[1,3],[3,0],[2],[0]]
Output: false
Explanation: We can never enter room 2.

04

Approach

DFS from room 0. Maintain a visited set. At each room i: insert i into visited, then recurse into each key (neighbor) that has not yet been visited. After DFS completes, check if visited.count == rooms.count.

Step-by-Step Solution

Initialise visited set —

var visited = Set(Int)().

Define DFS —

func dfs(_ i: Int): guard !visited.contains(i) else return. Insert i. For each key in rooms[i]: dfs(key).

Start from room 0 —

dfs(0).

Check all rooms visited —

Return visited.count == rooms.count.

07

Code Implementation

class Solution {
  func canVisitAllRooms(_ rooms: [[Int]]) -> Bool {
    var visited = Set<Int>()

    func dfs(_ i: Int) {
      guard !visited.contains(i) else { return }
      visited.insert(i)
      for key in rooms[i] {
        dfs(key)
      }
    }

    dfs(0)
    return visited.count == rooms.count
  }
}

09

Common Pitfalls

✕01
Starting DFS from room 1 instead of room 0
Room 0 is the only room that starts unlocked. DFS must begin at room 0.
Fix: Always call dfs(0).
✕02
Not guarding against already-visited rooms
Without the visited check, DFS can loop infinitely if rooms contain keys pointing back to earlier rooms.
Fix: Check visited.contains(i) at the top of dfs and return early if already visited.
✕03
Comparing visited.count to rooms.count - 1
All n rooms (including room 0) must be visited. Off-by-one returns true even when one room is unreachable.
Fix: Compare visited.count == rooms.count (not rooms.count - 1).
