Union-Find

Also called Disjoint Set Union (DSU). A data structure that efficiently tracks which elements belong to the same connected component, supporting two operations: union (merge two components) and find (identify a component's root).

Time O(α(N)) amortisedSpace O(N)Disjoint SetConnected ComponentsCycle Detection
01

Overview

Union-Find answers one question extremely fast: "Are these two nodes in the same connected component?" If you also need to merge components, it handles that too. It's the right choice when:

  • You process edges one by one and need to track component membership dynamically.
  • You need to detect whether adding an edge creates a cycle (Redundant Connection).
  • You need to count or enumerate connected components (Number of Provinces, Accounts Merge).

Unlike DFS/BFS which re-traverse the whole graph, Union-Find answers membership queries in nearly O(1) amortised time after the initial build.

Core Insight

Each component is represented as a tree, with one node as the root (representative). The parent[] array stores each node's parent. To find which component a node belongs to, walk up the tree to the root — that's find(). To merge two components, point one root to the other — that's union(). Two optimisations — path compression and union by rank — keep the trees nearly flat, making both operations practically O(1).

Implementation

The standard Swift implementation uses a class (reference semantics) so the parent and rank arrays are mutated in place without inout:

class UnionFind {
    private var parent: [Int]
    private var rank: [Int]
    private(set) var componentCount: Int

    init(_ n: Int) {
        parent = Array(0..<n)           // each node is its own root
        rank = Array(repeating: 0, count: n)
        componentCount = n
    }

    // Find with path compression — O(α(N)) amortised
    func find(_ x: Int) -> Int {
        if parent[x] != x {
            parent[x] = find(parent[x]) // compress: point directly to root
        }
        return parent[x]
    }

    // Union by rank — O(α(N)) amortised
    // Returns false if x and y are already in the same component
    @discardableResult
    func union(_ x: Int, _ y: Int) -> Bool {
        let px = find(x), py = find(y)
        guard px != py else { return false } // already connected

        if rank[px] < rank[py] {
            parent[px] = py
        } else if rank[px] > rank[py] {
            parent[py] = px
        } else {
            parent[py] = px
            rank[px] += 1
        }

        componentCount -= 1
        return true
    }

    func connected(_ x: Int, _ y: Int) -> Bool {
        find(x) == find(y)
    }
}

Usage example — Number of Provinces:

func findCircleNum(_ isConnected: [[Int]]) -> Int {
    let n = isConnected.count
    let uf = UnionFind(n)

    for i in 0..<n {
        for j in (i+1)..<n {
            if isConnected[i][j] == 1 {
                uf.union(i, j)
            }
        }
    }

    return uf.componentCount
}
Swift Notes

Using a class instead of struct avoids needing mutating on find and union — important because find is recursive and modifies parent[]. The @discardableResult attribute on union lets callers ignore the return value (whether a merge actually happened) without a compiler warning. For cycle detection problems, check if union returns false — that means the two nodes are already connected and adding the edge creates a cycle.

05

Path Compression Deep-Dive

Without path compression, find(x) walks up the tree one parent at a time — O(tree height). Path compression makes every node on the path point directly to the root after the first find, flattening the tree for all future queries:

// Without compression — tree stays tall
func findNaive(_ x: Int) -> Int {
    var x = x
    while parent[x] != x { x = parent[x] }
    return x
}

// With path compression — tree flattens on every find
func find(_ x: Int) -> Int {
    if parent[x] != x {
        parent[x] = find(parent[x]) // recursive: sets all ancestors to root
    }
    return parent[x]
}

The iterative version (avoids recursion overhead):

func find(_ x: Int) -> Int {
    var root = x
    while parent[root] != root { root = parent[root] }
    // second pass: compress all nodes on path to root
    var node = x
    while node != root {
        let next = parent[node]
        parent[node] = root
        node = next
    }
    return root
}
06

Complexity Analysis

Time Complexity — O(α(N)) per operation

With both path compression and union by rank, each find and union runs in amortised O(α(N)) time, where α is the inverse Ackermann function. For any practical input size (even N = 10^80), α(N) ≤ 4. This is effectively constant time. Without optimisations, find degrades to O(N) per operation.

Space Complexity — O(N)

Two arrays of size N — parent[] and rank[]. No additional space grows with the number of operations.

When to Use

Rule of Thumb

Reach for Union-Find when the problem gives you edges one at a time and asks about dynamic connectivity — whether two nodes are in the same group after each merge. If the graph is static and you only need one traversal, DFS/BFS is simpler.

Problem signalUse Union-Find
"Find connected components"Yes — count roots where parent[i] == i
"Detect cycle in undirected graph"Yes — union returns false = cycle found
"Group accounts / merge sets"Yes — union by a shared key, count final groups
"Redundant connection"Yes — the edge where union returns false
"Shortest path"No — use BFS or Dijkstra
"Topological ordering"No — use Kahn's algorithm
"All paths from A to B"No — use DFS
