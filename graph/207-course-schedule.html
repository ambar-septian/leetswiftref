01

Problem Overview

There are a total of numCourses courses you have to take, labelled from 0 to numCourses - 1. You are given an array prerequisites where prerequisites[i] = [a, b] indicates that you must take course b first if you want to take course a.

Return true if you can finish all courses. Otherwise, return false.

A cycle in the prerequisite graph makes it impossible to complete all courses — Kahn's algorithm detects this by checking whether every node was processed.

Constraints:

  • 1 <= numCourses <= 2000
  • 0 <= prerequisites.length <= 5000
  • prerequisites[i].length == 2
  • 0 <= a, b < numCourses
  • All the pairs prerequisites[i] are unique.
02

Examples

Example 1​

Input: numCourses = 2, prerequisites = [[1,0]]
Output: true
Explanation: Take course 0 first, then course 1.

Example 2​

Input: numCourses = 2, prerequisites = [[1,0],[0,1]]
Output: false
Explanation: Courses 0 and 1 each require the other — cycle detected.

Build an adjacency list and an in-degree array from prerequisites. Seed a queue with all courses that have in-degree 0 (no prerequisites). Process each course: decrement the in-degree of every dependent course; if a dependent course's in-degree reaches 0, enqueue it. After BFS, if results.count == numCourses, all courses were reachable and no cycle exists.

Why count equals numCourses means no cycle

In a DAG every node eventually reaches in-degree 0 and gets processed. In a cycle, nodes in the cycle never reach in-degree 0 — they remain in the graph and are never enqueued, so the processed count falls short of numCourses.

06

Step-by-Step Solution

Guard empty prerequisites —

If prerequisites is empty, all courses are independent — return true immediately.

Build adjacency list and in-degree array —

For each [course, prereq] pair: add course to adjList[prereq] and increment inDegree[course].

Seed queue with zero-in-degree nodes —

Iterate over inDegree; enqueue every index whose count is 0.

BFS processing —

Dequeue a node, append to results, then for each neighbor decrement its in-degree. If it reaches 0, enqueue it.

Check result count —

Return results.count == numCourses.

07

Code Implementation

class Solution {
  func canFinish(_ numCourses: Int, _ prerequisites: [[Int]]) -> Bool {
    guard !prerequisites.isEmpty else { return true }

    var results = [Int]()
    var inDegree = Array(repeating: 0, count: numCourses)
    var adjList = [Int: [Int]]()

    for pair in prerequisites {
      let course = pair[0]
      let preq   = pair[1]
      adjList[preq, default: []].append(course)
      inDegree[course] += 1
    }

    var queue = [Int]()
    for (index, count) in inDegree.enumerated() {
      if count == 0 { queue.append(index) }
    }

    while !queue.isEmpty {
      let node = queue.removeFirst()
      results.append(node)

      guard let neighbors = adjList[node] else { continue }
      for neighbor in neighbors {
        inDegree[neighbor] -= 1
        if inDegree[neighbor] == 0 { queue.append(neighbor) }
      }
    }

    return results.count == numCourses
  }
}

09

Common Pitfalls

✕01
Mixing up edge direction
prerequisite[i] = [a, b] means b must come before a. The edge goes b → a (b unlocks a), so a's in-degree is incremented.
Fix: adjList[prereq].append(course) and inDegree[course] += 1 — prereq is the source, course is the destination.
✕02
Not guarding against empty prerequisites
With no prerequisites, no edges exist, in-degree array is all zeros, and the guard is not strictly necessary — but forgetting to seed the queue correctly would cause an infinite loop.
Fix: Return true early for empty prerequisites, or ensure all indices with in-degree 0 are correctly seeded.
✕03
Using results.count instead of numCourses as the target
If prerequisites only mention a subset of courses, some courses may never appear in the graph and still need to be counted.
Fix: Compare results.count to numCourses. The in-degree array is of size numCourses, so unconstrained courses start at 0 and get enqueued at the seed step.
