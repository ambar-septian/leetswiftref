Number of Islands

#200MediumDFS Flood FillLeetCode ↗

Number of Islands

ArrayDepth-First SearchBreadth-First SearchUnion-FindMatrix
01

Problem Overview

Given an m x n 2D binary grid which represents a map of '1's (land) and '0's (water), return the number of islands.

An island is surrounded by water and is formed by connecting adjacent lands horizontally or vertically. You may assume all four edges of the grid are all surrounded by water.

Constraints:

  • m == grid.length
  • n == grid[i].length
  • 1 <= m, n <= 300
  • grid[i][j] is '0' or '1'.
Key Constraint

The grid can be up to 300 × 300 (90,000 cells). Recursive DFS can reach a depth of 90,000 in the worst case (all land), risking a stack overflow. In production, prefer BFS or an iterative DFS with an explicit stack for large grids.

Rule of Thumb

When counting connected components in a 2D grid, scan every cell and launch a flood-fill (DFS or BFS) from each unvisited land cell. Overwriting visited land with water eliminates the need for a separate visited set.

04

Approach

Core Insight

The outer double loop counts how many times a new unvisited island is discovered. Each DFS call sinks the entire island by overwriting every connected '1' with '0'. After DFS returns, the outer loop skips those cells naturally because they are now '0' — no separate visited set required.

Scan every cell. When a '1' is found, increment the island count and launch a DFS that overwrites all connected '1's with '0' (sinking the island). The DFS explores all four directions and stops at water, out-of-bounds, or already-sunk cells. The outer scan then continues to find the next unvisited island.

Why sink instead of a visited set

Overwriting with '0' avoids allocating a Set of coordinates. It is O(1) per cell instead of O(1) amortised with hash overhead, and keeps the logic minimal.

06

Step-by-Step Solution

Scan every cell —

Double loop over rows and columns. When grid[r][c] == '1', increment results and call dfs(r, c).

DFS guard conditions —

Return immediately if out of bounds, or grid[r][c] == '0'.

Sink the cell —

Set grid[r][c] = '0' and optionally add to visited (the sink serves as the visited marker).

Explore four directions —

Recurse into (r-1,c), (r+1,c), (r,c-1), (r,c+1).

Return count —

Return results after the full scan.

07

Code Implementation

class Solution {
  func numIslands(_ grid: [[Character]]) -> Int {
    guard !grid.isEmpty else { return 0 }

    var grid = grid
    var results = 0
    let rows = grid.count, cols = grid[0].count
    let dirs = [(-1,0),(1,0),(0,1),(0,-1)]

    func dfs(_ r: Int, _ c: Int) {
      guard r >= 0, c >= 0, r < rows, c < cols, grid[r][c] == "1" else { return }
      grid[r][c] = "0"
      for (dr, dc) in dirs { dfs(r + dr, c + dc) }
    }

    for r in 0..<rows {
      for c in 0..<cols {
        if grid[r][c] == "1" {
          results += 1
          dfs(r, c)
        }
      }
    }

    return results
  }
}
Swift Notes

Swift passes [[Character]] by value, so var grid = grid at the top creates a local mutable copy without affecting the caller. The nested dfs closure captures grid, rows, cols, and dirs from the outer scope — no explicit parameter passing required.

08

Complexity Analysis

Time Complexity — O(m×n)

Each cell is visited at most twice: once by the outer scan and once by a DFS call. The DFS overwrites cells with '0' so no cell is sunk more than once. Total work is O(m × n).

Space Complexity — O(m×n)

The DFS call stack can grow to O(m × n) in the degenerate case where the entire grid is a single island — the recursion depth equals the number of land cells. The var grid = grid copy is also O(m × n).

When to prefer DFS vs BFS

Prefer BFS when stack overflow is a concern — large grids filled with land can push DFS recursion depth to 90,000. BFS uses an explicit queue and avoids deep recursion. Prefer DFS for its concise recursive implementation when grid size is moderate.

09

Common Pitfalls

✕01
Forgetting to make grid mutable
Swift function parameters are constants by default. Assigning to grid[r][c] inside the function without var grid = grid at the call site causes a compile error.
Fix: Shadow the parameter with var grid = grid at the start of numIslands.
✕02
Using a visited set but also sinking cells
Double-tracking with both a Set and cell overwrite is redundant — either approach alone is sufficient.
Fix: Choose one strategy: either sink (overwrite with '0') or use a visited Set. The sink approach is simpler and avoids extra allocation.
✕03
Missing a direction in the DFS
Islands connect in four directions. Omitting any one direction (e.g. up or left) can leave part of an island unsunk and produce a count that is too high.
Fix: Always include all four offsets: (-1,0), (1,0), (0,1), (0,-1).
