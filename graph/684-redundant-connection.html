Redundant Connection

#684MediumUnion-FindLeetCode ↗

Redundant Connection

Depth-First SearchBreadth-First SearchUnion-FindGraph
01

Problem Overview

In this problem, a tree is an undirected graph that is connected and has no cycles.

You are given a graph that started as a tree with n nodes labelled 1 to n, with one additional edge added. The added edge has two different vertices chosen from 1 to n, and was not an edge that already existed in the tree.

The graph is represented as an array edges where edges[i] = [aᵢ, bᵢ] indicates that there is an undirected edge between nodes aᵢ and bᵢ.

Return an edge that can be removed so that the resulting graph is a tree of n nodes. If there are multiple answers, return the answer that occurs last in the input.

Constraints:

  • n == edges.length
  • 3 <= n <= 1000
  • 1 <= aᵢ < bᵢ <= n
  • aᵢ != bᵢ
  • There are no repeated edges.
  • The given graph is connected.
Key Constraint

Node labels are 1-indexed (1 to n) but the Union-Find arrays are 0-indexed. Every call to unite and find must subtract 1 from the edge values — forgetting this causes an off-by-one that corrupts the parent array. There is always exactly one redundant edge since the graph is a tree plus one extra edge.

Rule of Thumb

When detecting whether adding an edge creates a cycle in an undirected graph, Union-Find is the canonical tool: if both endpoints share the same root before the union, the edge is redundant. Process edges in order and overwrite the result on every failure — the last redundant edge wins.

04

Approach

Core Insight

Process edges in order. For each edge [a, b], check find(a-1) == find(b-1) before uniting — if both endpoints share the same root they are already connected, so this edge forms a cycle (redundant). Otherwise, unite them. Overwrite the result variable on every such failure; the final value is the last redundant edge.

Initialize Union-Find with n nodes (0-indexed). Process each edge [a, b]: call unite(a-1, b-1). If find(a-1) == find(b-1) before the merge (returned false), both nodes already share a root — this edge forms a cycle, so it is the redundant connection. Since we process edges in order, we record the last such edge and return it.

Path compression and union by rank

Path compression makes find() update every node on the path directly to the root, flattening the tree. Union by rank always attaches the smaller tree under the larger one. Together they achieve near-constant amortised time per operation.

06

Step-by-Step Solution

Initialise Union-Find —

Create parent = [0, 1, 2, ..., n-1] and rank = [0, 0, ..., 0] with n = edges.count.

Implement find with path compression —

find(i): if parent[i] == i, return i; else parent[i] = find(parent[i]) and return parent[i].

Implement unite with union by rank —

unite(a, b): find roots rootA, rootB. If equal, return false. Attach smaller rank under larger; increment rank on tie.

Process edges —

For each edge [a, b]: if unite(a-1, b-1) returns false, record [a, b] as result.

Return result —

Return the last edge that failed to unite — that is the redundant connection.

07

Code Implementation

class Solution {
  class UnionSet {
    var parent = [Int]()
    var rank = [Int]()

    init(_ n: Int) {
      self.parent = (0..<n).map { $0 }
      self.rank = Array(repeating: 0, count: n)
    }

    func find(_ i: Int) -> Int {
      if parent[i] == i { return i }
      parent[i] = find(parent[i])
      return parent[i]
    }

    func unite(_ a: Int, _ b: Int) -> Bool {
      let rootA = find(a)
      let rootB = find(b)
      guard rootA != rootB else { return false }

      if rank[rootA] < rank[rootB] {
        parent[rootA] = rootB
      } else if rank[rootA] > rank[rootB] {
        parent[rootB] = rootA
      } else {
        parent[rootA] = rootB
        rank[rootB] += 1
      }
      return true
    }
  }

  func findRedundantConnection(_ edges: [[Int]]) -> [Int] {
    let unionSet = UnionSet(edges.count)
    var result = [Int]()

    for edge in edges {
      if !unionSet.unite(edge[0] - 1, edge[1] - 1) {
        result = [edge[0], edge[1]]
      }
    }

    return result
  }
}
Swift Notes

UnionFind is an inner class of Solution — a clean way to scope the helper without polluting the module namespace. (0..<n).map { $0 } creates [0, 1, 2, ..., n-1] idiomatically. func find mutates parent[i] via path compression — this is safe because UnionFind is a class (reference type); if it were a struct the mutation would require mutating func.

08

Complexity Analysis

Time Complexity — O(n · α(n))

Processing n edges, each calling find and potentially unite. With path compression and union by rank, each find runs in amortised O(α(n)) — the inverse Ackermann function, which is effectively constant (≤ 4) for any practical n. Total: O(n · α(n)) ≈ O(n).

Space Complexity — O(n)

Two arrays of size n — parent and rank — are allocated at initialization. No other data structures grow with input size. Total: O(n).

09

Common Pitfalls

✕01
Off-by-one: nodes are 1-indexed in edges
Edges use 1-indexed node labels but the parent array is 0-indexed. Forgetting to subtract 1 causes out-of-bounds or incorrect union operations.
Fix: Always call unite(edge[0]-1, edge[1]-1) to convert to 0-indexed before touching the Union-Find arrays.
✕02
Returning the first redundant edge instead of the last
The problem asks for the redundant edge that appears last in the input when multiple exist.
Fix: Keep overwriting result on every failed unite — the final value will be the last redundant edge.
✕03
Missing path compression in find
Without path compression, find() is O(n) per call, degrading the overall algorithm to O(n^2).
Fix: Add parent[i] = find(parent[i]) before returning — this flattens the tree during every traversal.
