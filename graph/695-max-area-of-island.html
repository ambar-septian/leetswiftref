Max Area of Island

#695MediumDFS Flood-FillLeetCode ↗

Max Area of Island

ArrayDepth-First SearchBreadth-First SearchUnion-FindMatrix
01

Problem Overview

You are given an m x n binary matrix grid. An island is a group of 1s (land) connected 4-directionally (horizontal or vertical). You may assume all four edges of the grid are surrounded by water.

The area of an island is the number of cells with value 1 in the island.

Return the maximum area of an island in grid. If there is no island, return 0.

Constraints:

  • m == grid.length
  • n == grid[i].length
  • 1 <= m, n <= 50
  • grid[i][j] is either 0 or 1.
Key Constraint

The cell must be sunk (grid[r][c] = 0) as the very first action inside DFS — before any recursive calls. Sinking after recursing allows neighbors to re-enter the current cell, causing infinite recursion and an inflated count.

Rule of Thumb

When counting the area or cells of connected regions in a grid, DFS flood-fill with in-place sinking (set to 0 on entry) is the standard O(m×n) approach — each cell is visited at most once and no separate visited array is needed.

04

Approach

Core Insight

Scan every cell. On finding a 1, launch DFS: sink the cell to 0 immediately (in-place visited marker), recurse in all four directions, return 1 + sum of neighbor counts. The outer scan accumulates result = max(result, dfs(r, c)). Each cell is visited at most once — once sunk, it is never revisited.

Scan every cell. When a 1 is found, launch DFS. Inside DFS: immediately set grid[r][c] = 0 (sink), then recurse in all 4 directions, summing the return counts. Return 1 + counts from neighbors. Update results = max(results, dfs(r, c)) after each call. Return results at the end.

06

Step-by-Step Solution

Guard empty grid —

guard !grid.isEmpty else { return 0 }. Capture rows, cols, directions = [(0,1),(0,-1),(1,0),(-1,0)].

Define DFS —

func dfs(_ r: Int, _ c: Int) -> Int. If out-of-bounds or grid[r][c] == 0, return 0. Sink: grid[r][c] = 0. Recurse all 4 directions; return 1 + sum of recursive calls.

Outer scan —

For each (r, c): if grid[r][c] == 1, results = max(results, dfs(r, c)).

Return max area —

Return results.

07

Code Implementation

class Solution {
  func maxAreaOfIsland(_ grid: [[Int]]) -> Int {
    guard !grid.isEmpty else { return 0 }
    var grid = grid
    let rows = grid.count
    let cols = grid[0].count
    let directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
    var results = 0

    func dfs(_ r: Int, _ c: Int) -> Int {
      if r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] == 0 {
        return 0
      }
      grid[r][c] = 0
      var count = 1
      for dir in directions {
        count += dfs(r + dir.0, c + dir.1)
      }
      return count
    }

    for r in 0..<rows {
      for c in 0..<cols {
        if grid[r][c] == 1 {
          results = max(results, dfs(r, c))
        }
      }
    }

    return results
  }
}
Swift Notes

var grid = grid creates a mutable copy — Swift 2D arrays are value types (array of arrays), so each row is also copied. The nested func dfs captures grid, rows, cols, and directions from the outer scope via closure. let directions = [(0, 1), (0, -1), (1, 0), (-1, 0)] uses anonymous tuple types — clean for 4-directional traversal without a named struct.

08

Complexity Analysis

Time Complexity — O(m·n)

The outer scan visits every cell once. Each cell that is a 1 triggers one DFS call, but once sunk (set to 0) it is never revisited. Every cell is processed at most twice — once by the outer scan and once by DFS. Total: O(m·n).

Space Complexity — O(m·n)

The recursion stack depth is bounded by the size of the largest island — in the worst case (one giant connected island filling the entire grid) the stack reaches depth m·n. No additional visited array is used.

09

Common Pitfalls

✕01
Forgetting to sink the cell before recursing
If you do not mark the cell as visited (sink to 0) before recursing, the DFS will revisit it via a neighbor, causing infinite recursion or an inflated count.
Fix: Set grid[r][c] = 0 as the very first action inside DFS — before any recursive calls.
✕02
Checking bounds after the recursive call
Calling dfs without bounds-checking first causes an index-out-of-bounds crash.
Fix: Return 0 immediately at the top of DFS if r or c is out of range, or if grid[r][c] == 0.
✕03
Updating maxArea with the wrong variable
Comparing results to the count inside DFS (before the recursion unwinds) gives a partial count.
Fix: Call results = max(results, dfs(r, c)) from the outer scan loop — after the full DFS for the island returns its total area.
