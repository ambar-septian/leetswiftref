Minimum Number of Vertices to Reach All Nodes

#1557MediumGraph — In-Degree AnalysisLeetCode ↗

Minimum Number of Vertices to Reach All Nodes

Graph Theory
01

Problem Overview

Given a directed acyclic graph (DAG) with n vertices numbered from 0 to n-1, and an array edges where edges[i] = [from_i, to_i] represents a directed edge from node from_i to node to_i.

Find the smallest set of vertices from which all nodes in the graph are reachable. It's guaranteed that a unique solution exists.

Note: vertices with no incoming edges (in-degree 0) cannot be reached by any other vertex, so they must be in the starting set. Every other vertex can be reached via an incoming edge.

Constraints:

  • 2 <= n <= 10^5
  • 1 <= edges.length <= min(10^5, n * (n-1) / 2)
  • edges[i].length == 2
  • 0 <= from_i, to_i < n
  • All pairs (from_i, to_i) are distinct.
Key Constraint

The graph is a DAG (no cycles) with a guaranteed unique solution. Nodes with in-degree 0 are exactly the required minimum starting set — no DFS or reachability traversal is needed to verify this.

02

Examples

Example 1​

Input:  n=6, edges=[[0,1],[0,2],[2,5],[3,4],[4,2]]
Output: [0,3]
Explanation: 0 and 3 are the only nodes with no incoming edges.

Example 2​

Input:  n=5, edges=[[0,1],[2,1],[3,1],[1,4],[2,4]]
Output: [0,2,3]

Rule of Thumb

In a DAG, nodes with in-degree 0 are unreachable from any other node — they must always be in the minimum starting set. Collect all destination nodes in a set, then return every node not in that set.

04

Approach

Core Insight

A node with in-degree > 0 has at least one incoming edge — it can be reached from another node and need not be in the starting set. Only in-degree-0 nodes (never appearing as a destination) are required. No DFS or reachability check needed: collect all destination nodes, then find which nodes are never a destination.

Collect all nodes that appear as edge destinations (non-zero in-degree). Remove them from the full node set. The remaining nodes have in-degree 0 and form the minimum starting set.

O(E) solution

Build a set of all source nodes (edge[0]). Build a list of all destination nodes (edge[1]). Remove each destination from the source set. What remains are nodes with no incoming edges.

06

Step-by-Step Solution

Build candidate set from sources —

Start with a set of all nodes that appear as edge[0] (source side of any edge).

Collect all destinations —

Gather all nodes that appear as edge[1] (destination side) into a list.

Remove destinations from candidate set —

For each destination, remove it from the candidate set. What remains has in-degree 0.

Return result —

Convert the remaining set to an array and return it.

07

Code Implementation

class Solution {
  func findSmallestSetOfVertices(_ n: Int, _ edges: [[Int]]) -> [Int] {
    var sources = Set<Int>()
    var destinations = [Int]()

    for edge in edges {
      sources.insert(edge[0])
      destinations.append(edge[1])
    }

    for dest in destinations {
      sources.remove(dest)
    }

    return Array(sources)
  }
}
Swift Notes

var sources = Set&lt;Int&gt;() accumulates source nodes with O(1) average insert/remove. sources.remove(dest) silently does nothing if dest was never a source — no guard needed. Array(sources) at the end converts without sorting, which is fine since the problem accepts any order.

08

Complexity Analysis

Time Complexity — O(E)

One pass over all E edges to build the sources Set and destinations list. One pass over destinations to remove entries from sources. Each Set insert/remove is O(1) average. Total: O(E).

Space Complexity — O(n)

The sources Set holds at most n entries (one per node). The destinations list holds E entries but is processed in one pass. Total auxiliary: O(n + E), dominated by O(n) for the sources Set.

09

Common Pitfalls

✕01
Starting with all n nodes instead of just sources
If you iterate 0..<n and try to remove destinations, you correctly identify in-degree-0 nodes. The solution above uses a shorter path: only collect nodes that appeared as sources.
Fix: Either approach works. The key is: any node that appears as a destination has in-degree > 0 and should not be in the result.
✕02
Assuming the output must be sorted
The problem does not require a sorted output. Returning any order of the in-degree-0 nodes is accepted.
Fix: Return Array(sources) directly without sorting.
