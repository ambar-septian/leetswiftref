Math

Problems solved primarily with arithmetic, number theory, and integer properties — no complex data structures required. The key skill is recognising which mathematical primitive (modulo, GCD, digit extraction, or integer reversal) reduces the problem to a one-liner.

What is a Math Problem?

Math problems on LeetCode rely on properties of integers rather than on data-structure traversal. They are usually O(log n) or O(n) in complexity, but the insight is purely mathematical.

Four recurring primitives:

PrimitiveSwift idiomTypical use
Modulo / remaindern % kFizzBuzz, cycle detection, circular indexing
Digit extractionn % 10, n / 10Palindrome number, digit sum, reverse integer
GCD (Euclidean)gcd(a, b) via while b != 0GCD of strings, coprime checks
Integer palindromeReverse half, comparePalindrome Number without string conversion
Key Constraint

Most math problems have an integer overflow trap. In Swift, Int is 64-bit on all modern platforms, so Int.max ≈ 9.2 × 10¹⁸ — this covers the vast majority of LeetCode constraints. When a problem says values can exceed 2³¹ - 1, check whether the intermediate computation overflows Int before reaching the result.

Terminology

TermDefinition
Moduloa % b — the remainder after dividing a by b; always in [0, b-1] for positive b
Integer divisiona / b — truncates toward zero in Swift (-7 / 2 == -3, not -4)
GCDGreatest common divisor — largest integer dividing both a and b with no remainder
Digit extractionRepeatedly taking n % 10 to read the last digit, then n / 10 to shift right
OverflowResult exceeds the type's representable range; Swift traps on overflow by default

Rule of Thumb

If the problem works entirely on integer values with no array or string structure — modulo, digit sums, divisibility, GCD — reach for a math primitive before considering any data structure.
