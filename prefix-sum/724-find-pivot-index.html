#724EasyPrefix SumLeetCode ↗

Find Pivot Index

ArrayPrefix Sum
01

Problem Overview

Given an array of integers nums, calculate the pivot index of this array.

The pivot index is the index where the sum of all the numbers strictly to the left of the index is equal to the sum of all the numbers strictly to the right of the index.

If the index is on the left edge of the array, then the left sum is 0 because there are no elements to the left. This also applies to the right edge of the array.

Return the leftmost pivot index. If no such index exists, return -1.

Constraints:

  • 1 <= nums.length <= 10^4
  • -1000 <= nums[i] <= 1000
02

Examples

Example 1​

Input:  nums = [1,7,3,6,5,6]
Output: 3
Explanation: The pivot index is 3.
Left sum = nums[0]+nums[1]+nums[2] = 1+7+3 = 11
Right sum = nums[4]+nums[5] = 5+6 = 11

Example 2​

Input:  nums = [1,2,3]
Output: -1
Explanation: No index satisfies the conditions.

Example 3​

Input:  nums = [2,1,-1]
Output: 0
Explanation: Pivot index 0. Left sum = 0, right sum = 1+(-1) = 0.

Pattern Recognition

?
Can we precompute cumulative sums?
✓ Yes — Build a prefix sum array. Then for any index i, left sum = prefix[i-1] and total - prefix[i] gives right sum.
?

✓ Yes — At index 0, left sum is 0. Check if prefix[0] equals total - prefix[0]: prefix[0]*2 == total.

Can we do this in O(1) extra space?
✓ Yes — Track running left sum and compute right sum from total - leftSum - nums[i]. But the prefix array is clearer and still O(n) space.

04

Approach

Build prefix where prefix[i] = nums[0] + ... + nums[i]. For index i = 0: left sum is 0, right sum is prefix[n-1] - prefix[0] — check if they are equal. For i > 0: left sum is prefix[i-1], right sum is prefix[n-1] - prefix[i] — check equality. Return the first matching i, or -1.

Pivot condition

At index i: leftSum == prefix[i-1] (or 0 for i=0) and rightSum == totalSum - prefix[i]. Pivot is where leftSum == rightSum, i.e. prefix[i-1] == totalSum - prefix[i].

06

Step-by-Step Solution

Build prefix array —

prefixs[0] = nums[0]. For i in 1..count-1: prefixs[i] = prefixs[i-1] + nums[i].

Check index 0 —

If prefixs[0] == prefixs[count-1] - prefixs[0] (i.e., prefix[0]*2 == total), then leftSum = 0 = rightSum. The implementation checks prefixs[0] - prefixs[count-1] == 0 which simplifies to prefixs[0] == total/2 only when signed — check the exact condition carefully.

Check remaining indices —

For i in 1..count-1: if prefixs[i] == prefixs[count-1] - prefixs[i-1], return i.

No pivot found —

Return -1.

07

Code Implementation

class Solution {
  func pivotIndex(_ nums: [Int]) -> Int {
    let count = nums.count
    var prefixs = Array(repeating: 0, count: count)

    for i in 0..<count {
      if i == 0 {
        prefixs[0] = nums[0]
        continue
      }
      prefixs[i] = prefixs[i - 1] + nums[i]
    }

    if prefixs[0] - prefixs[count - 1] == 0 { return 0 }  // left=0, right=0

    for i in 1..<count {
      if prefixs[i] == prefixs[count - 1] - prefixs[i - 1] {
        return i
      }
    }

    return -1
  }
}

09

Common Pitfalls

✕01
Including nums[i] in the left or right sum
The pivot element is excluded from both sides. leftSum is the sum strictly before i, rightSum is strictly after i.
Fix: leftSum = prefix[i-1], rightSum = total - prefix[i]. The element at i is excluded from both.
✕02
Not handling index 0 specially
For i=0, there is no prefix[i-1]. Accessing prefix[-1] would be out of bounds.
Fix: Check i=0 separately: leftSum=0, rightSum = total - prefix[0]. Or use a running sum approach that starts leftSum=0 naturally.
✕03
Returning the wrong pivot when duplicates exist
There may be multiple valid pivot indices. The problem asks for the leftmost one.
Fix: Scan left to right and return the first match — the loop order guarantees this.
