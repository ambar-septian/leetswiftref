Sliding Window

🪟

Sliding Window

A technique for processing contiguous subarrays or substrings in linear time by maintaining a window of elements that slides across the input. Instead of recomputing the window from scratch each step, you add the incoming element and remove the outgoing one — amortising the cost to O(n) total.

SubarraySubstringO(n)Frequency MapTwo Pointers
01

What is Sliding Window?

A sliding window is a contiguous range [L, R] inside the array. As R advances right, the window grows; when the window violates a constraint, L advances right to shrink it. The key insight is that each element enters and leaves the window at most once, so the total work across all iterations is O(n).

Two flavours:

FixedConstant kEvery step"Subarray of exactly k elements"

Key Constraint

Sliding window only works when the window property is monotone: adding an element can only make the window more or less valid (not toggle back and forth). If the property is non-monotone (e.g., the sum must equal exactly k on an unsorted array with negatives), use a prefix sum + hash map instead.

Algorithm Variants

Fixed Window
Window size is exactly k. Initialise the first window, then slide one step at a time — add nums[R], remove nums[L-1]. Maximum average, DNA repeated sequences, count subarrays.
O(n)O(1) or O(k)
→
Variable Window
Window expands by advancing R; shrinks by advancing L when the window violates the constraint. Finds the shortest or longest subarray or substring satisfying the condition.
O(n)O(1) or O(k)
→
Rule of Thumb

Fixed window when the problem says "exactly k". Variable window when the problem says "at most", "at least", "longest", or "shortest". If the window needs a frequency map (characters, distinct elements), it's still sliding window — just with O(k) extra space.
