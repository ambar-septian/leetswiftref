Bit Manipulation

Bit manipulation uses bitwise operators (AND, OR, XOR, shift) to inspect and transform integer values at the binary level. These operations are O(1) and often replace more complex arithmetic. Common patterns: parity check, bit counting, isolating the lowest set bit, and halving via right shift.

ANDORXORShiftParityO(1)
01

What is Bit Manipulation?

Every integer is stored as a sequence of bits. Bitwise operators act on individual bits simultaneously across the entire integer — an entire 32-bit OR takes a single CPU instruction.

OperationSymbolCommon use
ANDn & mMask bits, check parity (n & 1), clear lowest bit (n & (n-1))
ORn | mSet a specific bit (n | (1 << k))
XORn ^ mToggle bits, find unique element (a ^ a == 0)
Left shiftn << kMultiply by 2^k
Right shiftn >> kDivide by 2^k (integer), extract upper bits
NOT~nFlip all bits
Key Constraint

In Swift, the right-shift >> operator on signed integers performs an arithmetic right shift — it copies the sign bit, preserving the sign of negative numbers. For bit manipulation on non-negative integers (the typical LeetCode case), this is not an issue. Be careful with ~n on signed integers: ~0 == -1 in two's complement, not the unsigned maximum. Use UInt32 or UInt64 when you need unsigned bit patterns.

Rule of Thumb

Reach for bit manipulation when the problem involves even/odd checks, dividing by 2, counting set bits, or finding a unique element among duplicates. A right shift is a divide-by-2; AND with 1 is a parity check. These O(1) tricks often replace slower arithmetic.
