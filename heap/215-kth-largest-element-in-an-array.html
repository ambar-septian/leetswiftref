Kth Largest Element in an Array

ArrayDivide and ConquerSortingHeap (Priority Queue)Quickselect
01

Problem Overview

Given an integer array nums and an integer k, return the kth largest element in the array. Note that it is the kth largest element in sorted order, not the kth distinct element.

You must solve it without sorting.

Constraints:

  • 1 <= k <= nums.length <= 10^5
  • -10^4 <= nums[i] <= 10^4
02

Examples

Example 1​

Input: nums = [3,2,1,5,6,4], k = 2
Output: 5

Example 2​

Input: nums = [3,2,3,1,2,4,5,5,6], k = 4
Output: 4

04

Approach

Build a max-heap from all elements. Call popMax() k times; the value returned on the kth pop is the kth largest.

Swift Heap

Swift's Heap type (from the swift-collections package) is a min-heap by default. To get max-heap behaviour, wrap values in a struct that reverses the comparator, or call popMax() which is provided on Heap for the maximum element.

06

Step-by-Step Solution

Build a max-heap —

Construct Heap(nums) — Swift's Heap supports popMax() to retrieve the largest element.

Pop max k times —

Decrement k each time. Store the value returned by popMax() as result.

Return result —

After k pops, result holds the kth largest element.

07

Code Implementation

Swift Collections Heap

The Heap used in this solution comes from the Swift Collections package. Note that the API might be slightly different if you implement your own heap.

import Collections

class Solution {
  func findKthLargest(_ nums: [Int], _ k: Int) -> Int {
    var heap = Heap(nums)  // Swift Collections Heap; popMax() available
    var k = k
    var result = -1

    while k > 0 {
      guard let max = heap.popMax() else { continue }
      result = max
      k -= 1
    }

    return result
  }
}

09

Common Pitfalls

✕01
Using a min-heap without size capping
Calling popMin() k times gives the k smallest elements, not the kth largest.
Fix: Use popMax() for max-heap approach, or maintain a min-heap of exactly k elements and return heap.min at the end.
✕02
Off-by-one: popping k-1 times
The kth largest is the value at the kth pop. Stopping after k-1 pops returns the (k-1)th largest.
Fix: Loop while k greater than 0, decrement k after each pop, and capture result each time. The final value of result after the loop is the kth largest.
✕03
Not handling the guard let failure
If the heap is empty unexpectedly and popMax() returns nil, an infinite loop results when using continue inside the while loop.
Fix: For a valid input where k is at most nums.count, the heap will never be empty during k pops. The guard is defensive — in practice, break or fatalError is cleaner.
