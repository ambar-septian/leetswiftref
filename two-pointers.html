Two Pointers

A technique that uses two index variables to traverse an array or string in a single pass, eliminating the need for nested loops. The two pointers may start at opposite ends and converge, or both start from the left and move in the same direction at different speeds.

In-placeO(n)Sorted ArrayPalindromePartitioning
01

What is Two Pointers?

Two Pointers replaces O(n²) brute-force search with an O(n) single-pass scan by exploiting the structure of the problem — usually sorted order or symmetric structure.

The two main variants:

VariantPointers startMove towardTypical trigger
Opposite endsLeft = 0, Right = n-1Each other (converging)Palindrome, sorted-array pair sum, max area
Same directionBoth at leftSame direction (fast races ahead)In-place removal, partitioning, merge
Key Constraint

The opposite-ends variant requires the array to be sorted (or have symmetric structure like a palindrome). Moving a pointer only makes sense if you can predict which direction brings you closer to the answer. On an unsorted array, use a hash map instead.

Terminology

TermDefinition
Left pointer (L)Starts at index 0; advances when the current element is processed
Right pointer (R)Starts at index n-1 (opposite ends) or just ahead of L (same direction)
Slow pointerThe write cursor in same-direction problems — marks where the next valid value should go
Fast pointerThe read cursor — scans the entire array looking for valid values
In-placeModifying the array without allocating a separate output array
InvariantThe property that holds before and after each pointer move (e.g. "everything left of slow is unique")
03

Algorithm Variants

Opposite Ends
Left and right pointers start at the two ends of the array and converge inward. Used for palindrome checks, two-sum on sorted arrays, container with most water, and reversals.
O(n)O(1)
→
Same Direction
Slow (write) and fast (read) pointers both start at the left and move right, with fast racing ahead. Used for in-place removal, deduplication, partitioning, and merge of sorted arrays.
O(n)O(1)
→
Rule of Thumb

Opposite ends for symmetric or sorted-pair problems; same direction for in-place write / partition problems where you need to skip or overwrite elements.
