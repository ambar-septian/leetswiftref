Set Matrix Zeroes

#73MediumHash SetLeetCode ↗

Set Matrix Zeroes

ArrayHash TableMatrix
01

Problem Overview

Given an m x n integer matrix matrix, if an element is 0, set its entire row and entire column to 0s. You must do it in place.

The key constraint is that the zeroing must be based on the original positions of zeros — not on zeros introduced by the operation itself. Setting a cell to zero must not cascade to zero out rows/columns that did not originally contain a zero.

Constraints

  • m == matrix.length
  • n == matrix[0].length
  • 1 <= m, n <= 200
  • -2^31 <= matrix[i][j] <= 2^31 - 1
Core Insight

Two-pass approach: first scan the matrix to record which rows and columns contain zeros into sets. Second pass: for every cell, if its row or column index is in either set, zero it out. The sets capture original zero positions before any modification.

04

Approach

Pass 1: Iterate all cells. When matrix[i][j] == 0, insert i into rowSet and j into colSet.

Pass 2: Iterate all cells again. If rowSet.contains(i) or colSet.contains(j), set matrix[i][j] = 0.

This guarantees the second pass only reads from the original zero positions captured in the sets.

O(1) space follow-up

You can achieve O(1) extra space by using the first row and first column of the matrix itself as markers. Record whether the first row/column originally contained a zero, then use row[0] and col[0] as flags for all other rows/columns. Apply zeroing in reverse order of marking to avoid overwriting the flags prematurely.

06

Step-by-Step Solution

First pass — record zero positions —

Iterate all (i, j). If matrix[i][j] == 0, insert i into row set and j into column set.

Second pass — apply zeros —

Iterate all (i, j). If row.contains(i) or column.contains(j), set matrix[i][j] = 0.

07

Code Implementation

class Solution {
  func setZeroes(_ matrix: inout [[Int]]) {
    var row = Set<Int>()
    var column = Set<Int>()

    // Pass 1: record original zero positions
    for i in 0..<matrix.count {
      for j in 0..<matrix[i].count {
        if matrix[i][j] == 0 {
          row.insert(i)
          column.insert(j)
        }
      }
    }

    // Pass 2: zero out recorded rows and columns
    for i in 0..<matrix.count {
      for j in 0..<matrix[i].count {
        if row.contains(i) || column.contains(j) {
          matrix[i][j] = 0
        }
      }
    }
  }
}

09

Common Pitfalls

✕01
Zeroing in place during the first pass
If you set row/column to zero as soon as you find a zero in pass 1, subsequent cells in those rows/columns appear to be original zeros when they are not, causing over-zeroing.
Fix: Separate the scan from the mutation: complete the full first pass into sets before any zeroing.
✕02
Checking matrix[i][j] == 0 in the second pass
After zeroing, the second-pass check for zero would include cells set to 0 in the current pass, not just original zeros.
Fix: Check membership in the row/column sets — these capture only original zero positions.
✕03
Modifying matrix dimensions mid-traversal
The problem says in-place modification. Be careful not to change matrix.count or matrix[i].count during iteration.
Fix: Only change cell values — never structural dimensions. The two-pass approach does this safely.
