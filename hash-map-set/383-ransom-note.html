Ransom Note

#383EasyHash Map Frequency CountLeetCode ↗

Ransom Note

Hash TableStringCounting
01

Problem Overview

Given two strings ransomNote and magazine, return true if ransomNote can be constructed by using the letters from magazine, and false otherwise.

Each letter in magazine can only be used once in ransomNote.

Constraints:

  • 1 <= ransomNote.length, magazine.length <= 10^5
  • ransomNote and magazine consist of lowercase English letters.
Key Constraint

Both strings consist only of lowercase English letters — at most 26 distinct characters can appear. This bounds the frequency map to at most 26 keys, making space O(1) regardless of string length.

02

Examples

Example 1​

Input:  ransomNote = "a", magazine = "b"
Output: false

Example 2​

Input:  ransomNote = "aa", magazine = "ab"
Output: false
Explanation: Only one 'a' in magazine; two needed.

Example 3​

Input:  ransomNote = "aa", magazine = "aab"
Output: true

Rule of Thumb

When checking if one string's characters can construct another, build the frequency map from the supplier and consume from the requester. Return false as soon as a required character is unavailable.

04

Approach

Core Insight

Count every character in magazine into a frequency map. For each character in ransomNote, check if it is available (positive count) and decrement. If any character is missing, return false immediately. The magazine can be longer than the note — only per-character availability matters.

Count every character in magazine into a hash map ([Character: Int]). Then iterate through ransomNote: for each character, check if it exists in the map with a positive count. If yes, decrement (and remove when it hits zero). If not found, return false. Return true after consuming all ransom note characters.

Why remove zero-count entries

Removing the entry when count reaches zero keeps the map clean and makes the presence check (dict[char] != nil) equivalent to checking a positive count. Alternatively, just check dict[char, default: 0] > 0.

06

Step-by-Step Solution

Build frequency map from magazine —

Iterate magazine; for each character increment dict[char, default: 0].

Consume ransom note characters —

For each character in ransomNote, look it up in dict.

Check availability —

If the character is not in dict, return false immediately.

Decrement and prune —

Decrement the count. If it reaches zero, remove the entry.

Return true —

If the loop completes, every character was available — return true.

07

Code Implementation

class Solution {
  func canConstruct(_ ransomNote: String, _ magazine: String) -> Bool {
    var dict = [Character: Int]()
    for char in magazine {
      dict[char, default: 0] += 1
    }

    for char in ransomNote {
      if let value = dict[char] {
        dict[char] = value - 1
        if value - 1 <= 0 {
          dict[char] = nil
        }
        continue
      }
      return false
    }

    return true
  }
}
Swift Notes

dict[char, default: 0] += 1 is idiomatic for initialise-and-increment without an explicit if-let. Setting dict[char] = nil removes the key when the count reaches zero, keeping dict[char] != nil equivalent to a positive-count check. For purely lowercase-letter inputs, a fixed 26-slot [Int] array keyed by asciiValue - 97 is a faster alternative.

08

Complexity Analysis

Time Complexity — O(m + n)

One pass over magazine (length m) to build the frequency map, then one pass over ransomNote (length n) to consume characters. Each character triggers an O(1) hash map operation. Total: O(m + n).

Space Complexity — O(1)

The frequency map holds at most 26 keys — the size of the lowercase English alphabet. Regardless of input length, storage is bounded by a constant.

09

Common Pitfalls

✕01
Building frequency map from ransomNote instead of magazine
The magazine supplies the letters; the ransom note consumes them. Building the map from the wrong string inverts the check.
Fix: Always build from magazine, then iterate ransomNote to consume letters.
✕02
Forgetting to remove zero-count entries
If entries stay at count 0, a subsequent check of dict[char] != nil incorrectly reports the letter as available.
Fix: Set dict[char] = nil when the count reaches zero, or consistently check dict[char, default: 0] > 0.
✕03
Checking length equality first
ransomNote.count == magazine.count is NOT a required condition — the magazine can be longer than the note.
Fix: Only check that each required letter is available in sufficient quantity. Length equality is neither necessary nor sufficient.
