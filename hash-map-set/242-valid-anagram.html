Valid Anagram

Hash TableStringSorting
01

Problem Overview

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

An anagram is a word or phrase formed by rearranging the letters of another — using all the original letters exactly once.

Constraints:

  • 1 <= s.length, t.length <= 5 * 10^4
  • s and t consist of lowercase English letters.
02

Examples

Example 1​

Input: s = "anagram", t = "nagaram"
Output: true

Example 2​

Input: s = "rat", t = "car"
Output: false

04

Approach

Guard early if lengths differ. Iterate s and t in lockstep using zip: increment counts[s[i] - 'a'] and decrement counts[t[i] - 'a']. After the loop, all counts must be zero.

Single-pass with zip

Using zip(s, t) processes both strings simultaneously in one loop — no need for separate loops or an index variable.

06

Step-by-Step Solution

Length guard —

Return false immediately if s.count != t.count.

Build frequency delta —

var counts = Array(repeating: 0, count: 26). For each (sChar, tChar) in zip(s, t): increment index for sChar, decrement index for tChar.

Verify all zeros —

Return counts.allSatisfy { $0 == 0 }.

07

Code Implementation

class Solution {
  func isAnagram(_ s: String, _ t: String) -> Bool {
    guard s.count == t.count else { return false }

    var counts = Array(repeating: 0, count: 26)
    for (sChar, tChar) in zip(s, t) {
      counts[Int(sChar.asciiValue! - 97)] += 1
      counts[Int(tChar.asciiValue! - 97)] -= 1
    }

    return counts.allSatisfy { $0 == 0 }
  }
}

09

Common Pitfalls

✕01
Not checking lengths first
Without the length guard, two strings of different lengths could pass the frequency check if the extra characters cancel each other — which is impossible, but skipping the guard is defensive practice.
Fix: guard s.count == t.count else { return false } at the top.
✕02
Using asciiValue without subtracting the base
'a' has ASCII value 97. Without subtracting 97, indices range from 97–122 instead of 0–25 and cause an out-of-bounds crash.
Fix: Int(char.asciiValue! - 97) maps 'a'→0, 'b'→1, ..., 'z'→25.
✕03
Checking counts != 0 instead of allSatisfy { $0 == 0 }
A count array where values cancel to zero overall but have non-zero individual entries (e.g. [1, -1, 0...]) would wrongly appear valid.
Fix: Each element must individually be zero: counts.allSatisfy { $0 == 0 }.
