Valid Sudoku

#36MediumHash SetLeetCode ↗

Valid Sudoku

ArrayHash TableMatrix
01

Problem Overview

Determine if a 9 x 9 Sudoku board is valid. Only the filled cells need to be validated according to the following rules:

  1. Each row must contain the digits 1-9 without repetition.
  2. Each column must contain the digits 1-9 without repetition.
  3. Each of the nine 3×3 sub-boxes of the grid must contain the digits 1-9 without repetition.

Note:

  • A Sudoku board (partially filled) could be valid but is not necessarily solvable.
  • Only the filled cells need to be validated according to the mentioned rules.

Constraints

  • board.length == 9
  • board[i].length == 9
  • board[i][j] is a digit 1-9 or '.'.
Key Constraint

The board is always 9×9 — both time and space are bounded by the constant 81 cells. Only filled cells (non-'.') need validation; a partially filled board is a valid input and does not need to be solvable.

Core Insight

One pass over all 81 cells. For each filled cell, check and insert into three sets — one for its row, one for its column, and one for its 3×3 box. The box index is derived by the formula (row / 3) * 3 + (col / 3), mapping each cell to one of nine boxes.

Rule of Thumb

When validating multiple independent constraint groups simultaneously, think one set per group — check and insert for row, column, and box in a single pass using the box index formula (row/3)*3 + (col/3).

04

Approach

Core Insight

One pass over all 81 cells. For each filled cell, check and insert into three independent sets simultaneously — one for its row, one for its column, and one for its 3×3 box. The box is identified by the formula (row/3)*3 + (col/3), which maps each cell to one of nine boxes numbered 0–8.

Initialise three arrays of 9 empty sets: rows, cols, boxes. Iterate over every cell (i, j). Skip '.'. For each digit, check containment in rows[i], cols[j], and boxes[(i/3)*3 + (j/3)] — return false on any hit. Otherwise insert into all three. Return true after the full traversal.

Box index formula

The 9 boxes are numbered 0–8 left-to-right, top-to-bottom. Row i falls in box-row i/3 (integer division) and column j falls in box-column j/3. Combining: boxIndex = (i/3)*3 + (j/3).

06

Step-by-Step Solution

Initialise 27 sets —

rows = [Set x9], cols = [Set x9], boxes = [Set x9].

Iterate over all 81 cells —

Double loop: for i in 0..8, for j in 0..8.

Skip empty cells —

guard item != '.' else continue.

Check all three constraints —

If rows[i], cols[j], or boxes[(i/3)*3+(j/3)] contains the digit, return false.

Insert into all three sets —

Add the digit to rows[i], cols[j], and the appropriate boxes entry.

Return true —

No duplicate found after all 81 cells.

07

Code Implementation

class Solution {
  func isValidSudoku(_ board: [[Character]]) -> Bool {
    var rows  = Array(repeating: Set<Character>(), count: 9)
    var cols  = Array(repeating: Set<Character>(), count: 9)
    var boxes = Array(repeating: Set<Character>(), count: 9)

    for i in 0..<9 {
      for j in 0..<9 {
        let item = board[i][j]
        guard item != "." else { continue }

        if rows[i].contains(item)  { return false }
        rows[i].insert(item)

        if cols[j].contains(item)  { return false }
        cols[j].insert(item)

        let boxIndex = (i / 3) * 3 + (j / 3)
        if boxes[boxIndex].contains(item) { return false }
        boxes[boxIndex].insert(item)
      }
    }

    return true
  }
}
Swift Notes

Array(repeating: Set<Character>(), count: 9) creates 9 independent Set values — because Array and Set are value types in Swift, each element is a distinct copy, not an alias of the same set. guard item != "." else { continue } cleanly skips empty cells without extra nesting.

Complexity Analysis

Time Complexity — O(1)

Exactly 81 cells are processed with O(1) set operations per cell. The board size is fixed at 9×9, so the total number of operations is bounded by a constant — time is O(1).

Space Complexity — O(1)

27 sets (9 rows + 9 cols + 9 boxes) each holding at most 9 digit characters. The total storage is bounded by 243 entries — a constant. Space is O(1).

09

Common Pitfalls

✕01
Wrong box index formula
A common mistake is using (i * 3) + j or similar expressions that don't correctly map all 81 cells to their 3×3 box.
Fix: Use (i / 3) * 3 + (j / 3). Integer division by 3 maps rows 0-2 to box-row 0, rows 3-5 to box-row 1, etc.
✕02
Not skipping '.' cells
Inserting '.' into the sets and checking for duplicates would falsely invalidate boards with multiple empty cells.
Fix: Guard item != '.' and skip to the next iteration with continue.
✕03
Checking all three sets before inserting into any
This is actually fine — but inserting into rows[i] before checking cols[j] and boxes[...] means a duplicate in cols or boxes is still caught. The order of check-then-insert per set matters within each set, but the three sets are independent.
Fix: For clarity and correctness, always check containment first, then insert — do this for each set individually.
