Group Anagrams

ArrayHash TableStringSorting
01

Problem Overview

Given an array of strings strs, group the anagrams together. You can return the answer in any order.

An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, using all the original letters exactly once. For example, "eat", "tea", and "ate" are all anagrams of each other.

Constraints

  • 1 <= strs.length <= 10^4
  • 0 <= strs[i].length <= 100
  • strs[i] consists of lowercase English letters
Core Insight

Two strings are anagrams if and only if their sorted characters are identical. Sort each string to produce a canonical key and group all strings that share the same key into the same bucket in a hash map.

02

Examples

Example 1​

Input:  strs = ["eat","tea","tan","ate","nat","bat"]
Output: [["bat"],["nat","tan"],["ate","eat","tea"]]

Example 2​

Input:  strs = [""]
Output: [[""]]

Example 3​

Input:  strs = ["a"]
Output: [["a"]]

04

Approach

Build a dictionary mapping String → [String]. For every input string, sort its characters to produce the canonical key. Append the original string to the array at that key (creating the array if absent). Return all values of the dictionary.

Alternative: character-frequency key

Instead of sorting, build a 26-element frequency array and use its string representation as the key (e.g., "1,0,0,...,1,0,0"). This runs in O(k) per string instead of O(k log k) and is preferable for very long strings.

06

Step-by-Step Solution

Initialise dictionary —

var dict = [String: [String]]() — maps sorted key to list of anagram strings.

Iterate and key each string —

For each str in strs: compute key = String(str.sorted()).

Bucket by key —

dict[key, default: []].append(str) — appends to existing bucket or creates a new one.

Return all groups —

return Array(dict.values) — each value is one anagram group.

07

Code Implementation

class Solution {
  func groupAnagrams(_ strs: [String]) -> [[String]] {
    var dict = [String: [String]]()

    for str in strs {
      let key = String(str.sorted())   // O(k log k) canonical key
      dict[key, default: []].append(str)
    }

    return Array(dict.values)
  }
}

09

Common Pitfalls

✕01
Using the original string as the key
The original string cannot serve as the key because anagrams have different original strings by definition. The key must be a canonical form that all anagrams share.
Fix: Use String(str.sorted()) — the sorted characters — as the key.
✕02
Returning dict.keys instead of dict.values
The question asks for groups of original strings, not the canonical keys. Returning keys gives sorted strings, not the originals.
Fix: Return Array(dict.values) — the arrays of original strings.
✕03
Not handling empty strings
An empty string sorts to an empty string. Multiple empty-string inputs should be grouped together — the canonical key approach handles this naturally since sorted("") == "".
Fix: No special handling needed. The algorithm is correct for empty strings.
