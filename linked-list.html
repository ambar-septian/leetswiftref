Linked List

A sequence of nodes where each node holds a value and a pointer to the next node. Unlike arrays, linked lists support O(1) insertion and deletion at known positions — but require O(n) traversal to reach a node by index. Most LeetCode linked list problems reduce to pointer manipulation and two-pointer traversal.

01

What is a Linked List?

A linked list stores elements in nodes scattered throughout memory, connected by next pointers. The trade-off vs arrays:

OperationArrayLinked List
Access by indexO(1)O(n)
Insert / delete at known nodeO(n) shiftO(1) pointer update
Insert / delete at headO(n) shiftO(1)
Memory layoutContiguousScattered
Extra memoryO(1)O(n) for pointers
Key Constraint

Linked list problems almost always require in-place pointer manipulation — no extra array is allowed. The two most useful tools are the sentinel (dummy) head (eliminates edge cases for head removal/insertion) and the fast/slow pointer (finds middle, detects cycles, removes nth-from-end in one pass).

Fast / slow pointer — finding the middle node:

Fast / Slow Pointer — Find Middle Node1 / 4
Initialise
Both slow and fast start at the head (index 0). Fast will move twice as quickly as slow.
var slow = head, fast = head
while fast != nil && fast!.next != nil {
    slow = slow!.next
    fast = fast!.next!.next
}

Rule of Thumb

Reach for the sentinel head when the problem modifies the list head (removal, insertion at front). Reach for fast/slow pointers when the problem asks for middle, cycle, or nth-from-end. Use in-place reversal when the problem asks you to reverse a sublist or alternate segments.
