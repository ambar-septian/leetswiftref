Container With Most Water

ArrayTwo PointersGreedy
01

Problem Overview

You are given an integer array height of length n. There are n vertical lines drawn such that the two endpoints of the ith line are at (i, 0) and (i, height[i]).

Find two lines that together with the x-axis form a container that holds the most water. Return the maximum amount of water a container can store.

Note: You may not slant the container.

Constraints

  • n == height.length
  • 2 <= n <= 10^5
  • 0 <= height[i] <= 10^4
Core Insight

Start with the widest possible container (pointers at both ends). The area is always limited by the shorter line. Moving the taller line inward can only decrease width without increasing the height limit — so always move the shorter pointer inward to have any chance of finding a larger area.

02

Examples

Example 1​

Input:  height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation: Lines at index 1 (height 8) and index 8 (height 7).
             Area = min(8, 7) * (8 - 1) = 7 * 7 = 49.

Example 2​

Input:  height = [1,1]
Output: 1
Explanation: Only two lines. Area = min(1,1) * 1 = 1.

Place left at index 0 and right at index n - 1. At each step compute area = min(height[left], height[right]) * (right - left) and update the running maximum. Then move whichever pointer points to the shorter line — if they are equal, move either (here we move left). Repeat until the pointers meet.

Why moving the taller pointer is always wrong

Suppose left is shorter. Moving right inward shrinks the width AND keeps the same height limit (still capped by the shorter left). The area can only decrease or stay the same. Moving left is the only way to possibly increase the height cap.

06

Step-by-Step Solution

Initialise pointers and result —

left = 0, right = height.count - 1, total = 0.

Compute area at current window —

minHeight = min(height[left], height[right]), width = right - left, area = minHeight * width. Update total = max(total, area).

Move the shorter pointer —

If height[left] > height[right] move right -= 1, else move left += 1.

Repeat until pointers meet —

Loop while left != right. Return total.

07

Code Implementation

class Solution {
  func maxArea(_ height: [Int]) -> Int {
    var left = 0
    var right = height.count - 1
    var total = 0

    while left != right {
      let minHeight = min(height[left], height[right])
      let width = right - left
      total = max(width * minHeight, total)

      if height[left] > height[right] {
        right -= 1
      } else {
        left += 1
      }
    }

    return total
  }
}

09

Common Pitfalls

✕01
Moving the taller pointer instead of the shorter one
Moving the taller pointer inward only reduces width while keeping the same height cap (limited by the shorter line). The area cannot increase.
Fix: Always move the pointer with the smaller height: if height[left] > height[right] move right, else move left.
✕02
Using left < right instead of left != right
Both conditions are equivalent here since left and right only move toward each other. Either works, but left != right is more explicit about the termination condition.
Fix: No correctness issue — either condition terminates the loop correctly.
✕03
Computing width as right - left + 1
Width is the number of gaps between lines, not the number of lines. For indices 1 and 8, width = 8 - 1 = 7, not 8.
Fix: Use width = right - left (no +1).
