Same Direction

Two Pointers›Same Direction

Same Direction

Slow (write) and fast (read) pointers both start at the left and advance rightward. The fast pointer scans every element; the slow pointer only advances when a valid element is found, overwriting in-place. Achieves O(n) time and O(1) space for partition and deduplication problems.

Time O(n)Space O(1)In-placePartitionDeduplicationSlow-Fast
01

Overview

The same-direction pattern maintains a clean invariant:

Everything at indices [0, slow) satisfies the condition (unique, non-zero, matching, etc.). Everything at [slow, fast) has been scanned and discarded. fast is the current element being inspected.

At each step:

  • If nums[fast] is valid → write it to nums[slow], advance slow, advance fast.
  • If nums[fast] is invalid (duplicate, zero, etc.) → skip: advance fast only.

When fast reaches the end, slow is the count of valid elements.

Core Insight

Think of slow as the "write head" and fast as the "read head". The write head only advances when it has something worth keeping. By the time the read head finishes, every valid element has been compacted to the front in-place — no extra array needed.

02

Key Concepts

ConceptDescription
Slow pointerThe next write position; nums[0..slow-1] holds the result
Fast pointerReads every element; checks validity before deciding to write
Invariantnums[0..slow-1] is always a valid prefix (deduplicated, non-zero, etc.)
In-placeResult occupies the front of the original array; return slow as the new length
PartitionA variant where elements are split into two groups (0s / non-0s, negatives / positives)
Key Constraint

The same-direction pattern produces a stable result (relative order of kept elements is preserved) only when fast scans left to right and writes to slow in the same order. If you need an unstable partition (e.g. Dutch National Flag), use a three-way partition instead.

04

Implementation

Remove duplicates from sorted array:

func removeDuplicates(_ nums: inout [Int]) -> Int {
    guard !nums.isEmpty else { return 0 }
    var slow = 1
    for fast in 1..<nums.count {
        if nums[fast] != nums[fast - 1] {   // new unique value
            nums[slow] = nums[fast]
            slow += 1
        }
    }
    return slow
}

Move zeroes to end (stable order of non-zeroes):

func moveZeroes(_ nums: inout [Int]) -> Void {
    var slow = 0
    for fast in 0..<nums.count {
        if nums[fast] != 0 {
            nums[slow] = nums[fast]
            slow += 1
        }
    }
    // fill rest with 0
    while slow < nums.count {
        nums[slow] = 0
        slow += 1
    }
}

Merge strings alternately (two arrays, same-direction):

func mergeAlternately(_ word1: String, _ word2: String) -> String {
    let a = Array(word1), b = Array(word2)
    var i = 0, j = 0
    var result = [Character]()
    while i < a.count || j < b.count {
        if i < a.count { result.append(a[i]); i += 1 }
        if j < b.count { result.append(b[j]); j += 1 }
    }
    return String(result)
}

Rotate array in-place (triple reverse):

func rotate(_ nums: inout [Int], _ k: Int) {
    let k = k % nums.count
    nums.reverse()
    nums[0..<k].reverse()
    nums[k...].reverse()
}
Swift Notes

Swift's inout arrays are passed by value-copy-on-write — mutations inside the function propagate back to the caller through inout. Use nums[slow] = nums[fast] directly (no swap needed when you are discarding the old value at slow). For removeDuplicates, comparing nums[fast] != nums[fast - 1] is idiomatic; you do not need to track the last written value separately because the comparison against the previous array slot is equivalent.

05

Complexity Analysis

Time Complexity — O(n)

The fast pointer advances exactly once per element — n steps total. The slow pointer advances at most n times. Combined, both pointers make at most 2n moves, giving O(n) time.

Space Complexity — O(1)

The result is written back into the original array. Only two integer pointers are needed as extra state — O(1) space regardless of input size.

When to Use

Rule of Thumb

Same-direction when you need to compact valid elements to the front in one pass — deduplication, filtering, partitioning. Opposite-ends when the problem involves a pair from both sides of the array (palindrome, max area, sorted pair sum).

Problem shapeUse same direction?
Remove duplicates from sorted arrayYes
Move all zeroes to the endYes
Filter elements matching a condition in-placeYes
Merge two arrays alternatelyYes (two fast pointers, one per array)
In-place reverse or palindrome checkNo — use opposite ends
Pair sum in sorted arrayNo — use opposite ends
